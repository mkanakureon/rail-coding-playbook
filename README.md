# Rail Coding Playbook (AI日産1万行実現プレイブック)

本プレイブックは、AIエージェント（Claude Code & Gemini CLI 等）を活用し、人間が設計書を書かずに「仕様策定」から「テスト・監査」までを自律実行させるための開発手法**「Rail Coding（レール駆動開発）」**の完全ガイドです。

「kaedevn-monorepo」プロジェクトで実証された、**人間が要求（ゴール）を与え、AIが自律的に仕様定義・実装・テストを完遂する**ための運用基盤とプロンプト群を、他プロジェクトでも再現・導入できるように一般化・体系化しています。

## 「Rail Coding」の概要

「Rail Coding」とは、AIに自由にコードを書かせるのではなく、以下の**強固なレール（Guardrails）**を敷くことで、AIを「迷わせず、バグを生ませない」状態に強制する手法です。

1. **型とLintによる物理的強制** (TypeScript `strict: true`, Biome等でエラー時のコミットをブロック)
2. **2体のAIによる相互監視** (実装担当: The Doer / 監査担当: The Auditor)
3. **段階的コミットによるコンテキスト同期** (Git履歴をAIの地図として活用)
4. **No-CLI原則** (人間がターミナルを叩かず、エラーログもAIに読ませる)
