# 技術評価報告書：コマンドバトル・システムの現状とポータビリティ分析

**日付**: 2026-02-27
**レビュアー**: Gemini CLI
**対象**: `packages/battle` および各エンジンの実装

---

## 1. 概要

本プロジェクトにおける「コマンドバトル」機能は、現在 **Web (PixiJS) 版において先行実装されており、動作可能な状態** にある。
一方で、ネイティブエンジン (`packages/native-engine`) 側には未移植であり、現在の「1行の修正もなく動く」というポータビリティ原則からは一時的に外れた状態（Web専用機能）となっている。

---

## 2. 現在の実装状況の評価

### 2.1 Web (PixiJS) 版の実装：🌟 良好
*   **ロジック**: `@kaedevn/battle` という独立したパッケージに切り出されており、数学的なバトルロジック（ダメージ計算、AI、勝敗判定）はプラットフォーム非依存で実装されている。
*   **UI**: `WebOpHandler.ts` 内で DOM Overlay を用いた簡易的な UI が実装されている。これにより、複雑な PIXI 描画を介さずに、実用的なバトル画面が構築されている。
*   **制御**: `Interpreter` 側で `battle` コマンドが定義されており、`battle("slime_pack", "win_label", "lose_label")` という形式でシナリオから呼び出し可能。

### 2.2 Native (C++) 版の実装：⚠️ 未着手
*   **現状**: `IEngineAPI.hpp` に `battleStart` が定義されておらず、`Interpreter.cpp` にも `battle` 命令のハンドラが存在しない。
*   **課題**: C++ 側で `@kaedevn/battle` と同等のロジックを再実装するか、JSON 形式でデータを受け渡しする共通ロジックを構築する必要がある。

---

## 3. 「完全ポータビリティ」に向けた課題と設計案

現状の実装をネイティブエンジンに移植し、完全なポータビリティを実現するためのロードマップを以下に提案する。

### 3.1 バトルロジックの共通化（C++移植）
`@kaedevn/battle` は非常にシンプルな TypeScript コード（約 300 行）であるため、C++ への移植は容易である。
*   **戦略**: `native-engine/src/interpreter/BattleEngine.cpp` として移植。
*   **データ構造**: `ActorState`, `EnemyState`, `SkillDef` 等の構造体を共通化し、Web 版と 100% 同一のダメージ計算ロジック（RNG 消費順序含む）を保証する。

### 3.2 UI 層の抽象化（最大の難所）
Web 版は DOM Overlay に頼っているが、ネイティブ版（Switch 等）では SDL_Renderer による直接描画が必要になる。
*   **改善案**: 
    1.  `IEngineAPI` に `battleStart` を追加。
    2.  `SDL2Engine` 側で、ノベルのテキストウィンドウを流用した「コマンド選択 UI」と、`CharacterLayer` を流用した「敵の表示」を実装する。
    3.  将来的な「07-MULTI_GENRE_SPEC_AND_DESIGN.md」で提案した **Dynamic UI (JSON定義)** を導入することで、コードを書き換えずに両プラットフォームの UI を同期させる。

### 3.3 データ駆動への移行
現在は `skills.ts` や `troops.ts` にデータがハードコードされている。
*   **改善案**: これらを `assets/battle_data.json` 等の外部ファイルに切り出し、Web/Native 両方のエンジンが実行時に読み込む形式にする。これにより、「敵の強さを変えたのに Native 版に反映されない」という不整合を未然に防ぐ。

---

## 4. 総評と次のステップ

### 評価：
現状のバトルシステムは **「コアロジックの分離」という最も重要な第一歩が完了している**。
Web 版での成功は、アーキテクチャの妥当性を証明している。これをネイティブエンジンに「正しく」引き継ぐことが、現在の最優先事項である。

### 次のアクション：
1.  **IEngineAPI の更新**: `battleStart` 仮想関数の追加。
2.  **Native 版 Interpreter の更新**: `battle` 命令のハンドラ実装。
3.  **BattleLogic の C++ 移植**: `packages/battle` の C++ 版実装。
4.  **簡易的な SDL2 バトル UI**: テキストベース（既存の TextWindow 拡張）での第一歩。

この実装が完了すれば、kaedevn は「ノベル＋バトル」という、より表現力の高いクロスプラットフォームエンジンへと進化する。
