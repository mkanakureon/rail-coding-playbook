# スタンドアロン配布仕様書：バイナリとフォルダの構成

**作成日**: 2026-02-26
**核心**: リポジトリや開発環境なしでゲームを動作させるための最小構成

## 1. 概念：エンジンとコンテンツの分離
kaedevn ネイティブエンジンは、**「エンジンバイナリ（EXE/バイナリ）」** と **「コンテンツデータ（フォルダ）」** が完全に分離されています。これにより、エンジン自体を一度ビルドしてしまえば、あとはデータフォルダを差し替えるだけで異なるゲームを動かすことができます。

## 2. 最小配布構成 (The Bundle)

エンドユーザーに配布する際のフォルダ構成は以下の通りです。

| 構成要素 | 内容 | 役割 |
| :--- | :--- | :--- |
| **実行ファイル** | `kaedevn_native(.exe)` | SDL2 ベースの描画・実行エンジン本体。 |
| **`assets/` フォルダ** | `main.ksc`, `assets.json` | シナリオとアセット名簿。ゲームの核。 |
| **アセット実体** | `*.webp`, `*.ogg` 等 | 実際の画像や音楽ファイル。 |
| **共有ライブラリ** | `SDL2.dll` 等 | 実行に必要な動的リンクライブラリ。 |

## 3. 実行時の動作
実行ファイルが起動されると、以下の順序でリソースを探索します：
1.  カレントディレクトリの `assets/assets.json` を読み込み、名簿を作成。
2.  `assets/main.ksc` を読み込み、インタプリタを開始。
3.  セーブデータが必要になったら、同じ階層に `saves/` フォルダを自動作成。

## 4. 利点
- **ポータビリティ**: フォルダごと USB メモリに入れて持ち運んでも動作します。
- **軽量性**: 巨大な Electron やブラウザを同梱する必要がないため、配布サイズが極めて小さくなります。
- **セキュリティ**: ソースコード（C++）や DB 接続情報を隠蔽したまま、完成したゲーム体験のみを配布できます。
