# PixiJS 最適化リサーチ報告書：低スペック・Switch 環境への適合

**作成日**: 2026-02-26
**対象**: PixiJS v8 を用いた Nintendo Switch および低スペック端末向けの最適化

## 1. 調査背景
kaedevn のレンダリング基盤である PixiJS を、リソース制限の厳しい Nintendo Switch 環境で安定動作（60FPS 維持）させるための最新のベストプラクティスを調査しました。

## 2. PixiJS v8 における主要な最適化手法

### 2.1 WebGPU の優先活用
- **メリット**: WebGPU は WebGL に比べ CPU 負荷を大幅に軽減し、バッテリー消費を抑えます。
- **実装**: Switch のブラウザエンジンがサポートしている場合、`preference: 'webgpu'` を明示的に指定して初期化します。

### 2.2 テクスチャ管理の最適化 (VRAM 節約)
- **圧縮テクスチャ (KTX2/Basis) の導入**: VRAM 消費を最大 75% 削減できます。ノベルゲームは背景や立ち絵といった静止画資産が多いため、この最適化は**最優先事項**です。
- **AssetPack の活用**: ビルド時に自動的に `.ktx2` フォーマットを生成するパイプラインを構築することを推奨します。

### 2.3 レンダリング負荷の軽減
- **Bitmap Text の徹底**: 通常の `PIXI.Text` は文字列変更のたびにキャンバスを再生成するため低速です。頻繁に更新されるテキストには Bitmap Text を使用してください。
- **cacheAsBitmap**: 静的な UI（メニュー画面、複数の装飾がある背景など）は `container.cacheAsBitmap = true` に設定し、ドローコールを 1 に抑えます。
- **アンチエイリアスの無効化**: Switch の解像度（携帯モード 720p）では、`antialias: false` に設定して計算負荷を下げることが現実的な選択肢となります。

## 3. kaedevn への適用勧告
1.  **アセットパイプラインの更新**: `packages/tools` 等に KTX2 変換ロジックを追加し、`packages/web` でこれらをロードするように変更する。
2.  **初期化オプションの見直し**: `packages/web` のエントリーポイントで、デバイスの解像度に応じた `resolution` の制限（最大 2 等）を導入する。
3.  **オブジェクトプーリング**: 演出で多用されるパーティクルやエフェクトに対して、`new` と `destroy` を繰り返さないプール実装を徹底する。
