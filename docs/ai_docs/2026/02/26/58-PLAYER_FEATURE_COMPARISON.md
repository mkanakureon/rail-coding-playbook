# プレイヤー機能比較報告書：ブラウザ版 vs ネイティブ版 (macOS)

**作成日**: 2026-02-26
**対象**: `packages/web` と `packages/native-engine` の機能差異分析

## 1. 概要
ブラウザ版は **PixiJS** と **KSC VM (バイトコード実行)** を基盤とし、リッチな UI と柔軟な実行能力を持っています。一方、ネイティブ版は **SDL2** と **自前インタプリタ (直接実行)** を基盤とし、低レイテンシ・高パフォーマンスと Nintendo Switch への移植性に特化しています。

## 3. ネイティブ版の優位性 (Native Advantages)
- **Switch 最短距離**: C++ で `IEngineAPI` を実装済みのため、Switch SDK への移植コストが最小。
- **デバッグ効率**: CLI から直接テストバイナリを実行でき、ビルドと検証のサイクルが高速。
- **決定論的動作**: ブラウザの GC や JIT コンパイルによる揺らぎがなく、常に安定したフレームレート。

## 4. 今後の強化ポイント
ブラウザ版に追いつくために、ネイティブ版で優先すべきタスクです。
1.  **アニメーション (FrameSet)**: スプライトシートを用いた目パチ・口パクの再現。
2.  **システム UI**: 保存スロット一覧や環境設定画面のビジュアル実装。
3.  **アセットマッピング**: DB から直接 `assets.json` を生成するフローの完全自動化。

## 5. 結論
ネイティブ版は、短期間でブラウザ版の核心機能を網羅することに成功しました。特に **「ポータビリティと堅牢性（RAII）」** においては、ブラウザ版を超える基盤を構築できています。今後は、ブラウザ版が持つ「演出の細やかさ」を取り込んでいくフェーズに入ります。
