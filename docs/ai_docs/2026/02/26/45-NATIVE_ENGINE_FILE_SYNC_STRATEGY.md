# ネイティブエンジン：アセット・シナリオ同期戦略

**作成日**: 2026-02-26
**対象**: データ連携パイプライン設計
**核心**: TS エディタで作り、Native エンジンで遊ぶシームレスな環境

## 1. 同期対象のデータ
エディタ (`apps/editor`) から出力される以下のデータをネイティブ版の `assets/` ディレクトリに同期します。

1.  **`.ksc` ファイル**: シナリオテキスト。
2.  **`assets.json`**: slug とファイルパス/URL の対応表。
3.  **物理アセット**: PNG, OGG, WAV ファイル。

## 2. 同期方法（開発環境）
シンボリックリンクまたはビルドスクリプトによる自動コピーを行います。

- **Mac 開発時**: 
    - `ln -s ../../projects/demo/output ./packages/native-engine/assets/current_project`
    - これにより、エディタで保存した瞬間に C++ 版でも最新のシナリオが読み込めるようになります。

## 3. 本番パッケージング（Switch/PC配布）
- **仮想ファイルシステム (VFS)**: 将来的に、全アセットを一つのアーカイブ（例: `.pak` または `.bundle`）にまとめ、暗号化と高速ロードを実現します。
- **データ形式の統一**: C++ 側では `nlohmann/json` を使用し、TS 版と全く同じ JSON スキーマをパースします。

## 4. 検証：データ駆動型回帰テスト
- **仕組み**: 
    1. TS 版で特定のシナリオを実行し、各行終了時の変数状態をログ出力。
    2. C++ 版でも同一のシナリオを実行し、同様のログを出力。
    3. 両者のログを `diff` し、1ビットの差異も許さない厳格な挙動一致テストを実施。
