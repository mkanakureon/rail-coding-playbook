## 1. 互換性の根拠：共通の「命令セット」
TS 版インタプリタと C++ 版インタプリタは、同一の「組み込み関数（Built-in Commands）」を実装しています。これにより、エディタで配置したブロックがネイティブ版でも同一の挙動を示します。

## 2. データ構造の同期
ネイティブエンジンは、エディタが出力する JSON メタデータを解釈できるよう設計されています。

- **アセット解決**: `assets.json` を介して、スクリプト上の `slug` がハッシュ化された物理ファイル名へ正しく変換されます。
- **変数状態**: `GameState` は Web/Native 共に JSON シリアライズを採用しており、データの相互乗り入れが可能です。

## 3. 注意が必要な相違点 (Implementation Gaps)
エディタで以下の高度な機能を使用した場合、ネイティブ版では現在無視されるか、簡易的な動作になります。

1.  **高度な制御構文**: `else if` や `switch` などの複雑なチェーン（ネイティブ版は現在単純な `if` と `jump` で構成）。
2.  **複雑なタイムライン**: `timeline()` コマンドによる多段アニメーション。
3.  **高度な演出エフェクト**: 特定のカスタムフィルター（セピア、ノイズ等）は、シェーダーの完全移植まで Web 版限定となります。
