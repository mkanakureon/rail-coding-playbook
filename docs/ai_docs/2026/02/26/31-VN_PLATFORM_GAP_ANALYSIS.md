# ノベルゲーム PF：機能ギャップ分析報告書

**作成日**: 2026-02-26
**対象**: 現状の kaedevn と商用 VN プラットフォームの差異分析

## 1. 概要
kaedevn は「エディタ」「AI 執筆支援」「マルチプラットフォーム実行」という強力な基盤を持っています。しかし、「完成したゲームを一般ユーザーが快適に遊ぶ」ためのシステム機能において、いくつかの重要な欠落（ギャップ）が確認されました。

## 2. 核心的な不足機能 (Critical Gaps)

### 2.1 プレイ体験 (Player Experience)
- **システム UI の欠如**: タイトル画面、セーブ/ロード画面、設定（環境設定）画面がエンジン側に標準実装されていません。
- **バックログ (メッセージ履歴)**: 読み飛ばしたセリフを遡る機能がありません。
- **オート / スキップモード**: 現代の VN において必須の QOL 機能が未実装です。

### 2.2 演出力 (Visual & Audio Richness)
- **画面遷移 (Transitions)**: 現在は単純なフェードのみが想定されています。ルール画像（グレースケール）を用いた複雑なトランジションが必要です。
- **レイヤー構造の高度化**: 前景・中景・遠景の分離や、フィルタ（セピア、回想など）の動的な適用。
- **立ち絵の動的演出**: 目パチ・口パク、または Spine/Live2D のようなボーンアニメーションへの対応。

### 2.3 エディタ・制作支援 (Authoring Gaps)
- **変数・フラグの可視化**: 実行中にどの変数がどう変化したかをデバッグ時に確認する UI が不足しています。
- **AI 執筆の中間介入 (Human-in-the-Loop)**: AI が生成したプロットを「修正してから」次のステージ（本文生成）へ進むためのワークフローが不完全です。

## 3. 今後の優先順位
1.  **High**: プレイ QOL（バックログ、セーブ/ロード、オート/スキップ）。
2.  **Medium**: 演出力強化（トランジション、フィルタ）。
3.  **Low**: 外部アセット形式（Live2D等）への対応。
