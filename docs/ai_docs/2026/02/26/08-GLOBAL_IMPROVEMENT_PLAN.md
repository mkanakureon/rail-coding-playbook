# AI 執筆支援システム：全体的な改善計画

**作成日**: 2026-02-26
**目的**: 複数ジャンルの出力レビューに基づく、生成品質の底上げとシステム改善

## 1. 課題分析と解決策

### 1.1 定型フレーズの過剰な繰り返しの抑制
- **課題**: キャラクターの口癖や特定の知識説明が、全編を通じて一字一句同じ形式で繰り返される傾向がある。
- **解決策**: 
    - `Stage 3` プロンプトに「直近のシーンで使用した特徴的なフレーズを避ける」指示を追加。
    - プロンプト内に「口調ガイド」だけでなく「状況に応じた言い換え例」を求める。

### 1.2 ビジュアルノベルとしての演出指示の強化
- **課題**: 小説としては成立しているが、ゲームとしてのSE、背景、立ち絵の表情指示が不足している。
- **解決策**:
    - 演出ルールを強化し、地の文の中に `[se: 扉の開く音]` や `[表情: 驚き]` といったメタデータを埋め込むよう LLM に指示する。
    - `ks-generator.ts` でこれらを正確にタグ変換する。

### 1.3 選択肢の論理的帰結の深化
- **課題**: 選択肢が単なる「会話の分岐」に留まっており、物語の結末やフラグに影響を与える「ゲーム性」が薄い。
- **解決策**:
    - `Stage 3` で選択肢を生成する際、必ず `flagSet`（変数の増減）をセットで提案させる。
    - 選択肢の `result` フィールドに、その後の展開への「伏線」を含めるよう指示。

## 2. 実装タスク

### 2.1 `prompts.ts` の修正 (P0)
- `buildStage3Prompt` に、反復抑制と演出指示の具体化に関するシステムインストラクションを追加する。

### 2.2 `_default.yaml` の更新 (P0)
- 全ジャンル共通の「執筆禁止事項（反復禁止など）」と「演出指示のフォーマット」を定義する。

### 2.3 `types.ts` と `schemas.ts` の拡張 (P1)
- 演出メタデータを保持するためのフィールド追加。
