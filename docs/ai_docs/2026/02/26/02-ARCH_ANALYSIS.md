# AI 執筆支援システム：設計・実装・品質向上戦略（2026-02-25 時点）

## 1. システム概要
本システムは、作者が用意した Markdown 形式の「作品設定」をインプットとし、多段階のプロンプトチェーンを通じて、最終的に演出コマンド付きのノベルゲームスクリプト（`.ks` ファイル）を自動生成するものです。

### 生成パイプライン（4 Stage）
1.  **Stage 1: 章プロット生成** (`gpt-4o-mini`)
    - 全体の章構成、各章の主要イベント、感情曲線を作成。
2.  **Stage 2: 話プロット生成** (`gpt-4o-mini`)
    - 各章を「話（Episode）」および「シーン（Scene）」に分割。場所、時間、登場人物、展開を定義。
3.  **Stage 3: シーンテキスト生成** (`claude-3.5-sonnet`)
    - プロットに基づき、地の文・セリフ・選択肢を生成。キャラの口調再現やコンテキストの維持を重視。
4.  **Stage 4: .ks スクリプト変換** (TypeScript テンプレート)
    - 生成されたテキストを、エンジンが解釈可能なスクリプトに決定論的に変換（LLM不使用）。

2026年2月25日時点で、コアロジックおよび CLI テスト環境の実装が完了しています。

- **CLI Tool**: `scripts/assist-cli.ts` により、DB不要で一気通貫の生成テスト（`all` コマンド）が可能。
- **自動テスト**: 86件のテストケースをパス済み。

## 3. コンテキスト維持と長期記憶の仕組み
長編制作において「設定の矛盾」や「記憶喪失」を防ぐため、高度なコンテキスト管理を導入しています。

- **補助プロンプトの活用**: シーン終了ごとに「シーン要約」と「キャラ状態（知識・感情・関係性）」を抽出。
- **階層的圧縮**: 「話要約」→「章要約」→「篇（Arc）要約」と段階的に圧縮。
- **長期記憶の制御**: 篇（数章単位）を跨ぐ際に情報を圧縮し、トークン上限による情報の欠落を防止。
- **伏線管理**: 未解決の伏線（Plot Threads）を抽出し、後の章のプロンプトへ自動的に注入。

## 4. 品質向上戦略（レビューに基づく改善）
2026-02-25 の全話レビュー（B+評価）を受け、A評価（プロレベル）への引き上げ策が策定されました。

- **外部 YAML ルール**: ジャンル別（ミステリー、恋愛、ホラー等）の執筆・演出ルールを外部ファイル化し、プロンプトへ動的に注入。
- **五感描写の強化**: `world.md` に場所ごとの視覚以外の情報（聴覚・嗅覚・触覚）を追加。
- **キャラクターの内面描写**: 単なる台詞だけでなく、微反応（Micro Reactions）や内面の声（Inner Voice）の指示を追加。
- **コスト最適化**: 次世代モデル（`gpt-5-nano` 想定）の採用を見据えた費用見積もり（1章約1.8円、36章全編で約75円）を実施。
