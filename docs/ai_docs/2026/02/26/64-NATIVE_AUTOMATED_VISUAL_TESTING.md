# ネイティブエンジン：自動ビジュアルテスト戦略

**作成日**: 2026-02-26
**対象**: 品質保証・自動テスト
**核心**: SDL2 画面の自動キャプチャと画像比較による挙動保証

## 1. 目的
ブラウザ版の `page.screenshot()` によるテストと同様に、ネイティブエンジンでも「意図した背景とキャラが、意図した座標に表示されているか」を自動で検証します。

## 2. 実装アプローチ

### 2.1 ヘッドレス・キャプチャ (Headless Capture)
物理的な画面を表示せずにテストを実行します。
- **Mac**: `SDL_VIDEODRIVER=dummy` 環境変数を使用して実行。
- **仕組み**: レンダラーを `SDL_RENDERER_SOFTWARE` またはオフスクリーンターゲットとして初期化し、描画完了後に `takeScreenshot()` を呼び出します。

### 2.2 シナリオ・スクリプトによる自動進行
テスト用の KSC スクリプトに、特定のタイミングで撮影を行う特殊なデバッグコマンドを追加します。
```ksc
bg('sky')
ch('hero', 'normal', 'none', 640)
debug_screenshot('01-initial-state.png') // 自動撮影
```

### 2.3 画像比較 (Visual Regression)
1.  **参照画像 (Gold Master)**: あらかじめ「正解」とされる画像を `tests/gold_masters/` に用意します。
2.  **比較**: テストで生成された画像と参照画像を、ピクセル単位（または Hash 値）で比較します。
3.  **判定**: 差異が閾値（例: 0.1%）を超えた場合にテスト失敗とします。

## 3. テスト項目例
- **背景・キャラ合成**: 透過 PNG が正しく重ね合わされているか。
- **座標精度**: `ch` コマンドの X 座標が、ピクセルレベルで正解画像と一致するか。
- **テキスト描画**: `SDL_ttf` による文字が、指定したフォントとサイズで正しく描画されているか。

## 4. 自動化ワークフロー
```bash
# ネイティブテストの全自動実行（将来像）
./scripts/run-native-visual-tests.sh
```
1.  テスト用バイナリをヘッドレスモードで起動。
2.  各シナリオの節目でスクリーンショットを連写。
3.  `ImageMagick` 等を用いて、既存のスクリーンショットと比較・レポート出力。
