# ネイティブエンジン：制御構文（if/choice）の実装設計

**作成日**: 2026-02-26
**対象**: `Interpreter.cpp` のフロー制御拡張
**核心**: 条件分岐とブロックのスキップロジック

## 1. if 文の実装ロジック
KSC の if 文は `{ ... }` によるブロック構造を持ちます。

1.  **条件評価**: `Evaluator` で条件式を評価。
2.  **真の場合**: 通常通り次の行（ブロック内）へ進む。
3.  **偽の場合**: 対応する閉じ括弧 `}` の次の行まで `pc` を一気に進める。
4.  **入れ子の管理**: カウンタを用いて、正しく対応する `}` を見つけるロジックを `Parser` に追加。

## 2. choice 文の実装ロジック
1.  **選択肢の抽出**: `choice { ... }` 内の文字列とジャンプ先をリスト化。
2.  **エンジンへの委譲**: `IEngineAPI.showChoices(options)` を呼び出し、ユーザーの入力を待つ。
3.  **ジャンプ**: 選択された項目のラベル、またはブロックへ `pc` を移動。

## 3. テスト計画
- 単純な `if`。
- 偽のときのブロック全スキップ。
- `if` の中にある `if` (入れ子) の正しい処理。
- 変数の値を変えて分岐が変わることの検証。
