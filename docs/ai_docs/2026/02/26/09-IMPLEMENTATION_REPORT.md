# AI 執筆支援システム：システム改善実装報告書

**作成日**: 2026-02-26
**対象**: 複数ジャンルレビューに基づくシステム改善の実装

## 1. 実施内容

### 1.1 `prompts.ts` の修正
- **反復抑制の強化**: `buildStage3Prompt` のシステムプロンプトに、「同じキャラの決め台詞・口癖を1シーン内で2回以上繰り返さない」という厳格なルールを追加しました。
- **演出指示の自動化**: 地の文（narration）の中に `[se: ...]`（効果音）や `[表情: slug]`（立ち絵表情）を埋め込むように LLM へ指示するセクションを追加しました。これにより、生成されたテキストから直接 `.ks` エンジン用の演出を抽出可能になります。

### 1.2 `_default.yaml` の更新
- **執筆クオリティの向上**:
    - **沈黙描写の制限**: 「……」のみの描写を1シーン1回までに制限し、代わりの行動描写を促すようにしました。
    - **文体リズムの改善**: 短文と長文を混ぜること、比喩を適度に使うこと（1シーン1〜2回）を義務付けました。
    - **表現の重複禁止**: 「静寂」「不穏」などの頻出ワードを前シーンから引き継がないよう指示を強化しました。

## 2. 期待される効果
- **キャラクター性の深化**: 単なる口癖の連発ではなく、状況に応じた多様な言い回しが生成されるようになります。
- **ゲーム演出の具体化**: これまで「地の文」として処理されていた部分にSEや表情指示が含まれるため、エンジニアやスクリプト制作者の工数が大幅に削減されます。
- **文章の「素人っぽさ」の解消**: 文のリズムや五感描写の強化により、読み物としてのクオリティが向上します。

## 3. 今後のステップ
- 改善されたプロンプトを使用して、再度 `projects/longstory` 等の生成テストを行い、品質の向上を確認する。
- `ks-generator.ts` を拡張し、新しく導入された `[se: ...]` などのタグを自動的に `@se` コマンド等へ変換するロジックを追加する。
