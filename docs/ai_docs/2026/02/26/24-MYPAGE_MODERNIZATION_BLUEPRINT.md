# マイページ・アセット管理：モダン化再設計図 (BluePrint)

**作成日**: 2026-02-26
**対象**: `apps/next/app/(private)/mypage` および `my-assets` の UI/UX 刷新

## 1. 現状の課題分析

### 1.1 マイページ (/mypage)
- **視覚的分断**: ヘッダーの鮮やかなグラデーションに対し、統計行 (StatsRow) が唐突な白背景で、デザインの連続性が欠けている。
- **情報の密度**: プロジェクト一覧が単純なリスト形式であり、プロジェクト数が増えた際の視認性が低い。

### 1.2 アセット管理 (/my-assets)
- **技術的負債**: 大部分がインラインスタイルで記述されており、メンテナンス性が低く、ダークモード対応も不十分。
- **UI の陳腐化**: フィルターやグリッドが標準的な HTML 要素に近く、モダンな Web アプリケーションとしての「心地よさ」が不足している。
- **機能不足**: 大量のアセットから目的のものを探すための検索機能や、一括操作機能が存在しない。

## 2. 再設計案 (Proposed Design)

### 2.1 共通デザイン言語：Glass & Gradient
- **テーマ**: `Red-Orange Gradient` + `Glassmorphism`。
- **カードデザイン**: 背景を `bg-white/70 dark:bg-gray-900/70 backdrop-blur-md` とし、境界線に微かな光沢を与える。

### 2.2 マイページの刷新内容
- **浮遊する統計カード**: 統計情報をヘッダーバナーに重なるように配置し、リッチな印象を与える。
- **AI 進捗バッジ**: プロジェクトごとに「AI 執筆: Stage 3 完了」などの進捗状況を表示し、次に何をすべきか一目でわかるようにする。

### 2.3 アセット管理の刷新内容
- **Tailwind 完全移行**: インラインスタイルを全廃。
- **インタラクティブ・フィルター**: 選択中のフィルターを視覚的に強調し、アニメーション（Framer Motion 等）を伴うスムーズな切り替えを導入。
- **クイック・プレビュー**: アセットをクリック（またはホバー）すると、大きな画面でプレビューや詳細情報（サイズ、使用箇所）を確認できるサイドパネル。
- **モダンな「プロジェクト追加」**: `select` タグを廃止し、アセットをドラッグ＆ドロップ、あるいは「＋」ボタンからの洗練されたモーダルで追加先を選択可能にする。

## 3. 実装優先順位

1.  **P0 (最優先)**: `my-assets` の TailwindCSS 移行と、インラインスタイルの除去。
2.  **P1 (重要)**: アセット検索バーの実装と、フィルターのコンポーネント化。
3.  **P2 (体験向上)**: モーダルベースの「プロジェクト追加」フローと、AI 進捗表示の統合。
