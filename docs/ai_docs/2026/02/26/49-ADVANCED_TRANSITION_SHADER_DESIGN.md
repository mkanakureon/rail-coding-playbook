# 高度なトランジション：シェーダー設計と実装指針

**作成日**: 2026-02-26
**対象**: レンダリングエンジンの高度化
**核心**: ルール画像を用いた多彩なワイプ演出（Mask Wipe）

## 1. 動作原理：Mask Wipe
グレースケールの「ルール画像（マスク）」の各ピクセルの輝度と、現在の「進行度（0.0 〜 1.0）」を比較します。

- **ロジック**: 
    - `輝度 < 進行度` ならば「遷移後の画像」を表示。
    - `輝度 >= 進行度` ならば「遷移前の画像」を表示。
- **エッジ処理**: `smoothstep` を用いることで、境界線をわずかにぼかし、高品質な仕上がりにします。

## 2. SDL2 における実装方法 (OpenGL Bridge)
`SDL_Renderer` のバックエンドとして OpenGL を使用し、テクスチャ描画時にカスタムシェーダーを「差し込む」手法を採用します。

### 2.1 必要なコンポーネント
- **Shaderクラス**: GLSL のコンパイルとリンクを管理。
- **TransitionManager**: 遷移前・後のテクスチャ、およびルール画像を保持し、時間経過（progress）を制御。

### 2.2 KSC コマンドの拡張案
```ksc
// bg(slug, transition_type, duration)
bg("forest", "mask:wave_01", 1000)
```

## 3. 実装上の課題と対策
- **シェーダーのポータビリティ**: Switch 実装時は `nn::gfx` または専用のシェーダーコンパイラを使用するため、GLSL コードを抽象化して管理する必要があります。
- **フォールバック**: シェーダーが利用できない（またはロードに失敗した）環境では、標準の `cross-fade` (Alpha 変更) に自動でフォールバックするロジックをエンジンに組み込みます。

## 4. 今後のステップ
1.  **Step 1**: SDL2 の初期化時に OpenGL ドライバーを強制する。
2.  **Step 2**: 最小限の「フェードシェーダー」を実装し、描画パイプラインをテスト。
3.  **Step 3**: ルール画像を用いた「ワイプシェーダー」へ拡張。
