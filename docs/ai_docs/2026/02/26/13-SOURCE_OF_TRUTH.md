## 1. サービス構成とポート割当

現在の kaedevn モノレポは、以下の 3 つの主要アプリケーションと共有パッケージ群で構成されています。

| サービス名 | ディレクトリ | ポート | 役割 |
| :--- | :--- | :--- | :--- |
| **Next.js** | `apps/next` | **3000** | 公式サイト、ユーザー管理、プロジェクト一覧、作品閲覧 |
| **Editor** | `apps/editor` | **5176** | ビジュアルノベルエディタ（制作ツール専用） |
| **Hono API** | `apps/hono` | **8080** | 唯一のバックエンド API。認証、DB、アセット管理を担当 |
| **Preview** | `packages/web` | **5175** | ゲームエンジンのプレビュー/実行環境 |

### ⚠️ 注意：廃止されたコンポーネント
- `apps/api` および `apps/server` は完全に廃止されました。
- `apps/public` は静的アセット管理用として存在しますが、API サーバーとしての機能は持っていません。

## 2. アプリケーション間の責任範囲

「1つのアプリに1つの責任」を徹底しています。

### apps/next (管理・公開)
- **認証**: ログイン、ユーザー登録。
- **管理**: プロジェクトの作成・削除、ダッシュボード。
- **公開**: 作品の公開設定、他ユーザーの作品プレイ。

### apps/editor (制作)
- **編集**: シナリオのブロック編集、タイムライン作成。
- **アセット**: キャラクター、背景、音楽のアップロードと割り当て。
- **プレビュー**: 編集中のシナリオを即座に確認（ポート 5175 を使用）。

### apps/hono (データ)
- **DB**: Prisma を使用した PostgreSQL 操作。
- **AI**: 執筆支援（Stage 1〜4）のロジック実行。
- **Storage**: Azure Blob Storage との通信。

## 3. 開発の基本ワークフロー

### 3.1 サーバーの起動
一括起動スクリプトを使用します。

```bash
./scripts/dev-start.sh all
```

### 3.2 認証の共有
`apps/next` (3000) でログインすると、ブラウザのクッキーを通じて `apps/editor` (5176) と `apps/hono` (8080) でも認証が有効になります（localhost ドメイン共有）。

---

## 4. 共有パッケージ (packages/)

- **@kaedevn/core**: 共通の型定義、タイムラインロジック、プラットフォーム抽象化インターフェース。
- **@kaedevn/interpreter**: Kaede Script (.ksc) の実行エンジン。
- **@kaedevn/web**: PixiJS を使用した Web 向けレンダリング実装。
- **@kaedevn/ai-gateway**: 執筆支援のための多段階 LLM クライアント。

## 5. 命名・設計規則

- **言語**: コード、識別子、コミットメッセージは **English**。ドキュメント、仕様、プロンプト内指示は **Japanese**。
- **移植性**: 常に Nintendo Switch への移植を意識し、`packages/core` の抽象化レイヤー（IInput, IAudio）を使用すること。
- **状態管理**: React の state は「共通の最小祖先」まで引き上げて管理すること。
