# エンジン QOL 機能：詳細仕様設計案

**作成日**: 2026-02-26
**対象**: バックログ、セーブ/ロード、オート、スキップの実装

## 1. バックログ (Message Log)
- **仕組み**: `OpRunner` または `Interpreter` がセリフを処理するたびに、履歴スタック（最大 100 件程度）へオブジェクトをプッシュします。
- **データ構造**: `{ speaker: string, text: string, voiceId?: string, timestamp: number }`
- **機能**: 過去の音声の再再生、特定の場所への巻き戻しジャンプ（将来拡張）。

## 2. セーブ / ロード
- **スナップショット方式**: 現在の PC (Program Counter)、変数、コールスタック、および「現在再生中のBGM/背景/キャラ」の状態をシリアライズして保存します。
- **ストレージ**: `IStorage` を通じて、Web では IndexedDB、Switch ではセーブデータ領域へ保存。
- **サムネイル**: 保存時のスクリーンショットを自動生成し、ロード画面で表示します。

## 3. オート & スキップ
- **オートモード**: 
    - 文字送り終了後、指定された待機時間（環境設定依存）を経て自動的に `OK` アクションを dispatch します。
    - 音声再生中は、音声終了まで待機するロジックを組み込みます。
- **スキップモード**:
    - 全てのウェイトを 0 にし、既読セリフのみ（または全セリフ）を高速で消化します。
    - `Interpreter` 側で高速ループ処理を行い、描画負荷を最小化します。

## 4. 共通システム UI (System UI)
- `packages/web` 内に、PixiJS コンテナとして実装された「共通メニューテンプレート」を導入することを推奨します。
