---

## 1. レビュー方針

本機能は「既存のツクールエディタの枠を超え、汎用的なゲームエンジンとしての外見（UI）を定義する」極めて重要なコンポーネントです。そのため、単なる動作確認に留まらず、**「ユーザーが自由に、かつ型安全にUIを構築できるか」**という設計思想の遵守を重点的にレビューします。

---

## 2. Step 1: 基盤（ストア + 型 + プリセット） レビュー項目

**目的**: レイアウト情報を保持するデータ構造と、初期プリセットが正しく定義されていること。

- [ ] **型定義**: `packages/core/types/PlayLayout.ts` に `screen-overlay` タイプが正しく追加されているか。
- [ ] **プリセット**: `smartphone-portrait.json` が 9:16 (720x1280) の解像度を正しく定義し、主要なUIパーツ（メッセージ、コマンド、金、場所等）を縦画面に最適化して配置しているか。
- [ ] **ストア**: `useTsukuruStore.ts` に `layout` 状態が追加され、Immutable な更新（`setLayout`, `updateElement`）が実装されているか。
- [ ] **シェル**: `LayoutEditor.tsx` が 3カラム構成（左:要素、中央:キャンバス、右:プロパティ）で初期表示されるか。

---

## 3. Step 2: キャンバス（react-rnd 操作系） レビュー項目

**目的**: WYSIWYG の根幹であるドラッグ＆ドロップおよびリサイズ操作の操作性。

- [ ] **ライブラリ**: `react-rnd` が正しく導入され、キャンバス内の各要素をラップしているか。
- [ ] **グリッド**: 8px 単位のスナップ機能が有効であり、配置がガタつかないか。
- [ ] **解像度同期**: ドロップダウンで解像度（16:9, 9:16等）を切り替えた際、キャンバスのアスペクト比が即座に反映されるか。
- [ ] **バリデーション**: 9:16 選択時に「Web/Android専用（Switch非対応）」の警告バッジが適切に表示されるか（指令 `24` の遵守）。
- [ ] **スケーリング**: ウィンドウサイズに合わせてキャンバス自体が縮小表示（Transform: scale）されても、要素のドラッグ座標が正しく計算されているか。

---

## 4. Step 3: パーツモック + ツリー + プロパティ レビュー項目

**目的**: エディタ上での見た目と、詳細なパラメータ編集の正確性。

- [ ] **モック**: `elementMocks.tsx` において、メッセージウィンドウや選択肢ウィンドウが「本物のランタイムに近い見た目」で描画されているか。
- [ ] **表示切替**: 左パネルの目玉アイコンで、エディタ上の表示/非表示を切り替えられるか。
- [ ] **並び順**: 左パネルでの D&D または zIndex 編集により、重なり順（zIndex）が即座に変更されるか。
- [ ] **プロパティ編集**:
    - [ ] 座標 (x, y) とサイズ (w, h) の数値入力が動作するか。
    - [ ] 不透明度 (opacity) がスライダーまたは数値で編集できるか。
    - [ ] **重要**: `borderVisible` や `backgroundColor` (transparent対応) が、脱ツクール要件（指令 `23`）を満たしているか。

---

## 5. Step 4: テストプレイ統合 + エクスポート レビュー項目

**目的**: 作成したレイアウトが、実際のゲーム画面（ランタイム）で 1:1 で再現されること。

- [ ] **ランタイム反映**: `TestPlayOverlay.tsx` がハードコードされた Tailwind クラスではなく、`PlayLayout` の `absolute` 座標を優先して描画しているか。
- [ ] **解像度追従**: 9:16 レイアウト時、テストプレイのオーバーレイ枠も縦長に制限されているか。
- [ ] **screen-overlay**: 画面全体を覆うオーバーレイパーツ（演出用）が最前面に正しく表示されるか。
- [ ] **インポート/エクスポート**:
    - [ ] 配置データを JSON ファイルとして保存（ダウンロード）できるか。
    - [ ] 外部の JSON をアップロードしてレイアウトを復元できるか。
- [ ] **下位互換性**: レイアウトが未設定（null）の場合、従来の「rpg-classic」配置で正しくフォールバックされるか。

---

## 6. 品質・ガバナンス レビュー項目

- [ ] **型安全**: `UiElementOptions` が各パーツごとに正しく型定義され、不正なプロパティが渡されないようになっているか（指令 `24`）。
- [ ] **Switch考慮**: `IInput` 等の抽象化レイヤーを破壊するような DOM 直接操作が、テストプレイ統合時に混入していないか。
- [ ] **テスト実行**: `layoutHelpers.test.ts` を含む 15 件の新規テストがパスし、カバレッジが維持されているか。
- [ ] **性能**: 要素のドラッグ中に、ストアの更新が高頻度すぎて UI がフリーズしないか（必要に応じて throttle/debounce を検討）。

---

## 完了の定義 (Definition of Done)

1. `npm run typecheck` でエラーがない。
2. 全ての Step のチェック項目が ✅ となり、指摘事項が修正されている。
3. `smartphone-portrait` プリセットを選択した状態でテストプレイを起動し、縦画面用のUIが意図通りに配置されていることが目視確認できる。
