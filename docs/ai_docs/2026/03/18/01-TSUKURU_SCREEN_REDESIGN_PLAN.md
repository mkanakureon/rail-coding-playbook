# ツクール型エディタ画面 再設計・画面設計ドキュメント

> **作成日**: 2026-03-18
> **担当**: Gemini CLI

## 1. 背景と目的
kaedevn を「RPGツクール」ユーザーにとって直感的で使いやすいツールにするため、エディタのUI/UXを本家ツクール（MV/MZ等）の画面構成に可能な限り寄せる形で再設計します。
本ドキュメントは、ツクールの画面概念を kaedevn のアーキテクチャ（ブロックエディタ＋GameDb＋PlayLayout）にマッピングし、画面ごとの機能を整理した画面設計の基礎資料です。

---

## 2. 画面構成サマリー（全4画面＋主要ダイアログ）

本家ツクールにおける「メインウィンドウ」「データベース」「イベントコマンド入力」の概念を、kaedevn では以下の4つの主要画面（タブ）と、それを補完する各種ダイアログに再構成します。

| # | 画面名（kaedevn） | ツクールの該当画面 | 主な役割・機能 |
|---|----------------|------------------|--------------|
| 1 | **マップエディタ画面** | メインウィンドウ（マップビュー） | タイル配置、レイヤー切替、マップイベントの配置 |
| 2 | **シナリオ（イベント）エディタ** | イベントエディタ / イベントコマンド | 物語の進行、イベント内容をブロックで構築 |
| 3 | **データベース画面** | データベースウィンドウ | キャラクター、敵、アイテム、スキルなどの各種データ定義 |
| 4 | **UIレイアウトエディタ** | （ツクールにはない独自機能） | プレイ画面のUI（メッセージウィンドウ等）の視覚的配置 |

※上記に加え、ゲーム制作に不可欠な**「素材管理」「変数・スイッチ管理」「テストプレイ＆デバッガ」**などの機能は専用ダイアログ・ウィンドウとして提供します。

---

## 3. 画面ごとの設計詳細・機能一覧

### 3.1. マップエディタ画面（Main Map Editor）
ツクールの起動時画面に相当します。直感的なマッピング体験を提供します。

*   **画面レイアウト**:
    *   **上部ツールバー**: ペン、四角形、塗りつぶし、消しゴム、ズーム、レイヤー切替（マップ / イベント）。
    *   **左側パネル（タイルパレット）**: `tilesets.json` から読み込んだタイルセット（A〜E相当）の一覧。オートタイル対応。
    *   **中央〜右側（キャンバス）**: グリッド表示つきのマップ描画エリア。
*   **レイヤー構成（ツクールライク）**:
    *   下層（Ground） / 中層（Deco-Lower） / 上層（Deco-Upper） / 特殊（Shadow, Collision, Region）
*   **マッププロパティ（設定ダイアログ）**:
    *   **BGM/BGS**: マップに入った瞬間に自動再生される音楽・環境音。
    *   **ランダムエンカウント**: 出現する敵グループ（トループ）と平均出現歩数の設定。
    *   **タイルセット指定**: そのマップで使用するタイルセット（フィールド、外観、内装など）の紐付け。
*   **イベント配置**:
    *   「イベントレイヤー」選択時にキャンバスをダブルクリックで「マップイベントエディタ（ダイアログ）」を開く。

### 3.2. データベース画面（GameDb Editor）
ツクールの心臓部である「データベース」ダイアログを、エディタの一機能（独立したタブまたは全画面モーダル）として実装します。

*   **画面レイアウト**:
    *   **上部タブ**: アクター / 職業（※1） / スキル / アイテム / 武器 / 防具 / 敵キャラ / トループ / ステート / アニメーション / タイルセット / コモンイベント（※2） / システム（※3）
    *   **左側パネル**: 選択中のタブに応じたデータ一覧（例：001 勇者、002 戦士）。下部に追加/削除ボタン。
    *   **右側パネル**: 選択中データのプロパティ編集フォーム。
*   **タブごとの主要機能**:
    *   **アクター**: 名前、初期レベル、初期装備（武器/防具）、特徴、グラフィック設定、成長曲線（グラフUI）。
    *   **敵キャラ**: 名前、画像設定、最大HP/MP等のステータス、ドロップアイテム、行動パターン定義。
    *   **トループ（敵グループ）**: 敵キャラの画面配置（フロントビュー/サイドビュープレビュー）、バトルイベント設定。
    *   **スキル**: 名前、消費MP、範囲（単体/全体）、ダメージ計算式（JavaScript式を入力）、効果、使用アニメーション。
    *   **アイテム / 武器 / 防具**: アイテムは種類（回復/大事なもの）や効果量。武器・防具はステータス変化と装備スロット種別（右手、盾、頭、体など）を定義。
    *   **アニメーション**: バトル中やマップで再生するエフェクト（Spineやパーティクル）の管理。
    *   **タイルセット**: 各タイル画像に対する「通行判定（○・×・☆）」「四方向通行」「カウンター」「茂み」などの当たり判定設定。
    *   **システム**: ゲームタイトル、初期パーティ構成、プレイヤーの初期位置、システムSE、通貨単位などの全体設定。
    *   *(※1) 現状の kaedevn では職業のステータス成長はアクターに統合されていますが、UI上はタブを分けるか検討。*
    *   *(※2) kaedevn の `templates` 機能を「コモンイベント」としてここに配置。*
    *   *(※3) ツクールでいう「システム」「用語」タブの役割を統合。*

### 3.3. シナリオ（イベント）エディタ画面
ツクールの「イベント実行内容」を構築する画面です。kaedevn のブロックエディタを、ツクールのコマンド入力ダイアログに似たカテゴリ分けで再設計します。

*   **画面レイアウト**:
    *   **左側（コマンドパレット）**: ブロック一覧をツクール風にカテゴリ分け（タブ化）して配置。
    *   **右側（実行内容）**: 追加されたブロックが上から下へリスト状に並ぶ。
*   **コマンドパレットのカテゴリ分け（ツクール風）**:
    *   **[メッセージ]**: 文章の表示 (`text`)、選択肢の表示 (`choice`)、スクロール文章 (`scroll_text`)
    *   **[ゲーム進行]**: スイッチ/変数の操作 (`set_var`)、条件分岐 (`if`)、コモンイベント呼出 (`call`)
    *   **[パーティ・アクター]**: 所持金・アイテム・武器・防具の増減、パーティメンバーの入れ替え、全回復。
    *   **[フロー制御・移動]**: 場所移動（別マップへの遷移）、移動ルートの設定（キャラクターをN歩動かす等の制御）、ラベルジャンプ (`jump` / `map_jump`)
    *   **[オーディオ・画面]**: BGM/SE演奏、画面のフェード/シェイク/フラッシュ (`effect`)、色調変更 (`screen_filter`)、ピクチャ (`overlay`)、背景 (`bg`)、キャラ (`ch`)
    *   **[システム・バトル]**: バトルの処理 (`battle`)、ショップの処理（売買画面の呼出）、セーブ画面を開く。
*   **編集体験**:
    *   ダブルクリックでプロパティ編集ダイアログが開き、引数（テキスト内容や変数設定）を入力する。

### 3.4. マップイベント設定ダイアログ（Map Event Editor）
マップエディタから呼び出される、マップ上のNPCや宝箱を設定する専用ダイアログです。

*   **機能とレイアウト**:
    *   **左側**: イベントページ一覧（[1] [2] [3]... とタブまたはリストでページ切替）。新規ページ作成ボタン。
    *   **上部（出現条件）**: スイッチ（グローバル変数）、アイテム所持、およびイベント固有の**セルフスイッチ（A, B, C, D）**によるページ有効化条件。
    *   **左下（画像・オプション・移動・トリガー）**:
        *   **画像**: キャラクター画像やオブジェクト画像の設定。
        *   **オプション**: 歩行アニメ、足踏みアニメ、向き固定、すり抜け、最前面に表示。
        *   **自律移動**: タイプ（固定、ランダム、近づく、カスタム）と、その速度・頻度。
        *   **トリガー**: 決定ボタン、プレイヤーから接触、**イベントから接触**（敵シンボル用）、自動実行、**並列処理**（バックグラウンド監視用）。
    *   **右側広域（実行内容）**: 3.3の「シナリオ（イベント）エディタ」と同じブロックリストコンポーネントを埋め込む。

### 3.5. 素材管理ダイアログ（Resource Manager）
ツクールの「素材管理」に相当し、プロジェクト内で使用する画像や音声を一元管理します。

*   **機能とレイアウト**:
    *   **左側（カテゴリ）**: キャラクター、顔グラフィック、マップチップ、BGM、SE、UI素材などのフォルダツリー。
    *   **右側（プレビュー）**: 選択した素材のプレビュー表示（画像の表示、音声の再生）。
    *   **下部（アクション）**: インポート（アップロード）、エクスポート、削除。

### 3.6. 変数・スイッチ管理ダイアログ
イベント構築時に不可欠な、変数やスイッチの名前付けと一覧管理を行う専用ダイアログです。

*   **機能とレイアウト**:
    *   シナリオエディタ内で「条件分岐」や「変数の操作」コマンドのプロパティ設定から呼び出されます。
    *   リスト状に番号（0001〜）と名前が表示され、クリックして名前を一括編集・管理できます。

### 3.7. テストプレイ＆デバッガUI
エディタからシームレスにゲームを起動し、動作確認を行うための機能群です。

*   **機能とレイアウト**:
    *   **プレビューウィンドウ**: エディタ上に重ねて、あるいは別タブでゲーム画面（1280x720基準）を起動。
    *   **デバッグツール**: テストプレイ中にF9キーなどで呼び出せる、現在の変数やスイッチの値をリアルタイムに確認・変更できるUI。

## 4. 類似性向上のための設計方針（デザイン原則）

現状の kaedevn を本家ツクールに似せ、学習コストを下げるためのガイドラインです。

1.  **用語の統一（ローカライズ）**:
    内部データ構造が `GameDb` や `templates` であっても、エディタのUI上は**「データベース」「コモンイベント」「スイッチ」「変数」「トループ」**といったツクールユーザーに馴染み深い語彙を徹底します。
2.  **リストビューとプロパティの分離**:
    データベース画面などでは、「左にリスト、右に詳細設定」というツクール伝統の2ペインレイアウトを厳守します。
3.  **ダブルクリックでの編集**:
    イベントの実行内容（ブロックリスト）は、インライン編集だけでなく、項目をダブルクリックすると詳細設定ダイアログが開く挙動をサポートし、ツクールの操作感に近づけます。
4.  **計算式・変数の互換的表現**:
    ダメージ計算式（例: `a.atk * 4 - b.def * 2`）など、ツクールの記述方式をそのまま使えるようにUIのヒントテキストを整備します。
5.  **エディタ操作の踏襲（暗黙のUX）**:
    マップキャンバス上で配置済みタイルを右クリックして選択状態にする「スポイト機能」や、マップ上のイベントを `Ctrl+C` `Ctrl+V` で複製できる機能など、ツクール作者の手に馴染んだ操作ショートカットをサポートします。

## 5. まとめ
この再設計により、kaedevn のエディタは「モダンなWebベースでありながら、UIの見た目と操作感は完全にRPGツクール」という状態を実現します。作者は新たなツール操作を覚えることなく、ブラウザ上で即座にゲーム制作を開始できるようになります。
