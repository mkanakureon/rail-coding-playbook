## 3. 主要な指摘事項と改善案

### 3.1. ネスト（入れ子）構造への挿入・編集ロジックの欠落（最重要）
現在の `ScenarioEditor.tsx` における `handleInsertCommand` 等の配列操作は、トップレベルの `commands` 配列のみを対象としています。
*   **課題**: 「条件分岐の中（`thenCommands` 等）」や「選択肢の中」に新しいコマンドを追加したり、中のコマンドを編集・削除・移動したりすることができません。
*   **改善案**: コマンドの特定を単なる `index: number` ではなく、ツリー構造上のパス（例: `[0, 'thenCommands', 1]`）で管理し、再帰的に配列を更新するヘルパー関数を実装してください。これはツクール体験において**絶対に妥協できない要件**です。

### 3.2. データベース参照のドロップダウン化
場所移動のマップ指定や、バトルのトループ指定が現在「テキストの手入力（slug や ID）」になっています。
*   **課題**: 作者が内部 ID を暗記しなければならず、制作効率が著しく低下します。また、存在しない ID を入力するバグの温床になります。
*   **改善案**: `useTsukuruStore` 等から `maps` や `gameDb.troops` の一覧を取得し、`select` タグを用いたドロップダウン選択式に変更してください。

### 3.3. コマンドプレビュー機能の追加
設計書（02）で提案された、kaedevn 独自の強みである「文章の表示のリアルタイムプレビュー」が実装されていません。
*   **改善案**: `ShowTextEditor` コンポーネント内に、実際のゲーム画面と同じアスペクト比・フォントサイズのメッセージウィンドウ領域（プレビュー枠）を設け、入力中のテキストがどのように表示されるか（改行位置など）をリアルタイムで確認できるようにしてください。
