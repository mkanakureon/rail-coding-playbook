# ツクール型エディタ 実装レビュー計画書

> **作成日**: 2026-03-18
> **担当**: Gemini CLI
> **対象ドキュメント**: `docs/09_reports/2026/03/18/03-TSUKURU_IMPLEMENTATION_PLAN.md`
> **目的**: 今後提出されるコミット群が、実装計画の各Phaseの要件を満たしているか、また設計書（画面レイアウト・UX要件）から逸脱していないかを体系的にレビューするためのチェックリスト。

## レビューの進め方
開発者（または実装エージェント）からコミットが上がってきた際、現在の進行Phaseを特定し、本計画書の該当Phaseのチェック項目に基づいてコミット履歴・差分をレビューします。各Phaseの「完了条件」を満たすまでは次のPhaseへの移行をブロックまたは警告します。

---

## Phase 1: シェル + マップエディタ接続 レビュー項目

**目的**: ツクール風の「ガワ（シェル）」が完成し、マップ描画の基本操作ができること。

### 1. DB/API連携
- [ ] `prisma/schema.prisma` に `projects.type` カラム（デフォルト "novel"）が追加されているか。
- [ ] GameDb用、変数・スイッチ用のJSONカラムが追加され、マイグレーションが作成されているか。
- [ ] `routes/gamedb.ts` が新規作成され、CRUDエンドポイントが実装されているか。

### 2. UIシェル・状態管理
- [ ] `useTsukuruStore.ts` が作成され、プロジェクト状態が管理されているか。
- [ ] `TsukuruEditorPage.tsx` が 3カラム（左:パレット/ツリー、中央:キャンバス、右:プロパティ）のレイアウトになっているか。
- [ ] メニューバー、ツールバー、ステータスバーのコンポーネントが存在し、見た目が再現されているか。
- [ ] ツクール風のダークテーマ用 CSSトークン が追加されているか。

### 3. マップエディタ統合
- [ ] 左パネルに `MapTree` コンポーネントが実装されているか。
- [ ] キャンバス上で「右クリックスポイト機能」が実装されているか。
- [ ] イベントモード時にマップレイヤーが半透明（alpha=0.5等）になる処理が入っているか。
- [ ] `LayerBar` で下層/中層/上層/影/R の切り替えUIが実装されているか。

---

## Phase 2: データベース拡張 レビュー項目

**目的**: ツクールの心臓部であるDBが「13タブ」構成で実装され、データ定義が完結すること。

### 1. 型定義
- [ ] `GameDb.ts` に新規タブ用（武器、防具、タイルセット等）の型が追加されているか。
- [ ] `EventCommand.ts` が新規作成され、コマンド型の定義が存在するか。

### 2. DBエディタ (GameDbEditor)
- [ ] 既存の `GameDbPanel.tsx` が拡張され、**13個のタブ**が切り替え可能になっているか。
- [ ] 新規タブコンポーネント（`ClassTab`, `WeaponTab`, `ArmorTab`, `StateTab`, `SystemTab`, `CommonEventTab`）が実装されているか。
- [ ] **（最重要）** `TilesetPassEditor.tsx` が実装され、タイル画像上でクリックによる ○/×/☆ の通行判定切り替えができるUIになっているか。
- [ ] `TroopPlacementEditor.tsx` で、敵のドラッグ配置UIが実装されているか。

### 3. ダイアログ群
- [ ] `VarSwitchManager.tsx` (変数・スイッチ管理ダイアログ) が実装されているか。
- [ ] `ResourceManager.tsx` (素材管理ダイアログ) が実装されているか。

## Phase 3: イベントコマンドシステム レビュー項目

**目的**: イベント内容を「ブロック（コマンド）」で構築できること。

### 1. コアコンポーネント
- [ ] `ScenarioEditor.tsx` (コマンドパレット + コマンドリストの2カラムUI) が実装されているか。
- [ ] コマンドリストが「◆ コマンド名: 引数」というツクール特有のテキスト表現（インデント付き）で描画されているか。

### 2. MVPコマンド (最低6種)
以下のコマンド編集フォーム（ダイアログ内コンテンツ）が実装されているか。
- [ ] `show_text` (文章の表示) ※プレビュー機能付きか確認
- [ ] `show_choices` (選択肢の表示)
- [ ] `control_switches` (スイッチ操作)
- [ ] `conditional_branch` (条件分岐)
- [ ] `transfer_player` (場所移動)
- [ ] `battle_processing` (バトルの処理)

### 3. マップイベント設定 (MapEventDialog)
- [ ] ダイアログ本体が 80vw × 85vh 程度の広さで作られているか。
- [ ] 左側に「出現条件（セルフスイッチ含む）」「オプション」「自律移動」「トリガー」、右側に `ScenarioEditor` が埋め込まれているか。
- [ ] 「1, 2, 3...」というページ切り替え（新規/コピー/削除）のUIが存在するか。

### 4. マッププロパティダイアログ
- [ ] マップの基本設定に加え、「BGM/BGS設定」と「ランダムエンカウント（敵グループ＋重み）」のUIが実装されているか。

## Phase 4: ランタイム統合 レビュー項目

**目的**: テストプレイボタンを押して、実際にゲームが動くこと。

### 1. インタプリタと状態管理
- [ ] `EventInterpreter.ts` が実装され、コマンド配列を逐次実行するロジックが存在するか。
- [ ] `GameState.ts` が実装され、変数やスイッチの状態を保持・変更できる設計になっているか。

### 2. バトルとエンカウント
- [ ] `EncounterManager.ts` 等による、歩数ベースのランダムエンカウントロジックが組み込まれているか。
- [ ] 既存のBattleSceneと連携し、バトルの結果（経験値・Goldなど）がGameStateに反映されるか。

### 3. システムUIとテストプレイ
- [ ] F9キーで起動する `DebuggerPanel.tsx` が実装され、変数のリアルタイム書き換えUIが存在するか。
- [ ] テストプレイ画面 (`TestPlayOverlay.tsx`) が起動し、マップ上の歩行やイベントトリガー（決定、接触など）が発火するか。
- [ ] `Ctrl+Shift+R` (現在地からテストプレイ) のショートカットが実装されているか。

---

## Phase 5: 仕上げ + デプロイ レビュー項目

**目的**: エディタのUXが磨かれ、本番デプロイが完了すること。

### 1. UXの磨き込み
- [ ] マップ上のイベントを右クリックした際のコンテキストメニューが存在するか。
- [ ] `Ctrl+C` / `Ctrl+V` によるマップタイル範囲コピー、およびイベントのコピーペーストが実装されているか。
- [ ] 描画ツール（四角形・楕円・塗りつぶし）のロジックが正常に動作するか。

### 2. デプロイとテスト
- [ ] E2Eテスト（ツクールエディタの基本フロー）が追加されているか。
- [ ] ビルドエラーがなく、`npm run typecheck` が通っているか。
