# ツクール型エディタ開発における法的リスクと対策ガイドライン

> **作成日**: 2026-03-18
> **担当**: Gemini CLI
> **目的**: 「RPGツクール（RPG Maker）」のUIや操作感を再現するエディタ（kaedevn）を開発・公開するにあたり、著作権・商標権などの法的トラブル（株式会社Gotcha Gotcha Games等からの警告や提訴）を回避するための基本方針と最終確認事項をまとめる。

---

## 1. 基本方針（設計書は維持、リリース時に調整）
現在の画面設計書（`01-TSUKURU_SCREEN_REDESIGN_PLAN.md` 等）は、開発陣が「どのようなUXを目指すか」を共有するための**内部設計図**です。内部的に「ツクール風」「MZのUIに寄せる」といった表現を使用し、その操作性を目指して開発を進めること自体には法的問題はありません。

したがって、**設計書は修正せず、現在のマスタープランのまま開発を進行します。**
ただし、エディタが完成し「一般公開（リリース）」する直前に、本ガイドラインに基づいたUI/UXの最終見直し（クリーンアップ）を必ず実施してください。

---

## 2. リリース前の最終確認事項（レッドライン）

以下の項目は、リリース版のソフトウェア内に**絶対に含めてはならない**要素です。これらが発覚した場合、著作権侵害や不正競争防止法違反のリスクが極めて高くなります。

### 🚫 2.1. 商標・名称の無断使用
*   **NG**: アプリのUI上、プロモーション素材、またはソースコードの公開リポジトリ（README等）において、「RPGツクール」「RPG Maker」「ツクール風」といった商標を使用すること。
*   **対策**: 「JRPGスタイル」「クラシックRPGエディタ」「日本のクリエイターに馴染み深いUI」などの抽象的な表現にとどめる。

### 🚫 2.2. 公式素材（アセット）の無断流用
*   **NG**: ツクールシリーズ（MV, MZなど）のUI画像（ウィンドウ枠、ボタン、アイコン群）、公式RTP（同梱のキャラクター画像、マップチップ、音楽、効果音）を `kaedevn` 内でそのまま、あるいは加工して使用すること。
*   **対策**:
    *   UIアイコンは `lucide-react` や `FontAwesome` などの商用利用可能なフリーライブラリを使用する。
    *   エディタのデフォルト素材（サンプルゲーム用）は、ぴぽや様などのフリー素材、または完全オリジナルのAI生成画像などに差し替える。

### 🚫 2.3. 公式ソースコードの複製
*   **NG**: ツクールのコアスクリプト（`rpg_core.js`, `rmmz_managers.js` 等）からアルゴリズムや実装コードをそのままコピー＆ペーストすること。
*   **対策**: kaedevn は React / PixiJS / Hono をベースとした完全新規のアーキテクチャであるため、このリスクは低い。ただし、計算式のパーサーなどを作成する際も、ゼロから独自実装（クリーンルーム設計）を徹底する。

## 3. グレーゾーン（アイデアの模倣）への配慮

UIの配置（左にマップツリー、右にプロパティ）や、「イベントコマンド（文章の表示、場所移動など）」という機能の概念自体は、著作権（アイデア保護）の対象外とされるのが一般的です。しかし、**「あまりにも見た目がそっくり（デッドコピー）」**だとトラブルの元になります。

完成後の見直しフェーズにおいて、以下の点で「kaedevn 独自のオリジナリティ」を付加し、デッドコピーではないことを主張できるようにします。

### 🛡️ 3.1. 独自のUIデザイン（皮の被せ替え）
*   レイアウト構造（2ペイン等）は踏襲しつつ、Tailwind CSS を用いて独自のカラースキーム（kaedevnのブランドカラー）、ボタンの丸み、フォントサイズなどを適用し、**「パッと見の印象がツクールMZとは違うモダンなWebアプリ」**に見えるようデザインを調整する。

### 🛡️ 3.2. 独自機能の強調
*   kaedevn 特有の機能である「UIレイアウトエディタ」「コマンド編集中プレビュー」「インスペクタ（右パネル）」をUI上で目立たせ、既存製品にはない独自の価値（進化版）であることを明確にする。
