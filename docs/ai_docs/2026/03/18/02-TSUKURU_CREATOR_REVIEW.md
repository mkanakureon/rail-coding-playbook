# ツクール作者視点での画面再設計プラン・レビュー報告

> **作成日**: 2026-03-18
> **対象ドキュメント**: `01-TSUKURU_SCREEN_REDESIGN_PLAN.md`
> **レビュー観点**: 「本家RPGツクール（MV/MZ等）の作者が、違和感なくゲーム1本を作り切れるか？」

## 1. 全体評価
現在の画面構成サマリー（マップエディタ、シナリオエディタ、データベース、UIレイアウトエディタ＋各種ダイアログ）は、ツクールの全体アーキテクチャを見事に捉えています。
特に「データベース」という概念を中心に据え、「リストビューとプロパティの分離（2ペイン）」を厳守する方針は、ツクールユーザーにとって学習コストを劇的に下げる素晴らしいアプローチです。

しかし、「RPGを実際に一本完成させる」という泥臭いワークフローを想定した場合、**ツクール作者の身体に染み付いている必須機能・概念**がいくつか設計から漏れています。これらが欠けていると「あれ？〇〇はどうやるの？」と制作がストップする原因となります。

---

## 2. 重大な不足機能と改善提案（Must-Have）

### 2.1. データベース内の不足タブ
ツクールのデータベースには、世界観を構築するための根幹設定がいくつか存在します。現在の案に追加すべきタブは以下の3つです。

1.  **「タイルセット」タブ（超重要）**
    *   **課題**: 現在の `3.1 マップエディタ` では `tilesets.json` を読み込むとありますが、タイルごとの「通行判定（○・×・☆）」「四方向通行」「カウンター判定」「茂み判定」を設定する場所が定義されていません。
    *   **改善案**: データベースに「タイルセット」タブを追加。作者はここで自作チップの当たり判定を設定してから、マップを描き始めます。
2.  **「アニメーション」タブ**
    *   **課題**: バトル中やマップ上で再生する「斬撃」「魔法」などのエフェクトを設定・管理する場所がありません。
    *   **改善案**: データベースに「アニメーション」タブを追加。kaedevn の場合、Spineデータやパーティクル設定をここで一元管理し、スキルやイベントから呼び出せるようにします。
3.  **「武器」「防具」の独立または明確な定義**
    *   **課題**: 3.2 の「アイテム」の中に武器・防具が含まれていますが、ツクールではこれらは独立したタブであり、「装備タイプ（右手、盾、頭、体など）」の概念を持ちます。
    *   **改善案**: 「システム」タブでの装備スロット定義と連携し、「武器」「防具」タブを分けるか、アイテム内でカテゴリとして厳密に区分できるUIが必要です。

### 2.2. マップイベントの必須オプションと概念
3.4 のマップイベント設定ダイアログにおいて、以下の概念が抜けていると「宝箱」や「一度話した人」を作るのに非常に苦労します。

1.  **セルフスイッチ（A, B, C, D）**
    *   グローバルなスイッチとは別に、そのイベント自身だけが持つ独立したスイッチ。宝箱の「開・閉」状態の管理などに不可欠です。出現条件として必須要件です。
2.  **イベントのオプション項目**
    *   「歩行アニメ」「足踏みアニメ」「向き固定」「すり抜け」「最前面に表示」のチェックボックス。これがないと、静止したNPCや空飛ぶ鳥などを表現できません。
3.  **自律移動の詳細設定**
    *   「ランダム」「近づく」「カスタム」といったタイプと、その「速度」「頻度」の設定項目。

### 2.3. イベントコマンドの欠落
1.  **「移動ルートの設定」コマンド**
    *   3.3 のシナリオエディタにおいて、キャラクターを「上に3歩動かしてジャンプさせる」といった制御を行うコマンドカテゴリが明記されていません。カットシーン制作において最も多用されるコマンドの一つです。

## 3. エディタUX（操作感）に関する暗黙の期待

機能面だけでなく、ツクール作者は「特定のマウス・キーボード操作」を無意識に期待しています。

*   **右クリックでのスポイト機能**:
    マップエディタのキャンバス上で、配置済みのタイルを右クリックすると、そのタイル（または複数タイルの塊）が自動的に選択状態になる機能。マッピングの作業効率に直結するため、ツクールライクを謳うなら実装必須です。
*   **イベントのコピペ**:
    マップ上でイベントを `Ctrl+C` `Ctrl+V` で複製できること。

---

## 4. 結論
現在の再設計プランは基盤として非常に優れていますが、本レビューで指摘した **「タイルセットの通行判定」「セルフスイッチ」「移動ルートの設定」「アニメーション管理」** の4点は、ツクール体験を再現する上で妥協できないコア機能です。
これらの要素を `01-TSUKURU_SCREEN_REDESIGN_PLAN.md` の設計に組み込むことで、名実ともに「Webで動くモダンなツクール」が完成すると確信します。
