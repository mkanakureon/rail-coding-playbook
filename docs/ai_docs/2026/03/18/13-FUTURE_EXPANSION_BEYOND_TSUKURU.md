# 本家ツクールを超越する：kaedevn 独自の次世代エディタ機能・将来展望

> **作成日**: 2026-03-18
> **担当**: Gemini CLI
> **目的**: 現在進行中の「ツクール互換UI（Phase 1-5）」の完成後を見据え、Webネイティブ・Reactベースである kaedevn ならではの「本家ツクールを圧倒する利便性」を言語化し、プロダクトの長期ビジョンとする。

---

## 1. コンセプト：学習コスト 0 から、圧倒的な自由度へ
現在の開発フェーズは、ツクール作者が迷わず移行できるように「慣れ親しんだ間取り（互換UI）」を構築することに主眼を置いています。しかし、土台が React / PixiJS / Hono というモダンな技術スタックであるため、この基盤を活かすことで、本家（デスクトップアプリ）では不可能な異次元のUXを提供できます。

---

## 2. 実現可能な次世代機能（イノベーション項目）

### 2.1. ビジュアルスクリプト（ノードエディタ）モード
ツクール最大の弱点である「深いネスト（インデント）によるイベントの可読性低下」を解決します。
*   **内容**: コマンドリスト（◆表記）と、Unreal Engine の Blueprint のような「ノード接続形式」をワンクリックで切り替え。
*   **利点**: 複雑な条件分岐や並列処理の全体像を、視覚的なフローチャートとして把握可能。

### 2.2. リアルタイム・マルチユーザー共同編集（エディタの Figma 化）
Webベースの強みを最大限に活かした、制作スタイルの革命です。
*   **内容**: 同一プロジェクトに対し、複数人が同時にログインして編集。誰がどのマップやイベントを編集中かアイコンで表示。
*   **利点**: チーム制作における「ファイルの競合」をゼロにし、素材担当・シナリオ担当・マップ担当が並行して作業を完結。

### 2.3. クロスデバイス制作（寝転がり・移動中マッピング）
PCへの拘束からの解放。
*   **内容**: タブレットやスマホに最適化されたタッチUIを提供。指でなぞってマップを描き、ペンシルで敵ステータスを調整。
*   **利点**: 通勤電車やリビングでのリラックスタイムなど、「思いついた瞬間にエディタを開く」という新しい制作習慣の創出。

### 2.4. WYSIWYG UIレイアウトエディタ（完全なデザイン自由度）
コード不要の画面カスタマイズ。
*   **内容**: プレイ画面のパーツ（メッセージ枠、HPゲージ、コマンドボタン等）をマウスのドラッグ＆ドロップで自由配置・装飾。
*   **利点**: ツクールではJavaScript（プラグイン）の知識が必要だった「画面デザインの差別化」を、誰でも視覚的に行える。

### 2.5. AI 制作エージェントの統合（Generative Game Creation）
Web/APIネイティブならではの強み。
*   **内容**: エディタ内に LLM (Gemini 等) を直結。「このマップに合う村人を3人置いて、魔王の噂をそれぞれ違う口調で話させて」という指示で、イベントコマンド（◆文章の表示等）を自動生成。
*   **利点**: 制作の「作業量」を劇的に減らし、作者が「企画・演出」というクリエイティブな活動に集中できる環境を提供。

---

## 3. 実現に向けた戦略的ロードマップ

1.  **Phase 1-5 (現在): 互換性の確立**
    *   ツクールユーザーの「安心感」と「移行のしやすさ」を最優先し、Web版ツクールとしての地位を確立する。
2.  **Phase 6 (次ステップ): UIレイアウトの解放**
    *   まずは「UIレイアウトエディタ」を公開し、kaedevn で作れば「自分だけの独自画面」が作れることをアピールする。
3.  **Phase 7以降: Web・AIパワーの全開**
    *   共同編集やAIアシスタントを順次投入し、「本家ツクールに戻るのが苦痛になるほど便利な体験」へ昇華させる。

---

## 4. 結論
kaedevn の最終目標は、ツクールの「真似」ではありません。ツクールの「使いやすさ」を入り口とし、最新のWeb技術によって**「RPG制作の定義を塗り替えること」**です。現在の Phase 1-3 の高い実装品質は、この壮大な将来像を実現するための、極めて強固な第一歩となっています。
