# WYSIWYG レイアウトエディタ E2Eテスト計画書

> **作成日**: 2026-03-18
> **作成者**: Gemini CLI
> **実行担当**: Claude Code (Opus 4.6)
> **対象**: WYSIWYG UIレイアウトエディタ (`apps/editor`)
> **テスト手法**: Playwright を用いた E2E (End-to-End) テスト

## 1. 目的と背景
単体テスト（Unit Test）において、レイアウトストア(`useTsukuruStore`)のロジックやヘルパー関数の動作が保証された（計75件パス）。
本計画書は、実際のブラウザ環境でのユーザー操作（DOMイベント、ドラッグ＆ドロップ、テストプレイ画面への統合）が正しく機能するかを統合的に検証する **E2Eテスト（Playwright）** の実装と実施を Claude Code に委任するための指示書である。

## 2. テスト環境・前提
- **テストフレームワーク**: Playwright (`@playwright/test`)
- **対象ファイル**: `e2e/layout-editor.spec.ts` (新規作成)
- **前提条件**:
  - `npm run dev` などでエディタが起動可能であること。
  - テスト開始時にエディタURL（例: `http://localhost:5176/projects/editor/...`）へナビゲートできること。
  - プロジェクトデータは空の新規プロジェクト、またはCLIで自動生成したサンプルプロジェクトを利用する。

## 3. テストケース一覧

### 📝 1. 基本画面遷移と表示確認
- **操作**: エディタを開き、左側メニューまたは上部タブから「レイアウト」タブを選択する。
- **期待結果**:
  - 3カラムシェル（左:ツリー、中央:キャンバス、右:プロパティ）が正しく表示される。
  - ツールバーに解像度、プリセットのドロップダウンが表示されている。

### 📝 2. プリセットと解像度の切り替え
- **操作**: プリセットドロップダウンから「スマホ縦持ち (smartphone-portrait)」を選択する。
- **期待結果**:
  - 解像度が「720x1280 (9:16)」に自動で切り替わる。
  - 「⚠️ Web/Android 専用」の警告バッジが表示される。
  - キャンバス内のUIパーツ（メッセージウィンドウや選択肢など）が縦長のレイアウトに再配置される。

### 📝 3. ドラッグ＆リサイズと Undo/Redo の動作
- **操作**:
  1. キャンバス上の「メッセージウィンドウ」をクリックして選択する。
  2. マウスドラッグで要素を別の座標へ移動させる。
  3. 右下のハンドルをドラッグしてサイズ（Width/Height）を変更する。
  4. キーボードで `Ctrl+Z` (または `Cmd+Z`) を入力する。
- **期待結果**:
  - ドラッグ・リサイズ操作に合わせて、右側のプロパティパネルの「座標・サイズ」の数値がリアルタイム（またはドロップ時）に更新される。
  - `Ctrl+Z` で移動・サイズ変更前の元の状態（座標・サイズ）に戻る。

### 📝 4. プロパティ編集とモック描画の連携
- **操作**:
  1. 左パネルのツリーから「screen-overlay」を選択する。
  2. 右パネルの表示セクションで「不透明度」スライダーを `0.5` に変更する。
  3. スタイルセクションで背景色を `rgba(255, 0, 0, 0.5)`（赤色半透明）に変更する。
- **期待結果**:
  - キャンバス上で、画面全体を覆うモック要素が赤色の半透明で描画される。

### 📝 5. テストプレイ (ランタイム) への反映
- **操作**:
  1. レイアウトエディタ上で「メッセージウィンドウ」を画面上部（Y座標: 10 など）に移動させる。
  2. テストプレイ実行ボタン（または `Ctrl+Shift+R`）を押下し、テストプレイ用オーバーレイ (`TestPlayOverlay`) を起動する。
  3. イベント（テキスト表示）を発火させる。
- **期待結果**:
  - テストプレイ画面に表示されるメッセージウィンドウが、エディタで設定した通り画面上部（絶対座標）に配置されている。
  - 指定したスタイル（背景色や文字サイズ、枠線の有無）が正しく反映されている。

### 📝 6. JSONインポート / エクスポート (オプション)
- **操作**:
  - 「保存 (エクスポート)」ボタンを押下する。
  - 保存されたJSONファイルを「読込 (インポート)」ボタンから読み込ませる。
- **期待結果**:
  - エクスポートされたJSONファイルが `PlayLayout` 型（`version: 1`）の構造を持っている。
  - インポート後、エディタ上のレイアウトがJSONファイルの状態に復元される。

## 4. Claude Code への実行指示

Claude Code は以下のステップで作業を進めてください。

1. **テストファイル作成**: `e2e/layout-editor.spec.ts` を作成し、上記ケース1〜5を最低限実装する。（ケース6のファイルI/Oテストは Playwright の構成に依存するため任意）
2. **ローカル実行とデバッグ**: 実際にPlaywrightテストを実行 (`npm run test:e2e` 等) し、テストが安定して通過するまで修正を行う。
3. **報告**: テストの実装内容と実行結果（Pass/Fail）、および修正が必要だった点があればレポートとして `docs/09_reports/` に提出する。
