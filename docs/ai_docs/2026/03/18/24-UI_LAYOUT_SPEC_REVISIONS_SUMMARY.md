# UIレイアウトエディタ要件：L3レビューに基づく修正・追記点サマリー

> **作成日**: 2026-03-18
> **担当**: Gemini CLI
> **対象**: `23-CLAUDE_CODE_STRATEGIC_DIRECTIVES.md` の更新内容

Claude からの L3 レビュー（`ui-layout-editor-review.md`）による差し戻し事項に対し、要件定義を以下の通り具体化・修正しました。

---

## 1. `options` プロパティの型定義（型安全性の確保）
「スタイリング自由度の向上」要件に対し、`Record<string, unknown>` だった箇所を具体的に定義しました。

*   **追加仕様**: `UiElementOptions` 型の定義
    *   `backgroundOpacity`: 背景の不透明度 (0.0〜1.0)
    *   `borderVisible`: 枠線（ウィンドウスキン）の表示有無
    *   `borderRadius`: 角の丸み (px)
    *   `backgroundColor`: 背景色 (Hexコード)
    *   `fontSize`: フォントサイズ (px)

---

## 2. プラットフォーム別「縦持ち (9:16)」対応状況の明示
「スマホ縦持ちサポート」要件に対し、プレイヤー側の制約を以下の通り定義しました。

| プラットフォーム | 対応状況 | 備考・制約 |
| :--- | :--- | :--- |
| **PixiJS (Web / Android)** | **対応可能** | Viewport スケーリングにより 9:16 レンダリングを行う。 |
| **SDL2 (Switch / ネイティブ)** | **非対応** | Switch 側の画面制約により横長固定。 |

*   **エディタ側の追加要件**: 縦持ちレイアウト選択時に「このレイアウトは Web/Android 専用になります」という警告ダイアログまたはバッジを表示すること。

---

## 3. `zIndex` のマスターデータ責務の明確化
「重なり順の制御」要件に対し、データの正（マスター）と各レイヤーの責務を定義しました。

*   **定義**: `UiLayoutElement.zIndex` は、実行時の **PixiJS (WebGL) レンダリングにおける重なり順のマスターデータ** である。
*   **各レイヤーの振る舞い**:
    *   **エディタ（React）**: `PlayLayout` データの `zIndex` 値をそのまま CSS の `z-index` に反映し、PixiJS での結果を WYSIWYG にシミュレーションする。
    *   **ランタイム（PixiJS）**: データの `zIndex` 値を `pixi.displayObject.zIndex` にセットして実描画を行う。

---

## 4. 結論
以上の修正により、抽象的だった「戦略的要件」が「技術仕様」へと昇華されました。実装担当はこれらの具体的数値を Props や State の初期値として採用することで、後続のシステム（プレイヤー側）との不整合を完全に回避できます。
