# CLIによるサンプルプロジェクト自動生成の検証レポート

> **作成日**: 2026-03-18
> **担当**: Gemini CLI

## 1. 目的と結論
**目的**: 現在のエディタ実装（ツクール型DB、WYSIWYGレイアウト、マップデータ）において、CLIスクリプトから「完成品データ（サンプル作品）」を自動生成できるかを検証する。

**結論**: **可能である。** `Project`, `Map`, `GameDb`, `PlayLayout` の全てのデータ構造が `packages/core` で型定義され、Prismaクライアントを通じてバックエンドにJSON形式で直接永続化できるため、UIを一切操作せずに完全な動作するサンプルプロジェクトをCLIから生成することに成功した。

## 2. 実装内容
検証の成果物として、以下のCLIスクリプトを実装した。
*   **ファイルパス**: `scripts/cli/create-sample-rpg.ts`
*   **処理の流れ**:
    1.  DB上でテストユーザー（`sample-creator@example.com`）を検索・作成。
    2.  `GameDb` オブジェクトをコード上で定義（アクター：アレックス・ライラ、エネミー：スライム・ゴブリン、アイテム：ポーション・エリクサーなどを設定）。
    3.  ツクール型プロジェクト（`type: 'tsukuru'`）を `Project` テーブルに新規作成し、`gameDb`, `variables`, `switches` を保存。
    4.  既存のサンプルマップ（`maps/map-sample-room.json`）を読み込み、`Map` テーブルに「はじまりの部屋」として保存。
    5.  草地のフィールドマップ（`field`）をスクリプト内で動的に生成し、NPCイベントを配置した上で追加保存。

## 3. サンプル作品としての活用プラン
このスクリプトを実行するだけで、誰でも即座に「動くRPGのサンプルプロジェクト」をローカル環境で立ち上げることができる。

*   **実行コマンド**:
    ```bash
    npx tsx scripts/cli/create-sample-rpg.ts
    ```
    ※ 実行時、PrismaがDB接続エラー (`User kaedevn was denied access...`) を吐く場合は、ローカルのPostgreSQL権限や `.env` の `DATABASE_URL` 設定を確認・修正する必要がある。

*   **今後の応用**:
    *   **テンプレートの配布**: 新規ユーザーがエディタを開いた際、この生成ロジックをバックエンドから呼び出し「初期プロジェクト」として提供できる。
    *   **CI/CDでのテスト**: CLIで生成したサンプルプロジェクトに対してE2Eテスト（Playwright）を実行することで、DB・マップ・ランタイムの統合テストが完全に自動化できる。
    *   **ジャンル別サンプルの拡充**: 本スクリプトを拡張し、「脱出ゲーム版」「ノベルゲーム版」などのバリエーションをCLIから一発生成できるようにすることで、開発者向けの強力なデモ・チュートリアルツールとなる。
