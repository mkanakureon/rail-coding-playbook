# ツクール型エディタ画面設計 最終レビュー：RPG制作シミュレーション報告

> **作成日**: 2026-03-18
> **対象ドキュメント**: `01-TSUKURU_SCREEN_REDESIGN_PLAN.md`, `02-TSUKURU_CREATOR_REVIEW.md`
> **レビュー観点**: 「マップ作成 → キャラ配置 → 敵と遭遇 → 戦闘 → エンディング」というRPG制作の全工程が現在の設計で完遂できるか？

## 1. 結論
前回のレビュー（02）で指摘した「タイルセットの通行判定」や「セルフスイッチ」を追加したとしても、RPGを1本完成させるには**「ゲームシステムを繋ぐ接着剤」となる要素が決定的に不足**しています。現在の設計では、マップは歩けても「ランダムエンカウントで戦闘になる」「宿屋で回復する」「お店でアイテムを買う」「仲間が加わる」といったRPGの王道フローが構築できません。

## 2. 不足している「RPG特有の接着剤」要素

### 2.1. マップのプロパティ設定（裏側の設定）
マップエディタ（3.1）には「絵」を描く機能しか定義されておらず、マップ自体の設定画面（マッププロパティダイアログ等）が欠落しています。
*   **ランダムエンカウント**: 「スライムの群れ」が「平均30歩」で出現する、といったマップ固有のエンカウント設定。
*   **マップBGM/BGS**: マップに入った瞬間に自動再生される音楽（町、ダンジョンなど）や環境音（風、雨など）の指定。
*   **タイルセットの紐付け**: そのマップが「フィールド」「外観」「内装」のどのタイルセットを使用するかの指定。

### 2.2. イベントトリガー（起動条件）の不足
マップイベント（3.4）のトリガーに、RPG特有の以下の条件が抜けています。
*   **イベントから接触（Event Touch）**: 徘徊するシンボルエンカウントの敵がプレイヤーにぶつかった際に戦闘を開始するために必須。
*   **並列処理（Parallel Process）**: プレイヤーの操作を妨げずに裏で動き続ける処理（毒の継続ダメージ、独自の時計や天候システム、カスタムUIの監視など）に絶対に必要。

### 2.3. RPG進行に必須の「イベントコマンド」
シナリオエディタ（3.3）のコマンド一覧に、システムを制御するRPGの根幹コマンドが不足しています。
*   **場所移動**: マップから別のマップ（町から家の中など）へ移動する処理。
*   **所持品の増減**: 所持金（Gold）、アイテム、武器、防具を増やす/減らす処理（宝箱やクエスト報酬）。
*   **パーティの変更**: 仲間が加入・離脱する処理。
*   **全回復**: 宿屋や回復の泉のイベントを作る処理。
*   **ショップの処理**: 購入・売却ができるお店システムを呼び出す処理。
*   **セーブ画面を開く**: セーブポイントを作るための処理。

## 3. 今後の対応方針（仕様の凍結に向けて）
設計情報が分散し、発散することを防ぐため、以下の対応を実施します。

1.  本レビュー（03）および前回のレビュー（02）で洗い出されたすべての不足要素を、**メイン設計書である `01-TSUKURU_SCREEN_REDESIGN_PLAN.md` の1ファイルに統合・追記**します。
2.  これにより `01` ドキュメントを「RPG制作が完遂できるマスタープラン」として完成させ、画面設計レベルの要件定義を**凍結（Freeze）**します。
3.  レビュー報告書（02, 03）は検討履歴（ログ）として残します。
