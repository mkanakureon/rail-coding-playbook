# 公式アセット自動生成パイプライン実装計画（AI Pixel-Art Pipeline）

> **作成日**: 2026-03-18
> **作成者**: Gemini CLI
> **対象エージェント**: Claude Code / Gemini CLI
> **目的**: ユーザーには提供しない「運営専用の公式アセット（歩行グラフィック・マップチップ）生成ツール」を構築し、最高品質のドット絵を全自動で量産する仕組みを実装する。

---

## 1. アプローチの選定：AI画像生成 × スクリプト加工（ハイブリッド方式）

LLMに直接SVGやCanvasコードを書かせる手法（前回の `gen-placeholder-tilesets.mjs`）は、無機質なマップタイルには有効ですが、**キャラクターや複雑な背景を作るには表現力の限界（のっぺりとした図形になる）**があります。

したがって、現状で「最高品質」を担保するための戦略として、**「画像生成AI（DALL-E 3 等）で生成した高解像度画像を、スクリプト（Sharp）で強制的にピクセルアート（ドット絵）の規格に落とし込む」** という強力なパイプラインを提案・実装します。

### パイプラインの流れ
1. **[Prompting]**: 生成AI（DALL-E 3 等）に「白背景、フラットカラー、真正面を向いたキャラクター」などの厳密なプロンプトを投げて 1024x1024 の画像を生成する。
2. **[Background Removal]**: サーバー側のスクリプトで白（または特定の色）背景を透過処理する。
3. **[Pixelation (Sharp)]**: `sharp` ライブラリを使用し、画像を `48x48` （または `144x144`）に `nearest-neighbor`（ニアレストネイバー）アルゴリズムで強制縮小し、アンチエイリアスを排除する。
4. **[Color Quantization]**: ツクール特有の「ドット絵らしさ」を出すため、16色または32色の固定パレット（例: DawnBringer 32）に強制減色する。
5. **[Spriting]**: （キャラの場合）静止画をベースに、上下の微細なピクセル移動（1pxずらし）をコードで適用し、3フレームの「足踏みアニメ」を自動生成してスプライトシート化する。

---

## 2. 実装要件（CLIツール）

`scripts/cli/asset/gen-official-asset.ts` という CLI ツールを新規に作成します。

### 必要な技術スタック
- **OpenAI API (`openai`)**: `.env` の `OPENAI_API_KEY` を使用し、DALL-E 3 でベース画像を生成。
- **画像処理 (`sharp`)**: `apps/hono/package.json` に既に導入されている `sharp` を利用して、ピクセル化・減色・スプライトシート結合を行う。
- **実行環境**: `npx tsx scripts/cli/asset/gen-official-asset.ts`

### コマンド体系のイメージ
```bash
# 剣士の歩行グラフィック（スプライトシート）を自動生成
npx tsx scripts/cli/asset/gen-official-asset.ts character "金髪のクールな剣士、青い鎧" --out ./assets/ch/knight.png

# マップチップ（木箱）を生成
npx tsx scripts/cli/asset/gen-official-asset.ts tile "古びた木箱" --out ./assets/tiles/box.png
```

## 3. Claude Code / Gemini CLI の役割分担（実装フロー）

本計画を実行するため、以下のステップで進めます。

### Step 1: DALL-E 3 連携基盤の構築 (Gemini CLI)
- OpenAI API をコールし、プロンプトを投げてベース画像（Base64 または URL）を取得するロジックを実装します。
- プロンプトには「2D game sprite, full body, facing camera, white background, flat colors, clear outlines, cel shading」といった、後でドット絵化しやすい呪文（プロンプトエンジニアリング）を固定で付与します。

### Step 2: ピクセル化・減色エンジンの実装 (Claude Code 担当)
- `sharp` を駆使した画像加工パイプラインを実装します。
- **処理内容**: `resize(48, 48, { kernel: 'nearest' })` によるドット化と、背景透過アルゴリズムの実装。

### Step 3: スプライトシート結合 (Claude Code 担当)
- 1枚の真正面画像から、ピクセルをプログラムで数ピクセル上下にズラすことで「待機アニメーション（3コマ）」や「歩行アニメーション（簡易版）」をでっち上げ、`144 x 192` などのツクール規格の PNG として出力するロジックを書きます。

---

## 4. この戦略の優位性
- **作者に提供しないからこその力技**: ユーザー向けAPIとして開放すると画像生成コスト（1枚数円〜数十円）が莫大になりますが、運営（我々）が公式アセットを月100体作るだけであれば全く問題になりません。
- **圧倒的な量産速度**: デザイナーに発注すれば1体数万円・数日かかるアセットが、スクリプトを1回叩くだけで10秒で生成されます。
- **規格の統一**: 最終出力が必ず「指定されたカラーパレット」と「48x48のニアレストネイバー」を通るため、どんなプロンプトを入れても「kaedevnのゲームエンジンで浮かない、統一感のあるドット絵」が保証されます。
