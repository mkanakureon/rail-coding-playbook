# 仕様レビュー：ノベルモバイル・SNS統合戦略

> **作成日**: 2026-03-19
> **担当**: Gemini CLI
> **対象文書**: `docs/09_reports/2026/03/19/08-novel-mobile-social-unified-spec.md`

## 1. 総合評価
**評価結果: 承認 (Approved with Caveats)**

本仕様は、ダウンロード不要なブラウザエンジンの利点と、現代のモバイル視聴習慣（縦画面・短尺動画）を高度に融合させています。RPG 機能をリリースから外すことで生じる「汎用性の低下」を、「圧倒的な拡散力」で補填し、V字回復を狙う戦略として非常に強力です。

## 2. 技術的フィードバック

### 2.1. エンジンの動的解像度対応
現在のエンジン実装は 1280x720 のハードコードに強く依存しています。
- **課題**: `VIRTUAL_WIDTH/HEIGHT` が `packages/core` の定数として定義されており、これに依存する座標計算が `WebOpHandler.ts` 等の各所に散在しています。
- **要件**: 解像度を `PlayLayout` オブジェクトから動的に読み込む仕組みへの移行。

### 2.2. モバイル・ブラウザ最適化
- **画面回転**: 現在の `apps/next` における強制ランドスケープ設定を解除し、プロジェクト設定に基づく動的な UI 切り替え（CSS/JS）の実装。
- **URL バーの隠蔽**: `play/[id]/page.tsx` にある URL バー隠蔽ロジックを、縦画面時にも最適に動作するように調整。

### 2.3. 動画クリップ生成 (Phase 3)
- **実現性**: WebCodecs API を利用したフロントエンド完結型の動画生成が、最新のスマホブラウザ（iOS Safari 15.4+ 等）で安定動作するか検証が必要。
- **代替案**: 初期段階では、Canvas キャプチャによる「動的 GIF」または「数秒のアニメーション WebP」の方が、SNS 互換性と実装コストのバランスが良い可能性があります。

## 3. ロードマップへの提言
Phase 1 の **「動的 OGP」** と **「ワンタップ起動」** は、技術的な障壁が低く、かつ導入直後から MAU 流入率に劇的な変化をもたらすため、最短での実装を推奨します。

## 4. 結論
本戦略の実行により、kaedevn は単なる「ゲーム制作ツール」から、SNS 時代の「ストーリー・プラットフォーム」へと昇華します。懸念点であるエンジンの定数リファクタリングを優先的に実施することで、本仕様の実現を強力に推進すべきです。
