# レビュー計画書：ノベルゲーム特化リリース設計仕様書

> **作成日**: 2026-03-19
> **担当**: Gemini CLI
> **対象文書**: `docs/09_reports/2026/03/19/11-release-design-spec.md`

## 1. レビューの目的
本リリース設計仕様書が、kaedevn の「ノベル特化・モバイル拡散戦略」を技術的に支え、かつ計画された期間（Phase 1: 1週間、Phase 2: 7週間）で安全に実装可能であることを検証する。

## 2. レビュー範囲
仕様書に記載された以下の項目を重点的にレビューする。

### 2.1. 技術的実現性 (Technical Feasibility)
- **Phase 1: 即時実装項目**
    - 動的 OGP 生成のサーバー負荷とキャッシュ戦略。
    - `?quick=1`（ワンタップ起動）時のアセットロード最適化手法。
    - `Web Share API` によるシーン共有のブラウザ互換性。
- **Phase 2: 基盤拡張項目**
    - ポイント課金システム（Prisma スキーマ、API 設計、モックフロー）。
    - 縦画面モード（720px 幅、ボトムアンカーUI）のレンダリング計算式。
    - 作者への 90% 分配ロジックの正確性と整合性。

### 2.2. 戦略的整合性 (Strategic Alignment)
- 「15分で感動、100円で完結」というコンセプトが UI/UX 設計に反映されているか。
- 作者分配 90% という差別化要因が、システム設計（Wallet/Transaction）に正しく組み込まれているか。
- Switch 出版（横画面）と Web（縦画面）の共存に関する設計。

### 2.3. リスク評価 (Risk Assessment)
- スケジュール（Week 1〜8）の妥当性とバッファの有無。
- モック決済から本番決済への移行パスの容易性。
- 既存の 16:9 アセットを縦画面で表示する際の視覚的課題。

## 3. レビュー実施体制
- **主査**: Gemini CLI
- **手法**: 
    1. 既存コードベース（`packages/web`, `apps/hono`, `apps/next`）との整合性チェック。
    2. 提案された Prisma スキーマのバリデーション。
    3. モバイルブラウザ（iOS/Android）の仕様制約との照合。

## 4. レビュー項目チェックリスト

| カテゴリ | チェック項目 |
|:---|:---|
| **OGP** | サーバーサイドでの画像合成ライブラリの選定は妥当か？ |
| **起動** | アセットの「先読み（Prefetch）」と「遅延読み込み（Lazy Load）」の境界は明確か？ |
| **課金** | トランザクションの ACIDC 特性（特に作者への分配時）は担保されているか？ |
| **UI** | 縦画面時の「ボトムアンカー」が、iPhone のホームインジケーターと干渉しないか？ |
| **作者** | 収益ダッシュボードの表示項目は、確定申告等の実務に耐えうるか？ |
