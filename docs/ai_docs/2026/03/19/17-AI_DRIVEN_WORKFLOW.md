# AI 主導型・超生産ワークフロー：開発者の「前頭葉化」宣言

> **作成日**: 2026-03-19
> **担当**: Gemini CLI

## 1. ワークフローのパラダイムシフト
本業から「専念」に切り替えた初日、開発者は「キーボードを打つ者」から「意思を確定させる者（決裁者）」へと進化する。AI エージェント群が「筋肉」として全速力で駆動し、1日で5万行のアウトプットを実現する。

## 2. 実践的「書かせない」フロー

### ステップ1：コンテキストの超圧縮（AM 9:00）
- **開発者の役割**: 「今日の完成形」を数行の指示（Prompt）とラフスケッチだけで提示する。
- **AIの役割**: `upstream-cli` を全開にし、必要な 5万行のファイル構成とコード生成の依存関係グラフを 5分以内に展開する。

### ステップ2：マルチスレッド同時実装（AM 10:00 - PM 3:00）
- **開発者の役割**: 複数のサブエージェントから上がってくる「方針の分岐点」に対し、0.5秒で判断を下し続ける。
- **AIの役割**: `codebase_investigator` で整合性を保ちつつ、`generalist` が並列でファイルを一括生成・修正する。型定義、テストコード、ドキュメントをセットで出力し、人間を「レビュー作業」にのみ追随させる。

### ステップ3：感覚的チューニング（PM 3:00 - PM 6:00）
- **開発者の役割**: 実機プレビューを確認し、AI には理解できない「心地よさ」「リズム」「エモーショナルな体験」の修正指示を出す。
- **AIの役割**: 指示を受けた瞬間に、数千行に及ぶ座標計算や演出タイミングを一括リファクタリングし、リアルタイムで反映する。

## 3. 人間が集中すべき「3つの聖域」
1. **Strategic Vision**: 「誰を幸せにするプロダクトか？」という方向性の堅持。
2. **Emotional Design**: ユーザーの心が動く瞬間の微調整。
3. **Critical Decision**: 破壊的変更やリスクのある実装に対する最終的な責任。

## 4. 結論
専念初日のゴールは、**「コードを1行も書かずに、5万行の成果物を検収し終えること」** である。我々 AI エージェントは、あなたの「意思」を最速で「現実」に変換するためのインターフェースとして、あなたの脳を拡張し続ける。
