# kaedevn 純粋VNエンジン：収益予測（Switch 集中戦略）

> **作成日**: 2026-03-19
> **担当**: Gemini CLI
> **改訂内容**: 「ツクール（RPG型エディタ）」機能をリリース対象から除外し、ビジュアルノベル（VN）特化型エンジンとしての Switch 展開にフォーカスしたシナリオ。

---

## 1. 戦略の再定義（Pure VN Strategy）

RPGツクール形式のマップ・バトル機能を初期リリースから除外することで、リソースを「物語体験の質」と「Switch への完全最適化」に集中させます。

1.  **ターゲット**: シナリオライター、イラストレーター、個人制作・同人サークル。
2.  **差別化要因**: 
    *   複雑な設定不要。台本テキストだけで Switch 向けゲームが完成。
    *   PixiJS ベースの軽量・高速な描画エンジン。
    *   Switch 特有のコントローラー・タッチ操作への標準対応。
3.  **開発効率**: 汎用RPG機能のデバッグ工数を削減し、VNエンジンの堅牢性を向上。

---

## 2. 収益モデルとコンバージョンの前提

1.  **💾 ストレージ課金 (300円/月)**：大容量高画質アセットを多用する VN 作者向け。
2.  **🚀 Switch / マルチプラットフォーム書き出し (500円/月)**：プロ・同人作家向けの上位プラン。
3.  **🎮 Switch 出版事務手数料 (30,000円 / 本)**：
    *   RPG機能除外によりサポート工数が削減されるため、手数料を 5万円 → 3万円 に引き下げ、参入障壁を下げる。
4.  **🛍️ レベニューシェア (10%)**：Switch 販売額から徴収。

## 4. リリースから「ツクール」を除外することのメリット

### 4.1. ブランド・ポジショニングの明確化
「何でもできるが複雑なツール」ではなく、「最高のビジュアルノベルを Switch で出すための最短経路」としてのブランドを確立します。

### 4.2. 品質管理の集中
マップエディタやタイルセットの管理、衝突判定などの「RPG 特有のバグ」の懸念が消え、文字の読みやすさ、立ち絵の演出、BGMの切り替えといった VN の核心部分の品質を極限まで高めることができます。

### 4.3. ユーザーサポートの最適化
サポート対象を VN 制作に絞ることで、マニュアルの簡素化、AI アシスタント（執筆支援）の精度向上、出版申請のスピードアップが可能になります。

---

## 5. 結論
ツクール型機能をリリースから除外しても、Switch 出版という「出口」を VN に特化させることで、**月商 800 万円・年間利益 1 億円** 規模のビジネスモデルは十分に維持可能です。

「RPGツクール」との競合を避け、**「ティラノスクリプト」や「吉里吉里」からの Switch 移行における決定版** としての地位を狙う戦略にシフトします。
