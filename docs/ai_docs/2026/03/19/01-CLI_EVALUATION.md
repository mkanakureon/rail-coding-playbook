# CLI ツール評価レポート

> **作成日**: 2026-03-19
> **担当**: Gemini CLI

## 背景と目的
本プロジェクト（kaedevn-monorepo）において、作者（シナリオライター）や AI アシスタントが効率的に開発・執筆を行うための CLI ツール群が `scripts/cli/` 配下に整備されている。これらのツールの現状を評価し、Gemini CLI からの利用可能性や改善点をまとめる。

## 評価内容

### 1. プロジェクト操作ツール (`scripts/cli/project/`)
- **`init-project.mjs`**: 非常に強力。テンプレートベースでアセット込みのプロジェクトを即座に作成できる。
- **`import-story.mjs`**: テキスト台本（独自フォーマット）からブロックへの変換機能。直感的であり、AI による一括投入と相性が良い。

### 2. ブロックエディタ操作ツール (`scripts/cli/block/`)
- **`editor-cli.mjs`**: JSON 直接編集を避け、コマンドベースで安全にブロックを追加・更新・削除できる。バリデーション機能 (`validate`) が組み込まれており、整合性チェックが可能。

### 3. AI 執筆支援ツール (`scripts/cli/ai/`)
- **`assist-cli.ts` / `upstream-cli.ts`**: Gemini API を活用した多段階パイプライン。プロットからエディタ JSON までの自動生成を実現しており、本プロジェクトの中核的な差別化要因となっている。

### 4. その他ツール
- **キャラクター・マップ操作**: API 経由でアセットと連携しており、ヘッドレス環境での素材管理が可能。
- **RAG インデックス**: 開発ドキュメントのベクトル化により、AI エージェントが最新の設計仕様を参照できる仕組みが整っている。

## 重要な発見
- **実装の完成度**: `scripts/cli/` 配下のツールはモックではなく、実際に API サーバーと通信してプロジェクトを操作できる完成度にある。
- **Gemini CLI との親和性**: これらのツールは `run_shell_command` 経由で Gemini CLI から直接呼び出すことができ、複雑なブロック操作を「プロンプト → CLI 呼び出し」のフローで自動化可能。
- **ドキュメントの乖離**: `docs/09_reports/` にある Claude Code 向けのリファレンス（Opus 4.6 生成）と、実際の実装は概ね一致しているが、一部のパラメータ（`--config` 等）に微細な差異が見られた。

## 結論・今後の課題
### 評価結果: **S (優良)**
極めて高度な CLI エコシステムが構築されており、エンジニア以外でも高度なゲーム開発が可能な状態にある。

### 今後の課題
1. **Gemini CLI 専用スキルの拡充**: `scripts/cli/` の機能を Gemini CLI のスキルとしてラップし、より直感的な自然言語操作を実現する。
2. **型安全性の強化**: `.mjs` ファイルが多いが、大規模化に伴い `scripts/` 全体の TypeScript 化（`.ts`）と型共有を推進すべきである。
3. **エラーハンドリングの改善**: 一部のツールにおいて、API サーバーが未起動の場合のメッセージが不親切なケースがあるため、接続チェックを強化する。
