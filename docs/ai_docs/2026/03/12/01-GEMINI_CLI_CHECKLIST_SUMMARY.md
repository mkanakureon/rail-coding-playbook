# Gemini CLI 確認事項サマリーレポート (2026-03-12)

本ドキュメントは、2026年3月12日時点でのAI生成パイプラインとエディタ連携に関する課題と確認事項をまとめたものである。

## 1. 生成パイプラインの品質課題
- **キャラクター名の表記揺れ解決**: `assist-cli.ts` のプロンプトに許可されたスラグ一覧（luca, garud 等）を注入し、綴りのブレを根絶する。
- **背景スラグの最適化**: 生成データ内の `$bg:` プレフィックスの扱いを、`manifest.json` との整合性を重視する方向に整理する。
- **動的なキャラクター定義**: ストーリーに登場する未定義キャラクターを自動検出し、`characters.json` へ登録すると同時に、公式アセットから適切な画像（老人系、悪役系等）を割り当てる仕組みを検討する。

## 2. インポートおよびエディタ表示の検証
- **インポートスクリプトの再試行**: `scripts/import-editor-json.ts`（ch-class メタデータ対応版）の実行と、全15ページにおけるサムネイルおよびテキストの表示確認。
- **プレビュー再生の完走**: プレビュー環境での背景・キャラ의切り替え、およびセリフ送りが JS エラーなしで動作することを実機確認する。

## 3. 公式アセット調査と案B（アセット拡充）への準備
- **公式アセットの網羅的調査**: `GET /api/official-assets` を用い、利用可能な背景・表情バリエーションをリストアップする。
- **マッピング戦略の構築**: AIが生成する多様なシチュエーションを、限られた公式アセット（10〜15種）へ効果的にマッピングするための対応表（マッピングテーブル）を設計する。

## 4. Playwright E2E テストの拡張
- **既存テストの保護**: インポート検証テストの継続的なパスを確認する。
- **再生テストの新規設計**: 単なるデータの存在確認だけでなく、プレビュー画面での「ユーザー体験」を模した自動テストの追加を検討する。
