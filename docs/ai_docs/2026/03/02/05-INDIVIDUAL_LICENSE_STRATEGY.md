# ノベルゲームエンジンのライセンス戦略：個人限定利用と商用縛りの分析

## 概要
kaedevnにおいて、「個人が制作したゲームのみ配信・配布を許可する」という制限と、それに伴うライセンスの「縛り」の影響を分析しました。

## 1. ライセンス形態：デュアルライセンス・モデル
個人と法人の利用を明確に区別するライセンス体系を構築します。
- **個人向け（Personal）**: 非営利または小規模な個人・サークルを対象に、無料でエンジンと配信権を提供。
- **商用向け（Commercial）**: 一定以上の売上がある個人、または法人を対象に、有料でライセンス（販売権・Switch展開権等）を提供。

## 2. 個人向けライセンスにおける「縛り」の内容
無料提供の対価として、以下の条件（縛り）を課すことが一般的です。
1. **ロゴ表示（スプラッシュスクリーン）**: ゲーム起動時にエンジンロゴを強制表示。
2. **クレジット表記**: スタッフロールやマニュアルへのライセンス情報の記載。
3. **売上上限の規定**: 累計売上が一定額（例：500万円）を超えた場合、商用ライセンスへの移行を義務付け。
4. **リーダーの保護**: 非公開の実行エンジン（リーダー）に対するリバースエンジニアリングの禁止。

## 3. 「個人限定」戦略のメリット
- **企業によるタダ乗りの防止**: 資本力のある企業がOSS資産を無償利用して利益を得ることを防ぎます。
- **クリエイター・コミュニティの保護**: 「個人の創作を応援するエンジン」としてのブランドを確立し、アマチュア層の支持を強固にします。
- **商用転換への誘導**: 個人の趣味で始まった作品がヒットした際に、自然な形で商用ライセンス（収益源）へと移行する導線が作れます。

## 4. リスクと対策
- **境界線の曖昧さ**: 同人サークルと法人の区別が難しい。
    - **対策**: 「法人格の有無」ではなく「売上高」を基準にすることで、公平な課金ポイントを設定します。
- **不正利用**: ライセンスを無視して企業が利用するリスク。
    - **対策**: コンソール機（Switch等）や各アプリストアへの展開において、「公式のライセンス証明」を必須とすることで物理的に制限します。

## 結論
「個人限定」の配信制限は、創作コミュニティの純度を高めつつ、ビジネスとしての収益性を確保するための極めて有効な戦略です。
**「制作はOSSで完全に自由（個人に寄り添う）」「配信・商用化はライセンスで管理（ビジネスを守る）」**というハイブリッドな姿勢が、kaedevnの独自性と競争力を生み出します。
