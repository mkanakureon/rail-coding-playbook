# AI職人技再構成：AIエージェントによる自動実装・運用計画書

## 1. 概要
Claude Code および Gemini CLI を活用し、「暗黙知の外部化」「パイプラインの自動検証」「ゴールデンデータの整備」を、人間が手を動かさずに遂行するための手順を定義します。

---

## 2. 計画①：「失敗モードの地図」の抽出とドキュメント化
**目的**: 人間の頭の中にある「過去のバグ」「PixiJSの罠」「演出のセオリー」をAIに吸い出させます。

### AIを使った進め方（Gemini CLI / Claude Code）
1. **履歴解析**: AIに対し「過去270コミットの diff を全て読み、繰り返し修正されている箇所や、特定のライブラリ（PixiJS等）に関連する修正パターンを抽出して」と指示。
2. **インタビュー形式**: 人間が AI に対し「背景切り替え（bg_new）で苦労した点は何か？」「5層パイプラインで最も忘れやすいのはどこか？」をチャットで伝え、AIに清書させる。
3. **成果物**: `docs/02_references/FAILURE_MODES.md` を生成。

---

## 3. 計画②：5層パイプラインの自動整合性バリデータ
**目的**: 命令（Op）を追加した際、Compiler, Interpreter, Renderer 全ての層が更新されているかを AI が自動チェックする仕組みを作ります。

### AIを使った進め方（Claude Code）
1. **メタデータ抽出ツールの作成**: AIに対し「`packages/core/src/types/Op.ts` をパースして、存在する命令名のリストを抽出するスクリプトを書いて」と指示。
2. **各層の対応チェック**: 
   - Compiler の `KNOWN_COMMANDS` に含まれているか？
   - Interpreter の `executeBuiltin` に case文があるか？
   - Renderer の `WebOpHandler` にメソッドがあるか？
   これらを比較し、不足があればエラーを出すテストコードを AI に生成させる。
3. **成果物**: `packages/tools/pipeline-linter.ts` および CI への組み込み。

---

## 4. 計画③：ゴールデンデータ（正解セット）の自動生成と検証
**目的**: 「正しい演出結果」を画像として保持し、AIが変更を加えた後に「見た目が壊れていないか」を自律的に確認させます。

### AIを使った進め方（Playwright + Gemini CLI）
1. **シナリオ作成**: AIに対し「全演出コマンド（@bg, @ch, @filter等）を網羅した、検証用のテストシナリオ（verify.ks）を作成して」と指示。
2. **自動キャプチャ**: 先日の `capture-preview.ts` を使い、AIに「verify.ks を実行して、全ページのスクリーンショットを `tests/golden/` に保存して」と指示。これが「正解」となります。
3. **自律検証**: AIがコードを修正した際、AI自身に「今のコードでプレビューを実行し、ゴールデン画像と比較して差異がないか報告して」と指示。
4. **成果物**: `tests/visual-regression/` スイート。

---

## 5. 運用サイクル：AIへの「検収」指示テンプレート
以後、新しい機能を実装する際は、Claude Code に対して以下のプロンプト（命令）を出します。

> 「新機能 X を実装してください。実装後、`pipeline-linter` を実行して断裂がないか確認し、さらに `verify.ks` を走らせて既存のゴールデンデータとの差分がないことを証明してください。」

## 結論
この計画により、開発者は「仕様を伝える」ことに集中し、AIが「過去の教訓を守り」「構造の整合性を保ち」「見た目の正確さを保証する」という、高度な職人的な品質管理を自動で代行するようになります。
