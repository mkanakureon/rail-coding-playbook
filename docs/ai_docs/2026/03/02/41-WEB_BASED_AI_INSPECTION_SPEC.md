# Web技術を活かしたローカルAI内部解析システムの仕様・設計書

## 1. 概要
kaedevnがWebベースであることを活かし、ローカルPC内での開発において、AIエージェントがゲームの「内部状態」をDOMメタデータとして直接解析し、自律的にテスト・修正ループを回せる仕組みを構築します。

## 2. コア機能：ローカルAIインスペクション（内部解析）

### A. プレビュー画面への「AI用メタタグ」埋め込み
Playwright等のツールでAIがローカルブラウザをスキャンした際、Canvas上の「どの座標に何が表示されているか」をDOMレベルで瞬時に把握できるようにします。

- **仕様**: PixiJSのレンダリング状態に同期して、不可視の `<div>` 要素をDOMツリーに配置。
- **データ構造例**:
  ```html
  <div id="ai-inspector" style="display:none">
    <span data-type="character" data-id="hero" data-x="640" data-y="720" data-pose="smile"></span>
    <span data-type="background" data-id="classroom_day"></span>
    <span data-type="filter" data-active="pc98"></span>
  </div>
  ```
- **メリット**: AIはスクリーンショット（視覚）に加え、このDOM構造（数値データ）を併用することで、「キャラが1pxズレている」「背景IDが間違っている」といった論理的なバグを確実に特定できます。

### B. ローカル・プレビューの状態ダンプ
実行中のエンジンの内部変数を、AIが読み取り可能な一時ファイル（`tmp/current_state.json`等）としてローカルに書き出します。
- **目的**: AIがブラウザを介さずとも、CLIから「今のゲームの状態」を把握できるようにします。

## 3. 開発フロー：ローカル・自律テスト

### A. スナップショットによる再現テスト
特定のシーン状態を即座に再現するための「シードパラメータ」を、ローカルのURLに追加します。
- **例**: `http://localhost:3000/preview/p123?page=5&force_bg=forest`
- **活用**: AIが「第5話の背景だけを変えてテストしたい」場合、このURLを叩だけで、他の手順を飛ばして目的の状態をローカルで即座に確認できます。

### B. 自動キャプチャと比較
AIがコード修正前と修正後のスクリーンショットをローカルで撮影し、差異を確認します。
- **動作**: 
  1. `npm run capture -- --before`
  2. コード修正。
  3. `npm run capture -- --after`
  4. AIが2枚の画像を比較して「意図した通りに変わったか」をローカルで判断。

## 4. アーキテクチャ設計

### レンダラー層 (`packages/web`)
- `WebOpHandler` 内に、状態変化を DOM に同期する `DOMSync` ロジックを実装。
- ローカル環境でのみ動作するフラグを設定し、製品版には影響を与えないようにします。

### CLI層 (`scripts/`)
- Playwright を用いたローカル専用の自動操作スクリプト。

## 結論
このシステムは、クラウドを一切介さず、ユーザーのローカルPC内だけで完結します。Web技術特有の「解析のしやすさ」を活かすことで、AIは人間が手動でデバッグするよりもはるかに速く、正確にローカルでの開発・テストを遂行できるようになります。
