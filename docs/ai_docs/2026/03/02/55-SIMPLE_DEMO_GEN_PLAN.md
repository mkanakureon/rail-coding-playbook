# 全シーン動画・画像自動生成ツール（目視確認用）の実装計画書

## 1. 目的
作者が制作したノベルゲームの全シーン（ページ）を自動的に実行・録画し、自分の目で演出を確認するための「プレビュー動画」と「代表画像」を一括生成します。

## 2. 今回実装する機能（最小構成）
- **全自動プレイ**: 第1話から最終話まで、Playwright が自動でクリックして進めます。
- **全編録画**: 実行の様子を一つの動画ファイル（.webm）として保存します。
- **代表シーンの静止画**: 各ページの開始時点（または終了時点）のスクリーンショットを連番で保存します。

## 3. 具体的な実装手順

### Step 1: 録画用 Playwright スクリプトの作成
`tests/generate-full-preview.spec.ts` を作成します。

```typescript
// 実装イメージ
test('generate-preview-material', async ({ page }) => {
  await page.goto('http://localhost:3000/preview/project-id');
  
  const pages = [1, 2, 3, 4, 5]; // 実際はプロジェクトのページ数分ループ
  for (const p of pages) {
    // 1. ページの開始画像を撮る
    await page.screenshot({ path: `./outputs/page_${p}.png` });
    
    // 2. 次のページまで自動進行（またはクリック待ち）
    // エンジン側の「次へ」ボタンを自動クリック
    await page.click('#next-button'); 
    await page.waitForTimeout(2000); // 演出の余韻
  }
});
```

### Step 2: package.json への追加
作者が迷わず実行できるように、専用のコマンドを追加します。

```json
"scripts": {
  "gen:preview": "playwright test tests/generate-full-preview.spec.ts --config=playwright.demo.config.ts"
}
```

## 4. 期待される「作者の体験」
1. 作者が `npm run gen:preview` を叩く。
2. ブラウザが自動で起動し、自分のゲームが最後まで「自動演奏」される。
3. 数分後、`demo-results/` フォルダに **「全編録画動画」** と **「シーンごとの画像一覧」** が出来上がっている。
4. 作者は動画を再生したり、画像をパラパラめくるだけで、演出のミス（キャラの出し忘れ等）を効率的に発見できる。

## 5. 工夫する点
- **等倍速実行**: 演出のテンポを確認するため、あえてスキップせず「普通のプレイ速度」で録画します。
- **ファイル管理**: 実行のたびに古いファイルを上書きせず、`20260302_1530/` のような日時フォルダに整理して保存するようにします。

## 結論
このツールは、作者にとっての「最強の試写室」になります。高度なプログラムによる検知よりも、まずは「自分の目で一気に確認できる」環境を提供することが、制作のモチベーションと品質向上に直結します。
