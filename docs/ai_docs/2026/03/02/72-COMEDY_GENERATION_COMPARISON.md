# コメディ作品生成結果の比較報告書：GPT-4o-mini vs Gemini 2.0 Flash

## 1. 概要
`projects/comedy` において、以前生成されたデータ（2026/02/25）と、今回 Gemini 2.0 Flash で生成されたデータ（2026/03/03）を比較し、品質、整合性、および演出の深度について評価しました。

## 2. 比較評価

### A. 文章の描写力と情緒
- **2026/02/25 (旧)**: 簡潔で事実を伝える文章が中心。テンポは良いが、情景描写は標準的。
- **2026/03/03 (新: Gemini 2.0 Flash)**: **情景描写が大幅に強化。** 「蝉の声が耳障りな中」「部室の雰囲気はまるで迷宮のよう」といった、読者の没入感を高める比喩や環境描写が豊富に含まれています。

### B. キャラクターの深掘りと演出
- **旧**: キャラクターのセリフは設定に忠実だが、やや平坦。
- **新 (Gemini)**: **演出情報の自己生成。** `[表情: smile]` といった、エンジンの演出（表情差分）を意識したタグが地の文に自然に組み込まれています。これにより、AIが「このセリフではこの表情」という演出指示まで自律的に行える可能性が示されました。

### C. 長期的な物語の整合性（プロット管理）
- **旧**: 章ごとのエピソードは独立性が高く、安定しているが、大きな驚きは少なめ。
- **新 (Gemini)**: **伏線の構築と回収が緻密。** 篇要約（arc1_summary）において、「入部届の提出状況」という小さな問題を中盤以降の大きな葛藤の火種として明確に位置づけています。また、各章の結末で「次篇への引き継ぎ事項」を具体的に言語化する能力に長けています。

## 3. 生成速度とコスト効率
- **旧 (GPT-4o-mini)**: 高速かつ低コストだが、一度に読み込めるコンテキストに限界があった。
- **新 (Gemini 2.0 Flash)**: **驚異的なスピード。** 15話分の複雑な執筆・分析を、旧モデルと同等以上の速度で完了。また、Google AI Studio の無料枠/クレジットを活用できるため、現状の kaedevn 開発において最もコストパフォーマンスが高いと判断できます。

## 4. 結論：今後の推奨モデル
今回の比較により、**Gemini 2.0 Flash がノベルゲームのメイン執筆エンジンとして極めて優秀である**ことが証明されました。

- **採用の決め手**: 膨大なコンテキスト（100万トークン〜）を活かした設定の一貫性と、演出指示（表情タグ等）を含む豊かな描写力。

今後は、Gemini が生成した `[表情: ...]` タグを自動的にエンジンの演出命令（Op）へ変換するブリッジ機能を強化することで、人間の手を介さない「完全自動演出」の実現を目指します。
