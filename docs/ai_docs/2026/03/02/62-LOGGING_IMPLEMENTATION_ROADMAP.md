# 実装計画書：AIエージェント（Claude Code）によるログ基盤導入

## 1. 開発フェーズと Claude Code への指示タスク

### フェーズ1：ロギング基盤の標準化（3日間）
**タスク**: `packages/core` への Logger 実装と `apps/hono` への適用。
- **Claudeへの指示**: 「`packages/core` に JSON 形式の構造化ログを出力する `UniversalLogger` クラスを実装して。次に、`apps/hono` の全エンドポイントでこの Logger を使い、リクエスト完了時に `event_type` を含むログを出すようにミドルウェアを修正して。」

### フェーズ2：フロントエンド・イベントの収集（2日間）
**タスク**: `apps/editor` での行動イベント発火。
- **Claudeへの指示**: 「エディタ内の主要なアクション（ブロック追加、プレビュー開始、保存）にログ送信イベントを埋め込んで。データは `POST /api/assist/logs` に送信するようにして。」

### フェーズ3：BigQuery 連携と分析 SQL の作成（2日間）
**タスク**: SQL ビューと ML モデルの定義。
- **Claudeへの指示**: 「BigQuery ML を使ってユーザー離脱予測を行うための `CREATE MODEL` 文と、過去のログから特徴量を抽出する SQL ビューを作成して。`FAILURE_MODES.md` にある分析要件を参考にして。」

## 2. 成功基準
- [ ] 全ての API ログが単一の JSON 形式で出力されている。
- [ ] エディタ上の操作が、欠落することなくバックエンドへ届いている。
- [ ] BigQuery 上で `SELECT` するだけで、前日のアクティブユーザー数が即座に集計できる。

## 3. リスク管理
- **ログの肥大化**: 開発環境ではデバッグレベルを上げ、本番環境では INFO 以上に制限する環境変数の切り替えを徹底する。
- **機密コードの混入**: `FAILURE_MODES.md` に「ログに平文のパスワードや秘密キーを出してはいけない」というルールを追加する。

## 結論
Claude Code を活用することで、複雑なデータエンジニアリング作業を、一貫したインターフェース定義と SQL 生成に集中させることができます。これにより、短期間で「データで語るプラットフォーム」へと進化させることが可能です。
