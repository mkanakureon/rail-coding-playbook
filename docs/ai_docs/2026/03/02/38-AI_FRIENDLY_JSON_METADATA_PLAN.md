# AI親和性を高めるJSONメタデータ付与の実装計画書

## 1. 目的
既存のゲームエンジン（実行系）やコンパイラのロジックを変更することなく、Claude Code等のAIエージェントがプロジェクトデータ（JSON）を正確に理解・編集できるようにするための「ヒント情報」を付与します。

## 2. 基本方針：既存ロジックへのゼロ・インパクト
- **非破壊的変更**: JSONの既存キーは一切変更・削除しません。
- **メタデータの追加**: JavaScript/TypeScriptのコメントに近い役割を果たす、AI専用のプロパティを追加します。
- **無視されるキーの活用**: エンジン側（WebOpHandler等）が読み取らないキーを追加することで、実行時のオーバーヘッドをゼロにします。

## 3. 具体的な実装案

### A. ブロック定義への「AI注釈」追加
各ブロックオブジェクトに `_debug_info` や `_ai_hint` といった、アンダースコアで始まるキーを追加します。

```json
{
  "id": "bg-12345",
  "type": "bg",
  "assetId": "classroom_day",
  "_ai_hint": "このブロックは背景を設定します。assetIdはbackgroundカテゴリのアセットを指定してください。"
}
```

### B. プロジェクトルートへの「スキーマ・ガイド」追加
JSONの先頭に、AIがこのファイルの全体像を把握するためのメタデータを追加します。

```json
{
  "title": "My Story",
  "updatedAt": 1740000000,
  "_metadata": {
    "version": "1.0",
    "description": "kaedevn project data",
    "ai_context": "このファイルはkaedevnエディタのプロジェクトデータです。pages配列内に物語の各話が格納されています。"
  },
  "pages": [...]
}
```

### C. 編集禁止・警告プロパティの導入
AIが誤って編集すべきでない箇所（自動生成されるID等）を触らないよう、明示的なフラグを立てます。

```json
{
  "id": "start-block",
  "type": "start",
  "_ai_read_only": true
}
```

## 4. エディタ（apps/editor）での対応
`useEditorStore.ts` の保存ロジックにおいて、JSONを書き出す直前にこれらのメタデータを付与するように修正します。

```typescript
// 修正イメージ
const exportData = {
  ...project,
  _metadata: {
    engine: "kaedevn",
    lastEditedBy: "AI-Assistant-Compatible-Editor"
  }
};
```

## 5. 期待される効果
- **AIの誤操作防止**: IDの重複作成や、不正な型（Stringを期待する場所にNumberを入れる等）の代入を AI が自律的に避けるようになります。
- **ドキュメント参照の削減**: AIがいちいち仕様書ファイルを読みに行かなくても、JSON自体を見るだけで「次に何をすべきか」を判断できるようになります。
- **開発速度の向上**: AIによる「ブロックの自動生成（例：このあらすじに基づいて全ブロックを生成して）」の精度が劇的に向上します。

## 6. 安全性の確認
後続の処理（`WebOpHandler`, `Compiler`, `Interpreter`）は、**「自分の知らないキー（プロパティ）は無視する」**という標準的な挙動をするため、プログラムがクラッシュする心配はありません。

## 結論
この改修により、kaedevnのデータ形式は「ただのデータ」から「AIと対話可能なセルフドキュメント形式」へと進化します。最小限の実装コストで、AI時代の開発スピードを最大化する基盤が整います。
