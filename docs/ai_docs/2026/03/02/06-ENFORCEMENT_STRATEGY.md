# ノベルゲームエンジンの運用戦略：不正利用・模倣配信への対抗策分析

## 概要
エンジンのOSS化と公開PF運営において、個人や法人の「勝手な配信・配布」を技術的に完全に防ぐことは困難です。本ドキュメントでは、技術的制限ではなく「ビジネス構造」による抑止と執行の戦略を分析しました。

## 1. 課題：技術的制限の限界
エンジンがOSSである以上、以下の行為を物理的に遮断することは不可能です。
- 法人が個人を装って公式PFを利用する。
- OSS版リーダーを流用し、外部サイトで独自にゲームを公開する。

## 2. 防御戦略：収益化ポイントの独占
技術的に防ぐのではなく、**「ビジネス上の出口」**を公式が管理することで実質的に制御します。
- **商用エクスポートの閉鎖性**: スマートフォン（iOS/Android）、PC（Steam）、家庭用ゲーム機（Switch等）向けのパッケージ化ツールを非公開・ライセンス制とします。
- **公式認証の必須化**: 各プラットフォームでの販売に際し、公式からの「ライセンス証明書」を必須とすることで、非公式な商用利用を物理的に排除します。

## 3. 抑止戦略：露出とブランドの管理
- **PF上のリーチ制御**: 公式PFでのランキング、おすすめ、検索結果のアルゴリズムは公式が管理します。規約違反が疑われる作品は露出を制限し、集客力を奪います。
- **事後的な法的執行**: 作品がヒットし、知名度が上がった段階で制作者の特定は容易になります。その時点でライセンス違反が発覚した場合、損害賠償や配信停止の法的措置を講じます。

## 4. ライセンスによる法的防御（AGPLの検討）
- コピーレフト条項（AGPL等）を採用することで、企業がエンジンを自社サイト等に組み込んで「クローズドな商売」をすることを法的に制限します。
- 「利用は自由だが、利用するなら透明性を確保せよ」というルールが、法人の不正利用に対する強い心理的障壁となります。

## 5. コミュニティによる自己浄化
- 公式PFに「通報システム」を設け、ユーザーコミュニティが不正な法人利用や著作権侵害を発見・報告する仕組みを作ります。
- 「公式PFで遊ぶのが最も安全で多機能」という読者側の体験を最大化し、非公式な配信場所の価値を相対的に低下させます。

## 結論
「勝手な利用」をすべて防ごうとするのではなく、**「Webでの自由な拡散（OSS）」をマーケティング資産として許容しつつ、「商用としての成功（パブリッシング）」の鍵を公式が握る**という構造が、最も持続可能なプラットフォーム運営を可能にします。
