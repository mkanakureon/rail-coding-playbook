# チーム開発における統合モノレポ運用とOSS同期ガイドライン

## 1. 概要
複数のコミッター（コア開発者）がAIエージェントを活用しながら、OSSエンジンと非公開プラットフォームを並行開発するための標準ワークフローを定義します。

## 2. 開発基盤の標準化
他のコミッターが迷わず参加できるよう、業界標準のモノレポ管理ツール（Turborepo等）を採用します。
- **共有パッケージ**: `packages/*` はOSS/非公開の双方から利用される共有資産。
- **公開用アプリ**: `apps/oss/*` (エディタ、ローカルポータル)。
- **非公開用アプリ**: `apps/private/*` (クラウド基盤、Switchビルド)。

## 3. チーム・AI 共通のルール
- **AIコンテキストの一致**: 全コミッターが `GEMINI.md` や `CLAUDE.md` に基づいたAI駆動開発を行い、コードの品質と一貫性を保ちます。
- **依存関係の自動検証**: OSSパッケージが非公開パッケージに依存していないかをCI（Lint）で常時監視し、エクスポート不能なコードの混入を未然に防ぎます。
- **自動同期 (No-Manual Sync)**: コミッターが `main` ブランチへプッシュした変更は、GitHub Actionsにより自動的にOSS用リポジトリ（Public）へ反映されます。

## 4. 貢献パスの分離
1. **コアメンバー (Committers)**: 統合モノレポ（Private）に直接アクセスし、AIと共に全機能を開発。
2. **コミュニティ (OSS Contributors)**: 公開リポジトリ（Public）に対しプルリクエストを送信。
3. **マージフロー**: OSSリポジトリのPRは、メンテナーによって統合モノレポの該当ディレクトリへ「逆同期（Backport）」され、全コンテキストでのテストを経て統合されます。

## 5. メリット：チームとしての開発加速
- **AIの知能共有**: リポジトリ全体のコンテキストが共有されているため、AIが生成するコードの整合性が全チームメンバーで保証されます。
- **摩擦の最小化**: コミッターは「同期」という特殊な工程を意識せず、標準的なモノレポ開発に集中できます。
- **透明性と秘匿性の両立**: コミットログのフィルタリングや機密情報の自動検知をCIに組み込むことで、チーム開発のスピードを落とさずにOSS運営を継続できます。

## 結論
他のコミッターがいる場合、独自の仕組みを排し、**「標準的なツール ＋ AI用コンテキスト ＋ 全自動同期」**という構成をとることが、チームの士気と開発速度を維持するための最適解となります。
