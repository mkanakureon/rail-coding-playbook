# 自動デモ動画録画システムの使用ガイド

## 1. 概要
Playwright を活用して、kaedevn エディタの操作デモ動画を自動的に録画するシステムを構築しました。UIの変更に合わせて常に最新のデモ動画を生成することが可能です。

## 2. セットアップと実行手順

### ① サービスの起動
録画を実行する前に、以下の全てのサービスがローカルで起動している必要があります。
```bash
./scripts/dev-start.sh all
```
※ API(8080), Editor(5176), Next.js(3000) が正常に動作していることを確認してください。

### ② 録画コマンドの実行
以下のコマンドを叩くと、Playwright がブラウザを自動操作し、動画を保存します。
```bash
npm run test:demo
```

## 3. 生成されるファイル
実行が完了すると、リポジトリ直下の `demo-results/` ディレクトリに動画ファイル（.webm 形式）が生成されます。

- **保存場所**: `demo-results/record-demo-record-editor-basic-ops-chromium/video.webm`
- **形式**: 1280x720, Chromium 録画

## 4. 録画内容のカスタマイズ
録画される操作の内容を変更したい場合は、以下のファイルを編集してください。
- `tests/record-demo.spec.ts`: クリックするブロックや、タブの切り替え、入力内容などのシナリオ。

## 5. 録画設定の変更
解像度や再生速度（SlowMo）を変更したい場合は、以下のファイルを編集してください。
- `playwright.demo.config.ts`: viewport（サイズ）や slowMo（操作の速度）の設定。

## 6. 次のステップ
- 生成された .webm ファイルを、必要に応じて GIF や WebP に変換して、プラットフォームの作者向けマニュアルや README に掲載してください。
- 音声合成（読み上げ）を別途合成して、完全な動画チュートリアルに仕上げることも可能です。
