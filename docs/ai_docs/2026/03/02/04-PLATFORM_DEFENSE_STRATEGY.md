# ノベルゲームプラットフォームの模倣リスクと防御戦略分析

## 概要
エンジンのOSS化に伴い、プラットフォーム（投稿サイト）やビジネスモデルが模倣されるリスクと、それに対する防御策（モート）を分析しました。

## 1. 模倣リスクの特定
- **エンジンのフォーク**: OSS化されたエディタとエンジンを流用し、他者が独自の投稿サイトを構築するリスク。
- **UI/UXのコピー**: 成功したプラットフォームの画面構成や機能を模倣されるリスク。
- **クリエイターの流出**: 競合がより安価な商用ライセンスや、自由なリーダーを提供してユーザーを奪うリスク。

## 2. 防御戦略：コードを超えた「価値の独占」

### A. ネットワーク外部性の構築（コミュニティ資産）
- コードはコピーできても、**「そこにいる読者」と「積み上げられた作品」**はコピーできません。
- pixivやYouTubeのように、ユーザーが互いに評価し、交流するソーシャルデータこそが最大の防御壁となります。

### B. 商用エクスポート・公式リーダーの優位性
- **クローズドな商用ビルド環境**: Web版リーダーはOSS化しても、Nintendo Switch等のコンソール機や高度に最適化されたネイティブアプリ用リーダーは非公開（プロプライエタリ）とします。
- **公式認証の価値**: コンソール機でのリリースにはプラットフォーマー（任天堂等）との契約と技術的信頼が必要です。公式PFが提供する「公式SDKとパブリッシング支援」は、模倣サイトには提供不可能な価値となります。

### C. リーダーの「性能差」による差別化
- OSS版リーダーは標準的な機能に留め、公式PFで提供するリーダーには「低消費電力」「高速ロード」「クラウドセーブ同期」などの高度な機能を搭載し、**「公式PFで読むのが最高の体験」**という状態を作ります。

## 3. 攻めの防御：デファクトスタンダード化
- 模倣を恐れるのではなく、kaedevnのスクリプト形式（.ksc）を世界標準のノベルゲーム形式に押し上げます。
- 形式が普及すればするほど、最も多くのユーザーが集まる「本家プラットフォーム」の価値が相対的に高まります。

## 結論
プラットフォームの模倣に対する最強の防御は、**「制作の入り口（OSS）を広く開放してシェアを奪い、出口（コンソール・商用展開）と場（コミュニティ）の価値を公式が独占する」**というハイブリッド戦略です。
「技術のコピー」は許容しつつ、「成功への最短ルート」としてのプラットフォームの地位を確立することで、模倣リスクを無効化できます。
