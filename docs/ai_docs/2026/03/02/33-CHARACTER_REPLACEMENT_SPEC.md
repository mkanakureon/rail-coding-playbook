# キャラクター入れ替えシステム（ch_new）の仕様と役割

## 1. 概要
本仕様における `ch_new` は、**「特定の立ち位置（左・中央・右）において、現在表示されているキャラクターを別のキャラクターへ滑らかに入れ替える」**ための暫定的なスプライト枠を指します。

表情差分の切り替えとは異なり、**「異なるキャラクターアセット同士の交代」**を視覚的に美しく行うことが目的です。

## 2. 実装の目的：立ち位置（ポジション）の継承
ノベルゲームでは、同じ位置（例：中央）にいるキャラAを消してキャラBを出す演出が多用されます。
この際、`bg_new` と同様の考え方を導入することで、以下のメリットが得られます。

- **チラつきの防止**: Aを消した瞬間に背景が透けて見えるのを防ぎ、Bが重なりながら現れるようにする。
- **配置の競合回避**: 内部管理 ID が重複しないように、入れ替え中のみ `ch_new`（または `pos_center_new` 等）という別枠で処理する。

## 3. キャラクター入れ替えのフロー（例：中央位置での交代）

1. **現状**: 中央（Center）にキャラAが表示されている。
2. **コマンド実行**: `@ch キャラB ポーズ 中央 fade 500`
3. **`ch_new` の生成**: 
   - キャラBを `ch_new` という一時的なキーで、中央位置に透明度 0% で作成する。
4. **クロスフェード**:
   - キャラA（旧）: 1.0 → 0.0 へフェードアウト。
   - `ch_new`（新キャラB）: 0.0 → 1.0 へフェードイン。
5. **交代完了**:
   - キャラAをメモリから削除。
   - `ch_new` のキーを、正規のキャラクター名（キャラB）に変更して確定。

## 4. 背景遷移（bg_new）との構造的な一致
この「入れ替え」としての `ch_new` は、`bg_new` と全く同じロジックで実装可能です。

| 項目 | 背景遷移 (bg_new) | キャラ入れ替え (ch_new) |
| :--- | :--- | :--- |
| **トリガー** | 次の背景指定 | 同じ位置への次のキャラ指定 |
| **一時枠** | `bg_new` | `ch_new` |
| **演出** | 画面全体を覆う | 特定の立ち位置を上書きする |
| **完了後** | `bg` に昇格 | キャラ名IDに昇格 |

## 5. 実装上の注意：複数ポジションの考慮
背景は1つしかありませんが、キャラクターは「左・中央・右」の3箇所にいる可能性があります。
そのため、実際の実装では `ch_new` という単一の名前ではなく、**「入れ替わる対象のポジション」**を意識した管理が必要になります。

```typescript
// イメージ
const oldCharAtPosition = this.findCharacterAt(pos); // 指定位置に誰かいるか？
if (oldCharAtPosition && fadeMs > 0) {
  // その位置で ch_new 方式の入れ替えを実行
}
```

## 結論
「違うキャラへの交代」としての `ch_new` は、エンジンの演出密度を背景演出と等しく高めるための重要なパーツです。`bg` における成功パターン（新旧共存 → アニメーション → 名前付け替え）をキャラクター表示にも適用することで、一貫性のある高品質な描画システムが構築できます。
