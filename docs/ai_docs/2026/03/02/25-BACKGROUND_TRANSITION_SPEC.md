# 背景遷移システム（bg_new）の仕様と機能解説

## 1. 概要
`bg_new` は、エンジンの描画層（WebOpHandler）において、**「古い背景」から「新しい背景」へ滑らかに遷移（トランジション）させるための一時的な作業用スプライト**です。

通常、背景を表示するスプライトのキーは `"bg"` ですが、遷移中のみ `"bg_new"` という別のキーを持つスプライトを生成することで、画面上に新旧2つの背景を同時に存在させ、アニメーションを可能にしています。

## 2. 主な演出機能

### A. クロスフェード (Cross-fade)
1. **旧背景 (`bg`)**: 不透明度 100% で表示中。
2. **新背景 (`bg_new`)**: 不透明度 0% で生成し、旧背景の上に重ねる。
3. **アニメーション**: 
   - `bg` を 100% → 0% へ。
   - `bg_new` を 0% → 100% へ。
4. **完了**: `bg` を削除し、`bg_new` のキーを `"bg"` に書き換えて常駐させる。

### B. スライド遷移 (Slide-in / Slide-out)
1. **新背景 (`bg_new`)**: 画面外（左または右）に配置。
2. **アニメーション**: 
   - `bg` を画面外へ移動。
   - `bg_new` を画面中央（0, 0）へ移動。
3. **完了**: フェードと同様にキーを `"bg"` に入れ替える。

## 3. 現在の技術的課題と不具合

調査（21-PIPELINE_CONSISTENCY_AUDIT.md）により、以下の問題が特定されています。

### レイヤー配置のミス
現在、`bg_new` は `WebOpHandler.ts` 内で以下のように処理されています。
- `isBackground` (`id === "bg"`) → `backgroundLayer` へ追加。
- **それ以外** → `characterLayer`（キャラ層）へ追加。

**結果として起こる問題**:
`bg_new` は `id === "bg"` ではないため、**キャラクターよりも前面（characterLayer）に表示されてしまいます。** 
このため、背景が切り替わる瞬間にキャラクターが一時的に背景の下に隠れてしまい、不自然な見た目になります。

## 4. 修正案
`WebOpHandler.ts` の `show()` メソッド内の判定ロジックを以下のように修正する必要があります。

```typescript
// 修正前
const isBackground = id === "bg";

// 修正後
const isBackground = id === "bg" || id === "bg_new";
```

これにより、遷移中の背景も正しく `backgroundLayer` に配置され、キャラクターの背後で安全にアニメーションが行われるようになります。

## 結論
`bg_new` は「消えるもの」と「現れるもの」を共存させるために不可欠な仕組みです。適切なレイヤー管理を行うことで、ユーザーに違和感を与えない高品質な背景演出を実現できます。
