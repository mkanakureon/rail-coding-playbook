# 既存機能の調査報告：クリック・プレビュー同期（CPC）の実装状況

## 1. 調査結果
提案していた「エディタのブロックをクリックするとプレビューが更新される」機能（CPC）は、既にリポジトリ内に高度な形で実装済みであることを確認しました。

### 実装の仕組み
- **イベント検知**: `EditorPage.tsx` の `onClick` ハンドラ（`editor-main` 領域）にて、クリックされた要素の `data-block-id` を取得し、`selectedBlockId` を更新しています。
- **データ生成**: `useEditorStore.ts` の `buildSnapshotScript(blockId)` が、該当ブロックまでの演出状態を累積した再現スクリプトを生成します。
- **通信と反映**: `SidebarPreview.tsx` が `selectedBlockId` の変化を検知し、`postMessage` 経由でプレビュー用 `iframe` へスクリプトを送信。プレビューエンジンがそれを即座に実行します。

## 2. 実装の優れた点
- **`previewTick` による強制更新**: 同じブロックを再度クリックした場合でも、`previewTick` カウンタをインクリメントすることで、確実にプレビューを最初から再生し直す工夫がなされています。
- **マルチレイヤー対応**: 背景、キャラクター、オーバーレイ、フィルター、タイムラインの全ての状態が正しく累積・再現されるようになっています。

## 3. さらなる改善の余地（地味だが効くブラッシュアップ）
既に基盤が完成しているため、以下の微調整を行うことで、より「成功」に近づくことができます。

### A. 編集時の「自動ライブ更新」の強化
現状は「クリック時」に更新されますが、**「入力中（文字を打っている最中やスライダーを動かしている最中）」** にも `previewTick` を回さずに内容だけを `postMessage` する「Live Link」を強化することで、よりシームレスな制作体験になります。

### B. スナップショット再生の高速化（演出スキップ）
スナップショット実行時に、`fadeMs` などの待機時間を強制的に `0` にするフラグをエンジン側に渡すことで、クリックした瞬間に「最終結果の静止画」を表示するようにし、体感速度を向上させます。

### C. ビジュアル・ハイライトの同期
プレビュー画面内の特定のオブジェクトをクリックした際、逆にエディタ側のブロックをスクロール・強調表示する「逆方向の同期」はまだ実装されていないため、今後の拡張項目として有望です。

## 結論
「クリックでその時点を再現する」という核心的な機能は既に最高水準で実装されています。この OSS の強みとして、この既存機能を「当たり前のもの」としてさらに磨き上げ、AI エージェントがこの仕組みを使いこなして自律デバッグする姿を見せることが、成功への近道となります。
