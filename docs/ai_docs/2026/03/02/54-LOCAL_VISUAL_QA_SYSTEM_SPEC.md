# ローカル自動ビジュアルQAシステム：仕様・設計書

## 1. システム概要
本システムは、ユーザー（作者）のローカル環境において、演出の変更前後のスクリーンショットを自動比較し、意図しない表示の崩れ（ビジュアル回帰）を検出・通知する「開発者のための安全網」です。

## 2. 主要コンポーネント

### A. キャプチャ・エンジン (`packages/web/scripts/qa-capture.ts`)
Playwright を使用し、指定されたシーン（ページ）のスクリーンショットを撮影します。
- **入力**: プロジェクトID、対象ページ、出力パス。
- **仕様**: 演出（フェード等）の完了を待機してから撮影する「インテリジェント・ウェイト」を搭載。

### B. ビジュアル比較モジュール (`packages/tools/src/visual-diff.ts`)
2枚の画像をピクセル単位で比較し、差異（差分）を抽出します。
- **技術**: `pixelmatch` ライブラリ（または同等のアルゴリズム）を使用。
- **出力**: 
  - 差異の割合（%）。
  - 差異部分を赤くハイライトした「Diff画像」。

### C. 状態管理（Baseline）
「正解の見た目」を Baseline 画像として保存します。
- **保存場所**: `projects/[id]/.qa/baselines/page_[n].png`
- **更新タイミング**: 作者が「この演出でOK」と承認した際、Baseline を上書き。

## 3. ワークフロー（自律デバッグ）

1. **ベースライン作成**: 
   - 作者が現在の演出を「完成」とする。
   - `npm run qa:approve` を実行し、現状を Baseline として保存。
2. **演出の変更・実験**: 
   - 作者がエディタで演出を調整。
3. **自動QA実行**: 
   - `npm run qa:check` を実行（または保存時に自動実行）。
   - 新しいスクリーンショットを撮影し、Baseline と比較。
4. **結果レポート**:
   - 差異がしきい値（例: 0.1%）を超えた場合、Diff画像を表示。
   - 「背景の座標が以前と 5px ズレています」といった警告を出す。

## 4. 設計のポイント

### A. 演出完了の判定
単なる `waitForTimeout` ではなく、エンジンの内部状態（`WebOpHandler` のアニメーション完了フラグ）を DOM メタデータ経由で監視し、最適なタイミングで撮影します。

### B. ヘッドレス実行の最適化
作者の作業を妨げないよう、GPUアクセラレーションを有効にしたヘッドレスブラウザで高速に実行します。

### C. UIへの統合（将来）
エディタの画面上に「QAステータス」ランプを設置。
- 🟢 緑: 前回承認時と変化なし。
- 🟡 黄: 微小な変化（意図的な調整の可能性）。
- 🔴 赤: 大きな変化（崩れの可能性）。

## 5. 実装ロードマップ
1. **フェーズ1**: Playwright による単一シーンのキャプチャスクリプト実装。
2. **フェーズ2**: `pixelmatch` を用いた画像比較 CLI の作成。
3. **フェーズ3**: エディタ（Store）との連携による、変更検知時の自動 QA 実行。

## 結論
このローカルQAシステムは、作者に「演出を大胆に変更しても、いつでも以前の状態に戻れる・確認できる」という自由を与えます。Web技術のポータビリティを活かし、人間とAIが協力して品質を守るための基盤となります。
