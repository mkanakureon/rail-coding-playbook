# シナリオ作者のための KS 背景コマンド（@bg）活用ガイド

## 1. 概要
シナリオ作者が KS スクリプトで記述する `@bg` コマンドは、エンジンの内部で `bg`（今の画面）と `bg_new`（次の画面）を自動的に操作し、滑らかな画面切り替えを実現します。

作者は内部の複雑な仕組みを意識することなく、パラメータを指定するだけで多彩な演出が可能です。

## 2. 基本的な記述例

### A. 瞬時に切り替える
フェード時間を指定しない場合、古い背景は消え、新しい背景が即座に表示されます。
```ks
@bg bedroom
```

### B. フェードで切り替える
`fade` とミリ秒を指定すると、内部で `bg` と `bg_new` が重なり合いながら入れ替わります。
```ks
@bg classroom fade 1000
```
- **内部の動き**: 1秒かけて今の背景が消え、新しい「教室」が現れます。

### C. スライド（流れる演出）で切り替える
`slide_left` や `slide_right` を使うと、背景が横に流れて入れ替わります。
```ks
@bg sea slide_left 500
```
- **内部の動き**: 今の背景が左へ去り、右から「海」がスライドしてきます。

## 3. 演出の組み合わせ（テクニック）

### キャラクターを表示したまま背景だけ変える
kaedevn の `@bg` は、キャラクター層（characterLayer）とは独立した背景層（backgroundLayer）を操作するため、立ち絵を表示したまま背景だけを演出付きで変えられます。

```ks
#hero
「よし、学校へ行こう」
@bg classroom fade 800
#hero
「着いたぞ」
```

### 背景を消して真っ暗にする
ID を指定せずに `@bg` だけを書くと、現在の背景をクリア（消去）します。
```ks
@bg fade 500
```
- **効果**: 暗転（ブラックアウト）のような効果が得られます。

## 4. 注意点：アセットIDの指定
`@bg` の後に続く単語（`bedroom`, `classroom` 等）は、プロジェクトに登録した **背景アセットの ID またはスラッグ** である必要があります。
存在しない ID を指定すると、画面にエラー（×印のプレースホルダー）が表示されます。

## 結論
KS スクリプトの `@bg` コマンドは、非常にシンプルな記述でプロフェッショナルな画面遷移を可能にします。`fade` や `slide` を使い分けることで、物語のテンポに合わせた最適な演出を行ってください。
