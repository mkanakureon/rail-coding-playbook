# ノベルゲームエンジンの市場戦略分析：成功しているOSS事例との比較

## 概要
kaedevnが採用しようとしている「オープンコア・モデル（OSSコア ＋ 非公開プラットフォーム）」が、既存の成功事例と比較して妥当であるかを分析しました。

## 1. 類似OSSプロジェクトの構成例

### A. WordPress (Automattic)
- **構成**: コアエンジンは完全OSS。大規模なホスティング・投稿サイト（WordPress.com）は非公開。
- **一致点**: 「配布・設置は自由だが、便利な場は提供側が管理する」という、kaedevn-cloudの戦略と一致。

### B. Godot Engine
- **構成**: エンジン・エディタはOSS。コンソール（Switch/PS5等）向けのエクスポート機能は、NDAの制約により非公開またはサードパーティ提供。
- **一致点**: 「Switchビルドを非公開リポジトリ（kaedevn-cloud）で管理する」という物理的制約の解決策と一致。

### C. Live2D
- **構成**: SDK（実行部）を広く配布し、制作ツール（Cubism Editor）のライセンスで収益を得る。
- **一致点**: 「高品質な制作体験」をビジネスの核とする姿勢と一致。ただしkaedevnはエディタをOSS化することで、より強力なコミュニティによる進化を目指す。

## 2. kaedevn 構成の優位性

### A. AIエージェントへの最適化
既存の古いプロジェクトと異なり、最初から AI（Claude Code等）による多リポジトリ横断開発を前提とした構成（Markdownベースのコンテキスト配置等）を採用しており、少人数での開発スピードが圧倒的です。

### B. 制作ツールの完全OSS化
競合（Live2D等）が制作ツールを非公開にしているのに対し、kaedevnはエディタをOSSとして開放することで、ユーザーが自身のワークフローに合わせてツールを改造できる「究極の自由度」を提供します。

### C. 「WebからSwitchへ」の一貫した導線
「Webでの拡散（OSS）」から「Switchでの製品化（Cloud）」までの技術的な障壁（アセット変換、ビルド）を最小限にするユニバーサル・ランタイム戦略は、既存エンジンに対する強力な差別化要因となります。

## 結論
kaedevnの構成案（OSS Studio/Engine + Private Cloud Builder）は、歴史的な成功事例の「良いとこ取り」をしつつ、最新のAI開発トレンドを組み込んだ、非常に競争力の高いアーキテクチャであると評価できます。
