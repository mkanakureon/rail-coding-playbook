# ノベルゲームエンジンの背景管理比較：kaedevn vs ティラノスクリプト (KAG)

## 概要
kaedevnの `bg_new` 方式が、業界標準であるティラノスクリプト（吉里吉里/KAG）の設計思想といかに一致しているかを分析しました。

## 1. ティラノスクリプトの「表（fore）」と「裏（back）」
ティラノスクリプトでは、背景（baseレイヤ）を含む各レイヤに「表」と「裏」の2つの状態を持たせています。
- **fore (foreground)**: ユーザーに見えている現在の画面。
- **back (background)**: 次の変化を準備するための隠れた画面。

## 2. 背景切り替えのプロセス比較

### kaedevn (`bg_new` 方式)
1. 現在の背景を `bg` として表示。
2. 新しい背景を `bg_new` に読み込む。
3. `bg` と `bg_new` の間でフェード等のアニメーションを実行。
4. 完了後、`bg_new` を `bg` へと役割変更（名前の付け替え）する。

### ティラノスクリプト (`fore/back` 方式)
1. 現在の背景を `fore` に表示。
2. 新しい背景を `back` に読み込む。
3. `trans`（トランジション）命令により、`fore` と `back` を入れ替える演出を実行。
4. 完了後、入れ替わった状態が新しい `fore` となる。

## 3. 共通する設計の利点
両者がこの「二面構造」を採用している理由は共通しています。
- **チラつきの防止**: 画像のロード待ち時間を「裏」で吸収するため、ユーザーに読み込み中を見せることがありません。
- **高度な演出**: 2枚の画像を同時に保持することで、クロスフェードだけでなく、ワイプや複雑なシェーダーによるトランジションが可能になります。
- **状態の確定**: アニメーションが完全に終わるまで「今の主役」を維持できるため、演出の途中で不整合が起きにくくなります。

## 結論
kaedevn の `bg_new` は、ティラノスクリプト等の先行エンジンが長年培ってきた「確実な画面遷移」のノウハウを、現代的なスプライト管理（PixiJS）の文脈で再実装したものです。この設計は、プロレベルの演出を目指す上で非常に正しい選択です。
