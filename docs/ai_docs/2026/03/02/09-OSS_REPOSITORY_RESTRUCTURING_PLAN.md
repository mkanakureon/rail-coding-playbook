# kaedevn-monorepo OSS化およびリポジトリ再編計画：個人開発環境の確立

## 概要
kaedevnを個人向け開発環境（OSS）と、商用プラットフォーム（非公開）に分離し、開発スピードと収益性を両立させるためのリポジトリ再編案です。

## 1. リポジトリ分割構成

### A. `kaedevn-engine` (OSS / SDK)
**【実行コア】** ノベルゲームを動かすための純粋なライブラリ。
- **構成**: `core`, `web` (PixiJS Renderer), `interpreter`, `compiler`, `battle`.
- **提供価値**: 開発者が自分のWebサイトやアプリにノベルエンジンを組み込むための基盤。

### B. `kaedevn-studio` (OSS / Local Development Environment)
**【制作環境】** ユーザーのローカルPCで動作するエディタおよび管理ツール。
- **構成**:
  - `apps/editor`: GUIエディタ。
  - `apps/hono-local`: ローカルファイルシステム操作用API。
  - `apps/dashboard-lite`: ローカルプロジェクト管理（マイページ）機能。
- **提供価値**: インストール不要（ブラウザ実行）で、セキュアかつ高速に自分のPCでゲームを制作できる環境。

### C. `kaedevn-cloud` (非公開 / Proprietary)
**【配信・商用プラットフォーム】** ユーザーの作品を公開し、収益化するための基盤。
- **構成**:
  - `apps/next`: 投稿サイト、SNS、ランキング、ユーザー認証。
  - `apps/reader`: プラットフォーム専用の高機能プレイヤー。
  - `services/build`: Switch/iOS/Android向けネイティブビルドパイプライン。

## 2. OSS版における機能範囲
OSS版（kaedevn-studio）では、以下の機能を完全に開放します。
- **プレイ画面**: 最新のPixiJSベースの描画エンジンによるプレビュー。
- **エディタ画面**: 全てのビジュアル編集機能（タイムライン、キャラ配置等）。
- **マイページ**: ローカルPC内の複数プロジェクトの作成・管理。
- **テスト実行**: 自分のブラウザ（localhost）での完全な動作確認。
- **ローカルアセット利用**: PC内の画像・音声ファイルの読み込みとJSON出力。

## 3. 技術的実装戦略：AIネイティブな開発
- **AIコンテキストの標準化**: `CLAUDE.md` や `GEMINI.md` を各パッケージに配置し、AIエージェント（Claude Code等）が即座にコード修正や機能追加を行える状態を維持します。
- **抽象化レイヤーの導入**: ストレージ（Local FS vs Cloud Blob）やDB（SQLite vs PostgreSQL）をインターフェース化し、OSS版とクラウド版で同一のコードベースを共有します。

## 4. ビジネスモデルとの整合
- **OSSの役割**: ユーザーベースの拡大、エディタのUI/UX改善、バグ修正のコミュニティ化（AIエージェントの活用）。
- **非公開版の役割**: 「作品の出口（配信場所）」と「商用化（ビルドサービス）」の独占による収益化。

## 結論
リポジトリを「エンジン(SDK)」「スタジオ(制作環境)」「クラウド(配信基盤)」に切り分けることで、個人の創作自由度を最大化しつつ、プラットフォームとしての競争力を保護します。AIエージェントが活躍しやすいOSS構造を作ることで、少人数でも世界レベルの進化を遂げるエンジンを目指します。
