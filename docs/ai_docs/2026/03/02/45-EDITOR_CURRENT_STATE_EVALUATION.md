# エディタ実装状況の精査に基づくOSS成功戦略：既存資産の最大化と未実装項目の特定

## 1. 現状評価：すでに実装・着手済みの項目
リポジトリの精査により、以下の機能はすでに高い完成度で存在、あるいは土台が構築されていることを確認しました。

- **インメモリ・プレビュー**: `useEditorStore` のスナップショット生成機能により、DB保存なしでのプレビュー基盤は完成済み。
- **マルチプラットフォーム基盤**: `packages/core` で描画（Op）とロジックが分離されており、Webベースでのライブデバッグが容易な構造。
- **アセット管理**: アセット定義とメタデータ管理の仕組みは `hono` および `editor` の双方に実装済み。

## 2. Web構成の優位性を活かした「一歩先」の地味な改善提案
既存の実装を最大限に活かしつつ、OSS利用者が「これは便利だ」と唸る、未実装の地味な機能を提案します。

### A. 双方向ビジュアル・トレーサー
- **内容**: プレビュー画面とエディタの完全同期。
- **機能**: 
  - プレビュー画面内のキャラクターをマウスオーバーすると、エディタ側の対応する `ch` ブロックが光る。
  - 逆に、エディタのブロックを操作した瞬間に、プレビュー側の該当スプライトがアニメーションして強調される。
- **優位性**: 同一ブラウザ・同一モノレポ内での `postMessage` 連携により、リロード不要で実現可能。

### B. アセット・インテリジェンス（リファレンス管理）
- **内容**: 素材の「死蔵」と「破壊」を防ぐ。
- **機能**: 
  - **使用箇所検索**: 「この背景画像、どこで使ってる？」をエディタ内で一瞬で検索。
  - **削除ガード**: 使用中のアセットを消そうとした際に警告を出し、依存関係を壊さないようにする。
- **優位性**: JSON形式のプロジェクトデータをメモリ内で高速スキャンできるWeb技術の強みを活用。

### C. ポータブル・プロジェクト・エクスポート
- **内容**: 「環境」ごと作品を渡せる体験。
- **機能**: 
  - 全アセットとスクリプトを単一のパッケージファイル（ZIP等）として書き出し、他のPCの kaedevn-studio で即座にインポート・再開できる機能。
- **優位性**: ブラウザの `showSaveFilePicker` 等のモダンAPIを活用し、ローカルだけで完結。

## 3. 実装の優先順位とロードマップ
1. **短期**: 「アセット使用箇所検索」の実装（既存のStoreに検索ロジックを追加するだけなので低コスト）。
2. **中期**: 「双方向ビジュアル・トレーサー」のUI実装。
3. **長期**: ローカル完結型エクスポート機能。

## 結論
現在の kaedevn は、すでに「賢いエンジン」としての素養を十分に備えています。新しく派手な機能を足すよりも、**「今あるエディタとプレビューの距離をさらに縮める」** ことに注力することで、世界で最も「手になじむ」制作環境へと進化できます。
