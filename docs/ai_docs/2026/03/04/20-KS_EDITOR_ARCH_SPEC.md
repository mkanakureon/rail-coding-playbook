# 20-KS_EDITOR_ARCH_SPEC

## 1. 概要
KSエディタは、Kaede Script (.ks) を唯一の正解（Single Source of Truth）として扱う、ハイブリッド型のシナリオ編集環境である。テキスト編集の自由度と、GUIによる視覚的な微調整の両立を目指す。

## 2. アーキテクチャ

### 2.1 データフロー（一方向同期）
1.  **Input**: ユーザーが `.ks` テキストを編集またはコピペ。
2.  **Parse**: サーバーまたはフロントエンドで KS を AST（抽象構文木）に変換。
3.  **Generate**: AST から以下の2つを自動生成。
    - **GUI JSON**: ブロックUI表示用の構造データ。
    - **Scenario IR**: エンジン実行用の最適化済み JSON。
4.  **Feedback**: GUI で値を変更した場合、対応する AST の行を更新し、`.ks` テキストを再生成する。

### 2.2 KS 文法の定義（最小セット）
- **コマンド形式**: `@command key=value key2="string value"`
- **テキスト形式**: 行頭が `@` でない場合はセリフ（地の文）として扱う。
- **制御構文**:
  - `@label name` (ジャンプ先)
  - `@jump target=label` (分岐)
  - `@if condition`, `@else`, `@endif` (条件分岐)

## 3. エディタ UI コンポーネント
- **Text Layer**: Monaco Editor または CodeMirror。シンタックスハイライト（@コマンドの色分け）を実装。
- **Block Layer**: KSから変換されたブロック表示。各ブロックは個別に編集可能なプロパティパネルを持つ。
- **Asset Panel**: 使用可能な背景・キャラIDを選択し、カーソル位置に `@bg` などのコマンドを挿入する支援機能。

## 4. バリデーション
- **構文チェック**: 未定義のコマンドや不正な引数形式をリアルタイムで検出。
- **資産チェック**: `@bg` や `@ch` で指定された ID が実際に存在するかを RAG または DB と照合。
