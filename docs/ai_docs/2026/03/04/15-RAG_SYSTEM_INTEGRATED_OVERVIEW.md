# 15-RAG_SYSTEM_INTEGRATED_OVERVIEW

## 1. システムの全体像
本プロジェクトで構築されたハイブリッド検索RAG（Retrieval-Augmented Generation）システムは、モノレポ内の膨大な知識（設計書、実装記録、過去の意思決定）を統合し、エージェント（AI）が「熟練エンジニア」のような文脈理解を持って開発を遂行できるようにする基盤である。

## 2. 実装の主要コンポーネント
- **ハイブリッド検索エンジン**: `pgvector` による意味検索と、`tsvector` によるキーワード検索を 0.6 : 0.4 の比率で統合。
- **自律型インデクサ**: Markdown の見出し構造を解析し、`content_hash` による高速な差分更新を実現。
- **DB分離アーキテクチャ**: `kaedevn_rag` データベースを新設し、メインDBのパフォーマンスを保護。
- **エージェント最適化インターフェース**: 検索結果に `read_file` アクションを付与し、エージェントの次の行動を直接誘導。

## 3. 現状のステータス（2026/03/04）
- **インデックス済みファイル数**: 658件
- **検証済みコアケース**: 10件（IEngineAPI, Tween, JWT等）
- **平均検索精度**: 高（Rank 1 に設計意図が集中）
- **インフラ**: Docker コンテナ（Postgres 16 + pgvector）にて安定稼働中。

## 4. 開発・運用へのインパクト
エージェントは「どこに何があるか」を人間に聞く必要がなくなり、自ら `search_docs` ツールを介して情報を収集し、正確な実装を提案できるようになった。これは、開発のターン数削減とコンテキスト窓の劇的な節約に直結している。

## 5. 今後の展望
- **コードベースのインデックス化**: `.ts` ファイル等のソースコード自体をチャンク化し、より実装に密着した検索を可能にする。
- **Claude Code 完全統合**: MCP サーバーの実装により、Claude Code の標準ツールとして提供する。
- **自動再インデックス**: CI/CD パイプラインや Git Hook への組み込み。
