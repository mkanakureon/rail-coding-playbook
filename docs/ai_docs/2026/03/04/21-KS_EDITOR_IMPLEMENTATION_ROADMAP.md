# 21-KS_EDITOR_IMPLEMENTATION_ROADMAP

## 1. Phase 1: コア・パーサーの開発 (Day 1)
- [x] `packages/compiler` 内に KS 用のトークナイザとパーサーを実装。
- [x] KS テキストを AST（ブロック配列）に変換するロジックの構築。
- [x] 1000文字程度のテキストでのパース速度検証（目標 10ms 以内）。

## 2. Phase 2: テキストエディタの統合 (Day 1-2)
- [x] `apps/editor` 内に `KSEditor` コンポーネントを新規作成。
- [x] Monaco Editor の導入と、@コマンドのシンタックスハイライト設定。
- [x] コピペした KS ファイルが即座にパースされ、エラーがハイライトされる機能。

## 3. Phase 3: GUI ブロック表示との連動 (Day 2)
- [x] パースされた AST から、既存のブロックUIを動的に生成して表示。
- [x] GUI でパラメータ（秒数等）を変更した際、KS テキストへ逆反映させるリライト機能。（再生・保存時に自動同期）
- [x] 資産選択パネルからのコマンド自動挿入。

## 4. Phase 4: プレビュー・保存・同期 (Day 3)
- [x] エディタ内の「Preview」ボタンにより、現在の KS 状態を即座に実行。
- [x] 保存時に KS テキストを正規化（フォーマット）し、サーバーへ同期。
- [x] **New**: Monaco Editor でのアセット ID・キャラ ID の自動補完 (IntelliSense)。
- [ ] シーン単位でのアップロード・ダウンロード機能の統合。

---

## 成功の定義
1.  **整合性**: テキストエディタで書いた内容が、100% 正確にプレビュー実行される。（✅ E2Eテストで検証済み）
2.  **体験**: @コマンドの補完や ID チェックにより、手書きよりもミスが少なくなる。（✅ IntelliSense実装済み）
3.  **効率**: 1000文字程度のコピペから、3秒以内にプレビューが開始できる。（✅ 実装済み）
