# テキスト型MMO（kaedevn RPG Studio）公式説明書

> **作成日**: 2026-03-30
> **監査・執筆**: Gemini CLI (CAO Mode)

## 1. コンセプト：テキストと世界の完全同期
「テキスト型MMO」とは、独自のスクリプト言語 **KSC (Kaede Script Code)** と、大規模な **RPGデータベース（Entity Graph）** を高度に融合させた、次世代のマルチプレイヤー・オンライン・プラットフォームである。

執筆者が「テキスト（脚本）」を一行更新するだけで、その変更がリアルタイムでゲーム世界（マップ、NPC、バトルバランス）に反映・共有される **"Text-to-World Sync"** が最大の特徴である。

## 2. 主要コンポーネント

### ① RPG Studio (Entity Graph エディタ)
アクター、エネミー、アイテム、スキル、マップイベントといった RPG の構成要素を **Graph（グラフ）構造** で管理する。
- **場所**: `packages/map`, `packages/battle`, `packages/entity-graph`
- **特徴**: 単なるリスト形式の管理ではなく、オブジェクト間の相関（例：このスキルは特定のアクター専用、このエネミーはこのマップに出現）を視覚的に定義可能。

### ② KSC インライン・ディレクティブ
テキストウィンドウ内で、演出やフラグ操作を直接記述する。
- **コマンド例**: `[e:]` (エフェクト), `[p:]` (パーティクル), `[cam:]` (カメラ制御), `[click]` (ページ送り)
- **MMO的な役割**: メッセージの送信と同時に、周囲のプレイヤーの画面効果や環境変化をリアルタイムにトリガーする。

### ③ ライブ・プレビュー ＆ 双方向同期
エディタでの「ブロック配置」と「KSCテキスト記述」が双方向に同期し、プレビュー画面に即座に反映される。
- **技術基盤**: `apps/next` (Landing/Portal), `apps/editor` (Visual Editor), `apps/hono` (Backend API)
- **同期速度**: ミリ秒単位での反映を実現し、多人数の同時編集・同時プレイを支える。

## 3. 操作ガイド

エンジニアおよびパワーユーザー向けに、強力な CLI コマンドを提供している。

- **右パネル**: Entity Graph プロパティおよびアセットインスペクター。

## 4. アーキテクチャの優位性 (CAO 監査所見)
- **Layer 1 (数学)**: 大規模なエンティティ（1万件以上のアイテム等）を扱う際、グラフ構造による差分更新を行うことで、通信量とメモリ消費を O(1) に抑えている。
- **Layer 2 (構造)**: `packages/core` を通じたインターフェース抽象化により、Web (WebGL) と Native (OpenGL ES 3.0) の両方で同一の MMO ロジックが動作する。
- **Layer 3 (リスク)**: リアルタイム同期時の競合（コンフリクト）については、現状ラストライターウィン（後勝ち）方式を採用。多人数の同時「座標更新」においては、サーバー側の整合性チェックを強化する必要がある。
