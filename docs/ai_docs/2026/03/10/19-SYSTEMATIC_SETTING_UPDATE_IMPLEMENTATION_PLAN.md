# システム的設定更新機能 実装計画書

## 1. 目的
AI レビューの結果（JSON パッチ）を、Markdown の構造を維持したまま、かつナラティブな文章を含まない「純粋な要件データ」として設定ファイルに自動反映する仕組みを構築する。

## 2. 実装詳細

### A. upstream-cli.ts の拡張
- **新コマンド**: `apply-review`
- **引数**:
    - `--input <patch.json>`: 修正要件が並んだ JSON ファイル。
    - `--target <file_path>`: 更新対象の Markdown ファイル（フルパス）。
- **内部プロンプト**: 「ドキュメントエンジニア」としての役割をハードコード。
    - 箇条書き形式（- **要件**: 内容）の強制。
    - 台詞や描写の執筆を厳禁する制約。

### B. ワークフローの標準化
1. 開発者がレビュー結果から `patch.json` を作成（または AI に生成させる）。
2. `apply-review` を実行。
3. Markdown が「データ」として更新され、履歴が担保される。

## 3. 安全性の担保
- 更新前に必ずファイルを読み込み、API エラーが発生した場合は書き込みを中断する。
- `prompts.ts` などの共通ライブラリは変更せず、CLI ツール側の独立したロジックとして実装する。

## 4. スケジュール
1. 本計画書の作成（完了）。
2. `upstream-cli.ts` へのコマンド実装。
3. 変更内容のコミット（実行前）。
4. `fantasy` プロジェクトでの試験運用。
