# シナリオ生成・統合開発計画書 (2026-03-10)

## 1. 概要
これまでの Gemini Vision による画像解析の成果を基盤とし、最新のレビューに基づいた「感覚描写強化プロンプト」を用いて、ファンタジープロジェクトのシナリオ品質を商用レベルまで引き上げる。また、開発プロセスの透明化（デバッグ情報の保持）と資産管理の統合を同時に進める。

## 2. 実施フェーズ

### Phase 1: 画像解析の完遂とDB同期
- **内容**: 実行中の 561 件の一括解析を完了させ、`official_assets` テーブルを最新状態にする。
- **成果物**: 全アセットの `fromVision` メタデータ、キーワード検索可能なタグ群。

### Phase 2: 感覚描写を強化したシナリオ生成
- **内容**: 強化済みプロンプト（五感・対比の指示追加）を使用し、`projects/fantasy` のシナリオを再生成または続行する。
- **重点**:
    - [s_04_WORLD] の感覚パラメーター（匂い、温度）のト書きへの反映。
    - 「かつての泉」と「枯れた現在」のような対比描写の強調。
- **成果物**: `_ai_debug`（プロンプト情報）付きの Stage 3 JSON 群。

### Phase 3: 資産リストの統合と検証
- **内容**: 修正済み CLI を使用して `assets_list.json` を生成し、プロジェクトで使用される全リソースを可視化する。
- **成果物**: `assets_list.json` (背景・キャラの統合リスト)。

### Phase 4: 品質比較レポートの作成
- **内容**: 以前の出力と強化後の出力を比較し、ナラティブの質（没入感、一貫性）を評価する。

## 3. スケジュール
1. **本日午前**: 一括解析の完了確認、DB更新。
2. **本日午後**: `projects/fantasy` の Stage 3 実行（感覚描写強化版）。
3. **本日夕刻**: 統合資産リストの生成、比較レポートの作成。

## 4. 成功の定義
- すべての JSON 出力に生成時のプロンプトが記録されていること。
- シナリオのト書きに、視覚以外の感覚（匂い・温度・音）が 1 つ以上含まれていること。
- 背景・キャラの資産リストが 1 つの JSON で出力され、開発資料として利用可能であること。
