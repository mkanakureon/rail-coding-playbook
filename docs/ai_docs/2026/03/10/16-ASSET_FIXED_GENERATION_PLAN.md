# 資産確定型シナリオ生成 実施計画書

## 1. 目的
Gemini Vision による画像解析データを活用し、シナリオ生成前に使用アセットを特定・固定することで、ゲームエンジンへの統合コストを最小化し、演出の説得力を最大化する。

## 2. 実施ステップ

### Step 1: 資産の選定と ID 抽出
- **内容**: 企画(S00)に基づき、必要な背景とキャラクターを DB からキーワード検索。
- **ツール**: psql または検索用スクリプト。
- **目標**: 登場する全場所・全キャラに 1 つ以上の `official_asset.id` を割り当てる。

### Step 2: 設定ファイルへのマッピング（資産固定）
- **内容**: `s_02_characters.md` および `s_04_world.md` に `assetId` 項目を追加し、Step 1 で選定した ID を書き込む。
- **効果**: これにより、設定ファイルが「ゲーム資産の構成表」を兼ねるようになる。

### Step 3: 資産情報を考慮した執筆 (Stage 3)
- **内容**: `assist-cli` を実行。プロンプトに「固定された資産の Vision 解析結果」を自動注入する。
- **利点**: AI が「実際の画像」を見て書くため、「背景は昼なのにト書きは夜」といった矛盾が構造的に発生しなくなる。

### Step 4: ID 完全一致でのゲームデータ出力 (Stage 4)
- **内容**: エディタ用 JSON 生成時に、設定ファイルに書かれた `assetId` をそのまま出力。
- **成果物**: 修正不要で即座にプレビュー可能な `page-xxx.json`。

## 3. スケジュール
1. **フェーズ 1**: `fantasy` プロジェクトの主要アセット検索と ID 固定（本日中）。
2. **フェーズ 2**: 固定された ID に基づく第 1 章の再生成テスト。
3. **フェーズ 3**: 他プロジェクトへの水平展開。

## 4. 期待される効果
- **工数削減**: 手動での背景・キャラ ID 修正作業が 90% 削減。
- **品質向上**: 画像の詳細（小物の配置や表情）に基づいた密度の高いナラティブ。
- **一貫性**: 複数人で開発しても、設定ファイルを見るだけで使用素材が明確になる。
