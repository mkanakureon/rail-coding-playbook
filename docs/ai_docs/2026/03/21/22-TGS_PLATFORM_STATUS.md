# 東京ゲームショウ (TGS) 出展に向けたプラットフォーム機能現状分析レポート

> **作成日**: 2026-03-21
> **担当**: Gemini CLI

## 1. はじめに
本ドキュメントは、ノベルゲーム制作・公開プラットフォームとしての「kaedevn-monorepo」の現状を、東京ゲームショウ（TGS）等の展示会への出展を見据えて分析したものである。エンジニアリングの観点から、作者が作品を公開し、収益を得るための一連のフローがどの程度完成しているかを整理する。

## 2. プラットフォームとしての強み
- **真のマルチプラットフォーム:** 同一コードベースで Web (PixiJS) と Nintendo Switch (Native) の両方に対応。
- **制作から公開までの統合体験:** エディタ (`apps/editor`)、ポータル (`apps/next`)、API (`apps/hono`) が密結合しており、作者はツールを切り替えることなく公開まで完結できる。
- **収益化基盤の標準装備:** ポイントシステムと作者への収益分配ロジックが既にバックエンドに組み込まれている。

## 3. 機能別の実装状況 (2026-03-21 時点)

### A. プロジェクト管理・公開 (充足度: 90%)
- **実装済み:**
    - エディタからのワンクリック公開エンドポイント。
    - 作品メタデータ（タイトル、説明、タグ、カテゴリー）の管理。
    - サムネイルの自動キャプチャ機能。
    - **価格設定:** 0〜10,000pt の範囲で作品価格を自由に設定可能。
- **今後の課題:** 公開範囲（限定公開、パスワード保護など）の細分化。

### B. 読者体験・視聴環境 (充足度: 85%)
- **実装済み:**
    - Web上でのシームレスな再生環境 (`/play/[id]`)。
    - **ペイウォール (Paywall):** 未購入のリッチコンテンツに対する購入誘導UI。
    - セーブデータのクラウド同期機能 (`cloud-saves`)。
- **今後の課題:** オフライン再生への対応、モバイルアプリ版との連携強化。

### C. 収益化・エコシステム (充足度: 70%)
- **実装済み:**
    - **Walletシステム:** ユーザーごとの残高管理。
    - **収益分配ロジック:** 購入時に「作者へ90%加算、プラットフォーム手数料10%」を自動実行。
    - 取引履歴（チャージ、購入、収益）の完全な管理。
- **未実装:**
    - **外部決済連携:** ポイントチャージ機能は現在モック実装（Stripe等との接続が必要）。
    - **収益の引き出し (Payout):** 作者が貯まったポイントを現金として引き出すフロー。

## 4. TGS展示におけるキラーデモ案
展示会でのインパクトを最大化するため、以下のデモを推奨する。
1. **「爆速公開」の実演:** エディタでシーンを編集し、保存・公開後、即座にスマホやSwitchでその変更が反映されている様子を見せる。
2. **「収益サイクル」の可視化:** 読者役の端末でポイント購入し、作者役の管理画面で即座に「収益」として残高が増えるサイクルを実演。
3. **Switch実機展示:** 「Webプラットフォームで公開したものが、コンシューマー機でも動く」という技術的優位性の強調。

## 5. 結論と提言
本プロジェクトは、インディーゲーム作者が「最も手軽に、かつ本格的に（Switch等へ）展開できる」プラットフォームとしての骨組みを既に備えている。

**直近の推奨アクション:**
- Stripe API との統合による「本物のチャージ体験」の実装。
- 作者向けダッシュボード（収益グラフ等）の視覚化による「プラットフォーム感」の向上。
