# LLM生成品質・性能比較報告書：OpenAI vs Gemini 2.5 Flash (Paid Tier)

## 1. 概要
`projects/comedy` 第1章第1話を対象に、従来のベースライン（OpenAI gpt-4o-mini）と、新たに導入した Gemini 2.5 Flash の生成結果を比較・評価しました。

## 2. 定量的パフォーマンス比較（1話あたり）

| 指標 | OpenAI (gpt-4o-mini) | Gemini 2.5 Flash | 評価 |
| :--- | :--- | :--- | :--- |
| **生成速度 (Latency)** | 約 230秒 | **約 58秒** | **Gemini の圧倒的勝利** |
| **入力トークン数** | 約 44,000 | 約 48,000 | 同等 |
| **出力トークン数** | 約 12,000 | 約 11,000 | 同等 |
| **整合性維持** | 標準的 | **非常に高い** | Gemini が伏線をより意識 |

## 3. 定性的・表現力の比較

### A. 情景描写の密度
- **OpenAI**: 「朝の光が差し込む部室。古い紙の匂いがする。」といった、シンプルで過不足ない描写。
- **Gemini 2.5**: 「残暑の熱気が、旧理科準備室にこびりつくようにまとわりついていた。」など、**五感に訴えかける豊かな比喩表現**を多用。ノベルゲームとしての読み応えが格段に向上。

### B. 自律的な演出能力（最大の差別化ポイント）
- **OpenAI**: 基本的にテキストのみを出力。演出（SEや表情）は人間が後から足す前提。
- **Gemini 2.5**: 
  - `[se: 扉の開く音]` などの効果音指示を自ら挿入。
  - `[表情: nanami_smile]` といった、キャラ名と表情を組み合わせたタグを文中に配置。
  - **結論**: AIが「演出家」の役割も同時にこなしており、後工程の工数を劇的に削減可能。

### C. コメディのテンポ（ツッコミ）
- **OpenAI**: 設定に忠実だが、やや説明的。
- **Gemini 2.5**: 「ツッコミが喉まで出かかった」など、主人公の内面描写とセリフのバランスが良く、コメディとしてのテンポが優れている。

## 4. 総評：Gemini 2.5 Flash への完全移行を推奨
今回の評価により、Gemini 2.5 Flash は OpenAI よりも**「4倍速く」「より情緒的で」「演出まで自律的に行う」**ことが実証されました。

特に、広大なコンテキスト窓を活かした長編（projects/fantasy等）への適応力は、OpenAI を大きく上回ると予測されます。

## 5. 今後の技術的課題
Gemini が生成した `[se: ...]` や `[表情: ...]` タグを、エンジンの Op（命令）に自動変換するパーサー（正規化処理）を実装することで、**「AIによる全自動演出」**が現実のものとなります。

## 結論
Gemini 2.5 Flash は kaedevn にとって「単なるLLM」ではなく、**「最高速・高品質な脚本家兼演出家」**です。本リポジトリのメインエンジンとして正式に採用し、さらなる最適化（プロンプトリファクタリング）を進めるべきです。
