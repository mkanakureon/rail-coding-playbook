# Gemini 2.5 Flash プロンプト最適化計画書：演出自動化と描写密度の極致

## 1. 目的
Gemini 2.5 Flash の「広大なコンテキスト保持能力」と「自律的な演出タグ生成能力」を最大限に引き出すため、現在のプロンプトを全面的にリファクタリングし、その効果を 3 回の反復試行で検証する。

## 2. 最適化の 3 つの柱

### ① 命令の構造化（XML/Markdown テンプレート）
- **現状**: 平文の指示が混在。
- **改善**: 役割（Persona）、世界観（Knowledge）、出力形式（Format）を明確なタグで区切り、AIが指示の優先順位を誤らないようにする。

### ② 演出タグ（Staging Tags）の標準化
- **現状**: AIが勝手に `[表情: ...]` や `[se: ...]` を作っている。
- **改善**: エンジンが解釈可能なタグ一覧をシステムプロンプトに注入し、「このセリフの裏で鳴らすべき音」や「立ち絵のポーズ」を論理的に推論させる。

### ③ 伏線管理プロンプトの導入
- **現状**: 過去の要約を渡しているだけ。
- **改善**: 「未解決のプロットスレッド」を明示的に入力に含め、Gemini に「この回でどの伏線を回収すべきか」を宣言させてから執筆を開始させる（Chain of Thought）。

## 3. 検証プロセス（3回の反復改善）

| 試行 | 最適化のポイント | 期待される成果 |
| :--- | :--- | :--- |
| **Run 1** | 現行プロンプトのまま実行 | ベースラインの再確認。 |
| **Run 2** | 構造化（XMLタグ）と演出リスト注入 | 演出タグの正確性と描写密度の向上。 |
| **Run 3** | 伏線管理（スレッド意識）の追加 | 物語の整合性と意外性の強化。 |

## 4. 評価指標
- **演出密度**: 1話あたりの演出タグ（SE/表情）の数。
- **文章の質**: 比喩表現や五感に訴える描写の豊富さ。
- **整合性**: 過去の設定との矛盾の有無。
- **スピード**: 1話あたりの生成秒数。

## 結論
このプロセスを通じて、Gemini 2.5 Flash を単なる「代筆屋」から、エンジンの仕様を熟知した「専属監督」へと昇華させます。
