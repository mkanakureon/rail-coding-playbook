# 原因分析：生成トーンの不一致とブレの問題

## 1. 現象
同一の設定ファイル（`fantasy` 等）から生成を開始しても、実行のたびに「素朴な冒険もの」「ダークファンタジー」「王道少年もの」のように、作品の文体や世界観の厳しさが大きく変動する。

## 2. 根本原因の特定

### A. Stage 0 (骨格) におけるトーン定義の欠落
Stage 0 は「何が起きるか（論理）」のみを決定し、「どのような空気感で語るか（情緒）」を決定していない。
- **現状**: 「泉が枯れる」という事実のみを固定。
- **問題**: その事実を「悲劇（ダーク）」として描くか、「試練（少年漫画）」として描くか、「日常の異変（素朴）」として描くかが LLM のアドリブに委ねられている。

### B. エモーションポイント（EP）の「方向性」の未定義
本日の EP 実装により「熱量（強・弱）」の制御は可能になったが、「ベクトルの向き」が定義されていない。
- **現状**: EP 80 は「盛り上がる」ことを意味する。
- **問題**: 「絶望で盛り上がる（ダーク）」のか「友情で盛り上がる（王道）」のかが指定されていないため、各シーンでトーンが漂流する。

### C. ジャンル・プロンプトの抽象度
`prompts.ts` における `settings.genre` や `settings.tone` の扱いが単純な文字列の流し込みに留まっている。
- **現状**: `ジャンル: ファンタジー` という指示。
- **問題**: LLM にとっての「ファンタジー」の解釈（ハイ・ファンタジー、ロー・ファンタジー、なろう系等）が広すぎる。

### D. 否定的な制約（Negative Prompt）の不足
「やってはいけない描写」が定義されていない。
- **現状**: 「描写を豊かにせよ」という加点方式の指示。
- **問題**: 素朴な物語において「過度な残酷描写」や「過激な台詞」を排除するブレーキが存在しない。

## 3. 解決の方向性
1. **作品トーンの「具体言語化」 (Stage 0)**: 作品固有のキーワード（例：温かみ、静寂、泥臭さ）を 3 つ抽出して固定する。
2. **章別トーン・指針の策定 (Stage 1)**: 各章のプロットに「この章で維持すべき空気感」を明記する。
3. **禁止事項の注入 (All Stages)**: ジャンルに応じた「避けるべき文体・語彙」をプロンプトに組み込む。
