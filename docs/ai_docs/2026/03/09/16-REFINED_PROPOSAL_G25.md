# 改善提案：Gemini 2.5 Flash による高品質シナリオ生成の最大化

## 1. 概要
架空のモデルに依存せず、現在利用可能な最強モデル **`gemini-2.5-flash`** の性能を 100% 引き出すための 2 パス生成・マルチプラットフォーム戦略を定義する。

## 2. 2パス・パイプラインの最適化 (Single Model Strategy)

同一モデル（2.5 Flash）を 2 回通すことで、役割を完全に分離する。

### パス 1：Stage 3 (Drafting)
- **役割**: ストーリーの「勢い」と「プロットの完遂」。
- **改善点**: 2.5 Flash の長いコンテキスト窓を使い、過去の全エピソードの要約を常に参照させながら、設定矛盾のない初稿を書かせる。

### パス 2：Stage 4 (Refining & Rich JSON)
- **役割**: 「演出」と「構造化」。
- **改善点**: 初稿を読み込み、2.5 Flash の高い演出力を活かして以下の Rich Data を強制的に出力させる。
  - `emotionDetail`: 立ち絵の表情を日本語で細かく指定（Nano Banana 用）。
  - `cameraAngle`: シーンの迫力を出すための構図指定。
  - `wait`: 読者の読了スピードを予測した最適な「間」の数値化。

## 3. マルチプラットフォーム展開の自動化
2.5 Flash は「文体模写」に優れているため、生成された Rich JSON を元に以下の変換を AI 自身に行わせる機能を強化する。

- **Web小説モード**: 2.5 Flash に「なろう人気作家風のリライト」を指示。
- **書籍ラノベモード**: 2.5 Flash に「三人称多視点の重厚な文体への変換」を指示。

## 4. 結論
最新の安定モデル `gemini-2.5-flash` に役割を絞り込むことで、API エラーやモデル間の性能差に悩まされることなく、商業レベルの物語資産を高速に量産することが可能となる。
