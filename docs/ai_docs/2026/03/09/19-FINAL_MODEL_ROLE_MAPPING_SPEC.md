# 決定事項：モデル役割（Role）マッピングと動的設定仕様

## 1. 目的
生成パイプラインの各ステージに対し、モデル名を直接ハードコードするのではなく、知能ランクに応じた「役割（Role）」を介してモデルを割り当てることで、柔軟な運用とコスト最適化を実現する。

## 2. モデル役割（Role）の定義

| 役割名 | ランク | 想定される性能と特性 |
| :--- | :--- | :--- |
| **`pro`** | **Premium** | 最高知能。複雑な因果関係の把握、キャラの深み、高品質なリライト。 |
| **`flash`** | **Standard** | 標準知能。論理的な構成、データ分析、正確な構造化。 |
| **`lite`** | **Economy** | 速度・コスト優先。大量のテキスト生成、単純な要約作業。 |

## 3. ステージ別マッピング表 (P01 Pipeline)

各ステージの実行時に、指定された「役割」に対応するモデル ID が動的に解決される。

| ステージ | 割り当て役割 | 具体的な業務内容 |
| :--- | :--- | :--- |
| **Stage 0** | **`pro`** | 作品骨格（アーク・章）の設計、伏線の配置。 |
| **Stage 1** | **`pro`** | 章ごとの詳細プロット作成、感情曲線の設計。 |
| **Stage 2** | **`flash`** | 話ごとのシーン分割、因果関係の構築。 |
| **Stage 3** | **`lite`** | 本文ドラフト（初稿）の大量生成（64Kトークン出力を活用）。 |
| **Stage 4** | **`pro`** | Rich Editor-JSON への成形、高品質な演出・リライト。 |
| **要約工程** | **`lite`** | 次話・次章への橋渡しのためのコンテキスト圧縮。 |

## 4. プロジェクト個別設定 (Project Config)

`scripts/cli/configs/{genre}.json` にて、プロジェクトの特性に合わせた「モデルセット」を定義できる。

### 設定例 (`fantasy.json`)
```json
{
  "projectTitlePrefix": "ファンタジー",
  "models": {
    "pro": "gemini-2.5-pro",
    "flash": "gemini-2.5-flash",
    "lite": "gemini-2.5-flash-lite"
  }
}
```

## 5. 解決の優先順位
1. **CLI 引数** (`--model-pro` 等)
2. **環境変数** (`ASSIST_MODEL_PRO` 等)
3. **Project Config JSON**（上記定義）
4. **システムデフォルト** (`gemini-2.5-flash` シリーズ)
