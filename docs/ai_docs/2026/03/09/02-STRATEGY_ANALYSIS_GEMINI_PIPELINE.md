# 戦略分析：Gemini 多段階シナリオ生成とベンダーロックイン MVP

## 1. 概要
`docs/01_in_specs/2026/03/0309/gemini.md` に記された対話ログに基づき、ノベルゲーム制作における Gemini CLI の優位性と、コスト・品質を最大化するための具体的な「多段階生成パイプライン」戦略を分析・定義する。

## 2. 多段階生成（Multi-Stage Pipeline）のモデル最適配置
「費用を抑えつつクオリティを最大化する」ため、各フェーズの特性に合わせてモデルを使い分ける。

| ステージ | 内容 | 推奨モデル | 戦略的理由 |
| :--- | :--- | :--- | :--- |
| **Stage 0/1** | 全体骨格・章プロット | **Gemini 2.5 Flash** | 長大なコンテキスト窓を活用し、長編での設定矛盾（ハルシネーション）を排除する。 |
| **Stage 2** | 話プロット（詳細構成） | **Gemini 2.5 Flash** | 安定した推論能力により、起承転結のロジックと伏線の配置を行う。 |
| **Stage 3** | 本文ドラフト生成 | **Gemini 2.5 Flash** | 爆速・格安の量産。最もトークンを消費する工程を Flash モデルで回す。 |
| **Stage 4** | エディタ JSON 成形・演出 | **Gemini 2.5 Flash** | 初稿を読み込み、キャラ口調の反映・エモい描写・演出ブロックへの正確な変換を担当する。 |

## 3. 「ベンダーロックイン MVP」戦略
特定のプラットフォーム（Google Cloud / Gemini）の機能を最大限に活用し、抽象化レイヤーを介さないことで、開発スピードと品質を最大化する。

### A. ネイティブ機能の徹底活用
- **JSON Mode**: AI に直接 `editor-json`（manifest, page-XXX.json）を出力させ、変換の不安定さを解消。
- **Context Caching**: 世界観（`world.md`）やキャラ設定（`characters.md`）をキャッシュし、全生成工程で一貫したトーンを維持。
- **Grounding (Google Search)**: 実在の場所や最新の知識に基づいた、リアリティのあるシナリオ描写。

### B. Nano Banana (画像生成) との連携
- **マスター JSON 管理**: キャラクター容姿や背景設定を JSON で定義。
- **プロンプトエンジニアリング**: Gemini CLI が「ディレクター」となり、Nano Banana (Flash/Pro) が最も反応しやすい英語プロンプトを自動生成する。

## 4. 他 CLI (Claude Code, etc.) との比較優位性
- **圧倒的なコンテキスト窓**: 1M〜2M トークンにより、プロジェクト全体の全ファイルを読み込ませた上での「あうんの呼吸」が可能。
- **自動化とコスト効率**: 無料枠の広さと、Flash-Lite による「そこそこのクオリティ」の大量生産。

## 5. 本プロジェクトへの適用
現在の `assist-cli.ts` の修正において、以下のフローを実装する：
1. `config/*.json` を読み込み、`manifest.json` と `characters.json` を自動生成。
2. Stage 3 (Flash-Lite) でシナリオの初稿を書き出す。
3. Stage 4 (3.1 Pro) で初稿を読み込み、config のアセットID（`$bg:bg_hall` 等）を正確にマッピングした `page-XXX.json` を生成する。
