# モデル戦略：Gemini 2.5 シリーズによる多段階生成最適化

## 1. 目的
2026年3月時点で実在する最新モデル「Gemini 2.5 シリーズ」をパイプラインの各工程に最適配置し、コスト・速度・品質（特に長文出力能力）を最大化する。

## 2. パイプライン別モデル構成 (P01 Pipeline)

| ステージ | 内容 | 推奨モデル ID | 選定理由 |
| :--- | :--- | :--- | :--- |
| **Stage 0/1** | 骨格・章プロット | **`gemini-2.5-pro`** | **最高知能 + Thinking。** 200万トークン超のコンテキストキャッシュを利用し、長編での設定矛盾を徹底排除。 |
| **Stage 2** | 話プロット | **`gemini-2.5-flash`** | **論理構築。** Thinking 機能により、起承転結の整合性と「引き」の強さを Pro に次ぐ精度で実現。 |
| **Stage 3** | 本文ドラフト | **`gemini-2.5-flash-lite`** | **爆速・最安・超長文出力。** 出力上限が 65,536 トークンに拡大されており、長いシーンの「尻切れ」リスクを解消。 |
| **Stage 4** | JSON成形・演出 | **`gemini-2.5-flash`** | **構造化の安定性。** Rich JSON への変換速度が速く、かつ JSON モードの出力が極めて安定している。 |

## 3. 主要な改善点 (vs 2.0世代)

### A. Stage 3 の安定化
`gemini-2.0-flash-lite` から **`gemini-2.5-flash-lite`** へアップグレードすることで、出力上限が 8,192 → 65,536 トークンへと大幅に拡張される。これにより、描写の多い長大なシーンも一度の API コールで完結可能となる。

### B. Stage 4 の適正化
存在しない「3.1 Pro」等の架空モデルを排し、実績のある **`gemini-2.5-flash`** を採用。JSON 成形タスクにおいては Pro よりも高速であり、実用上の精度も十分であることが実証されている。

### C. 初動の知能強化
全体設計（S0/S1）に **`gemini-2.5-pro`** を固定することで、物語の「種」の段階で最高レベルのクリエイティビティを確保する。

## 4. 運用ルール
- `assist-cli.ts` の内部デフォルト値を上記 ID に準拠させる。
- 特定の環境で `gemini-2.5-pro` が利用できない場合は、フォールバックとして `gemini-2.5-flash` を使用する。
