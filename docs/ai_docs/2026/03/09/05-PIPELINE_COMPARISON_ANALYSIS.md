# パイプライン比較分析：現状（v1）vs 提案（v2）

## 1. 概要
現在のシナリオ生成パイプライン（v1）と、Gemini 3.1 世代の特性を最大限に活かした新世代パイプライン（v2）を比較する。最大の違いは、**「Stage 4 の LLM による高品質化（Refinement）」** ステージの追加と、出力形式の **「editor-json 化」** である。

## 2. パイプライン構造の比較

### 現状：パイプライン v1（.ks 出力）
ローカルスクリプトによる単純な変換に頼っているため、品質のバラツキやパースエラーが発生しやすい。

| ステージ | 内容 | モデル / 処理 | 出力形式 |
| :--- | :--- | :--- | :--- |
| **Stage 0/1** | 全体構成 | Gemini (汎用) | JSON |
| **Stage 2** | 話プロット | Gemini (汎用) | JSON |
| **Stage 3** | 本文生成 | Gemini (Flash 等) | JSON (lines) |
| **Stage 4** | **KS 変換** | **ローカルスクリプト** | **.ks (KaedeScript)** |

### 提案：パイプライン v2（editor-json 出力）
本文生成を「ドラフト」と「成形・演出」の 2 段階に分けることで、コストを抑えつつプロ級の品質を実現する。

| ステージ | 内容 | モデル (推奨) | 出力形式 | 役割の変更点 |
| :--- | :--- | :--- | :--- | :--- |
| **Stage 0/1** | 全体構成 | **1.5 Pro** | JSON | 長大なコンテキストを維持。 |
| **Stage 2** | 話プロット | **2.5 Pro** | JSON | 論理的な構成と引き（フック）。 |
| **Stage 3** | **本文ドラフト** | **3.1 Flash-Lite** | JSON | **[変更]** 爆速・格安で大量生成。 |
| **Stage 4** | **成形・演出** | **3.1 Pro** | **editor-json** | **[新規]** ドラフトをエモく成形し、JSON化。 |

## 3. 主要な変更点：Stage 4 の役割の変化

### 【旧】Stage 4：機械的な変換 (Local Script)
- **処理**: Stage 3 の JSON を読み込み、単純な置換で `.ks` を作成。
- **欠点**: キャラクターの口調の微調整、複雑な演出タグの挿入、アセットIDの厳密な解決ができない。

### 【新】Stage 4：LLM による高品質化 (Gemini 3.1 Pro)
- **処理**: Flash-Lite が書いた「そこそこ」のドラフトを、最高知能の Pro が「リライト」する。
- **利点**:
  - **品質**: キャラクター設定に忠実な「エモい」台詞回しに修正。
  - **演出**: `text`, `ch`, `bg` ブロックへの正確なマッピング。
  - **整合性**: `config/*.json` に基づく正しいアセットID（`$bg:bg_hall` 等）の埋め込み。
  - **制約遵守**: 46文字×3行の物理的制約を AI が理解した上で改行。

## 4. コストと品質のシミュレーション

| 項目 | パイプライン v1 | パイプライン v2 |
| :--- | :--- | :--- |
| **生成コスト** | 中 (Flash 1.5) | **低 (Flash-Lite + Pro リライト)** |
| **文章の質** | そこそこ | **プロ級 (Pro による磨き上げ)** |
| **エラー率** | 高 (正規表現の失敗) | **極低 (JSON Mode による構造化)** |
| **エディタ連携** | 変換（ks-convert）が必要 | **直接投入（ks-upload）が可能** |

## 5. 結論
パイプライン v2 は、**「最も安価なモデルで大量の素材（ドラフト）を作り、最も賢いモデルで最後の仕上げ（JSON成形）をする」** という、ベンダーロックイン MVP における最強のコストパフォーマンス戦略である。

この Stage 4 の LLM 化により、人間による手直しを最小限に抑えつつ、商業レベルのノベルゲーム素材を高速に量産できる体制が整う。
