# 仕様書：パイプライン ID とファイル命名規則

## 1. 目的
複数の生成ロジックやプロンプトのバージョンが混在する中で、どの成果物がどのロジックで生成されたかを明確に区別し、比較・検証を容易にする。

## 2. パイプライン ID 定義

| ID | 名称 | 内容 |
| :--- | :--- | :--- |
| **P00** | Legacy Pipeline | 初期実装。Gemini 1.5 Flash 中心、`.ks` (KaedeScript) 出力。 |
| **P01** | 2-Pass Rich Pipeline | 本日(2026/03/09)実装。2パス生成、Rich Editor-JSON 直接出力。 |
| **P02** | Image Integrated | (予定) P01 に Nano Banana (画像生成) 連携を統合したもの。 |

## 3. ファイル・ディレクトリ命名規則

### A. 出力ディレクトリ
ディレクトリ名の先頭にパイプライン ID を付与する。
- 形式: `output/{PIPELINE_ID}_{TIMESTAMP}/`
- 例: `output/P01_20260309_120000/`

### B. 主要ファイル
出力ディレクトリ直下のファイルにプレフィックスを付与する。
- 例: `P01_skeleton.json`
- 例: `P01_chapters.json`
- 例: `P01_ch1_full_script.md`

### C. エディタ用 JSON (editor-json/)
エディタへの投入を容易にするため、`editor-json/` 配下のファイル名は変更しない。
その代わり、`manifest.json` 内のメタデータにパイプライン ID を記録する。

## 4. 運用のメリット
- 異なるモデル設定での生成結果を同一プロジェクト内で共存・比較できる。
- エラーが発生した際、どのバージョンの CLI ロジックに問題があるか特定が容易になる。
