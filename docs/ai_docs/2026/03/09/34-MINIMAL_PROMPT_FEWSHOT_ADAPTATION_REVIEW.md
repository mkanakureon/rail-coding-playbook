# 最終レビュー報告：極小プロンプト（実例誘導のみ）への移行と AI の過剰適応

## 1. 実施内容
プロンプトエンジニアリングの究極的なスリム化を断行した。
- **言葉による指示の撤廃**: 「〜禁止」「〜してください」「改行ルール（[r], [p]）」といった長文のテキスト指示を 90% 以上削除。
- **API の jsonMode への依存**: JSON 形式に関する冗長な説明を「Output Schema:」のヘッダーのみに集約。
- **Few-shot（出力例）の絶対化**: AI に対する唯一の、かつ強力な指針として JSON の実例（Few-shot）を 1 つ提示。

## 2. 実験結果：AI の驚異的な模倣能力

### A. 名前混入の完全解消（再確認）
- **結果**: 名前に関する「禁止」指示を一切削除したにも関わらず、出力例（Few-shot）に名前が含まれていなかったため、**AI は 1 度も名前を混入させることなく生成を完了した。**
- **結論**: AI にとって「言葉でのルール説明」は「実例」という圧倒的な証拠の前では無視される（あるいは二の次になる）ことが証明された。

### B.Few-shot への「過剰適応」の発見
- **現象**: 修正した Few-shot の全行末尾が `[p]` で終わっていたため、AI は「本文は必ず [p] で終わるべきである」というルールを自律的に抽出し、**全ての行を [p]（クリック待ち）で出力した。**
- **洞察**: これは AI の高い模倣能力の裏返しであり、**「実例に含まれる些細な特徴が、AI にとっての『絶対的なルール』として解釈される」** という強力な特性（Few-shot 適応）を浮き彫りにした。

### C. トーンと内容の安定
- **結果**: 負の制約（禁止ワード）を解いても、世界観リファレンスと「正のキーワード」による誘導のみで、王道冒険ものの「温かみ」と「危機感」のバランスが最適に保たれた。

## 3. 今後の運用指針（ベストプラクティス）

1. **Instruction よりも Example**: プロンプトを長くするのではなく、**「理想的な出力例（Few-shot）」を 1 つ、完璧に作り込んで提示する** ことが最も重要である。
2. **実例の「ノイズ」に注意**: Few-shot 内の改行、タグの打ち方、句読点の位置、文字数などは、すべて AI に「ルール」として学習される。実例を作成する際は、その細部が「汎用的なルール」として解釈されても良いか、細心の注意を払う必要がある。
3. **動的な Few-shot**: 必要に応じて、現在の EP や Vibe に合わせた「異なるパターンの Few-shot」を動的に生成・注入することで、より高度な制御が可能になる。
