# レビューと計画：生身の文章クオリティの評価と自動整形への道

## 1. 文章クオリティの評価 (Review)

プロンプトから機械的な整形ルール（文字数制限、[r], [p]タグの強制）を撤廃したことで、文章の質が劇的に向上した。

### 成果：
- **表現のダイナミズム**: 46文字の「檻」から解放されたことで、長文による重厚な描写と短文によるテンポの良い進行が共存し、文学的なリズムが生まれた。
- **比喩の解像度**: 「古びた土器の底のように乾ききっていた」など、状況を的確かつ情緒的に伝える比喩が多用されるようになった。
- **執筆への集中**: AI がテクニカルな計算（タグ打ち、文字数カウント）に意識を割かなくなった結果、キャラクターの心理描写や情景の深みが明らかに増している。

## 2. さらなるスリム化：背景演出の完全分離

現状、AI は本文冒頭に `[bg: ...]` タグを自主的に挿入しているが、これは冗長である。

### 根拠：
- JSON の `location` プロパティに「場所名」が記述されており、システム側でアセット ID への変換が可能。
- 本文内に `[bg]` が混在すると、執筆中に AI が「背景 ID を思い出そうとする」ノイズが発生する。

### 対策：
- Few-shot から `[bg]` を削除し、背景の切り替えは JSON メタデータに基づいたプログラム制御（後付け）に 100% 委ねる。

## 3. 実装計画：スマート・タギング・エンジン (Next Steps)

AI が生成した「生身の文章」を、ゲームエンジン（Nintendo Switch / Web）に最適な形式へ自動変換するロジックを実装する。

### ステップ 1: プロンプトの最終スリム化
- Few-shot から `[bg]` を削除。
- AI への期待値を「純粋なナラティブの出力」に完全にシフトさせる。

### ステップ 2: KS-Generator の拡張
`apps/hono/src/lib/assist/ks-generator.ts` に以下の自動整形機能を実装する。
1. **自動背景挿入**: シーン開始時に `location` から `@bg` を生成。
2. **自動改行 (@r)**: 文の切れ目（。！？）および 46 文字制限に基づいた自動改行。
3. **自動ページ送り (@p)**: 3行ごとの自動クリック待ち挿入。
4. **タグの正規化**: 文中に残った `[se]`, `[表情]` タグを適切な位置に再配置。

## 4. 期待される効果
- **作家性の最大化**: 人間が書いたかのような、質の高いシナリオの自動生成。
- **エンジンの完全制御**: 端末の解像度やフォントサイズに合わせて、生成済みテキストから動的に最適な改行を再計算可能になる。
