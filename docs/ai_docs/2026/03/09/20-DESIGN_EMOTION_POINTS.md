# 設計：エモーションポイント（EP）連動型 生成パイプライン

## 1. 目的
物語の「論理（出来事）」と「感情（熱量）」を分離し、数値（0-100）による精密な演出制御と執筆トーンの自動調整を実現する。

## 2. 階層別 EP 定義

| 階層 | パラメータ | 役割 |
| :--- | :--- | :--- |
| **Chapter (Stage 1)** | `targetMaxEp` (0-100)<br>`emotionalArc` (Text) | 章全体の感情目標と曲線の設計。 |
| **Scene (Stage 2)** | `ep` (0-100)<br>`vibe` (Enum) | シーンごとの具体的な熱量と感情の方向性。 |
| **Text (Stage 3)** | `appliedEp` (0-100) | 実際に描写に反映された最終的な熱量。 |

### vibe の定義
- `calm`: 静寂、客観的、日常
- `tension`: 緊張、不安、予兆
- `climax`: 最高潮、熱狂、激動
- `relief`: 安堵、和解、余韻
- `shock`: 衝撃、絶望、急転

## 3. 数値による執筆・演出制御ロジック

### A. 執筆トーン (Stage 3)
- **EP 0 - 30 (Low)**: 短文、事実の羅列、客観的な三人称的視点。
- **EP 31 - 70 (Mid)**: 標準的な描写、比喩の使用、キャラクターの内面への言及。
- **EP 71 - 100 (High)**: 重厚な語彙、五感の強調、畳み掛けるような心理描写、主観的な熱量。

### B. 演出指示タグの自動連動 (Staging)
- **EP 80+**: `[flash]`, `[shake]` の積極使用。BGM を Climax 系に指定。
- **EP 50-79**: 表情変化（驚き、怒り、泣き）の頻度向上。効果音の強調。
- **EP 0-49**: 環境音メイン。表情は通常、微笑、微かな変化に留める。

### C. キャラクター一貫性と伏線配置の連動 (NEW)
- **Role-Based Dialect Locking**:
    - 特殊キャラクター（コン等）に対し、低EP時でも現代語へのフォールバックを禁止する「語彙ロック」を適用。
    - EPの変動に関わらず、指定された語尾（〜ですな、〜ゆえ）の一貫性を数値的に強制する。
- **Setup-Peak Payoff (伏線の自動配置)**:
    - 特定キャラクターの「高EPシーン（過去の告白など）」が予定されている場合、その $N$ 話前の低EPシーンに「視覚的接触（顔出し）」を自動的に義務付けるロジック。
    - 感情のピーク（Payoff）を支えるための事前の仕込み（Setup）を、EP曲線の波形から逆算して生成。

## 4. データ構造拡張案 (schemas.ts / types.ts)

### ScenePlot (Stage 2)
```typescript
{
  "sceneNumber": number,
  "ep": number, // 0-100
  "vibe": "calm" | "tension" | "climax" | "relief" | "shock",
  "event": string
}
```

## 5. 実装ロードマップ
1. **Schema 拡張**: `types.ts`, `schemas.ts` へのフィールド追加。
2. **Stage 2 拡張**: 感情曲線から各シーンの EP を算出するプロンプトへの更新。
3. **Stage 3 拡張**: EP 数値に基づいて執筆ガイドを動的に切り替えるロジックの導入。
4. **Validation**: ファンタジープロジェクト等で EP の推移と文章の熱量が連動しているかを確認。
