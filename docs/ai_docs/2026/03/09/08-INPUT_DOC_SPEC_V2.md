# 入力ドキュメント仕様 v2（文書 ID ・ 拡張性対応）

## 1. 目的
プロジェクト設定（`settings/`）内のドキュメントを整理し、生成 AI が文脈を把握しやすくするとともに、将来的な入力情報の増加に柔軟に対応できる構造を定義する。

## 2. ファイル命名規則
ファイル名は `ID_CATEGORY.md` の形式とし、すべて小文字（またはスネークケース）で統一する。

- 形式: `s_{2桁連番}_{カテゴリー名}.md`
- 例: `s_01_overview.md`, `s_02_characters.md`

## 3. 文書ヘッダー仕様（AI フレンドリー）
各ファイルの 1 行目には、必ず文書 ID とカテゴリー名を含むヘッダーを記述する。AI はこのヘッダーを見て情報の種類を特定する。

- 形式: `# [ID_ID名] カテゴリー名（日本語）`
- 例: `# [s_01_OVERVIEW] 作品概要`

## 4. 標準入力ドキュメント一覧 (Input)
`settings/` ディレクトリに配置し、人間が AI に与える「材料」となる。

| 文書 ID | ファイル名 | カテゴリー名 | 内容 |
| :--- | :--- | :--- | :--- |
| `s_01_OVERVIEW` | `s_01_overview.md` | 作品概要 | タイトル、ジャンル、トーン、目標章数など。 |
| `s_02_CHARACTERS` | `s_02_characters.md` | キャラクター設定 | 登場人物の slug、性格、口調、関係性。 |
| `s_03_PLOT` | `s_03_plot.md` | プロット設定 | ラフプロット、章ごとの必須新情報。 |
| `s_04_WORLD` | `s_04_world.md` | 世界観・場所設定 | 舞台背景、各場所（location）の詳細描写。 |
| `s_05_HISTORY` | `s_05_history.md` | 歴史・段階開示 | 年表、物語の進行に合わせて開示する事実。 |

## 5. 拡張と運用
- **拡張**: 新しい種類の情報（例：アイテム一覧、魔法体系など）が必要な場合は、`s_06_xxx.md` のように連番を増やして追加する。
- **読み込み**: `md-parser.ts` はディレクトリ内の `s_XX_*.md` をスキャンし、ヘッダーの ID に基づいて WorkSetting にマッピングする。
- **アセット管理**: `s_06_ASSETS` はインプットとしては廃止。アセットリストは Stage 4 のアウトプット（画像生成プロンプト集）として自動生成される。
