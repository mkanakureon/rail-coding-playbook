# ツクール機能実装状況・比較分析レポート (kaedevn vs RPG Maker)

**作成日:** 2026-03-16
**対象コミット:** `a300068ce282c1b8d6b4d569e6fe843fa6988017`

## 1. 概要
kaedevn-monorepo は、ノベルゲームエンジンをベースとしつつ、RPGツクール（以下「本家」）に匹敵する RPG 制作機能を統合している。本家が独自のバイナリや Ruby/JavaScript で動作するのに対し、kaedevn は TypeScript/PixiJS ベースのモダンな Web 技術スタック上で「ブロック型エディタ」と「KSCスクリプト」のハイブリッド構成をとっている。

---

## 2. 実装済み機能一覧

### A. マップシステム (`MapSystem.ts`)
- **タイルベースレンダリング**: PixiJS による高速な描画。
- **イベントページ**: 複数の実行条件（変数・スイッチ）に基づくページの動的切り替えを再現。
- **オートタイル**: 周囲のタイル状況に応じた自動パターン選択（Phase 9 で実装）。
- **移動ルート**: ランダム、接近、逃走、カスタムパスなどの NPC 自律移動。

### B. データベース (`GameDb.ts`)
- **アクター/敵キャラ**: ステータス（HP, MP, ATK, DEF等）および成長曲線の定義。
- **スキル/アイテム**: 消費MP、効果量、ターゲット（単体/全体）の設定。
- **トループ (敵グループ)**: 敵キャラの配置とエンカウント単位の管理。

### C. バトルシステム (`BattleScene.ts`)
- **ターン制サイドビュー**: PixiJS によるアニメーション対応の戦闘画面。
- **報酬システム**: 経験値（累計・次レベルまで）およびゴールドの計算とレベルアップ。
- **ステータス異常**: 毒（割合ダメージ）、麻痺（行動不能）、気絶のロジック。

### D. イベント制御 (`callTemplate`)
- **コモンイベント**: 汎用的な処理をテンプレート化し、スクリプトやマップイベントから呼び出し可能。
- **変数操作**: 数値・文字列変数の代入、加減算、条件分岐。

## 3. 本家ツクール（MV/MZ）との比較

| 機能カテゴリ | 本家ツクール (MV/MZ) | kaedevn (現状) | 比較分析 |
| :--- | :--- | :--- | :--- |
| **描画エンジン** | Canvas/WebGL (PixiJS 4/5) | PixiJS 8 (最新) | kaedevn の方が描画パフォーマンスと最新ブラウザ対応で優位。 |
| **スクリプト** | JavaScript (直接編集) | KSC (独自のDSL) | kaedevn は非プログラマ向けの簡潔さと、コードによる高度な制御を両立。 |
| **エディタ** | デスクトップ専用アプリ | Webベース・Reactエディタ | kaedevn はブラウザのみで制作・共有が可能。 |
| **イベント実行** | 並列処理・自動実行など | シーケンシャル・コール | 現状 kaedevn は「並列イベント」が未実装。 |
| **データベース** | 数千件規模の固定長データ | JSONベースの動的データ | kaedevn の方がデータの拡張性（カスタムプロパティ追加等）が高い。 |
| **プラグイン** | .js ファイルによる拡張 | システムパッケージの追加 | kaedevn はモノレポ構成により、コア機能自体の拡張が容易。 |

### kaedevn 独自の強み
1.  **ノベル×RPGの融合**: 本来ノベルエンジンであるため、リッチな立ち絵表示、演出、複雑なテキスト表示（バックログ、既読スキップ）が RPG 部分とシームレスに統合されている。
2.  **マルチプラットフォーム**: Web/Switch の抽象化レイヤー（IInput, IAudio）が初期から設計されており、移植性が高い。

## 4. 本家と比較して不足している要素（今後の課題）

1.  **並列処理イベント**: 「特定の変数が100になったら画面を赤くする」といった、メインシナリオの裏で動き続けるロジック。
2.  **高度な衝突判定**: 現状はタイル単位の判定のみ。ピクセル単位や半歩移動への対応。
3.  **プラグイン・スクリプト実行**: スクリプト内から任意の JS 関数を呼び出す `eval` 相当の機能（セキュリティとのトレードオフ）。
4.  **アセット管理**: タイルセットやキャラクター素材のインポート・トリミング機能の充実。

---

## 5. 結論
kaedevn は本家ツクールのコア・ゲームプレイ（探索、会話、戦闘、成長）を Web モダンな技術で再構築することに成功している。特に Phase 7〜9 の実装により、「RPG らしい遊び」に必要な最小セット（MVP）はほぼ出揃ったと言える。
今後は、既存ツクールからの移行ユーザーが違和感を感じないよう、UIの洗練とパフォーマンスの最適化（特に TextureCache の修正など）に注力すべきである。
