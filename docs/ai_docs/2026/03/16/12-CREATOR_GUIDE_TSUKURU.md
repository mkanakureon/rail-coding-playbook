# kaedevn ツクール機能・制作者向けガイド (v1.0-RC)

このドキュメントは、kaedevn エンジンを使用して RPG 要素を含む作品を制作するクリエイター向けの**詳細な技術解説書**です。システムの内部ロジックを理解することで、より緻密なゲームバランスの構築が可能になります。

---

## 1. マップエディタ：世界構築の詳細
PixiJS による高速な描画を活かした、モダンなマップ制作が可能です。

- **マルチレイヤー構造**: 
  - **背景レイヤー**: タイルセットのベース（地面など）を配置。
  - **装飾レイヤー**: 背景の上に重ねるオブジェクト（家具、木など）を配置。
  - **イベントレイヤー**: プレイヤー、NPC、宝箱などの動的オブジェクト。
  - **衝突判定 (Collision)**: 通行不可領域を設定。
  - **リージョン (Region)**: エンカウント率や特殊イベントの範囲指定に使用。
- **オートタイル機能**: 
  - 47通りの隣接パターンを自動認識し、水辺や壁の角を自動的に描画します。
- **テストプレイ**: 
  - エディタ上で `T` キーまたはボタンを押すことで、現在のマップのスポーン位置から即座に操作確認が可能です。

## 2. データベース（GameDb）：計算式の仕様
kaedevn の戦闘や成長は、以下の固定式に基づいて計算されます。

### A. ダメージ計算式
通常攻撃およびスキルのダメージは以下のロジックで算出されます。
```text
基本ダメージ = max(1, 攻撃者のATK - 対象のDEF)
最終ダメージ = floor(基本ダメージ * スキル威力 * 分散度)
```
- **分散度**: `1.0 ± variance` の範囲でランダムに変動します。
- **命中判定**: `Random(0.0, 1.0) <= 命中率` で判定されます。

### B. 経験値・レベルアップ
レベルアップに必要な**累計経験値**は、ツクール 3 次関数方式を採用しています。
```text
必要累計経験値(L) = floor(L^3 * 0.8 + 10 * L)
```
- **最大レベル**: 現状 99 に制限されています。

### C. ステータス成長
レベル $L$ における各ステータスは以下の式で算出されます。
```text
現在の値 = 初期値 + 上昇量 * (L - 1)
```
- エディタの成長曲線パネルで設定した「上昇量」が、1レベルごとの加算値となります。

## 3. バトルシステム：演出とロジック
- **サイドビュー戦闘**: プレイヤー（左）と敵（右）が対峙する形式。
- **ターン順**: 現状は「プレイヤー側全員 → 敵側全員」の順で行動しますが、内部的には AGI（素早さ）による順序制御の拡張が可能です。
- **状態異常ロジック**:
  - **毒 (Poison)**: ターン終了時に最大HPの10%ダメージ（HP 1 で踏みとどまる）。
  - **麻痺 (Paralyze)**: 行動不能。
  - **気絶 (Stun)**: 1ターンのみ行動不能。

## 4. KSCスクリプト：高度な制御コマンド
スクリプトやマップイベントの「実行内容」で使用できる主要なコマンド一覧です。

| コマンド | 引数例 | 解説 |
| :--- | :--- | :--- |
| `@map_load` | `id x y dir` | マップ `id` の座標 `(x,y)` に、向き `dir` で移動。 |
| `@battle` | `troopId onWin onLose` | トループと戦闘開始。勝利時は `onWin`、敗北時は `onLose` ラベルへ。 |
| `@call` | `templateId` | コモンイベント（テンプレート）をサブルーチンとして実行。 |
| `@ch` | `slug pose pos` | ノベル形式の立ち絵を表示。RPGの会話シーンに最適。 |
| `@scroll_text` | `text speed=1` | 指定したテキストを画面下から上へスクロール表示。 |
| `@filter` | `gray 0.5` | 画面全体にフィルター（モノクロ、セピア等）を適用。 |

## 5. セーブデータと永続化
セーブデータ（Schema v2）には以下の情報が自動的に含まれます。
- **パーティ状態**: 各キャラの現在HP/MP、レベル、累計経験値、現在の装備。
- **インベントリ**: 所持ゴールド、各アイテムの所持数。
- **マップ状態**: 現在のマップID、座標、向き、および訪問済みのマップ一覧。

## 6. クリエイターへのアドバイス
- **変数の積極活用**: スイッチだけでなく、数値変数を活用して「好感度」や「名声」を管理し、イベントページで条件分岐させることで、深みのあるシナリオが構築できます。
- **ノベルとRPGの切り替え**: 
  - ダンジョン探索中は `@filter` で雰囲気を出し、ボス戦直前の会話では `@camera_set` でキャラクターにズームする、といった演出が可能です。

このガイドに記載されていない詳細な挙動や、特定の計算式の調整については、開発チーム（または Issue）へフィードバックをお寄せください。
