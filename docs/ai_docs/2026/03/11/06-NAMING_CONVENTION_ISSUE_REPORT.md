# ファイル命名規則の不一致に関する調査報告書 (2026-03-11)

## 1. 発生している事象
各プロジェクトの出力ファイル名において、プレフィックス（接頭辞）が `P01_` だったり、プロジェクトID（`POLITICS_`等）だったりと、統一されていない。

## 2. 原因分析

### A. ツール改修前の生成（Sangokushi 等）
動的ID命名ロジックを実装する前に生成されたプロジェクトは、プログラム内に `const PIPELINE_ID = 'P01'` がハードコードされていたため、一律で `P01_` が付与された。

### B. `resume` による過去遺産の引き継ぎ（Fantasy 等）
ツール改修後に `--resume` オプションを使用してエクスポート等を行うと、`initOutputDir` 関数が過去の出力ディレクトリからファイルを全てコピーする。この際、古い `P01_` プレフィックスのファイルも一緒にコピーされ、新しく生成された動的IDファイルと混在する結果となった。

## 3. 現在の各ディレクトリの状態

- `projects/politics/output/20260311_121903`: **正常** (`POLITICS_`)
- `projects/sangokushi/output/20260311_084121`: **旧形式** (`P01_`)
- `projects/fantasy_generated/output/20260311_161005`: **混在** (コピーされた `P01_` が残存)

## 4. 解決策：完全新規生成によるクリーンアップ
過去のファイルを一切コピーせず、設定（s_01-s_04）のみから **一から生成をやり直す（all コマンドを resume なしで実行）** ことで、すべてのファイルが正しいプロジェクトIDで統一される。

## 5. 今後の運用ルール
- 命名規則を検証する際は、`--resume` を使わず新規ディレクトリで生成を開始すること。
- 過去の `P01_` 成果物は、必要に応じて手動でリネームするか、再生成を行う。
