# 地の文強化戦略：Few-shot 移行計画書 (2026-03-11)

## 1. 背景
当初計画していたファインチューニング（FT）について、Google AI Studio の Web UI および API 経由での実行が困難である（モデル権限やスキーマの不整合）ことが判明した。プラットフォームの初期サンプル（ファンタジー）を高品質に量産するという目標を最短で達成するため、モデルの「重み」を更新する FT 方式から、コンテキスト内に「正解例」を示す **Few-shot 方式** へ切り替える。

## 2. 役割の定義
本計画は、シナリオ生成パイプラインの **Stage 4（本文執筆）** を担当するモデルの筆力を向上させることを目的とする。

- **インプット**: Stage 3 の演出指示 JSON
- **アウトプット**: Few-shot で示された「黄金の文体」をトレースした情緒豊かな本文

## 3. 実装手順（ロードマップ）

### Step 1: 精鋭例示の選定
`<PROJECT_ROOT>/docs/10_ai_docs/2026/03/11/ft_fantasy_narrative/tuning_data.jsonl` から、以下の 4 カテゴリを網羅する **10 件のエピソード** を抽出する。
- **情景描写**: 密度・特盛（廃墟、自然）
- **身体感覚**: 五感重視（匂い、温度、重さ）
- **アクション**: テンポ・超速（戦闘、魔法）
- **心理描写**: 感情・極振（葛藤、決意）

### Step 2: プロンプトへの組み込み（フルパス）
`<PROJECT_ROOT>/apps/hono/src/lib/assist/prompts.ts` の `buildStage4Prompt`（または相当する執筆指示）に、選定した 10 件を「お手本（Examples）」として挿入する。

### Step 3: コンテキストキャッシュの有効化
Few-shot によりプロンプトが長くなるため、Gemini の **Context Caching** を活用し、例示部分のトークンコストを最小化する。

## 4. 期待される効果
- **作家性の統一**: 30個の正解データに基づく「あなたらしい文体」が、全章にわたって安定して出力される。
- **劣化の防止**: 以前発生した「ゲーム形式にしようとして文章が劣化する」問題を、自然言語ベースの例示によって克服する。
- **即時性**: 学習待ち時間ゼロで、修正したその瞬間から新モデルと同等の品質で量産が開始できる。
