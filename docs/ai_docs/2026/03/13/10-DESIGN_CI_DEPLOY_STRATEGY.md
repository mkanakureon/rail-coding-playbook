# 詳細設計書: デプロイ & CI 戦略 (RPG/マップ機能)

**作成日**: 2026-03-13
**対象**: `scripts/`, `.github/workflows/`

## 1. 目的
RPG 機能の追加に伴い、コードベースが大規模化する。これらを安全にデプロイし、各プラットフォームで動作を自動検証する体制を構築する。

## 2. CI パイプラインの拡張

### 2.1 自動テストの追加
- **マップデータ整合性テスト**: `packages/map/src/validate.ts` を用いて、リポジトリ内の全 `.json` マップデータが仕様に準拠しているか PR 時にチェック。
- **ヘッドレス描画テスト**: `native-engine` のスクリーンショット機能を CI で実行し、マップのタイル描画結果に意図しない崩れがないか視覚的に検証（Visual Regression Testing）。

### 2.2 ビルド検証
- **Monorepo Dependency Check**: `npm run build` を実行し、`core` -> `web` -> `editor` のビルドチェーンが壊れていないか確認。
- **Android APK 生成**: `native-engine` の Android ビルドを CI 上で実行し、コンパイルエラーを早期検知。

## 3. デプロイ戦略

### 3.1 Azure Container Apps へのデプロイ
- **Web 版エディタ**: マップ編集機能を含む新しいエディタを、既存の `apps/editor` パイプラインで Azure へデプロイ。
- **静的アセット**: `PNG/公式アセット` およびマップデータを CDN (Azure Front Door 等) を通じて配信し、ランタイムでのロードを高速化。

### 3.2 モバイル版の配布
- **Android (TestFlight相当)**: ビルドされた APK を社内配布サーバーに自動アップロードし、QR コードで実機テストを容易にする。

## 4. 監視とエラー報告
- **Runtime Error Tracking**: マップ読み込み失敗や衝突判定のスタックなど、ランタイムでのエラーを Sentry 等で収集し、マップID・座標と共に報告する。

## 5. ロールバック計画
- **データ互換性**: マップ形式のバージョン管理を徹底。古いエンジンで新しいマップを開かないよう、`SaveData` にエンジンバージョン情報を付与し、非互換時は警告を表示する。
