# RPGエンジン & UIレイアウト 統合テスト計画書

**作成日**: 2026-03-13
**作成者**: Gemini CLI
**対象**: マップシステム、UIレイアウトエンジン、ゲームプレイ統合ロジック

---

## 1. テスト戦略

「ロジックの正当性」「データの整合性」「ユーザー体験の品質」の 3 軸で検証を行う。

| テストレベル | 手法 | 主な検証項目 |
| :--- | :--- | :--- |
| **単体テスト** | Vitest | オートタイル判定、条件評価式、エンカウント計算、補間移動の数学的正確性。 |
| **統合テスト** | Vitest / Supertest | コンパイル結果の整合性、API (CRUD) の動作、パケット構造。 |
| **E2E テスト** | Playwright | マップ遷移フロー、イベント発火シーケンス、UI レイアウト切り替えの視覚的確認。 |
| **視覚・感覚テスト** | 手動 / 録画 | 移動の滑らかさ、アニメーションの違和感、UI の配置バランス。 |

## 2. 検証項目詳細

### 2.1 マップ & 移動ロジック (MapSystem)
- **衝突判定**: 
    - 壁、他イベント、マップ境界での停止。
    - `through` フラグが有効なイベントとの重なり。
- **補間移動**:
    - 低速/高速移動時のカクつきの有無。
    - 移動中の入力予約（先行入力）の挙動。
- **オートタイル**:
    - 47 パターンすべての接続整合性（特に孤立タイルや T 字路）。

### 2.2 UI レイアウトエンジン (PlayLayout)
- **プリセット切り替え**:
    - `@layout_set` 命令実行時の瞬時かつ正確な再配置。
    - 切り替え時の zIndex 順序の維持。
- **リサイズ耐性**:
    - ブラウザのウィンドウサイズ変更に伴う UI 要素の相対/絶対位置の維持。
- **NameBox / ClickWaitIcon**:
    - メッセージ表示と非表示に同期した正確な可視性制御。

### 2.3 イベント & ゲームシステム
- **条件評価 (Conditions)**:
    - スイッチ・変数の全比較演算子（==, !=, >, <, >=, <=）の正常動作。
    - 複数条件（AND）の複合判定。
- **エンカウント**:
    - 指定された歩数平均に基づいた出現確率の妥当性。
    - リージョン ID による出現モンスターの切り替わり。
- **インベントリ**:
    - アイテム獲得・消費時の内部データの正確な増減。

## 3. E2E テストシナリオ案

`tests/rpg/` ディレクトリに以下の自動テストを作成する。

1.  **Town Exploration Scenario**:
    - 村のマップをロード -> 町の人に話しかける（メッセージ窓表示） -> 宝箱を開ける（アイテム獲得） -> 家に入る（マップ遷移 & 暗転演出）。
2.  **Wilderness & Battle Transition**:
    - フィールドマップを走行 -> 一定歩数でエンカウント発生 -> バトル画面（モック）への遷移を確認。
3.  **UI Layout Stress Test**:
    - 会話中にレイアウトを次々と変更 -> UI が重なったり画面外に消えたりしないかを確認。

## 4. 実行環境とスケジュール

- **CI/CD 統合**: `npm test` および `npm run test:e2e` に RPG 関連のスイートを追加。
- **検証用アセット**: 4 方向歩行グラフィック、全パターンのタイルセットを含む「テスト専用プロジェクト」を維持。
