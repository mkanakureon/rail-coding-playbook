# フルスタックRPGエンジン完遂計画書 (改訂版)

**作成日**: 2026-03-13
**作成者**: Gemini CLI
**目的**: サーバー（API）およびデータベースをフル活用し、マップ制作、UIカスタマイズ、ゲームプレイ、クロスプラットフォーム展開までを網羅する完全なRPG/ADVハイブリッド環境を構築する。

---

## Phase 1: 高度なマップ表現とUI基盤 (Engine & API)

マップデータとUIレイアウトの永続化、およびビジュアル表現の確立。

### 1.1 マップデータのAPI連携
- **API (Hono)**: マップデータの CRUD エンドポイント実装。
- **DB (PostgreSQL)**: `maps` テーブルの作成（JSONB形式でのデータ保持）。
- **Web**: API 経由のマップデータロードへ完全移行。

### 1.2 UIレイアウトエンジン (Tsukuru Integration)
- **Core**: `PlayLayout` 型定義の追加と標準プリセット（ノベル/RPG）の作成。
- **Web**: `UiLayoutContainer` の実装と `WebOpHandler` の統合。
- **表現**: メッセージウィンドウ、名前枠、選択肢、ボタン群の配置を JSON で動的に変更可能にする。

### 1.3 高度な描画エンジンの完成
- **47タイル形式オートタイル**: 隅の接続判定を含む完全なオートタイル実装。
- **アニメーション**: キャラクターの 4 方向歩行アニメーションの精密化。

---

## Phase 2: 次世代エディタ群の構築 (Editor Integration)

制作体験を最大化する 2 つのエディタを統合。

### 2.1 マップペイント & イベントエディタ
- **ペイント**: タイルセットから範囲選択・スタンプ・塗りつぶしができる UI。
- **イベント**: マップ上をクリックして NPC やワープポイントを設置。プロパティパネルでの詳細設定。

### 2.2 ツクール型 UI レイアウトエディタ
- **ビジュアル編集**: プレイ画面の UI 部品（メッセージ窓、ボタン等）をマウスで配置変更。
- **プリセット管理**: プロジェクト設定からレイアウトプリセットを選択・保存。

---

## Phase 3: ゲームシステムとUIの完全統合 (Gameplay & UI)

### 3.1 本格的なインベントリ & メニュー
- **メニュー UI**: マップ上で開けるメニュー（アイテム、ステータス、セーブ/ロード）。
- **アイテム効果**: データ駆動型のアイテム使用ロジック。

### 3.2 エンカウント & バトル接続
- **リージョンベース**: エリアごとの敵出現設定。
- **歩数カウンタ**: ランダムエンカウント判定とバトル画面への遷移。

---

## Phase 4: マルチプラットフォーム最適化 (Native & Performance)

### 4.1 Native Engine (C++/SDL2) パリティ
- **ロジック移植**: `MapSystem` および `PlayLayout` 適用ロジックを C++ へ完全移植。
- **最適化**: Switch 上での描画パフォーマンスの最大化。

---

## 到達目標 (Definitions of Done)

1.  **エディタ**: マップを描き、イベントを置き、プレイ画面の UI レイアウトを自由に変更できる。
2.  **ゲーム**: 滑らかな移動、オートタイル、ランダムエンカウント、アイテム、カスタムUIを含む本格RPG。
3.  **保存**: プロジェクトデータ（マップ、レイアウト、シナリオ）がすべてサーバーで安全に管理される。
