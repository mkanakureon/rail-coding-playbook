# 詳細設計書: イベント & シナリオ連携ブリッジ

**作成日**: 2026-03-13
**対象**: `packages/web`, `packages/interpreter`

## 1. 目的
マップ上のイベント発火とノベルシナリオ（.ksc）をシームレスに連携させる。

## 2. イベントトリガー・フロー

### 2.1 発火プロセス
1.  **判定**: `MapSystem` がプレイヤーの移動終了時、または決定ボタン押下時に現在位置の `MapEvent` を検索。
2.  **条件**: `MapEvent.conditions` (スイッチ、変数、アイテム所持) をチェック。
3.  **実行**: 条件を満たす場合、`MapEvent.scenarioId` を取得。

### 2.2 シナリオの動的実行 (OpRunner Bridge)
- `OpRunner.start(scenarioId)` を呼び出す。
- **実行中**: マップ上のプレイヤー移動を一時停止（Input を会話ウィンドウにフォーカス）。
- **終了時**: 会話ウィンドウを閉じ、プレイヤーの移動制御を再開。

## 3. インタプリタ (KSC) との連携

### `IEngineAPI` の拡張
- `.ksc` スクリプトから以下の情報を取得・操作できるようにする。
    - `getMapId()`: 現在のマップID。
    - `getPlayerPos()`: プレイヤーの座標 {x, y, dir}。
    - `setPlayerPos(x, y, dir)`: 強制座標変更。
    - `loadMap(id)`: マップ切り替え。

### 変数の共有
- `Interpreter` の `vars` と、マップエンジンのスイッチ・変数は、同一の `SaveData.vars` を参照・更新するように設計する（データの二重持ちを避ける）。

## 4. 特殊イベントの設計
- **扉・ワープ**: `MAP_LOAD` を呼ぶだけの単純なシナリオファイルを自動生成（またはエディタで設定）。
- **宝箱**: `ITEM_ADD` 命令を含んだシナリオ。
- **並列イベント**: プレイヤーの移動を止めずに、背後で実行され続けるイベント（BGS 制御、時間経過など）。
