# サーバー不要のRPGエンジン強化計画書

**作成日**: 2026-03-13
**作成者**: Gemini CLI
**目的**: サーバー（API）が利用できない制限下で、フロントエンドおよびコアロジックの改善を最大化するためのロードマップを定義する。

---

## 1. 視覚演出の高度化 (Visual Polish)

サーバーなしで PixiJS の描画ロジックとローカルアセットのみで完結可能な項目。

### 1.1 歩行グラフィック・アニメーションの実装
- **内容**: キャラクターを単色矩形から、スプライトシート形式（例：3x4フレーム）へ変更。
- **実装**: `MapCharacter` にテクスチャ切り替えロジックを追加。`direction` と `moveProgress` に同期した足運びアニメーションを実現。
- **検証**: `web/public` へのダミー素材配置と静的実行。

### 1.2 高度なオートタイル（47タイル形式）への対応
- **内容**: 現在の 16 タイル（4bit）形式を、隅（コーナー）の接続まで考慮した 47 タイル形式（RPGツクール互換）へ拡張。
- **実装**: `MapSystem` のビットマスク計算を 8-bit 周辺判定へアップグレード。
- **検証**: 複雑な地形接続のパターンをユニットテストで網羅。

---

## 2. ゲームロジックの精緻化 (Game Logic)

Vitest によるロジックテストのみで 100% 検証可能な項目。

### 2.1 エンカウント & バトル接続ロジック
- **内容**: マップ上の通行座標に基づく「エンカウント歩数計算」と、`BATTLE_START` 命令への自動遷移。
- **実装**: `MapSystem` 内に歩数カウンタと確率計算ロジックを実装。
- **検証**: 期待する確率でバトル命令が発行されるかをユニットテストで確認。

### 2.2 アイテム・スキル効果の定数定義と計算
- **内容**: 回復アイテムや装備品がステータスに与える影響の計算式を定義。
- **実装**: `packages/core` または `packages/battle` への計算ユーティリティの追加。
- **検証**: ダメージ計算、回復計算の整合性をテスト。

---

## 3. UI/UX 基盤の構築 (UI Foundations)

React (Editor) または PixiJS Overlay (Web) の静的なコンポーネント実装。

### 3.1 インベントリ & ステータス画面 (Web)
- **内容**: マップ上で「I」キー等を押した際に開く、所持品一覧とキャラステータスのオーバーレイ UI。
- **実装**: `packages/web` の UI 層への追加。データは `InventorySystem` から取得。
- **検証**: 静的なモックデータを使用した表示確認。

### 3.2 マップエディタの React プロトタイプ (Editor)
- **内容**: `apps/editor` 内でのタイルパレット、レイヤー選択、イベント配置の UI パーツ実装。
- **実装**: API との通信を Mock に差し替え、React コンポーネントのみを構築。
- **検証**: Vite の HMR を利用したフロントエンドのみの動作確認。

---

## 4. 優先順位

1.  **アニメーション**: 見た目のインパクトが大きく、エンジンの「完成品感」が激増するため。
2.  **高度オートタイル**: 制作ツールの使い勝手（表現力）に直結するため。
3.  **UI/UX基盤**: ゲームとしての操作体系を確立するため。

---
 サーバーがなくても、ロジックと表現力の追求に限界はありません。
 「動くのは当たり前、触って心地よいエンジン」を目指して、一歩ずつ進めていきます。
