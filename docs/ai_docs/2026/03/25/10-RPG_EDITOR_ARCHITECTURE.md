# 次世代RPGエディタ設計方針：空間とデータの直感的統合

## 1. コンセプト：Visible Logic
RPG特有の複雑な「データ（数式・変数）」と「空間（マップ・イベント）」をすべて視覚化し、初心者には「直感的な配置」を、上級者には「精密な数理制御」を提供する。

---

## 2. 3つの専門編集モード

### ① プロパティ・データベース（データ主権）
*   **機能:** アクター（味方）、エネミー、アイテム、スキルの統合管理。
*   **設計:** 単なる数値入力ではなく、成長曲線エディタや、ダメージ計算式のリアルタイム・シミュレータを内蔵。
*   **ポータビリティ:** ここで設定した「属性」や「耐性」は、Switchの Native 側 `packages/battle` でそのまま高速計算に使用される。

### ② 空間イベントエディタ（マップ主権）
*   **機能:** タイルベースのマップ作成と、位置情報のイベント紐付け。
*   **設計:** 「このマスを踏んだら実行」というトリガーを、マップ上のドラッグ＆ドロップで定義。
*   **可視化:** スイッチ（フラグ）によるマップの変化やNPCの移動ルートを、エディタ上で「透視」できるようにする。

### ③ バトル・シーケンス・エディタ（演出主権）
*   **機能:** バトル中のアニメーション、SE、ダメージタイミングの微調整。
*   **設計:** タイムライン形式を採用。スキル発動から命中、演出終了までを 1/60 秒単位で同期させる。
*   **ロジック:** 敵の行動AI（条件分岐）をフローチャート形式で構築。

---

## 3. 開発体験（Developer Experience）の階層化
1.  **Level 1 (Entry):** プリセット（標準的なRPGセット）を配置し、テキストを変えるだけで完成。
2.  **Level 2 (Standard):** 変数とスイッチを駆使し、独自のクエストや成長システムを構築。
3.  **Level 3 (Pro):** GLSL シェーダーによる独自エフェクトの実装と、ダメージ計算式の完全カスタマイズ。
