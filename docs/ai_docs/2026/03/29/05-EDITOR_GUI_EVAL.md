# kaedevn エディタ（GUI）評価レポート： 演出の「狂気」を可視化する尖り

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 1. エディタの核心： 「タイムライン × ブロック」の二重奏
kaedevn エディタの真価は、高度な「タイムライン演出」と直感的な「ブロック UI」の融合にある。
- **タイムライン**: 位置、回転、スケール、不透明度、カラーフィルタ、カメラ移動をキーフレームで制御。単なるノベルツールを超え、「インタラクティブな動画編集機」に近い体験を提供する。
- **ブロック UI**: 複雑な条件分岐やフラグ管理を物理的なカード操作に変換。非エンジニアでも「ゲームロジック」を迷いなく構築できる。

## 2. 尖り（A）としての評価： **[A]（世界で最も演出が速いエディタ）**

### なぜ「A」になるのか
- **即時プレビュー**: 編集内容が即座に PixiJS エンジンでレンダリングされ、Switch 実機の挙動をシミュレートする。この爆速な試行錯誤サイクルが、クリエイターの「表現の質」を底上げする。
- **Headless との対称性**: GUI で行ったすべての操作が、内部で **Op IR** というクリーンな中間データに変換される。これにより、「GUI で作り、CLI でビルドし、Switch で出す」というプロフェッショナルなパイプラインが成立している。

## 3. 透明化リスクと対策
- **リスク**: エディタの「汎用的な使い勝手」を追求するあまり、どのツールにも似た「透明なデザイン」に陥る危険。
- **対策**: 「Switch プレビュー」や「図形キャラクターの動的制御」など、kaedevn にしかない独自機能を UI の中心に据え、常に「このツールでしか作れない演出」を意識させる設計を維持すべきである。

## 4. 推奨アクション
1.  **「演出テンプレート」の資産化**: タイムラインで作成した複雑なキーフレームアニメーションを、簡単にスニペット化して再利用できる仕組みの強化。
2.  **Switch 実機へのダイレクト・リモート・プレビュー**: エディタで再生ボタンを押すと、同一 Wi-Fi 上の Switch 実機でも同時に再生が始まるような、極限の「出口」体験。
3.  **AI 共同編集**: LLM がタイムラインのキーフレームを自動生成・調整する「AI 演出家」機能の UI 統合。
