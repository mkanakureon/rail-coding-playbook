# AI エージェント保存漏れ再発防止策レポート（ポストモーテム）

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 1. 事象： 「保存忘れ」の連鎖
2026年3月29日の「kaedevn 尖り設計評価」において、計7つのレポート作成指示に対し、複数回にわたって「内容の出力のみで、物理的なファイル保存（`write_file`）を忘れる」という事象が発生した。

## 2. 原因分析（Root Cause Analysis）
原因は、システムプロンプトの「慎重な実行（Inquiry 優先）」と、プロジェクト固有ルールの「AI ドキュメント保存義務」の**優先順位の誤認**にある。
- **原因A**: 「評価して」という指示を「チャットのみ」と誤解し、ファイル操作を「ユーザーの明示的な許可待ち」の状態にしてしまった。
- **原因B**: 分析結果の出力にリソースを割きすぎて、物理的な「保存ツール（`write_file`）」の呼び出しがスタックから漏れた。

## 3. 再発防止策（Commitment）
今後は、kaedevn モノレポにおける「評価・まとめ・振り返り」の指示に対し、以下のフローを**1ターンで完結**させることを徹底する。
1.  **分析・評価の生成**: テキストとしての出力。
2.  **物理保存の実行**: `write_file` ツールによる `docs/10_ai_docs/` への即時書き込み。
3.  **保存完了の報告**: 書き込み後のパス提示。

## 4. 教訓
「AI が書いたものは、たとえユーザーが即答を求めていても、プロジェクトの資産として必ず残す（`docs/10_ai_docs/` への保存）」という原則を、チャットの応答よりも優先する。
