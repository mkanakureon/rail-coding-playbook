# kaedevn CLI ツール評価レポート：Headless Editor という尖り

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 1. コンセプト評価： 「エディタができることは全部 CLI でできる」
`kaede-tools` の本質は、単なる補助ツールではなく「**UI を持たないエディタ（Headless Editor）**」である。GUI で行われる操作（シナリオ編集、アセット管理、ビルド、最適化）のすべてが CLI 経由でプログラマブルに実行可能であるという設計思想は、競合ツールに対する圧倒的な差別化要因（A）となっている。

## 2. 尖り（A）としての価値： **[S]（プロフェッショナル・グレード）**

### なぜ「替えがきかない」のか
- **AI 時代のワークフロー**: 生成 AI が出力した大量のテキストや画像を、手作業を介さず直接ゲームデータへ流し込むパイプラインを構築できる。
- **自動化の極致**: GitHub Actions 等の CI 環境で、シナリオの整合性チェックから Switch 用パッケージのビルドまでを全自動化できる。
- **エンジニア・ファースト**: GUI を嫌う高度なエンジニア層に対し、好みのエディタと CLI だけで「Switch 向けゲーム」を開発できる環境を提供している。

## 3. Switch パブリッシュ（本来のA）への貢献
- **実機転送の自動化**: 「ビルド → 最適化 → Switch 転送」を 1 コマンドで完結させ、検証サイクルを劇的に短縮する。
- **リソースの事前最適化**: Switch のメモリ制限に合わせ、CLI がビルド時にテクスチャ圧縮や音声変換を自動で行うことで、パブリッシュのハードルを最小化する。

## 4. 現状と今後の課題
- **現状**: 現在の CLI はアセット変換（MP4/PNG/WebP）に強みを持っている。
- **課題**: エディタが持つ「シナリオのパースと IR 変換」のロジックを CLI 側でも完全に同期し、`kaede-tools build` 一発で実行可能なバイナリ（game.pak）を生成するまでの「完結性」を強化すべきである。

## 結論
「エディタの全機能を CLI で」というコンセプトは、kaedevn を「アマチュア向けツール」から「プロフェッショナルなゲーム開発基盤」へと押し上げる、もう一つの核心的な「尖り」である。
