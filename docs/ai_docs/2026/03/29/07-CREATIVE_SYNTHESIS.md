# kaedevn 創造評価レポート：個人の熱量を Switch へ叩き込む「狂気のエコシステム」

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 1. 創造の核心（A-factor の統合評価）
kaedevn が創造しているのは、単なる「ゲーム制作ツール」ではない。それは、**「個人が、プロ級の技術的障壁（Switch 移植・動画最適化・物理アニメ）を無視して、表現の熱量だけに集中できる特権」**である。

- **「Switch」という出口**: これが全設計の北極星となっている。
- **「狂気」の技術レイヤー**: C++ エンジン、Headless CLI、図形キャラの物理演算。これら「裏側」の作り込みが、表側の「手軽さ」を支えている。

## 2. 制作プロセスのイノベーション
これまでのレポートで確認した各要素が、以下のように「創造」の形を変えている。

- **「描かなくていい」創造**: 図形キャラクターが、絵を描く時間を「演出を練る時間」に変えた。
- **「待たなくていい」創造**: 爆速の CLI とプレビューが、試行錯誤の回数を 10 倍に増やした。
- **「コードを知らなくていい、でもコードが味方」**: ブロック UI で積み上げ、必要なら `KscBlock` でコードの力を借りる。この柔軟性が、創造性の「天井」を撤廃した。

## 3. 「尖り（A）」の相乗効果
バラバラに見えた各機能は、実は一つの「鋭い刃」を形成している。

1.  **AI 素材 (動画)** を CLI で高速変換し、
2.  **図形キャラ (役者)** と組み合わせて、
3.  **タイムライン (演出)** で命を吹き込み、
4.  **C++ エンジン (心臓)** でスマホや Switch でヌルヌル動かす。

この流れに一切の「無駄な待ち時間」がない。この**「思考の速度で Switch ゲームができる」**こと自体が、kaedevn 最大の創造的価値である。

## 4. 創造的結論： 「透明化」への完全勝利
「文句のない 70 点」を目指すツールは、最後には誰も使わなくなる。kaedevn は、以下の「極端さ」を選択することで、透明化を回避し、独自の生態系を築きつつある。

- **極端な軽量性**: テクスチャを捨てて「図形」で動かす。
- **極端な自動化**: エディタを捨てて「CLI」でビルドする。
- **極端な出口戦略**: Web を超えて「Switch」に直接刺す。
