# kaedevn 総合評価レポート：Switch への最短距離を創る「尖り」の結集

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 1. 核心的価値（A-factor）： **[S]（代替不能）**
kaedevn の「一点突破」は、**「個人クリエイターの熱量を、そのまま Nintendo Switch という商業出口へ叩き込める唯一の極小エコシステム」**である。

- **Switch 直接パブリッシュ**: C++/SDL2 によるネイティブ・エンジン戦略が、理論上の理想ではなく、実装レベルの「最短距離」として確立されている。

## 2. エコシステムの統合評価： **[A+]**
「Switch に出せる」という出口を、以下の 3 つの「狂気」が支えている。

- **エディタ（GUI）の狂気**: タイムラインとブロック UI を融合し、Web 上で Live2D 級のアニメーションを数分で組み上げる「演出の爆速化」。
- **CLI（Headless）の狂気**: 「エディタでできることはすべて CLI でできる」という設計。AI アセットの自動変換からビルドまでをプログラマブルに完結させる、プロフェッショナル向け自動化環境。
- **図形キャラクターの狂気**: テクスチャを捨て、物理演算とベクトル描画だけで「生きているキャラ」を生成。アセット制作コストを 0 にしつつ、Switch のリソース制限を技術的に回避している。

## 3. スマホ・動画・AI 時代への適応： **[A]**
- **AI 動画連携**: Midjourney 等の AI 動画から背景を抜き、ゲーム用アセットへ自動変換するパイプラインを標準搭載。
- **スマホ最適化**: ハードウェア加速を前提とした C++ エンジンにより、低スペック端末でも「動画だらけのリッチな演出」を軽快に動かす。

## 4. 創造的結論： 透明化への勝利
kaedevn は「誰にでも使いやすい 70 点のツール」であることを拒否し、**「Switch で表現したい、狂った熱量を持つ個人のための 120 点の武器」**であることを選んでいる。

- **透明化リスクへの対策**: 汎用機能の追加ではなく、図形キャラや Headless CLI といった「kaedevn でしか体験できない制作手法」を研ぎ澄ますことで、代替不能な地位を築いている。

## 5. 推奨される次の一手
- **機能パリティの完遂**: Web (PixiJS) で実装されたリッチな演出を、100% 同期して C++ (Native) で動かすための完全なブリッジ化。
- **「Switch プレビュー」の徹底**: エディタの UI そのものを、常に Switch の「出口」を意識させるデザイン（セーフエリア・操作感）へ統一する。
