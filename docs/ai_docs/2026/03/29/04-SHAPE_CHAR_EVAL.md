# kaedevn 図形キャラクター評価レポート： 「生きている」汎用アセットの尖り

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 1. 機能の核心： 「SDK 無しの Live2D 風アニメーション」
`chara-editor.ts` で実装された図形キャラクターは、単なる「仮置きの図形」ではない。
- **動的生成**: パラメータ補間と物理演算（髪の揺れ、呼吸）により、数千行のコードだけで「生きている感」を演出。
- **低コスト・高表現**: Live2D SDK や重いテクスチャアセットを必要とせず、ベクトル描画（Graphics API）のみでリッチなアニメーションを実現している。
- **インライン制御**: シナリオテキスト中の `[e:smile]` などのタグで、ポーズや表情をリアルタイムに、かつ滑らかに遷移させることが可能。

## 2. 尖り（A）としての評価： **[A+]（制作の心理的障壁を破壊する）**

### なぜこれが「A」になるのか
個人開発者が Switch パブリッシュを目指す際、最大の壁は「全キャラクターの立ち絵アセット（表情差分含む）を揃えること」である。
- **アセット不要の即戦力**: 図形キャラクターは、絵が描けない開発者でも「表情豊かに動くキャラクター」を即座に配置できる。
- **「プロトタイプ」を「製品」に変える**: 単なる仮置きではなく、その独特のミニマルな造形が「kaedevn らしいスタイル」として成立しており、そのまま製品版としてリリース可能なクオリティに達している。
- **Switch への最適性**: 大量のテクスチャメモリを消費しないベクトル描画は、リソース制限の厳しい Switch 環境において圧倒的に有利である。

## 3. 「透明化リスク」へのカウンター
「汎用的な機能」を増やすことは透明化を招くが、この図形キャラクターは**「kaedevn でしか作れない独特の質感」**を生んでおり、プロダクトの個性を際立たせている。

## 4. 推奨アクション： 尖りをさらに研ぐ
1.  **C++ エンジンへの完全移植**: 現在 PixiJS 上で動いている「パラメータ補間アニメーション」のロジックを、`native-engine` (C++) 側でも 100% 同期させる。
2.  **AI シナリオ連携の強化**: LLM が感情分析を行い、自動的に `[e:smile]` などのタグを挿入する機能を強化し、「書くだけで演出が完成する」体験を極める。
3.  **スタイルの多様化**: 現在の「男女 4 人ずつ」のプリセットを超え、スライダーで体型や髪型をカスタマイズできる「図形キャラ・ジェネレーター」をエディタに統合する。
