# kaedevn エコシステム統合評価レポート

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 背景と目的
「Nintendo Switch パブリッシュ」という核心的な尖り（A-factor）を支える周辺エコシステム（エディタ、公式アセット、マルチプラットフォーム戦略、CLI ツール）の現状を評価し、その整合性と貢献度を明らかにする。

## 評価内容

### 1. エディタ機能： **[A]**
- **タイムライン & ブロック UI**: キーフレームアニメーションとシナリオフローを高度な React コンポーネントで実現。初心者への直感性と、内部的な **Op IR** 生成という専門性が高次元で両立している。
- **KS エディタ同期**: コードとビジュアルの双方向編集が、エンジニアとクリエイターの連携を「助け」ている。

### 2. 公式アセット & 互換性： **[A+]**
- **網羅性**: ファンタジーから学園、政治、中国短尺ドラマまで、ニッチかつ需要の高いジャンルを数千点規模で網羅。
- **狂気的な整合性**: Web (PixiJS) と Native (C++/SDL2) で全く同じ見た目を保証するレンダラーのミラーリングと、スラグベースのアセット管理により「1ピクセルの狂いもない」マルチプラットフォーム展開を可能にしている。

### 3. ４プラットフォーム戦略： **[A]**
- **対応状況**: Web, Switch, Android, iOS。
- **設計の鋭さ**: OS ネイティブのパフォーマンスを追求した C++ エンジン (`native-engine`) を持ちつつ、共通の Op IR インタープリタ方式を採用することで、挙動の一貫性を担保している。

### 4. CLI ツール (`kaede-tools`)： **[B+]**
- **実用性**: 「AI 動画からアニメーション立ち絵を生成（背景抜き・WebP 変換）」という、現代の制作フローの急所を突いた自動化を提供。
- **Aへの貢献**: アセット制作コストを劇的に下げることで、個人開発者による Switch パブリッシュの現実味を「助け」ている。

## 結論・今後の課題
kaedevn は、**「Switch に出せる」という尖りを、エディタの使い勝手とアセットの物量、そして狂気的なまでの互換性設計で支えている** 理想的なプロダクト構造である。

### 推奨アクション
1.  **アセットの「トーン」の整備**: 公式アセットの画風を AI 等で自動変換・統一できる機能の検討。
2.  **CLI のパブリッシュ機能強化**: `kaede-tools publish --platform switch` のような、デプロイまでを CLI で完結させる機能の追加。
3.  **Native エンジンの露出向上**: モバイル/Switch 向けビルドの進捗をより可視化し、「４プラットフォーム」の実在感を高める。
