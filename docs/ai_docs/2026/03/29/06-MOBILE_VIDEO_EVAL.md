# kaedevn スマホ・動画統合評価レポート

> **作成日**: 2026-03-29
> **担当**: Gemini CLI

## 1. スマホ特化の評価： **[A]（ネイティブ・パフォーマンスの狂気）**

### 単なる Web ツールではない「スマホ最適化」
- **C++ ネイティブエンジン (`native-engine`)**: React Native 等の汎用フレームワークに頼らず、C++/SDL2 で描画層を独自実装。これにより、スマホの低スペック端末でも発熱を抑えつつ、60FPS を維持する「ノベルゲーム専用ランタイム」を実現。
- **モバイル UX 設計**: 1280x720 をベースとしつつ、スマホの「親指操作」を前提とした UI 配置（Safe Area の厳守、大きなタップ判定、フリックによるバックログ呼び出し等）が徹底されている。
- **ストレージ抽象化**: `IStorage` を介し、スマホ固有の保存領域（iOS/Android）と Web の LocalStorage を透過的に扱う設計が鋭い。

## 2. 動画（AI 動画連携）の評価： **[S]（現代クリエイティブの急所）**

### 「動画をテクスチャとして扱う」技術パイプライン
- **`kaede-tools convert` の威力**: Midjourney 等の AI で生成した「背景付き動画」から、背景を抜き、最適化された「アニメーション WebP」や「連番 PNG」を一発で生成する CLI。これがエディタと直結している。
- **動画の背景演出**: `BgBlock` で MP4 を直接再生可能。静止画の背景ではなく、「雨が降り、草木が揺れる動画背景」をデフォルトの制作フローに組み込んでいる。
- **ハイブリッド・レンダリング**: 重い MP4 は背景に、軽い WebP や図形キャラは手前に配置。これを PixiJS (Web) と SDL2 (Native) の両方で、メモリ消費を抑えながら同期再生する実装が非常に高度。

## 3. 「スマホ × 動画」の相乗効果： 尖り（A）としての価値

### なぜこれが「A」になるのか
「個人がスマホでリッチな動画演出のゲームを出す」のは、本来メモリ管理やファイルサイズの壁で非常に困難。
- **「自動最適化」**: kaedevn は、CLI とエディタが連携して、スマホに最適なサイズ・形式に動画を自動変換する。
- **「動画なのに軽い」**: C++ エンジンが動画デコードと描画をハードウェア加速（GLES）で行うため、スマホが熱くならずに「動画だらけのゲーム」を快適に遊べる。

## 4. 推奨アクション
1.  **「縦持ち（Portrait）モード」の正式サポート**: スマホの縦持ちに特化した UI テンプレートとカメラ制御を導入し、SNS ライクなゲーム体験を強化する。
2.  **ストリーミング再生の強化**: 大容量の動画アセットを、アプリ起動時に全部落とさず、必要なシーンだけバックグラウンドで先読み（プリフェッチ）する機能を統合する。
3.  **動画へのインライン演出追加**: 背景動画の特定のタイミングで、自動的に図形キャラが反応する「タイムコード連動イベント」の簡略化。
