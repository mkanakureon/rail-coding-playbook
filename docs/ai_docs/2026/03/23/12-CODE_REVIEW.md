## Summary
急増するユーザー負荷に対応するための、バックエンド API 層の第1次スケーラビリティ改善 (Phase A)。

## Findings
1. **認証プロセスの効率化**:
    - JWT 自体にユーザーの状態（role/status）を保持させることで、認証時の DB Lookup を排除。
    - 大規模アクセス時の一番のボトルネックとなる DB 読み取り回数を大幅に削減。
2. **データベースクエリの最適化**:
    - ランディングページで多用される「人気順」「新着順」のクエリに対し、複合インデックスを適用。
    - Prisma の接続プール設定の最適化により、同時実行性を向上。
3. **リソース保護**:
    - API の取得件数にハードリミット (100件) を設定し、過度なデータ取得によるメモリ圧迫や DoS 攻撃を防止。

## Recommendations
- **ユーザー状態の即時反映**: JWT キャッシュの欠点として、ステータス変更（停止等）が即時反映されないリスクがある。クリティカルな操作（決済・公開等）においては、強制的に DB を参照する「二段構え」の設計を検討すべき。
- **カーソルベースの検討**: `skip` によるページネーションはデータ増大時に遅延するため、次フェーズでは `cursor` 方式への移行を推奨。
