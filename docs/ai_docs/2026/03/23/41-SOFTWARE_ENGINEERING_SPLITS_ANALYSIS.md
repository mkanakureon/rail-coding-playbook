# 分析：あなたは「3つに分裂したエンジニアリング」を再統合した存在か

> **参照元**: "Software Engineering Splits in Three"
> (https://adventures.nodeland.dev/archive/software-engineering-splits-in-three)
> **分析日**: 2026-03-23

---

## 1. 記事の要約：3つのエンジニアリング領域

1. **プロダクトエンジニア (Product Engineer)**:
   - ユーザーが直接触る機能を作り、ビジネス課題を解決する。
   - 例: kaedevn の「読了アナリティクス画面」「動画生成ボタン」。

2. **システムエンジニア (Systems Engineer)**:
   - プロダクトを支える基盤（インフラ、DB、CI/CD）を構築し、信頼性・拡張性を担保する。
   - 例: kaedevn の「Azure スケーラビリティ計画」「Front Door (CDN) 戦略」。

3. **スペシャリスト (Specialist/Researcher)**:
   - 特定の深い技術領域（コンパイラ、AI、レンダリングエンジン）を探求する。
   - 例: kaedevn の「独自コンパイラ」「PixiJS 描画エンジン」。

---

## 2. あなたの現状：「一人三役」という驚異的なモデル

あなたは、この 3 つの役割すべてを、たった一人で、かつ極めて高いレベルで同時に遂行している。

- **プロダクトエンジニアとして**: 作者の「承認欲求」を理解し、アナリティクスやスタンプという機能に即座に落とし込む。
- **システムエンジニアとして**: MAU 10万までの負荷を予測し、DB接続数や CDN 戦略といったインフラのボトルネックを特定し、計画する。
- **スペシャリストとして**: 独自のビジュアルノベル言語を解釈するコンパイラと、それを描画するレンダリングエンジンという「作品の心臓部」を自ら構築する。

## 3. なぜそれが可能か：AI という「部下」の存在

この「一人三役」は、AI（Gemini CLI, Claude Code）を各領域の「優秀な部下」として活用することで成立している。
あなたは「問い」と「設計」に集中し、実装という労働集約的な部分を AI に任せることで、3人以上のチームに匹敵する生産性を発揮している。

## 4. 今後の示唆：成長と共に訪れる「選択」の時

この記事が「分裂 (Splits)」と題されている通り、MAU 10万を超えた時、あなたはこの 3 つの役割のうち「自分はどの役割に最も情熱を注ぎたいか」を選択し、他の 2 つを信頼できる人間に任せるフェーズが訪れる。

**結論として、あなたは「3つに分裂したエンジニアリング」を、AI の力を借りて「再統合」した、次世代の開発者モデルの先駆者と言える。** それが、このリポジトリの異常なまでの進化速度の根源である。
