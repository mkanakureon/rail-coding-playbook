# 読者フィードバック戦略：管理コスト 0 の「承認」設計

> **作成日**: 2026-03-23
> **ステータス**: 戦略案（モデレーション負荷の回避）

---

## 1. 背景：自由コメントの「毒」を避ける

作者のモチベーションにとって読者の声は劇薬です。良い言葉は力になりますが、一言の誹謗中傷やスパムはプラットフォームの寿命を縮めます。MAU 10,000 に至るまでは、人力でのモデレーションは不可能です。

## 2. 代替案：エモーション・フィードバック

テキストコメントの代わりに、以下の「非テキスト」な反応機能を推奨します。

### ① シーン・リアクション（スタンプ）
- **機能**: プレイ中の特定ページで、読者が絵文字（❤️, 😂, 😭, ⚡️）をタップできる。
- **管理コスト**: 0（言葉ではないため、検閲不要）。
- **作者メリット**: アナリティクス画面で「どのページで読者が盛り上がったか」が可視化される。

### ② 読了後の「一言チップ」
- **機能**: 読み終わった後に「最高！」「天才」「全裸待機」などの運営が用意したポジティブな定型タグを選んで送る。
- **管理コスト**: 0。
- **作者メリット**: 語彙力に自信のない読者からも、確実にポジティブな反応が届く。

## 3. 実装のステップ

1. **Phase 1 (MAU 100〜)**: 「いいね」のカウントをページ単位でも取れるように拡張（アナリティクスへの統合）。
2. **Phase 2 (MAU 500〜)**: スタンプ機能の解放。
3. **Phase 3 (MAU 5,000〜)**: AI (LLM) による自動検閲を前提とした、テキストコメントの解禁。

---

## 4. 結論

「自由コメント」は諸刃の剣です。初期フェーズにおいては、**「悪意を物理的に排除した、純粋な好意だけが届くシステム」** を構築することが、デリケートな作者層を 100 人定着させるための最短ルートです。
