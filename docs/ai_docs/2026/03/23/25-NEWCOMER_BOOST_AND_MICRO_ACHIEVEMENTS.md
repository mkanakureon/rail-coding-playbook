# 作者定着戦略：新着ブースト ＆ マイクロ承認（小さな達成感）施策案

> **作成日**: 2026-03-23
> **ステータス**: 設計案（低実装難易度・高心理効果）

---

## 1. 「ゼロ・プレイ（誰にも読まれない）」の撲滅

作者が最も挫折するのは、公開したのに「プレイ数 0」が続く時です。これをシステムロジックで解決します。

### A. 「フレッシュ・ピックアップ」枠の設置
- **ロジック**: `playCount < 50` かつ `status === 'published'` の作品を優先的にトップページへ表示。
- **目的**: 初投稿作品でも 48 時間以内に **「最初の 10 プレイ」** を確実に発生させる。
- **実装**: `GET /api/works` に `sort=fresh` を追加し、新着かつ低再生数の作品をシャッフル取得。

### B. 「初プレイ」の豪華通知
- **施策**: 最初の 1 人がプレイした際、作者に「世界で最初の読者が訪れました！」と通知。
- **心理効果**: 「自分の物語が届いた」という成功体験を最小単位で提供。

---

## 2. マイクロ・マイルストーン（制作過程の「ご褒美」）

大きな「完成」を目指す前に、制作途中の小さな進捗を褒める仕組み。

- **達成バッジシステム**: 
    - 「1,000文字執筆達成！」
    - 「3キャラクター登場！」
    - 「5シーン突破！」
- **実装**: エディタ（frontend）の Zustand ストアでカウントし、達成時に祝祭感のある UI ポップアップを表示。

---

## 3. 「ジャンル別・はじめてのテンプレート」

「白い画面」からのスタートを回避し、書き始めの摩擦をゼロにする。

- **内容**: 
    - 「学園恋愛」「異世界ファンタジー」等の定番ジャンルのテンプレートプロジェクト。
    - 10行程度のセリフ、適切な背景・立ち絵が最初からセット済み。
- **メリット**: セリフを数行書き換えるだけで「自分の作品」として公開できる手軽さ。

---

## 4. なぜこの施策が「勝てる」のか

1.  **実装が低コスト**: 高度な非同期処理や外部 API 連携が不要。
2.  **作者の孤独を解消**: システムが常に「自分を見てくれている」という安心感。
3.  **コミュニティの活気**: 誰もが「最初の反応」を得られるため、初心者が定着しやすい。
