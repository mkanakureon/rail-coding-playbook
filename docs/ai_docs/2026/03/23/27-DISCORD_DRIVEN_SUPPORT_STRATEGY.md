# コミュニティ駆動型サポート（Discord 統合）戦略

> **作成日**: 2026-03-23
> **ステータス**: 仕様確定（自前サポートチャットからの転換）

---

## 1. 核心：なぜ Discord に集約するのか

自前のチャットシステムは「非公開」であり、そこでのやり取りは他のユーザーに役立ちません。Discord を使うことで、サポートを「一対一のコスト」から「コミュニティ全体の資産」へと変換します。

## 2. 実装イメージ（UI 誘導）

- **エディタ・サイドバー**: 「困ったら Discord で相談」ボタンを常設。
- **プロジェクト詳細ページ**: 「Discord コミュニティに参加」セクションを追加。
- **エラーハンドリング**: 保存エラー等の際に「Discord で問題を報告」というリンクを表示し、シームレスに誘導。

## 3. 運用のメリット

1. **自己解決の促進**: 過去の質問が検索可能になり、同じ質問への対応コストが下がる。
2. **信頼の構築**: 開発者とユーザーのやり取りがオープンになることで、プラットフォームへの信頼感が増す。
3. **コスト削減**: チャット用のサーバーリソースや DB スキーマ管理が不要になり、開発を「エンジン機能」に集中できる。

## 4. アクションアイテム

- [ ] エディタのヘッダー/サイドバーに Discord 招待リンクを配置。
- [ ] Discord Webhook を活用し、システム上の重大なエラー（5xx）を特定の管理用チャンネルに自動通知する（早期発見のため）。
- [ ] FAQ チャンネルを作成し、頻出する質問をストックする。

---

## 5. 結論

「仕様変更した」という判断は、MAU 100 〜 1,000 を目指す上での **運用効率化とコミュニティ形成の最大化** に直結する英断です。これ以後、自前チャットの実装は一切行わず、Discord との高度な連携にリソースを集中します。
