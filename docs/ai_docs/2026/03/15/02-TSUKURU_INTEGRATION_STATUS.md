# ツクール型機能の現状整理と実装ロードマップ

> **作成日**: 2026-03-15
> **担当**: Gemini CLI
> **ステータス**: 最新の概念マップ（20-rpg-tkool-concept-map.md）を正として統合

本ドキュメントは、kaedevn における「RPGツクール型機能」の調査結果、既存の実装状況、および今後の開発優先順位をまとめたものである。

---

## 1. kaedevn の4本柱（再定義）

最新の設計に基づき、システムを以下の4つの独立したモジュールとして再定義した。

| 柱 | 役割 | 現状 |
|----|------|------|
| **シナリオ** | テキスト演出・分岐・フラグ管理 | **完成** |
| **ゲームDB** | アクター・敵・技・アイテムの定義 | 未着手 (Phase 3) |
| **マップ** | タイル描画・歩行・マップイベント | **基盤完成** (Phase 5) |
| **UIレイアウト** | `PlayLayout JSON` による画面配置 | **エンジン完成** (Phase 6) |

## 2. 既存の実装状況（2026-03-15 時点）

### エンジン・ランタイム (packages/web, core)
- **PlayLayout システム**: JSON 定義に基づく UI 配置の動的変更が可能。4つの標準プリセットが実装済み。
- **マップエンジン**: PixiJS によるタイル描画、8方向オートタイル、物理衝突判定、スムーズな補間移動が実装済み。
- **バトル基盤**: `packages/battle` にて Actor/Enemy の型定義と静的なデータ構造が存在。

### エディタ (apps/editor)
- **マップエディタ**: マップ作成、タイル配置、イベント設置の基本機能が実装済み。
- **UIレイアウトエディタ**: **未実装**。ただし、HTML形式の動くプロトタイプ（`10-tsukuru-layout-preview.html`）が存在する。
- **Game Database**: **未実装**。現在は TypeScript コード内に静的なデータが定義されているのみ。

---

## 3. 実装ロードマップ（最新版を「正」とする）

旧計画と新計画を比較し、最新の概念マップ（`20-rpg-tkool-concept-map.md`）に従った優先順位を確定した。

### Phase 2: コモンイベント（再利用ブロック）
- **内容**: どこからでも呼び出せる共通処理（ショップ、宿屋、演出等）の実現。
- **追加要素**: `templates[]` 配列（プロジェクトデータ）、`call` ブロック型。

### Phase 3: ゲームデータベース
- **内容**: プロジェクトごとに Actor / Enemy / Skill / Item を編集可能にする。
- **追加要素**: `data.gameDb` スキーマの策定、エディタでの編集画面。

### Phase 4: バトルランタイムの高度化
- **内容**: ダメージ計算式の eval 実装（`a.atk * 4 - b.def * 2` 等）、行動AI、勝敗分岐。
- **追加要素**: セーブスキーマ v1.2（パーティ状態の保存）。

### Phase 5: マップ連携の完成
- **内容**: シナリオからマップへの遷移、エンカウント、複数ページを持つマップイベント。
- **追加要素**: `map_jump` ブロック、セーブスキーマ v1.3（マップ座標の保存）。

### Phase 6: UIカスタマイズの統合
- **内容**: `PlayLayout` のエディタ統合。ドラッグ＆ドロップによる自由な UI 配置。
- **追加要素**: `LayoutEditorView` の実装（HTMLプロトタイプからの移植）。

---

## 4. 結論と次の一手

調査の結果、**エンジン側の基盤は先行して完成しているが、エディタ側の「動的なデータ定義（DB）」と「再利用性（テンプレート）」が次のボトルネック**であることが判明した。

### 直近のアクションプラン
1. **`call` ブロックの実装**: `packages/core` の型定義に `call` ブロックを追加し、`templates` を介した再利用ロジックを実装する。
2. **`gameDb` のスキーマ定義**: `packages/battle` の型をベースに、JSON シリアライズ可能なデータベース構造を策定する。

これらの開発により、kaedevn は単なる「歩けるノベル」から、本格的な「RPG/ADV制作ツール」へと進化する。
