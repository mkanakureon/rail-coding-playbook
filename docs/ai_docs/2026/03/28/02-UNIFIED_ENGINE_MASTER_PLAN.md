# kaedevn 統一エンジン・マスタープラン：Web と Native の完全同期設計

## 1. はじめに
本ドキュメントは、Web (PixiJS) と Native (SDL2/Switch) の両プラットフォームにおける開発効率と実行品質を最大化するための、究極の基盤統一設計（マスタープラン）を定義したものである。

目標は、プラットフォーム間の差異を「描画 API」の 1 点のみに凝縮し、それ以外のすべてのロジック（命令実行、状態管理、アニメーション計算）を完全同期させることにある。

---

## 2. 3 つの核心的統一

### ① 実行エンジンの統一 (Universal OpRunner)
*   **設計:** Native 側の KSC テキストパーサーを廃止し、Web と共通の「コンパイル済み Op JSON」を唯一の入力とする。
*   **意義:** 命令の解釈ロジック（分岐、待機、変数演算）が全プラットフォームで数学的に一致することを保証する。
*   **運用:** Web 側のコンパイラ (`packages/compiler`) を「唯一の正解」とし、Native 側はそれを忠実に実行する「ランタイム」に徹する。

### ② 画面状態の統一 (GameScreenState)
*   **設計:** 画面上のすべての要素（背景、キャラ、テキスト、フィルター、タイマー）を、`GameScreenState` という単一のデータ構造に定義する。
*   **意義:** 「状態（何を）」と「描画（どう）」を完全に分離する。
*   **効果:** エディタの状態（JSON）をコピペするだけで、Native 側の画面を 100% 再現できる「ポータブル・スナップショット」を実現する。

### ③ アニメーション進行の共通化 (Pure Logic Updater)
*   **設計:** 「フェードの進捗」や「文字送りのカウント」などの時間経過に伴う計算を、描画 API から切り離し、`packages/core` の純粋な TypeScript ロジック（および同等の C++ ロジック）に集約する。
*   **意義:** 「Web では滑らかだが Native ではカクつく」といった挙動の差異を、計算式レベルで排除する。

## 3. マイルストーン：Zero-Gap への道

### Phase 1: 心臓の移植 (A-1)
*   C++ 版 OpRunner の実装。JSON 形式の命令を実行可能にする。
*   Native 側の独自パーサーを廃止。

### Phase 2: 神経の整理 (A-2)
*   Native 側のノンブロッキング化。
*   `GameScreenState` に基づくシングルループ（16ms）への完全移行。

### Phase 3: 皮膚の分離 (A-3)
*   `WebOpHandler` から計算ロジックを抽出し、`GameScreenUpdater` として共通化。
*   Web と Native の Handler を「状態を読んで描画するだけ」の薄いレイヤーへ。

## 4. 結論：未来の創作体験
このマスタープランが完了した時、kaedevn は以下の状態に到達する。

> **「演出家（作者）は、Web エディタという最高の鏡で自分を磨き、その魂（JSON）を Switch という最高の舞台へ送り出す。そこには一切の歪み（バグ）も、劣化（不一致）も存在しない。」**
