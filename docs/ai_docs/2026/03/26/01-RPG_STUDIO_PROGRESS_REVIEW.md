## 1. 概況
直近のコミット（`78d7be4` 等）により、kaedevn はノベルエンジンから「次世代 RPG 制作プラットフォーム」への劇的な転換を果たした。特に、RPG ツクールの長年の課題であった「データの分断」と「管理の不透明さ」を解消する強力な機能群が実装されている。

## 2. データベース進化：13 タブ統合とリレーショナル設計
`apps/editor/src/components/panels/GameDbPanel.tsx` において、以下の進化が確認された。
*   **全要素の統合:** アクター、職業、スキル、アイテム、敵キャラ、トループ（敵グループ）、ステート、アニメ、タイルセット、コモンイベント、システム。RPG 構築に必要な全 13 要素が 1 枚のパネルで完結。
*   **相互参照の自動化:** 職業設定からスキルセットへ、トループ配置から敵データへ。ID 番号ではなく「名前とリンク」でデータを繋ぐ設計が徹底されており、ツクール特有の「メモ帳での ID 管理」を完全に不要にしている。

## 3. イベント・ランタイム：可視化されたロジック
*   **EventInterpreter の独立:** ノベル用の `OpRunner` とは別に、RPG 特有の並列・逐次処理を司るエンジンが実装された。
*   **デバッガの統合:** テストプレイ中に変数やスイッチの状態をリアルタイムに監視できる機能がエディタ内に組み込まれた。これにより「なぜこのイベントが動かないのか」という探索時間を劇的に短縮する。

---

## 4. 演出の革新：Entity Graph
最新のコミットで導入された **Entity Graph エディタ** は、バラバラだったデータベース項目を「視覚的な繋がり」として描画する。
*   **意義:** 「このスキルがどの敵に使われているか」「このアクターがどの職業に就いているか」という全体構造を一目で把握できる。これはツクールを凌駕する kaedevn 独自の強みである。
