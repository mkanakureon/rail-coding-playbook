# 内部仕様案：2.5D 表現による視覚的次元上昇と空間設計

## 1. コンセプト：Depth & Atmosphere (奥行きと空気感)
完全な 3D モデルに頼るのではなく、「2D アセットを 3D 空間に配置・投影する」ことで、制作のしやすさを維持したまま、メメントモリや HD-2D のような圧倒的なリッチさを実現する。

---

## 2. 2.5D 表現の 4 つの柱

### ① パララックス・レイヤリング (多層視差)
*   **仕様:** 背景を「遠景・中景・近景」の最低 3 層に分離。
*   **効果:** カメラのパン（横移動）に合わせて各層の移動速度を変えることで、2D 静止画に奥行きを与える。
*   **エディタ対応:** 各ブロックに「Z 深度」プロパティを追加し、スライダーで直感的に奥行きを設定。

### ② 被写界深度 (Depth of Field: DoF)
*   **仕様:** キャラクターの Z 深度に合わせて、背景や手前のオブジェクトを動的にぼかす（Blur）。
*   **効果:** プレイヤーの視線を特定のキャラクターやイベントに誘導し、映画的な「フォーカス演出」を可能にする。
*   **技術:** `IRenderer` で定義した Bloom / Blur フィルターを Z 深度に連動させる。

### ③ ダイナミック・プロジェクション (台形変換・パース)
*   **仕様:** 地面タイルや背景画像を、消失点（Vanishing Point）に向かって斜めに歪ませる。
*   **効果:** RPG のマップにおいて、単なる真上からの視点ではなく、奥行きのある「広大なフィールド」を表現。
*   **利点:** 3D モデルを作らずとも、2D タイルセットの工夫だけで 3D 的な空間を作れる。

### ④ 空間連動ライティング
*   **仕様:** 3D 空間上の「光源」を設定し、2D スプライトに影（ドロップシャドウ）を投影。
*   **効果:** キャラクターがマップ上を移動する際、街灯の下を通ると影が伸び縮みする等の「実存感」を演出。

## 3. エディタでの「作りやすさ」の担保

2.5D は設定が複雑になりがちだが、kaedevn Next では以下のように簡略化する。

*   **「自動パース」モード:** 初心者は背景を選ぶだけで、AI（またはプリセット）が適切な Z 深度とボケ味を自動設定。
*   **ビジュアル・デプス・エディタ:** 画面を横から見た図（サイドビュー）で、アセットの重なり順をドラッグで調整。
*   **脱・座標計算:** 「奥へ移動」「手前へ表示」といった演出用語のブロックで、複雑なスケール計算を隠蔽。

## 4. 技術的実現性 (Web & Native)

| レイヤー | Web (PixiJS) | Native (GLRenderer) |
| :--- | :--- | :--- |
| **投影** | `Projection` コンテナを使用. | 射影行列 (Perspective Matrix) を適用. |
| **深度管理** | `zOrder` と `scale` の自動連動. | Z-Buffer を活用した深度テスト. |
| **負荷** | 低（2D スプライトの変形のみ）. | 極低（モバイル/Switch でも 60fps 維持可能）. |

## 5. 結論：なぜ 2.5D か？

1.  **アセットコストの抑制:** 3D モデル制作（高コスト）を回避し、既存の 2D イラストレーターの資産を最大限に活かせる。
2.  **Unity への差別化:** Unity で 2.5D を作るのは「2D と 3D の混在」による設定の複雑さがあるが、kaedevn は最初から「2.5D 専用の最適化」を施すことで、設定コストを 1/10 にする。
3.  **Switch での映え:** 大画面テレビで出力した際、単なる 2D よりも 2.5D の方が「ハードウェアの性能を活かしている」という満足度が高い。
