# 09_reports - 作業報告書

日々の作業報告・リリース記録・障害対応記録を日付ベースのフォルダで管理します。

## フォルダ構造

```
09_reports/
├── README.md
└── {YYYY}/
    └── {MM}/
        └── {DD}/
            ├── <report-name>.md
            └── ...
```

## 命名規則

- フォルダ: `YYYY/MM/DD` (例: `2026/02/18`)
- ファイル: `<種別>-<概要>.md` (例: `azure-container-apps-release.md`)

### ファイル種別の例

| 種別 | 説明 |
|------|------|
| `release-*` | リリース報告 |
| `hotfix-*` | 緊急修正報告 |
| `incident-*` | 障害対応報告 |
| `migration-*` | マイグレーション報告 |
| `azure-*` | Azure 関連作業報告 |

## 報告書テンプレート

```markdown
# タイトル

- **日時**: YYYY-MM-DD HH:MM JST
- **実施者**:
- **対象環境**:

## 概要
## 更新前の状態
## 実施内容
## 更新後の状態
## 動作確認結果
## 再現手順
## 教訓・注意事項
```
