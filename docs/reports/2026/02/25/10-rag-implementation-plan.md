## 背景

kaedevn（ノベルゲーム PF）に AI 機能を追加する。**まず「作者向け執筆支援」のみ実装する。** モデレーション・NPC 会話は後回し。

作者が作品設定をインプットすると、多段階プロンプトで `.ks` ファイルまで自動生成する。

## 生成パイプライン

```
作品設定（インプット）
  │  概要, ジャンル, 世界観, キャラ設定, 歴史的事実, ラフプロット
  ↓
Stage 1: 章プロット生成      ← gpt-4o-mini（構造化出力、コスト優先）
  │  章タイトル, 要約, 主要イベント, 感情曲線
  │  ※作者が確認・編集
  ↓
Stage 2: 話プロット生成      ← gpt-4o-mini（構造化出力、コスト優先）
  │  話タイトル, シーン構成, 場所, 登場人物, イベント
  │  ※作者が確認・編集
  ↓
Stage 3: 話テキスト生成      ← claude-sonnet（テキスト品質・口調再現重視）
  │  地の文, セリフ（キャラ口調反映）, 選択肢
  │  ※シーン単位で分割生成（前シーンをコンテキストとして渡す）
  │  ※生成後に自動要約 → 次話の previousEpisodeSummary に使う
  │  ※作者が確認・編集
  ↓
Stage 4: .ks ファイル変換    ← TS テンプレート（LLM 不使用）
  │  @bg, @ch, セリフ（全角コロン）, choice{flagSet,jumpTo}, @l/@p
  │  ※ compile() で構文チェック
  ↓
.ks ファイル出力
```

## レビュー反映事項

| # | レビュー指摘 | 対応 |
|---|------------|------|
| 1 | LLM クライアントの抽象化が足りない | `LLMClient` インターフェース + `OpenAIClient` / `AnthropicClient` を実装 |

| 7 | choice の構造がフラット | `flagSet` / `jumpTo` フィールドを nullable で予約 |

## 実装ステップ（全 8 ステップ）

### Step 1: `packages/ai-gateway` 作成

OpenAI / Anthropic を統一的に扱う LLM クライアント。

```
packages/ai-gateway/
  src/
    index.ts              # エクスポート
    types.ts              # LLMClient, GenerateParams, GenerateResult
    providers/
      openai.ts           # OpenAIClient implements LLMClient
      anthropic.ts        # AnthropicClient implements LLMClient
    factory.ts            # createClient(provider) → LLMClient
  package.json
  tsconfig.json
```

**LLMClient インターフェース**:

```typescript
interface LLMClient {
  generate(params: GenerateParams): Promise<GenerateResult>;
}

interface GenerateParams {
  model: string;
  systemPrompt: string;
  userPrompt: string;
  jsonMode: boolean;
  maxTokens?: number;
  temperature?: number;
}

interface GenerateResult {
  content: string;
  usage: { tokensIn: number; tokensOut: number };
  model: string;
  provider: 'openai' | 'anthropic';
}
```

**責務**:
- プロバイダ抽象化（OpenAI / Anthropic 切替）
- リトライ（429 / 5xx）、タイムアウト（30 秒）
- JSON mode（OpenAI: `response_format`、Claude: XML ラッパー）
- トークン使用量の返却

**環境変数**:
```
OPENAI_API_KEY=<YOUR_OPENAI_API_KEY>...
ANTHROPIC_API_KEY=<YOUR_ANTHROPIC_API_KEY>...
ASSIST_STAGE1_MODEL=gpt-4o-mini
ASSIST_STAGE2_MODEL=gpt-4o-mini
ASSIST_STAGE3_MODEL=claude-sonnet-4-6
```

### Step 2: DB テーブル追加

Prisma マイグレーションで 3 テーブルを追加する。

#### `ai_requests`（監査ログ）

```sql
CREATE TABLE ai_requests (
  id           VARCHAR(31) PRIMARY KEY,
  user_id      VARCHAR(31) NOT NULL,
  endpoint     VARCHAR(50) NOT NULL,
  provider     VARCHAR(20) NOT NULL,     -- 'openai' | 'anthropic'
  model        VARCHAR(50) NOT NULL,
  tokens_in    INT NOT NULL,
  tokens_out   INT NOT NULL,
  latency_ms   INT,
  status       VARCHAR(20) NOT NULL,
  error_msg    TEXT,
  created_at   BIGINT NOT NULL
);

CREATE INDEX ai_requests_user_idx ON ai_requests(user_id, created_at);
```

#### `work_settings`（作品設定）

```sql
CREATE TABLE work_settings (
  id           VARCHAR(31) PRIMARY KEY,
  project_id   VARCHAR(31) NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  user_id      VARCHAR(31) NOT NULL,
  version      INT NOT NULL DEFAULT 1,
  settings     JSONB NOT NULL,
  created_at   BIGINT NOT NULL,
  updated_at   BIGINT NOT NULL,
  UNIQUE(project_id)
);

CREATE INDEX work_settings_user_idx ON work_settings(user_id);
```

#### `generation_results`（生成結果）

```sql
CREATE TABLE generation_results (
  id           VARCHAR(31) PRIMARY KEY,
  project_id   VARCHAR(31) NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  user_id      VARCHAR(31) NOT NULL,
  stage        VARCHAR(20) NOT NULL,
  stage_key    VARCHAR(50) NOT NULL,
  input_hash   VARCHAR(64) NOT NULL,
  result       JSONB NOT NULL,
  is_edited    BOOLEAN NOT NULL DEFAULT false,
  created_at   BIGINT NOT NULL,
  updated_at   BIGINT NOT NULL,
  UNIQUE(project_id, stage, stage_key)
);

CREATE INDEX gen_results_project_idx ON generation_results(project_id, stage);
```

**マイグレーション手順**:
```bash
cd apps/hono
npx prisma migrate dev --name add_assist_tables
# 本番: npx prisma migrate deploy
```

> **pgvector は不要**。v1 では RAG を使わない。

### Step 4: .md パーサー + 型定義 + Zod スキーマ + プロンプトテンプレート

```
apps/hono/src/lib/assist/
  md-parser.ts          # .md ファイル → WorkSetting 変換
  types.ts              # WorkSetting, CharacterSetting, AssetMapping
  schemas.ts            # Zod スキーマ（各 Stage 出力バリデーション）
  prompts.ts            # 各ステージのプロンプトテンプレート
  parser.ts             # LLM レスポンスの JSON 抽出 + Zod バリデーション
```

**md-parser.ts**:
- settings ディレクトリから .md ファイルを読み込む
- YAML frontmatter を抽出（overview.md の genre, era, tone, themes 等）
- characters.md の `## 名前（slug）` ヘッダー + 箇条書きをパース
- assets.md のマークダウンテーブルから AssetMapping を抽出
- 結果を `WorkSetting` 型に変換

**parser.ts の処理フロー**:
1. プロバイダに応じて JSON 文字列を抽出（OpenAI: そのまま、Claude: `<json>` タグ抽出）
2. フォールバック: マークダウンコードブロック抽出
3. `JSON.parse`
4. Zod `safeParse` でバリデーション
5. 失敗時: LLM を 1 回リトライ

### Step 5: .ks ジェネレーター作成（Stage 4）

```
apps/hono/src/lib/assist/
  ks-generator.ts       # EpisodeText → .ks 変換
```

LLM は使わない。TS テンプレートで機械的に変換する。

**変換ルール**:
- `narration` → `テキスト\n@l\n`
- `dialogue` → `{name}：{text}\n@l\n`（slug → name に変換、全角コロン）
- `choice` → `choice { "テキスト" { flagSet / result / jumpTo } }`
- シーン変更時 → `@bg {bgId} fade 500`（assetMapping がある場合）
- 話の末尾 → `@p`

**バリデーション**: 生成後に `packages/compiler` の `compile()` を実行して構文チェック。

### Step 6: API ルート実装

```
apps/hono/src/routes/assist.ts
```

7 エンドポイント:

| # | Method | Path | 用途 |
|---|--------|------|------|
| 1 | `PUT` | `/api/assist/settings/:projectId` | 作品設定を保存（version インクリメント） |
| 2 | `GET` | `/api/assist/settings/:projectId` | 作品設定を取得 |
| 3 | `GET` | `/api/assist/assets/:projectId/backgrounds` | 背景アセット候補を取得 |
| 4 | `POST` | `/api/assist/generate/chapters` | Stage 1: 章プロット生成 |
| 5 | `POST` | `/api/assist/generate/episodes` | Stage 2: 話プロット生成 |
| 6 | `POST` | `/api/assist/generate/text` | Stage 3: 話テキスト生成（シーン単位分割） |
| 7 | `POST` | `/api/assist/generate/ks` | Stage 4: .ks 変換 |

**共通パターン**:
1. `authMiddleware` + `aiAssistLimiter`
2. Zod バリデーション
3. プロジェクト所有権チェック
4. AI Gateway で生成（Stage 1-3）/ TS 変換（Stage 4）
5. `generation_results` に保存
6. `ai_requests` に監査ログ記録（provider 付き）
7. レスポンス返却

### Step 7: CLI テストスクリプト作成

```
scripts/assist-cli.ts     # CLI エントリポイント
```

ローカル .md ファイルを読み込み → ai-gateway を呼び出し → 結果を `output/` に保存する。API サーバーを経由せず、直接 ai-gateway を使う。

```bash
# 章プロット生成
npx tsx scripts/assist-cli.ts chapters --settings projects/demo/settings/

# 話プロット生成
npx tsx scripts/assist-cli.ts episodes --settings projects/demo/settings/ \
  --chapters output/chapters.json --target 1

# 話テキスト生成
npx tsx scripts/assist-cli.ts text --settings projects/demo/settings/ \
  --chapters output/chapters.json --episode output/ch1_episodes.json --target 1

# .ks 変換
npx tsx scripts/assist-cli.ts ks --settings projects/demo/settings/ \
  --text output/ch1_ep1_text.json
```

CLI テストで全ステージの動作を確認してから API ルートに統合する。

## リスクと対策

| リスク | 対策 |
|--------|------|
| LLM が不正な JSON を返す | OpenAI: `response_format`、Claude: XML タグ抽出 → Zod `safeParse` → リトライ 1 回 |
| .ks 構文エラー | Stage 4 は TS テンプレート（LLM 不使用）+ `compile()` バリデーション |
| 全角/半角コロン混在 | `ks-generator.ts` で全角コロン（：）に統一 |
| Stage 3 トークン超過 | シーン単位で分割生成。前シーンのコンテキストを次に渡す |
| コスト暴発 | `aiAssistLimiter` + `ai_requests` で全量記録（provider 別集計可能） |
| マイグレーション漏れ | Step 2 で `prisma migrate dev` → 本番 `migrate deploy` を必ず実行 |
| API キー漏洩 | API サーバー経由のみ。フロントエンドからは直接呼ばない |
| Anthropic API 障害時 | Stage 3 のモデルを環境変数で切替可能にしておく |
