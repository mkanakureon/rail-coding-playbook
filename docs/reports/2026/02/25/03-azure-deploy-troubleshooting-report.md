### エラー 4: Preview — Dockerfile COPY 漏れ

| 項目 | 詳細 |

## 根本原因の分析

### なぜローカルで気づけなかったか

| ツール | ローカル (dev) | 本番 (build/Docker) |
|---|---|---|
| Vite | esbuild でトランスパイルのみ | `tsc -b` で型チェック |
| Next.js | ESLint を実行しない | `next build` で ESLint 実行 |
| npm (モノレポ) | symlink で全パッケージ参照可能 | Dockerfile に COPY した分だけ |
| Prisma | `db push` で直接同期 | `migrate deploy` でファイル必須 |

### 共通パターン

**開発ツールが「開発体験 (DX) のために省略していること」が、本番ビルドで牙を剥く。**

CI/CD パイプラインがないため、これらのチェックがデプロイ前に走らなかった。

## 教訓

1. **「ローカルで動く」は「正しい」の証明にならない** — dev server が省略しているチェックを把握する
2. **誤修正は元のエラーより危険** — エラー 4 で tsconfig を変えたとき、別パッケージを参照してしまい連鎖エラーになった。修正前に依存関係を確認する
3. **`prisma db push` は本番移行のタイミングで `migrate` に切り替える** — プロトタイピング期と運用期の境界を意識する
4. **テストは「壊れていること」に気づくためのもの** — schema-sync テストがあれば、デプロイ前に発見できた
