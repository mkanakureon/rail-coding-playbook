# AI 執筆支援 SaaS ビジネス分析

> Generated by Claude Opus 4.6

## 背景

AI 執筆支援パイプライン（4 Stage 構成）の実装が完了し、gpt-4.1-nano での実行コストが判明した。
1 章（約 1 万文字）あたり約 1.8 円という低コストから、SaaS サービスとしての事業性を検討した。

## 原価構成

| 項目 | 単価 |
|------|------|
| 1 章（約 1 万文字） | 約 1.8 円 |
| 36 章（1 作品フル） | 約 65 円 |
| 月 10 作品生成 | 約 650 円 |

### 前提条件

- モデル: gpt-4.1-nano（$0.05/1M input, $0.40/1M output）
- 4 Stage パイプライン: 章プロット → 話プロット → 話テキスト → .ks 変換
- 1 章あたり約 3,000 トークン入力 + 6,000 トークン出力

## 価格設定シミュレーション

| プラン | 月額 | 含む章数 | 原価 | 粗利率 |
|--------|------|----------|------|--------|
| ライト | 980 円 | 50 章 | 90 円 | 91% |
| スタンダード | 1,980 円 | 200 章 | 360 円 | 82% |
| プロ | 4,980 円 | 無制限（目安 500 章） | 900 円 | 82% |

API コスト面では粗利率 80〜90% を確保できる。

## 競合優位性

### 1. 長編一貫性（最大の差別化要素）

既存の AI 執筆ツール（NovelAI、AIのべりすと等）は 1 シーン単位の生成が主流。
本パイプラインは以下の仕組みで 36 章の一貫性を維持する:

- **コンテキスト階層**（篇サマリー → 章サマリー → 話コンテキスト）
- **伏線追跡**（plotThreads の open/resolved 管理）
- **キャラクター状態追跡**（知識・関係性・感情の引き継ぎ）

### 2. ジャンル別品質最適化

外部 YAML ファイルによるジャンルルール切替:

- ミステリー: 手がかり配置、ミスリード、微反応
- 恋愛: 距離感の緩急、心理描写
- ファンタジー: 世界観用語の一貫性
- ホラー: 五感描写、不穏な伏線

ビルド不要で YAML を差し替えるだけで調整可能。

### 3. ビジュアルノベルエンジン統合

テキスト生成 → .ks スクリプト → エンジンでプレビューまで一気通貫。
「書いたらすぐ動く」体験は他ツールにない。

## リスク分析

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| **品質の天井** — nano モデルでは B+ が限界、文芸レベルには届かない | 高 | 「下書き生成→人間が仕上げる」ポジショニング。上位モデル（sonnet等）オプション提供 |
| **プラットフォームリスク** — OpenAI の値上げ・API 変更 | 中 | ai-gateway による抽象化で Anthropic/OpenAI 切替可能。マルチプロバイダ対応済み |
| **コンテンツ責任** — AI 生成物の著作権・品質問題 | 中 | 利用規約で「生成物は下書き」と明記。最終責任はユーザー |
| **競合参入** — 大手が同機能を安価に提供 | 中 | 長編一貫性 + VN エンジン統合という組み合わせで差別化 |
| **市場規模** — VN 作家 × AI に抵抗なし、の層は限定的 | 高 | VN 以外にも Web 小説・ラノベ作家へ展開。.ks 出力はオプション化 |

## 推奨ポジショニング

### 「AI が書く」ではなく「作家の生産性を 10 倍にするツール」

- ターゲット: 個人 VN 開発者、Web 小説作家、同人ゲーム制作者
- 価値提案: 「36 章の長編を一貫性を保って下書き生成」
- 価格帯: 980〜1,980 円/月（個人が払える範囲）
- 差別化: 長編コンテキスト管理 + ジャンル最適化 + VN エンジン統合

### 段階的展開

| Phase | 内容 | 期間 |
|-------|------|------|
| Phase 1 | CLI ツールとして無料公開、フィードバック収集 | 1〜2 ヶ月 |
| Phase 2 | Web UI 化、ユーザー登録・課金導入 | 2〜3 ヶ月 |
| Phase 3 | VN エンジン統合、プレビュー機能 | 3〜4 ヶ月 |
| Phase 4 | Web 小説プラットフォーム連携 | 4〜6 ヶ月 |

## 結論

API コスト面では十分に事業性がある。課題は「品質の天井」と「市場のニッチさ」。
まずは CLI 無料公開でフィードバックを集め、ニーズの深さを検証するのが現実的な第一歩。
