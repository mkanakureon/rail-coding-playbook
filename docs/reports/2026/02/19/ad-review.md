# 広告仕様レビュー結果

対象: `docs/01_in_specs/0219/02_広告.md`
日付: 2026-02-19

## 決定事項

### 1. 広告プロバイダー: Google AdSense（初期）

- 初期フェーズはAdSenseを採用
- 導入が簡単（script1行 + スロット定義）、自動最適化で運用コスト低い
- トラフィック増加後に Google Ad Manager への移行を検討

**リスク:**
- UGC（ユーザー投稿作品）サイトはAdSense審査が厳しめ
- 不適切コンテンツが表示されるページに広告が出るとアカウント停止リスクあり
- → 公開作品のモデレーション（公開審査）の仕組みとセットで運用する

### 2. 作品閲覧ページの広告配置

プレイ画面はiframeで表示。広告はiframeの外（親ページ側）に配置する。

```
┌─────────────────────────────┐
│  iframe (VNプレイ画面)       │
│  ┌───────────────────────┐  │
│  │  PixiJS キャンバス     │  │
│  └───────────────────────┘  │
├─────────────────────────────┤
│  作品情報エリア              │
│  (タイトル、作者、説明など)   │
├─────────────────────────────┤
│  広告枠                     │
└─────────────────────────────┘
```

- iframe内にAdSenseコードは入れない（ポリシー違反）
- sticky-bottom の場合、iframeの高さとの関係に注意（広告がiframeに被らないようにする）

### 3. 広告表示ページの追加

元仕様では広告対象は3ページ（作品閲覧、作品一覧、トップ）のみだったが、以下の4ページにも広告を表示する：

- ログインページ
- 利用規約・プライバシーポリシーページ
- 404 / エラーページ
- 設定ページ

配置ルールは同様（最大2枠、CTA近くに置かない、CLS対策）。

**広告を入れないページ（変更なし）:** 作者マイページ、エディタ、プレビュー、購入画面

## レビュー指摘事項

### 高優先度

| # | 指摘 | 内容 |
|---|------|------|
| 1 | 作品閲覧ページの定義が曖昧だった | → iframe + 親ページ構成で決定（上記） |
| 2 | 広告プロバイダー未指定だった | → AdSense で決定（上記） |

### 中優先度

| # | 指摘 | 内容 |
|---|------|------|
| 3 | レスポンシブ広告サイズが限定的 | PC: 728x90、モバイル: 320x50 のみ。in-content枠では 300x250（レクタングル）の方がRPMが高い傾向。検討の余地あり |
| 4 | SPA対応の具体策が不足 | チェックリストに「二重ロード防止」とあるが、具体的な実装方法（usePathname検知、useEffectクリーンアップ等）が未記載 |
| 5 | 計測のログ送信先が未定義 | GA4カスタムイベント or 独自エンドポイント（Hono API）か未決定 |

### 低優先度

| # | 指摘 | 内容 |
|---|------|------|
| 6 | CSS重複宣言 | セクション4.2で `.ad-sticky-wrap` が2回定義。`padding-bottom: env(...)` を1つ目に統合すべき |
| 7 | `position: sticky` の注意 | 親要素の `overflow` 設定次第で効かない場合がある。コメントに「レイアウトに依存しにくい」とあるが実際は依存する |
| 8 | 広告非表示ページの網羅性 | → 決定済み: ログイン、利用規約、404、設定ページに広告を表示する（決定事項3参照） |
| 9 | UGC + AdSense のリスク | 仕様書にモデレーション要件を記載すべき |

## 仕様書の評価

**良い点:**
- 配置ルールが明確（最大2枠、禁止ページ、禁止配置）
- CLS対策が最初から考慮済み
- 「計測値でのみ仕様変更」のルールが実践的
- PRレビュー用チェックリストが実用的

**総評:** 実装に進められるレベル。上記の高優先度2件は本レビューで決定済み。中優先度の項目は実装着手時に詰める。
