# バトル画面 UI設計書

> 現在のプレイ画面（ノベルモード）を基に、戦闘モード時のUI配置を定義する。

## 1. 現在のプレイ画面（ノベルモード）

```

+----------------------------------------------------------+
|                                                          |
|  [背景レイヤー — 全画面カバー]                             |
|                                                          |
|     x=320         x=640          x=960                   |
|     "left"       "center"       "right"                  |
|       |             |              |                     |
```

**既存レイヤー構成（z-order）:**

| Z | レイヤー | 内容 |

## 3. 各UIコンポーネント仕様

### 3.1 敵表示エリア

```
最大4体を画面上部に等間隔で配置。

敵の数  →  配置X座標
  1        [640]
  2        [427, 853]
  3        [320, 640, 960]
  4        [256, 512, 768, 1024]

各スプライト:
  サイズ: 最大 200x200（アスペクト比を維持して収まるようスケール）
  アンカー: (0.5, 1.0) — 足元中央

敵HP/名前:
  HPバー: スプライト直下, 幅 160px, 高さ 8px
    塗り: 緑→黄→赤のグラデーション（HP%に応じて）
    背景: 灰 alpha 0.5
  名前: HPバー下, 白, 14px, 中央揃え
```

**選択カーソル（ターゲット指定時）:**
```
  選択中の敵の上に ▽ マーク（黄色, 点滅）
  矢印キー左右で移動
  Enterで確定
```

**倒された敵:**
```
  alpha を 0.3 に下げる
  HPバーを赤 0% に
  名前を灰色に
```

### 3.6 ターゲット選択モード

Attack / Skill 選択後、対象を選ぶ:

```
敵選択:
  敵スプライトの上に ▽ カーソル（黄色, 点滅）
  ←→ で移動、Enter で確定、Esc/Back で戻る
  倒れた敵はスキップ

味方選択（回復スキル）:
  パーティステータスの対象行を青枠で囲む
  ↑↓ で移動、Enter で確定
```

## 5. 実装方針

### 5.1 レイヤー追加

既存の `uiLayer` 内に `battleContainer` を追加:

```
uiLayer
  ├ textWindow.container      ← 戦闘中は非表示
  ├ choiceOverlay.container   ← 戦闘中は非表示
  ├ battleContainer           ← 【新規】戦闘中のみ表示
  │   ├ enemyArea             敵スプライト + HP
  │   ├ messageArea           戦闘メッセージ
  │   ├ partyStatusArea       味方ステータス
  │   ├ commandPanel          コマンドボタン
  │   └ subMenu               スキル/アイテム選択
  ├ gameUI.container
  └ diagnostics.container
```

### 5.2 BattleIO とUIの対応

| BattleIO メソッド | UI動作 |
|-------------------|--------|
| `show(text)` | messageArea にテキスト表示（タイプライター） |
| `choose(title, choices)` | subMenu に選択肢表示 |
| `promptCommand(state, actor)` | commandPanel を有効化、actor を強調 |

### 5.3 既存コンポーネントの再利用

| 既存 | バトルでの用途 |
|------|--------------|
| TextWindow のタイプライター | messageArea のテキスト表示 |
| ChoiceOverlay のボタンスタイル | commandPanel / subMenu |
| 背景レイヤー | 戦闘背景の表示 |
| InputManager | コマンド選択・ターゲット選択の操作 |

### 5.4 Phase対応

| Phase | 追加されるUI要素 |
|-------|-----------------|
| Phase 1 | なし（テスト用 — CLIのみ） |
| Phase 2 | 全UI要素の基本実装 |
| Phase 3 | 状態異常アイコン、トリガーテキスト（話者名付き） |
| Phase 4 | なし（AI内部のみ） |
| Phase 5 | アイテムサブメニュー、Guard ボタン動作、エフェクト充実 |

## 6. エフェクト（Phase 5 以降）

MVP では最小限。Phase 5 で追加:

| エフェクト | 実装方法 |
|-----------|---------|
| ダメージ数字 | 対象の上に数字ポップアップ（赤, フェードアウト↑移動） |
| 回復数字 | 対象の上に数字ポップアップ（緑, フェードアウト↑移動） |
| 被ダメージ | 対象スプライトの明滅（白フラッシュ 0.2秒） |

| 状態異常付与 | 状態アイコン出現 + SE |
| 勝利演出 | 画面フラッシュ + ファンファーレ |
| 敗北演出 | 画面暗転（黒フェード 1.5秒） |

## 7. レスポンシブ対応

既存のスケーリング方式をそのまま使用:

```typescript
const scale = Math.min(screenWidth / 1280, screenHeight / 720);
app.stage.scale.set(scale);
```

- バトルUIも全て 1280x720 仮想座標で配置
- 物理解像度の違いはステージスケールで吸収
- タッチ操作: コマンドボタンのタッチ領域は 240x60px（スケール後）で十分な大きさ
