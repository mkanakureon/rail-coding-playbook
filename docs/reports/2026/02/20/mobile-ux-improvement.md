# エディタ モバイルUX改善仕様書

**作成日:** 2026-02-20
**対象:** `apps/editor` — 場面（ページ）編集・ブロック編集・画像選択・キャラクター選択

---

## 1. 現状の課題まとめ

### 1.1 場面（ページ）編集

| 課題 | 深刻度 | 現状 |
|------|--------|------|
| ページ名が編集不可 | 中 | `第N話` の自動採番のみ。PageListModal で startBlock の label を読もうとしているが、StartBlock に label フィールドが存在せず常に undefined |
| ページ一覧の情報が不足 | 中 | ブロック数のみ表示。どのページにどんな背景・キャラがいるか分からない |
| スワイプでページ切替時のフィードバックがない | 低 | ページが切り替わったことが視覚的に分かりにくい |

### 1.2 ブロック編集

| 課題 | 深刻度 | 現状 |
|------|--------|------|
| ChoiceBlock のインライン編集が狭い | 高 | 変数名 `w-16`(64px)、値 `w-12`(48px) — スマホでは入力困難 |
| IfBlock のセクション切替が分かりにくい | 高 | 条件式/TRUE/FALSE を下部トグルで切替。同時に見られない |
| BlockMenu（⋯）のタッチ操作 | 中 | `mousedown` で外側クリック検出。タッチデバイスで閉じない場合あり |
| 一部ブロックで削除確認がない | 中 | BgBlock, ChoiceBlock, IfBlock, JumpBlock, SetVarBlock は `window.confirm` または確認なし |
| ブロックの並び替えが↑↓ボタンのみ | 中 | ドラッグ&ドロップ非対応。離れた位置への移動に何度もタップが必要 |

### 1.3 画像選択（背景）

| 課題 | 深刻度 | 現状 |
|------|--------|------|
| InlineAssetSelector が6件固定 | 中 | アセットが多い場合に「もっと」で全画面モーダルを開く必要があり、操作が2段階 |
| AssetSelectModal がデスクトップ向けレイアウト | 高 | モバイルでは `fixed inset-0` のフルスクリーンだが、グリッドが2列で画像が小さい。カテゴリフィルタがドロップダウンで一覧性が低い |
| iOS Safari でのファイルアップロード | 高 | `document.createElement('input').click()` はジェスチャーコンテキスト外で失敗する場合あり |

### 1.4 キャラクター選択

| 課題 | 深刻度 | 現状 |
|------|--------|------|
| ChBlockCard がネイティブ `<select>` のみ | 高 | キャラクター・表情の選択にビジュアルプレビューがない。テキストのみで選ぶ |
| CharacterEditModal の表情操作ボタンが密集 | 中 | 「静止画アップロード」「既存から選択」「ZIP(アニメ)」「削除」が `flex-wrap` で狭い画面に詰め込まれる |
| 削除が `window.confirm()` | 低 | モバイルブラウザでスタイル不統一 |

## 2. 改善提案

### 2.1 場面（ページ）編集の改善

#### A. ページ一覧のリッチ化

**現状:** テキストのみのリスト
**改善:** サムネイル付きリスト

```
┌─────────────────────────────────────┐
│ ページ一覧                    ✕     │
├─────────────────────────────────────┤
│ ┌──────┐                            │
│ │ BG   │ 第1話「出会い」            │
│ │ thumb│ 5ブロック                   │
│ └──────┘                     ● 現在  │
│─────────────────────────────────────│
│ ┌──────┐                            │
│ │ BG   │ 第2話「決意」              │
│ │ thumb│ 8ブロック                   │
│ └──────┘                            │
│─────────────────────────────────────│
│        ＋ ページ追加                 │
└─────────────────────────────────────┘
```

- 各ページの最初の背景ブロックの画像をサムネイル表示（60×34px、16:9）
- ページ名をタップで編集可能（インライン input）
- ボトムシートのまま（モバイル）、スクロール可能

#### B. ページ切替トーストフィードバック

スワイプでページ移動時、画面上部に短いトースト表示：
```
「ページ 2/5 に移動しました」（1.5秒で自動消去）
```

### 2.2 ブロック編集の改善

#### A. ブロックカードの統一的なモバイルパターン

全ブロックカードで以下を統一する：

1. **コンパクト表示（デフォルト）:** 1行サマリー + タイプバッジ
2. **タップで展開:** 編集UI表示
3. **長押しでコンテキストメニュー:** 移動・複製・削除（既存の `useLongPress` + `ContextMenu` を活用）
4. **削除は全て `DeleteConfirmSheet`（ボトムシート）に統一**

```
┌─────────────────────────────────────┐
│ [背景] 宮殿の広間          ⋯       │  ← コンパクト（サムネイル付き）
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ [キャラ] 主人公 / 笑顔 / C  ⋯      │  ← コンパクト
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ [テキスト] 「ここはどこだ...」⋯     │  ← コンパクト
└─────────────────────────────────────┘

 タップで展開 ↓

┌─────────────────────────────────────┐
│ [テキスト]                   ▲ 閉じる│
│                                     │
│ キャラ名  [主人公        ▼]         │
│ ┌─────────────────────────────────┐ │
│ │ ここはどこだ...                 │ │
│ │                                 │ │
│ └─────────────────────────────────┘ │
│                        [完了]       │
└─────────────────────────────────────┘
```

#### B. ChoiceBlock の編集を全画面シートに変更

**現状:** インラインで狭いフォーム
**改善:** 「編集」タップで全画面ボトムシートを開く

```
┌─────────────────────────────────────┐
│ 選択肢を編集                 ✕      │
├─────────────────────────────────────┤
│                                     │
│ 選択肢 1                            │
│ ┌─────────────────────────────────┐ │
│ │ はい、行きます                  │ │
│ └─────────────────────────────────┘ │
│ ジャンプ先: [ページ2  ▼]           │
│                                     │
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│                                     │
│ 選択肢 2                            │
│ ┌─────────────────────────────────┐ │
│ │ いいえ、やめます                │ │
│ └─────────────────────────────────┘ │
│ ジャンプ先: [ページ3  ▼]           │
│                                     │
│       ＋ 選択肢を追加               │
│                                     │
│              [保存]                  │
└─────────────────────────────────────┘
```

- 各フィールドがフル幅で使える
- 変数アクションも十分なスペースで編集可能

#### C. IfBlock のセクション表示改善

**現状:** 下部トグルで条件式/TRUE/FALSE を切替
**改善:** スワイプ可能なタブ + バッジ

```
┌─────────────────────────────────────┐
│ [IF] 条件分岐                ⋯      │
│                                     │
│  [条件式]  [TRUE (3)]  [FALSE (1)]  │
│  ─────────────────────────────────  │
│                                     │
│  flag_met == true                   │
│  ← スワイプでTRUE/FALSEに切替 →    │
│                                     │
└─────────────────────────────────────┘
```

- 各セクションのブロック数をバッジ表示
- 横スワイプでセクション切替（react-swipeable を再利用）

#### D. ブロック並び替えの改善

長押しで並び替えモードに入る方式を追加：

```
┌─────────────────────────────────────┐
│ ≡ [背景] 宮殿の広間                │  ← ドラッグハンドル表示
│ ≡ [キャラ] 主人公 / 笑顔           │
│ ≡ [テキスト]「ここはどこだ...」     │  ← 長押しでつかんで移動
│ ≡ [選択肢] 2個の選択肢             │
│                                     │
│              [完了]                  │
└─────────────────────────────────────┘
```

- 実装: `@dnd-kit/sortable` のタッチセンサー使用
- 通常時は非表示。ブロックメニューまたはヘッダーの「並び替え」ボタンから起動

### 2.3 画像選択（背景）の改善

#### A. AssetSelectModal のモバイル最適化

**現状:** 2列グリッド、小さなサムネイル
**改善:** モバイルでは専用レイアウト

```
┌─────────────────────────────────────┐
│ 背景画像を選択               ✕      │
├─────────────────────────────────────┤
│ [マイアセット] [公式アセット]       │
│                                     │
│ カテゴリ:                           │
│ [すべて] [和風] [洋風] [自然] [街]  │ ← 横スクロールチップ
│                                     │
│ ┌────────┐ ┌────────┐              │
│ │        │ │        │              │
│ │  BG 1  │ │  BG 2  │              │ ← 2列、16:9比率
│ │        │ │        │              │
│ │ 名前   │ │ 名前   │              │
│ └────────┘ └────────┘              │
│ ┌────────┐ ┌────────┐              │
│ │        │ │        │              │
│ │  BG 3  │ │  BG 4  │              │
│ │        │ │        │              │
│ │ 名前   │ │ 名前   │              │
│ └────────┘ └────────┘              │
│                                     │
└─────────────────────────────────────┘
```

変更点：
- **カテゴリフィルタをチップ（横スクロール）に変更** — ドロップダウンより一覧性が高く1タップで切替可能
- **タップ即選択** — モバイルでは選択→確認の2ステップを廃止（現状の動作を維持）
- **サムネイルを16:9比率に統一** — 背景画像に適した比率で見やすく

#### B. InlineAssetSelector の改善

**現状:** 横スクロール6件 + 「もっと」ボタン
**改善:** 最近使った画像を優先表示 + カウント表示

```
┌─────────────────────────────────────┐
│ [背景] 宮殿の広間                   │
│                                     │
│ 最近使用:                           │
│ [img1] [img2] [img3] [img4] [+12]  │ ← 残数表示
│                                     │
│         [全て表示]                   │
└─────────────────────────────────────┘
```

- 最近使用した画像（直近の編集順）を先頭に表示
- 残りアセット数をバッジで表示（`+12` のように）

#### C. iOS Safari ファイルアップロード修正

```typescript
// 現状（問題あり）
const input = document.createElement('input');
input.click(); // ジェスチャーコンテキスト外で失敗する場合あり

// 改善案
// ボタンの onClick ハンドラ内で直接 input.click() を呼ぶ
// または、非表示の <input> を DOM に事前マウントしておく
<input ref={fileInputRef} type="file" className="hidden" onChange={handleFile} />
<button onClick={() => fileInputRef.current?.click()}>アップロード</button>
```

## 3. 共通の改善事項

### 3.1 `window.confirm()` の全廃

すべての確認ダイアログを `DeleteConfirmSheet`（ボトムシート）に統一する。

対象箇所:
- `CharacterPanel.tsx` — キャラクター削除
- `PageMenu.tsx` — ページ削除
- `BgBlockCard.tsx` — ブロック削除
- `ChoiceBlockCard.tsx` — 選択肢削除
- `IfBlockCard.tsx` — ブロック削除
- `JumpBlockCard.tsx` — ブロック削除
- `SetVarBlockCard.tsx` — ブロック削除

### 3.2 BlockMenu のタッチイベント対応

```typescript
// 現状
document.addEventListener('mousedown', handleOutsideClick);

// 改善
document.addEventListener('mousedown', handleOutsideClick);
document.addEventListener('touchstart', handleOutsideClick, { passive: true });
```

### 3.3 未使用コンポーネントの活用

以下のコンポーネントは実装済みだが未接続。今回の改善で活用する：

| コンポーネント | 用途 |
|---------------|------|
| `ContextMenu` | ブロック長押しメニュー（移動・複製・削除） |
| `useLongPress` | ブロックの長押し検出 |
| `Drawer` | 設定パネルやフィルタ表示（必要に応じて） |

### 3.4 スクロール検出の修正

```typescript
// 現状: window.scroll を監視（editor-main の overflow-y-auto と不一致）
// 改善: editor-main の scroll イベントを監視
const editorMain = document.getElementById('editor-main');
editorMain?.addEventListener('scroll', handleScroll);
```

`StickyHeader` の `hideOnScroll` が正しく動作するようになる。

## 5. 技術的な注意事項

### 既存アーキテクチャとの整合性

- **状態管理:** すべて `useEditorStore`（Zustand）経由。新UIも同じストアを使用
- **レスポンシブ判定:** `useMediaQuery('(max-width: 640px)')` を継続使用
- **アニメーション:** Framer Motion + `useReducedMotion` フック対応を維持
- **アセット API:** 既存の `AssetSelectModal` のデータ取得ロジック（マイアセット/公式アセット）を再利用
- **ボトムシート:** `react-modal-sheet` を全モーダルに適用（既に `DeleteConfirmSheet` で使用中）

### 新規依存パッケージ（Phase 3 のみ）

- `@dnd-kit/core` + `@dnd-kit/sortable` — ブロック並び替え用（Phase 3-12 のみ）
- それ以外は既存パッケージで実装可能
