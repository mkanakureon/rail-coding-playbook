# ミニDSL運用仕様の策定

> Generated by Claude Opus 4.6

## 背景

Claude Code への画面改修指示で「解釈ブレ」が発生する問題があった。自然言語の速さを維持しつつ、ブレだけ減らすための軽量フォーマットを検討した。

## 方針

3案を比較検討し、案C（Layout1）を採用。指示は自然言語のまま維持する。

| 案 | 特徴 | 採否 |
|---|------|------|
| A: PatchSpec | TARGET/GOAL/DO/DONT/CHECK の固定欄 | 不採用 — GOAL/DO/DONT は自然言語で十分 |
| B: ScreenCard | 画面仕様のカード化（8フィールド） | 不採用 — 記述量が多い |
| **C: Layout1** | **LAYOUT ブロックだけ固定 + 自然言語** | **採用** |

決め手: LAYOUT で「どこに何があるか」を固定すれば、自然言語で「何をして / しないで」と書くだけで Claude のブレが止まる。

## 実装の詳細

### フォーマット

```
TARGET: <URL or ページ名 or ファイル>
LAYOUT:
- <ブロック名>: <中身の要素>
- <ブロック名>: <中身の要素>
STATES: <該当する状態>（任意）

<自然言語の指示>
```

### 記述ルール

1. LAYOUT のブロック名は画面に見える単位で区切る
2. ブロック内の要素は `/` `+` `()` `,` で区切る
3. 自然言語は「〜して」で追加・変更、「〜しないで」で禁止
4. 1 指示 = 1 TARGET
5. 変えない部分も LAYOUT に書く（全体像を伝える）

### 実例（抜粋）

```
TARGET: /works (WorksPage)
LAYOUT:
- Header: 共通ヘッダー
- TitleSection: ページタイトル + 説明 + マイページへ戻るボタン
- SortControls: 「新着」/「人気」トグルボタン
- CategoryFilter: 「All」+ カテゴリpillボタン
- WorksGrid: カード一覧（サムネイル/タイトル/説明/カテゴリバッジ/タグ/作者リンク/再生数/いいね数）
- Pagination: Previous / Next + ページ番号
STATES: loading/empty/error/default

SortControls に「いいね順」ボタンを追加して。
CategoryFilter の pill は横スクロールにして、折り返しはしない。
WorksGrid のカードサイズは変えないで。
```

仕様書には8画面分の実例を収録（Next.js 6画面 + Editor 2パネル）。

## 教訓

- ミニDSL は形式を最小にして自然言語と組み合わせるほうが運用に乗りやすい
- LAYOUT のブロック名を自然言語の指示で参照すると位置指定が正確になる
- パネル・タブ単位で TARGET を絞ると EditorPage のような複雑画面でもブレない
