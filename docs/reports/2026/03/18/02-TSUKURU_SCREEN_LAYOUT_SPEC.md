### 1.2 各領域の仕様

| 領域 | 高さ/幅 | 内容 | ツクール対応 |
|------|---------|------|-------------|
| Menu Bar | h-8 (32px) | ドロップダウンメニュー | MV/MZ メニューバー |
| Toolbar | h-10 (40px) | アイコンボタン群 | MV/MZ ツールバー |
| Left Panel | w-56 (224px), min 180px, max 320px | タイルパレット or DB リスト | MV/MZ 左パネル |
| Center Canvas | flex-1 | マップ描画 / イベントリスト / DB 詳細 | MV/MZ メイン領域 |
| Right Panel | w-80 (320px), min 240px, max 450px | プロパティ / プレビュー | **kaedevn 独自** |
| Map Tree | 左パネル下部 1/3 | マップ一覧ツリー | MV/MZ マップリスト |
| Status Bar | h-6 (24px) | カーソル位置・マップ情報 | MV/MZ ステータスバー |

### 1.3 ツクールとの差分（kaedevn 独自機能）

| 項目 | ツクール MV/MZ | kaedevn | 理由 |
|------|---------------|---------|------|
| Right Panel | なし | プロパティ + ライブプレビュー | Web ブラウザの横幅を活用、リアルタイム確認 |
| UI レイアウトタブ | なし | ドラッグ & ドロップで UI 配置 | Switch 移植時の柔軟性 |
| ダークモード | なし | 対応 | Web エディタの標準 |
| レスポンシブ | デスクトップ固定 | 1024px 以上で 3 カラム | タブレット対応 |
| リサイズハンドル | なし | 4px ドラッグハンドル | 各パネル幅を可変 |

## 3. ツールバー

### 3.1 レイアウト

```
┌─────────────────────────────────────────────────────────────────────────┐
│ [💾][↩][↪] │ [✏️][▬][◯][🪣][🔲] │ [🔍+][🔍-][1:1] │ [▶ テストプレイ] │ [📍マップ][📌イベント] │
│  保存 Undo Redo  描画ツール        ズーム              実行              モード切替          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 ボタングループ仕様

```
Group 1: ファイル操作
  ├── 保存 (Ctrl+S)          アイコン: 💾
  ├── 元に戻す (Ctrl+Z)      アイコン: ↩  disabled: !canUndo
  └── やり直し (Ctrl+Y)      アイコン: ↪  disabled: !canRedo

Group 2: 描画ツール (マップモード時のみ active)
  ├── ペン                   アイコン: ✏️  ショートカット: P
  ├── 四角形                  アイコン: ▬   ショートカット: R
  ├── 楕円                   アイコン: ◯   ショートカット: E
  ├── 塗りつぶし              アイコン: 🪣  ショートカット: G
  └── 消しゴム               アイコン: 🔲  ショートカット: X

Group 3: ズーム
  ├── 拡大 (Ctrl++)          最大 400%
  ├── 縮小 (Ctrl+-)          最小 25%
  └── 等倍 (Ctrl+0)          100%

Group 4: 実行
  └── テストプレイ (Ctrl+R)   緑の ▶ ボタン、目立つ配色

Group 5: モード切替 (トグルボタングループ)
  ├── マップ (F5)            タイル配置モード
  └── イベント (F6)          イベント配置モード
```

### 7.3 イベントコマンドリストの表示

ツクール準拠の「◆ + コマンド名 + 引数要約」形式:

```
◆ 文章の表示
：  宝箱を開けた！
：  ポーションを手に入れた！
◆ アイテムの増減: ポーション, +1
◆ SE の演奏: Item1 (90, 100, 0)
◆ セルフスイッチの操作: A = ON
：
```

- `◆` = コマンド行のインデント記号（ツクール準拠）
- `：` = コマンドの引数（テキストの続きや空行）
- 条件分岐は自動インデント
- ダブルクリック → コマンド編集ダイアログ
- 空行（`：`）をダブルクリック → コマンド追加ダイアログ

## 14. レスポンシブ対応

### 14.1 ブレークポイント

| 幅 | レイアウト | 変更点 |
|-----|----------|--------|
| ≥ 1280px | フル 3 カラム | 左パネル + キャンバス + 右パネル + ステータスバー |
| 1024〜1279px | 2 カラム | 右パネルを下部に折り畳み |
| < 1024px | **非対応**（注意表示） | 「ツクールエディタは 1024px 以上の画面でご利用ください」 |

### 14.2 非対応の理由

ツクール型エディタはマップ描画・イベント配置・データベース編集など、デスクトップ前提の複雑な UI が多い。モバイル対応は投資対効果が低いため、1024px 未満は警告表示のみとする。（ノベルエディタのモバイル対応は維持）

## 19. 実装順序（レビュー提言反映）

Gemini CLI レビュー（04-TSUKURU_LAYOUT_SPEC_REVIEW_REPORT.md）の提言に基づく推奨実装順序:

### Step 1: シェル + マップエディタ（最優先）

1. `TsukuruEditorPage` のシェル（メニューバー + ツールバー + タブ切替 + ステータスバー）
2. 既存 `MapEditor` をマップタブに接続（左パネルにマップツリー追加）
3. `MapCanvas` に右クリックスポイト + イベントモード半透明表示を追加

### Step 2: 独立ダイアログ群（並行実装可）

以下は互いに依存しないため、並行して実装可能:

- `MapEventDialog`（ページ切替 + セルフスイッチ + コマンドリスト）
- `MapPropertyDialog`（BGM + エンカウント + 接続）
- `VarSwitchManager`（変数・スイッチ管理）
- `ResourceManager`（素材管理）

### Step 3: シナリオエディタ

- `CommandPalette` + `CommandListView`（◆ 表記）
- 最小コマンドセット: show_text, show_choices, control_switches, conditional_branch, transfer_player, battle_processing
- `CommandEditDialog`（各コマンドの編集フォーム）

### Step 4: データベース拡張

- `GameDbEditor` 13 タブ化
- `TilesetPassEditor`（通行判定エディタ）
- `TroopPlacementEditor`（敵配置エディタ）

### Step 5: テストプレイ + デバッガ

- `TestPlayOverlay`（ゲーム画面）
- `DebuggerPanel`（F9 変数・スイッチモニター）
- イベントコマンドインタプリタとの接続
