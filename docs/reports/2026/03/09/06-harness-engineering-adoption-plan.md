## 導入優先度

### Priority 1: PostToolUse Hook で自動リント（効果: 大、工数: 小）

記事で最も強調されているポイント。ファイル編集のたびに自動でフォーマット＋リントが走り、エラーがコンテキストに注入されるので手戻りが激減する。

```jsonc
// .claude/settings.json に追加
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'FILE=$(jq -r .tool_input.file_path); case \"$FILE\" in *.ts|*.tsx) npx oxlint --fix \"$FILE\" 2>&1 | head -20;; esac'"
          }
        ]
      }
    ]
  }
}
```

- oxlint + biome を先にインストールする必要あり
- 既存の ESLint でやるなら `npx eslint --fix` でも OK（速度は落ちる）
- 記事によると ESLint 比 50-100x 高速で PostToolUse 向き

### Priority 2: PreToolUse Hook で機密ファイル保護（効果: 中、工数: 小）

Claude がリンター設定や `.env` を書き換えるリスクをゼロにする。

```jsonc
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'FILE=$(jq -r .tool_input.file_path); case \"$FILE\" in *.env|*.env.*|.eslintrc*|biome.json|tsconfig.json) echo \"BLOCKED: $FILE is protected\" >&2; exit 1;; esac'"
          }
        ]
      }
    ]
  }
}
```
