# インラインディレクティブ仕様 — テキスト内タグで表情・ポーズ・リップシンク制御

> Generated by Claude Opus 4.6

## 背景

従来の KSC スクリプトでは、表情・ポーズ・リップシンクの変更は**テキストブロックの前**に `@ch` コマンドで設定する必要があった。

```
@ch hero smile center lip=talk
こんにちは！
@ch hero angry center lip=shout
やめろーー！！
```

この方式では、**1つのセリフの途中で**表情やポーズを変えるには別ブロックに分割するしかない。テンポが悪い。

## 提案: インラインディレクティブ

テキスト内にタグを埋め込み、文字送り中にリアルタイムで演出を切り替える。

```
こんにちは。[e:smile]よろしく！[p:wave][w]えへへ...[e:shy][s]またねー！
```

## タグ一覧

### リップシンク（既存設計を継承）

| タグ | モード | 備考 |
|------|--------|------|
| `[t]` | talk | デフォルト（省略可） |
| `[w]` | whisper | ささやき |
| `[s]` | shout | 叫び |

### 表情

| タグ | 動作 |
|------|------|
| `[e:名前]` | 表情を切り替え（スムーズ遷移 500ms） |

利用可能な名前（18種）:

| カテゴリ | 名前 |
|---------|------|
| Happy | `normal`, `smile`, `laugh`, `grin`, `wink`, `smirk` |
| Sad/Cry | `sad`, `cry`, `shy` |
| Angry | `angry`, `furious`, `serious` |
| Surprise | `surprised`, `shocked`, `blush` |
| Other | `sleepy`, `closed`, `cold` |

### ポーズ

| タグ | 動作 |
|------|------|
| `[p:名前]` | ポーズを切り替え（スムーズ遷移 800ms） |

利用可能な名前（16種）:

| カテゴリ | 名前 |
|---------|------|
| Basic | `standing`, `relaxed`, `cool` |
| Arms | `arms_crossed`, `hands_on_hips`, `wave`, `point`, `salute`, `think` |
| Action | `cheer`, `fighting`, `jump` |
| Emotion | `bow`, `shy_pose`, `tired`, `sit` |

### ウェイト

| タグ | 動作 |
|------|------|
| `[wait:ミリ秒]` | 文字送りを一時停止 |

### カメラ

| タグ | 動作 | 使用例 |
|------|------|--------|
| `[cam:shake]` | 画面揺れ（衝撃・怒り） | `[e:angry][cam:shake]ふざけるな！` |
| `[cam:zoom:倍率]` | ズーム（1.0=等倍、1.2=寄り） | `[cam:zoom:1.3]...まさか。` |
| `[cam:flash]` | 白フラッシュ（閃光・衝撃） | `[cam:flash]うわっ！` |
| `[cam:dark]` | 暗転（意識喪失・場面転換） | `[cam:dark]...意識が...` |
| `[cam:reset]` | カメラ状態を初期化 | `[cam:reset]気がつくと、` |

## 実行例

### 入力テキスト

```
@ch hero normal center
はじめまして。[e:smile]よろしくね！[e:shy][w]...えへへ。[p:wave][s]またねー！
```

### 実行タイムライン

| 時点 | 表示テキスト | 表情 | ポーズ | リップ |
|------|------------|------|--------|--------|
| 0ms | は | normal | standing | talk |
| ... | はじめまして。 | normal | standing | talk |
| 480ms | `[e:smile]` タグ検出 | → **smile** へ遷移開始 | standing | talk |
| 480ms~ | よろしくね！ | smile (遷移中) | standing | talk |
| 960ms | `[e:shy][w]` タグ検出 | → **shy** へ遷移開始 | standing | → **whisper** |
| 960ms~ | ...えへへ。 | shy (遷移中) | standing | whisper |
| 1320ms | `[p:wave][s]` タグ検出 | shy | → **wave** へ遷移開始 | → **shout** |
| 1320ms~ | またねー！ | shy | wave (遷移中) | shout |

## タグのルール

1. テキスト先頭にタグがない場合は `@ch` コマンドの設定を継承
2. タグは画面に表示しない（非表示文字として処理）
3. 複数タグは連続可能: `[e:smile][p:wave][s]`
4. 不明なタグ名は無視（エラーにしない）
5. 表情・ポーズの遷移は **chara-editor で実装済みのスムーズ補間** をそのまま使う

## ブロックエディタ UI

```
┌──────────────────────────────────────────────────┐
│ こんにちは。[e:smile]よろしく！[p:wave]          │
│ [w]えへへ...[e:shy][cam:shake][s]またねー！      │
└──────────────────────────────────────────────────┘
[ 表情 ▼ ] [ ポーズ ▼ ] [ リップ ▼ ] [ カメラ ▼ ] [挿入]
```

### 操作

1. テキストエリア内でカーソルを置く
2. プルダウンから項目を選ぶ（表情18種 / ポーズ16種 / リップ3種）
3. 「挿入」ボタンでカーソル位置にタグを挿入
4. カーソルはタグの直後に移動

### タグのハイライト表示

エディタ上ではタグを色分け表示:

| タグ種類 | 表示色 |
|---------|--------|
| `[e:...]` | ピンク（`#ff6699`）— 表情ボタンと同色 |
| `[p:...]` | グリーン（`#66cc99`）— ポーズボタンと同色 |
| `[t][w][s]` | オレンジ（`#ffaa33`）— リップボタンと同色 |
| `[cam:...]` | ブルー（`#4488ff`）— カメラ演出 |
| `[wait:...]` | グレー（`#888888`） |

## パーサー実装方針

テキストエンジンの文字送りループ内で、次の文字がタグかどうかを判定:

```typescript
function processNextChar(text: string, index: number): {
  consumed: number;       // 消費した文字数
  displayChar?: string;   // 表示する文字（タグなら undefined）
  directive?: {
    type: "expr" | "pose" | "lip" | "wait";
    value: string;
  };
}
```

- `[` を検出 → `]` までをパース → ディレクティブとして処理
- タグ内の文字は表示しない（consumed だけ進む）
- 不正なタグ（`]` が見つからない等）はそのまま表示

## 業界での先行事例

| エンジン | アプローチ |
|---------|-----------|
| 吉里吉里/KAG | `[emote exp=smile]` — テキスト内コマンド混在（本提案の元祖） |
| NScripter | `!sd`, `!w500` — 制御コードをテキスト内に |
| RPGツクール | `\!`(ウェイト), `\>`(瞬間表示) — 限定的なインライン制御 |
| Ren'Py | `{w=1.0}`, `{image=...}` — テキストタグで演出制御 |
| Unity Fungus | `{w=1}{punch}` — テキスト内でカメラ揺れ等も可能 |

kaedevn の `[e:smile]` は吉里吉里の `[emote exp=smile]` より簡潔で、タイプ量が少ない。

## KSC コマンドとの関係

| 設定方法 | 用途 |
|---------|------|
| `@ch hero smile center` | セリフ開始時の初期状態 |
| `[e:smile]` インライン | セリフ途中の表情変更 |

`@ch` コマンドは「初期状態の宣言」、インラインタグは「途中での変更」。共存する設計。

## Live2D 対応設計

インラインディレクティブは **レンダラーに依存しない抽象層** として設計する。
図形キャラ・Live2D・将来の 3D モデル、どのレンダラーでも同じタグが動く。

### アーキテクチャ

```
テキストエンジン（文字送り）
  ↓ タグ検出
ディレクティブディスパッチャー（共通）
  ↓ ICharacterController インターフェース
  ├── ShapeCharaController（図形キャラ — 現在の実装）
  ├── Live2DController（Live2D Cubism モデル）
  └── 将来: Spine / 3D モデル等
```

### ICharacterController インターフェース

```typescript
interface ICharacterController {
  setExpression(name: string, durationMs: number): void;
  setPose(name: string, durationMs: number): void;
  setLipSync(mode: "talk" | "whisper" | "shout" | "off"): void;
  setLipVowel(vowel: "A" | "I" | "U" | "E" | "O" | "N"): void;
}
```

### タグ → Live2D パラメータのマッピング

#### 表情 `[e:名前]`

| タグ | 図形キャラ | Live2D |
|------|-----------|--------|
| `[e:smile]` | ExpressionParams 補間 | `model.expression("smile")` — .exp3.json 再生 |
| `[e:angry]` | ExpressionParams 補間 | `model.expression("angry")` |
| `[e:cry]` | ExpressionParams 補間 + 涙エフェクト | `model.expression("cry")` + パーティクル |

Live2D の表情ファイル（.exp3.json）は Cubism Editor で作成。
名前の対応表をプロジェクトごとに定義する（18種の標準名 → モデル固有名）。

#### ポーズ `[p:名前]`

| タグ | 図形キャラ | Live2D |
|------|-----------|--------|
| `[p:wave]` | PoseParams 補間 | `model.motion("wave")` — .motion3.json 再生 |
| `[p:bow]` | PoseParams 補間 | `model.motion("bow")` |

Live2D のモーションファイル（.motion3.json）で実現。
ポーズ名 → モーションファイルのマッピングをプロジェクト設定で定義。

#### リップシンク `[t][w][s]` + 母音

| タグ | 図形キャラ | Live2D |
|------|-----------|--------|
| 母音 A | mouthOpen=0.8, mouthWidth=1.2 | `ParamMouthOpenY=0.8`, `ParamMouthForm=0.2` |
| 母音 I | mouthOpen=0.2, mouthWidth=1.4 | `ParamMouthOpenY=0.2`, `ParamMouthForm=1.0` |
| 母音 U | mouthOpen=0.4, mouthWidth=0.6 | `ParamMouthOpenY=0.4`, `ParamMouthForm=-0.5` |
| 母音 N | mouthOpen=0, mouthWidth=1.0 | `ParamMouthOpenY=0` |

Live2D の標準パラメータ:
- `ParamMouthOpenY`: 口の開き（0〜1）
- `ParamMouthForm`: 口の形（-1=すぼめ 〜 1=横広）

**テキスト駆動リップシンクは Live2D でそのまま使える。** `charToVowel()` → 母音 → Live2D パラメータ変換するだけ。既存の `KANA_VOWEL_MAP` と `LIP_SHAPES` がそのまま活用できる。

#### カメラ `[cam:種類]`

カメラはレンダラー共通。Live2D 固有の処理は不要。

### Live2D 固有の追加タグ（将来拡張）

| タグ | 動作 |
|------|------|
| `[l2d:param:名前:値]` | 任意の Live2D パラメータを直接制御 |
| `[l2d:motion:名前]` | 任意のモーション再生 |
| `[l2d:physics:on/off]` | 物理演算の ON/OFF |

これらは Live2D 専用の拡張タグ。図形キャラでは無視される。

### Ren'Py との差別化

| 機能 | Ren'Py + Live2D | kaedevn + Live2D |
|------|----------------|-----------------|
| 表情変更 | `show eileen e2`（別コマンド） | `[e:smile]`（テキスト内） |
| モーション | `show eileen m1`（別コマンド） | `[p:wave]`（テキスト内） |
| リップシンク | なし（手動 or 音声解析のみ） | **テキスト駆動自動リップシンク** |
| セリフ途中変更 | **不可能** | `こんにちは[e:smile]嬉しいな` |

**Ren'Py は Live2D をサポートしているが、セリフ途中での表情・モーション変更はできない。**
kaedevn のインラインタグ + Live2D は業界的にもユニークな組み合わせ。

### フォールバック設計

| 状態 | 動作 |
|------|------|
| Live2D モデルあり | Live2DController で描画 |
| Live2D モデルなし + 画像あり | 従来のスプライト表示 |
| 画像なし | ShapeCharaController（図形キャラ）で描画 |

1つのプロジェクトで混在可能。キャラごとに異なるレンダラーを使える。
例: 主人公は Live2D、モブキャラは図形キャラ。

## 今後の実装ステップ

### Phase 1: インラインタグ基盤
1. **パーサー**: テキスト文字送りにタグ検出を追加
2. **エンジン**: タグ検出時に `ICharacterController` メソッドを呼ぶ
3. **エディタ UI**: タグ挿入ボタン + ハイライト表示
4. **コンパイラ**: KSC → Op 変換でインラインタグをメタデータとして保持

### Phase 2: 図形キャラ統合
5. **ShapeCharaController**: 今の chara-editor の描画を ICharacterController に適合
6. **エンジン統合**: `@show` で図形キャラを表示、インラインタグで制御

### Phase 3: Live2D 統合
7. **Live2DController**: pixi-live2d-display で Cubism モデルを表示
8. **パラメータマッピング**: 表情名 → .exp3.json、ポーズ名 → .motion3.json
9. **リップシンク接続**: `setLipVowel()` → `ParamMouthOpenY` / `ParamMouthForm`
10. **Switch ネイティブ**: SDL2 + Cubism Native SDK での Live2D 描画
