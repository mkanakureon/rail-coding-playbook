# 図形キャラブロック（shape_ch）設計仕様書

> Generated by Claude Opus 4.6

## 背景

従来のブロックエディタでは、キャラ表示（ch ブロック）とセリフ（text ブロック）が別ブロックで、セリフ途中の表情・ポーズ変更ができない。図形キャラ＋インラインディレクティブを1ブロックで完結させる新型ブロックを設計する。

## ブロック型定義

```typescript
// apps/editor/src/types/index.ts に追加

export type ShapeChBlock = {
  id: string;
  type: "shape_ch";
  speaker: string;           // 話者名（"エミリ"）
  gender: "female" | "male"; // 図形キャラの性別
  preset?: string;           // プリセット名（"hanako" 等）
  body: string;              // セリフ本文（インラインタグ含む）
  initialExpr: string;       // 初期表情（"normal"）
  initialPose: string;       // 初期ポーズ（"standing"）
  initialLip: string;        // 初期リップ（"talk"）
};
```

## Block union への追加

```typescript
export type Block =
  | StartBlock | BgBlock | ChBlock | TextBlock | ChoiceBlock
  | ...
```

| ボタン | 挿入されるタグ | プルダウン内容 |

## Op 型

```typescript

export interface ShapeChOp {
  type: "SHAPE_CH";
  speaker: string;
  gender: "female" | "male";
  preset?: string;
  initialExpr: string;
  initialPose: string;
  initialLip: string;
  body: string;  // インラインタグ含む

```

## エンジン実行フロー

```
ShapeChOp
  ↓ OpRunner.dispatch
ShapeChHandler
  ├── 1. 図形キャラ描画（drawCharacter）
  ├── 2. body をパース（parseInlineText → buildTimeline）
  ├── 3. TextWindow.show() でテキスト表示開始
  ├── 4. 文字送り中にタグ検出 → executeDirective
  │       ├── [e:] → startExprTransition
  │       ├── [p:] → startPoseTransition
  │       ├── [t][w][s] → リップモード切替
  │       ├── [cam:] → カメラエフェクト
  │       └── [click] → waitClick（既存エンジンのクリック待ち）
  └── 5. テキスト完了 → 次の Op へ
```
