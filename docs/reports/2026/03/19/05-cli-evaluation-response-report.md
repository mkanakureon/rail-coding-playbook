## 指摘1: ドキュメントと実装のパラメータ差異

### 発見された差異

| CLI | ドキュメント（誤） | 実装（正） | 影響 |
|-----|-------------------|-----------|------|
| editor-cli `effect` | `--effectType "shake"` | `--effect "shake"` | コマンド失敗 |

### 対応

- `02-novel-cli-reference.md`: 全パラメータを実装に合わせて修正済み
- `03-novel-cli-from-claude-code-guide.md`: 同上
- import-story.ts: choice options を `label` フィールドに統一

## 指摘2: TypeScript 化

### 対応

作者向け CLI 5ファイルを TypeScript に変換し、共通ライブラリを新規作成。

| ファイル | 行数 | 変更 |
|---------|------|------|
| `scripts/cli/lib/api-client.ts` | 新規 | 共通 API クライアント（認証・通信・エラーハンドリング） |
| `scripts/cli/block/editor-cli.ts` | 1004→TS | ブロックエディタ CLI |
| `scripts/cli/character/character-cli.ts` | 386→TS | キャラクター管理 CLI |
| `scripts/cli/asset/asset-cli.ts` | 458→TS | アセット管理 CLI |
| `scripts/cli/project/init-project.ts` | 308→TS | プロジェクト初期化 CLI |
| `scripts/cli/project/import-story.ts` | 365→TS | テキスト台本インポート CLI |

### 共通ライブラリ（`api-client.ts`）の機能

- `getToken()` — 認証トークン取得（ECONNREFUSED 時のメッセージ改善込み）
- `apiFetch()` / `apiGet()` / `apiPost()` / `apiPut()` / `apiPatch()` / `apiDelete()` — 型付き API 通信
- `apiUpload()` — FormData ファイルアップロード
- `parseArgs()` — CLI 引数パーサー
- `runCli()` — エントリーポイント用エラーハンドラー
- `ApiConnectionError` / `ApiError` — 専用エラークラス

### 実行コマンドの変更

```bash
# 旧（.mjs）
node scripts/cli/block/editor-cli.mjs list

# 新（.ts）
npx tsx scripts/cli/block/editor-cli.ts list
```

元の `.mjs` ファイルは互換性のため残してある。
