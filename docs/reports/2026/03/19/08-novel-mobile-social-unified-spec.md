## エグゼクティブサマリー

kaedevn の最大の武器は「ブラウザで動く」こと。ダウンロード不要で URL を送るだけで即プレイできる。しかし現状この強みを活かしきれていない。スマホ縦持ちで黒帯が出る、OGP がない、シェア機能がない。

本戦略は **3本の柱** でこれを解決する:

1. **縦画面モード（9:19.5）** — 現代スマホ最適化 + 縦動画サイトとの親和性
2. **SNS 拡散インフラ** — OGP + シェア + 動画クリップで拡散ループを作る
3. **摩擦ゼロ体験** — ワンタップ起動 + PWA + 埋め込みプレイヤー

## 柱 1. 縦画面モード（9:19.5）

### なぜ必要か

- スマホユーザーの 90% は縦持ち
- 現状の横画面では上下に大きな黒帯 → 離脱率が高い
- 現代スマホの主流は **9:19.5**（iPhone 15/16、Galaxy S24 等）。9:16 は古い規格
- TikTok / YouTube Shorts / Reels は 9:16 だが、9:19.5 の画面内に 9:16 の動画領域を収められる
- **Switch は横画面のまま**。Web 版だけ縦画面を追加すれば矛盾しない

### 解像度の根拠

| 端末 | 解像度 | アスペクト比 |
|------|--------|------------|
| iPhone 15/16 | 1179x2556 | 9:19.5 |
| Galaxy S24 | 1080x2340 | 9:19.5 |
| Pixel 8 | 1080x2400 | 9:20 |
| iPhone SE 3 | 750x1334 | 9:16（旧世代） |

論理解像度: **720x1560**（≈ 9:19.5）を採用。16:9 より約 22% 縦に長く、現代スマホの画面全体をフルに使える。

### レイアウト設計：ボトムアンカー方式

固定解像度ではなく、**UI を下から積み上げ、背景領域を可変にする**。端末のアスペクト比（9:16〜9:21）に関係なく黒帯が出ない。

```
9:16 (SE)    9:19.5 (iPhone15)   9:20 (Pixel)    9:21 (Xperia)
┌────────┐   ┌────────┐          ┌────────┐      ┌────────┐
│ safe   │   │ safe   │          │ safe   │      │ safe   │
│        │   │        │          │        │      │        │
│ 背景    │   │        │          │        │      │        │
│ キャラ   │   │ 背景    │          │ 背景    │      │ 背景    │
│(cover) │   │ キャラ   │          │ キャラ   │      │ キャラ   │
│        │   │(cover) │          │(cover) │      │(cover) │
├────────┤   │        │          │        │      │        │
│テキスト  │   ├────────┤          ├────────┤      │        │
│ウィンドウ│   │テキスト  │          │テキスト  │      ├────────┤
│ 30%    │   │ウィンドウ│          │ウィンドウ│      │テキスト  │
│[選択肢] │   │ 30%    │          │ 30%    │      │ウィンドウ│
│ safe   │   │[選択肢] │          │[選択肢] │      │ 30%    │
└────────┘   │ safe   │          │ safe   │      │[選択肢] │
             └────────┘          └────────┘      │ safe   │
                                                └────────┘

背景+キャラ領域が伸縮。UI は常に下端固定。
```

### 各要素の配置ルール

| 要素 | アンカー | サイズ | 端末差への挙動 |
|------|---------|--------|-------------|
| 背景 | 上端から fill | cover（アスペクト比維持で拡大、はみ出し切り捨て） | 縦長端末ほど表示面積が広がる |
| キャラ | 背景領域の中央下寄せ | 背景領域の高さの 80%（腰上アップ基本） | 背景領域に合わせて自動調整 |
| テキストウィンドウ | **下端から固定** | 画面高さの 30%（最低 280px） | 全端末で同じ操作感 |
| 選択肢 | テキストウィンドウの上 | 1件 48px | テキストウィンドウを押し上げる |
| safe area | 上下 | `env(safe-area-inset-*)` | ノッチ・ホームバーに被らない |

### コードイメージ

```typescript
const screenH = app.renderer.height;
const safeTop = env.safeAreaInsetTop;
const safeBottom = env.safeAreaInsetBottom;

// テキストウィンドウ: 下から 30%（固定）
const textWindowH = Math.max(screenH * 0.3, 280);
const textWindowY = screenH - safeBottom - textWindowH;

// 背景+キャラ領域: 残り全部（可変）
const bgAreaH = textWindowY - safeTop;
// → 9:16 端末: bgAreaH ≈ 画面の 65%
// → 9:21 端末: bgAreaH ≈ 画面の 72%
// → 背景は cover で描画、キャラは bgArea の中央下寄せ
```

### 設計のメリット

- **黒帯が出ない**: どの端末でも画面全体を使う
- **UI が安定**: テキストウィンドウは常に親指の届く位置（下端固定）
- **背景が映える**: 縦長端末ほど背景の表示面積が広がる
- **作者の手間ゼロ**: 端末ごとの調整が不要。背景は cover で自動フィット

### 横画面モードとの対比

| | 横画面（Switch/PC） | 縦画面（スマホ Web） |
|---|---|---|
| 解像度 | 1280x720 固定 | 720 幅 × 端末高さに追従 |
| テキストウィンドウ | 背景の上に重なる | 背景と分離（下部固定） |
| アスペクト比差 | レターボックス（黒帯） | **ボトムアンカーで吸収（黒帯なし）** |
| UI 配置 | anchor-based（既存） | **ボトムアンカー（新規）** |

### 実装方針

| 項目 | 内容 |
|------|------|
| 解像度 | 幅 720 固定、高さは端末に追従（論理基準値 1560 ≈ 9:19.5） |
| 設定場所 | プロジェクト設定で作者が選択 |
| キャラ配置 | 背景領域の中央下寄せ（腰から上のアップ基本） |
| テキストウィンドウ | 下端アンカー固定、画面高さの 30% |
| Switch ビルド | 強制 landscape 変換（portrait 専用ブロックは無視） |
| 既存プロジェクト | 影響なし（デフォルトは横画面のまま） |

### 相性のいいジャンル

| ジャンル | 相性 | 理由 |
|---------|------|------|
| チャット風ノベル | ◎ | LINE/メッセージアプリと同じ体験 |
| 恋愛・乙女ゲーム | ◎ | 通勤通学の隙間時間にぴったり |
| ホラー | ○ | スマホの小さい画面が没入感を増す |
| 日記・手紙形式 | ◎ | 縦スクロール読みに近い |
| 脱出ゲーム的選択肢 | ○ | 片手操作と相性がいい |

## 柱 2. SNS 拡散インフラ

### 2-1. 動的 OGP（Open Graph Protocol）

URL を SNS に貼ったとき、リッチなプレビューカードが表示される。

```
┌─────────────────────────────┐
│  [作品のスクリーンショット]    │
│                             │
│  「深夜の廃校」              │
│  作者: 楓  |  ホラー         │
│  ▶ 今すぐプレイ（無料）      │
└─────────────────────────────┘
```

| 項目 | 実装 |
|------|------|
| エンドポイント | `GET /api/works/:id/ogp` — スクリーンショットをサーバーサイド生成 |
| Next.js | `generateMetadata` で動的に `og:image`, `og:title`, `og:description` |
| Twitter Card | `summary_large_image`（大きい画像カード） |
| カバー画像 | 作者がアップロード可能。未設定時は自動生成 |

### 2-2. シーン共有

プレイ中に「このシーンを共有」→ スクリーンショット + URL を SNS に投稿。

```
🎮 「深夜の廃校」を読んでる
💬 「この選択、どっちにした？」
▶ https://kaedevn.com/play/xxx?scene=42
```

| 項目 | 実装 |
|------|------|
| キャプチャ | Canvas の `toDataURL()` で現在フレームを取得 |
| シェア | Web Share API（`navigator.share()`）でネイティブ共有 |
| フォールバック | Twitter / LINE の share URL を生成 |
| ディープリンク | `?scene=42` パラメータで途中のシーンから再生 |
| スクリプトコマンド | Web-only `@share` コマンドを追加 |

### 2-3. 動画クリップ自動生成

作品の名場面を **9:16 の短尺動画（15〜30秒）** に自動変換し、TikTok / YouTube Shorts / Reels に投稿可能。ゲーム画面は 9:19.5 だが、動画出力時は 9:16（1080x1920）にクロップ/レターボックスして各プラットフォームの規格に合わせる。

#### 生成方法（2案）

| 方式 | メリット | デメリット |
|------|---------|-----------|
| サーバーサイド（Puppeteer） | 高品質、安定 | サーバー負荷 |
| ブラウザ内（WebCodecs API） | サーバー不要 | ブラウザ互換性 |

#### 動画の構成

```
┌──────────┐
│ 背景画像  │
│ キャラ    │  ← 9:16（1080x1920）動画出力用
│          │
│ セリフが   │  ← 1文字ずつ表示（演出付き）
│ 流れる    │
│          │
│ ▶ 続きを │  ← CTA（Call to Action）
│   読む   │
│ [QRコード]│  ← プレイ URL
└──────────┘
```

#### Phase 分け

| Phase | 内容 | 工数 |
|-------|------|------|
| 1 | 作者が手動でシーン範囲指定 → MP4 生成 | 中 |
| 2 | AI が「名場面」を自動検出（選択肢直前、感情的セリフ、CG 切替） | 大 |
| 3 | TikTok / YouTube API 連携で直接投稿 | 中 |

#### AI 音声合成との連携（将来）

テキストブロックを AI 音声で読み上げ → 「喋るショート動画」として書き出し。実況動画を作者が自動生成できる。

### 2-4. インタラクティブ拡散

動画の最後に「A か B か選んでね」→ コメント欄やプロフ欄のリンクから「その選択肢の続き」を直接ブラウザでプレイ。

- TikTok のコメント欄が投票場になる
- 「どっちルート派？」で議論が生まれ、再生回数が伸びる

## 柱 3. 摩擦ゼロ体験

### 3-1. ワンタップ起動

URL を踏んだ瞬間にゲームが始まる。ロード画面 → タイトル画面 → スタートの3ステップを省略。

| 項目 | 実装 |
|------|------|
| パラメータ | `?quick=1` で即プレイモード |
| アセット | 最初の 3 ページ分だけ先読み、残りはバックグラウンドロード |
| SNS 内ブラウザ | Twitter / LINE のインアプリブラウザで数秒以内に1ページ目を表示 |

### 3-2. お試しプレイ

ログイン不要、セーブ不要で最も盛り上がるシーンから直接開始できるディープリンク。

```
https://kaedevn.com/play/xxx?start=climax
→ クライマックスシーンから3分間プレイ
→ 「最初から読む」ボタンで全編へ
```

### 3-3. PWA + 通知

| 項目 | 実装 |
|------|------|
| ホーム画面追加 | 既存（manifest.json） |
| フルスクリーン | 既存（standalone モード） |
| Web Push 通知 | **新規** — 新章公開を通知 |
| 続き読み | 既存（IndexedDB セーブ）+ 「前回の続きから」ボタン強化 |

### 3-4. 埋め込みプレイヤー

ブログや Web サイトに iframe で埋め込める軽量プレイヤー。

```html
<iframe src="https://kaedevn.com/embed/xxx" width="360" height="640"></iframe>
```

- 縦画面モードで表示（スマホブログに最適）
- 自動再生なし、タップで開始
- 3〜5 シーンまで無料 → 「続きは kaedevn で」の導線

### 3-5. 読者向け縦スクロールモード（実験的）

「クリックで進む」を**縦スクロールで読める**モードに変換。漫画アプリ（ピッコマ/LINEマンガ）のような UX。

- 通常のノベルゲームモードと切り替え可能
- 「ゲームっぽさ」が苦手な読者向け
- 選択肢はインラインで表示

## 結論

ノベルゲームは「短い物語を何度も消費・共有する」文化と親和性が高い。RPG の「長時間遊ぶ」体験とは根本的に異なる。

kaedevn がノベル特化で勝つ公式:

```
縦画面 × ブラウザ即プレイ × SNS シェア × 縦動画拡散 = 他のツールに絶対作れない体験
```

Phase 1（OGP + ワンタップ + シェア）は**1週間で実装可能**。リリースに必ず含めるべき。
