# ノベルゲーム特化：モバイル＋SNS拡散戦略

## 現状分析

### ブラウザで動くメリット（活かせていないもの）

| メリット | 現状 | 活用度 |
|---------|------|--------|
| URL を送るだけで即プレイ | 機能としては動く | △（OGP なし、リンク先が殺風景） |
| ダウンロード不要 | 完全対応 | ○ |
| スマホでもPCでも同じURL | 動く（横画面固定） | △（縦持ちで黒帯） |
| SNS からワンタップで起動 | 可能だが導線がない | ✗ |
| プレイ途中のシーンを共有 | 未実装 | ✗ |

### 最大の課題：**横画面固定（1280x720）**

スマホユーザーの 90% は縦持ちで閲覧する。現状の横画面固定では：

- ブラウザで開くと上下に大きな黒帯
- 「URL 踏んだけど画面小さくてよくわからない」→ 離脱
- 縦動画（TikTok/Shorts/Reels）に切り出せない

**Switch は横画面が必須**だが、**Web 版は縦画面に対応できる**。これが「ブラウザで動くメリット」の最大の武器。

## 柱 1. 縦画面モード（Web 専用）

### コンセプト

Switch 向けの 1280x720 はそのまま維持。Web 版のみ、**縦画面レイアウト（720x1280）** を追加する。作者がプロジェクト設定で選択できるようにする。

```
横画面（デフォルト）: 1280x720 → Switch / PC / 横持ちスマホ
縦画面（Web専用）:     720x1280 → 縦持ちスマホ / 縦動画
```

### 縦画面レイアウトの構成

```
┌──────────────┐
│              │
│   背景画像    │  ← 上半分：背景＋キャラ
│   キャラ      │
│              │
├──────────────┤
│              │
│  テキスト     │  ← 下半分：メッセージウィンドウ
│  ウィンドウ   │
│              │
│  [選択肢1]   │
│  [選択肢2]   │
└──────────────┘
```

### 実装方針

- `VIRTUAL_WIDTH` / `VIRTUAL_HEIGHT` をプロジェクト設定から読む
- レイアウトプリセット: `landscape-16:9`（デフォルト）、`portrait-9:16`（Web専用）
- `portrait` モードでは UI 配置が自動で変わる（背景が上、テキストが下）
- **Switch ビルド時は強制的に landscape に変換**（portrait 専用ブロックは無視）
- 既存の横画面プロジェクトは影響なし

### 縦画面が効くジャンル

| ジャンル | 縦画面との相性 | 理由 |
|---------|--------------|------|
| チャット風ノベル | ◎ | LINE/メッセージアプリと同じ体験 |
| 日記・手紙形式 | ◎ | スクロール読みに近い |
| ホラー | ○ | スマホの小さい画面が没入感を増す |
| 選択肢重視（脱出ゲーム的） | ○ | 片手操作と相性がいい |
| 恋愛（乙女ゲーム） | ◎ | 通勤・通学の隙間時間にぴったり |

## 柱 2. SNS 拡散インフラ

### 2-1. 動的 OGP（Open Graph Protocol）

URL を SNS に貼ったとき、リッチなプレビューカードが表示される。

```
┌─────────────────────────────┐
│  [作品のスクリーンショット]    │
│                             │
│  「深夜の廃校」              │
│  作者: 楓  |  ホラー         │
│  ▶ 今すぐプレイ（無料）      │
└─────────────────────────────┘
```

**実装:**

- `GET /api/works/:id/ogp` — 作品のスクリーンショットをサーバーサイドで生成
- Next.js の `generateMetadata` で動的に `og:image`, `og:title`, `og:description` を設定
- Twitter Card: `summary_large_image`（大きい画像カード）
- 作者がカバー画像をアップロードできる（未設定時は自動生成）

### 2-2. シーン共有（「ここまで読んだ」シェア）

プレイ中に「このシーンを共有」ボタン → SNS にシーンのスクリーンショット + URL を投稿。

```
🎮 「深夜の廃校」を読んでる
💬 「この選択、どっちにした？」
▶ https://kaedevn.com/play/xxx?scene=42
```

**実装:**

- Web-only コマンド `@share` をスクリプトに追加
- Canvas の現在フレームを `toDataURL()` でキャプチャ
- Web Share API（`navigator.share()`）でネイティブ共有
- フォールバック: Twitter/LINE の share URL を生成
- `?scene=42` パラメータで途中のシーンから再生可能

### 2-3. 縦動画クリップ自動生成

作品の名場面を **9:16 の短尺動画（15〜30秒）** に自動変換。TikTok/YouTube Shorts/Reels に投稿可能。

```
┌──────────┐
│ 背景画像  │
│ キャラ    │
│          │
│ セリフ    │
│ テキスト  │
│          │
│ ▶ 続きを │
│   読む   │
│ QRコード │
└──────────┘
```

**実装（段階的）:**

1. **Phase 1**: 作者が手動でシーン範囲を指定 → サーバーサイドで MP4 生成
2. **Phase 2**: AI が「名場面」を自動検出（選択肢直前、感情的なセリフ、CG 切替等）
3. **Phase 3**: 作者が動画をそのまま TikTok/YouTube に投稿（API 連携）

**動画の構成:**
- 背景 + キャラが表示された状態でテキストが1文字ずつ表示される
- BGM 付き（著作権フリーの場合のみ）
- 最後に「続きはこちら」+ QR コード + URL
- 9:16（1080x1920）で出力

### 2-4. 埋め込みプレイヤー

ブログや Web サイトに iframe で埋め込める軽量プレイヤー。

```html
<iframe src="https://kaedevn.com/embed/xxx" width="360" height="640"></iframe>
```

- 縦画面モードで表示（スマホブログに最適）
- 自動再生なし、タップで開始
- 3〜5シーンまで無料 → 「続きは kaedevn で」の導線

## 柱 3. スマホ読者体験の最適化

### 3-1. ワンタップ起動

URL を踏んだ瞬間にゲームが始まる。ロード画面 → タイトル画面 → スタートの3ステップを省略。

```
SNS でリンクタップ
  ↓
即座にプロローグが始まる（ロード中はテキストを先に表示）
  ↓
続きが気になったら「フルバージョンで読む」
```

**実装:**
- `?quick=1` パラメータで即プレイモード
- 最初の 3 ページ分のアセットだけ先読み（残りはバックグラウンドでロード）
- タイトル画面スキップ

### 3-2. 通知・続き読み

PWA として「ホーム画面に追加」した読者に、新章公開を通知。

- Web Push API でブラウザ通知
- 「前回の続きから」ボタン（セーブデータは IndexedDB）
- 読了率トラッキング（作者ダッシュボードに表示）

### 3-3. 読者向けの縦スクロールモード（実験的）

ノベルゲームの「クリックで進む」を、**漫画アプリのように縦スクロールで読める**モードに変換。

```
┌──────────┐
│ [背景]    │  ← スクロールで次のシーンへ
│ [キャラ]  │
│ セリフ1   │
├──────────┤
│ [背景]    │
│ [キャラ]  │
│ セリフ2   │
├──────────┤
│ 選択肢    │
│ [A] [B]  │
└──────────┘
```

- 通常のノベルゲームモードと切り替え可能
- 「読む」体験に近づく（ゲームっぽさが苦手な読者向け）
- ピッコマ/LINEマンガのUXを参考に

## 優先順位

| 優先度 | 施策 | 工数 | 効果 | 理由 |
|--------|------|------|------|------|
| **1** | 動的 OGP | 小 | 大 | URL共有のたびに効く。やらないと損 |
| **2** | ワンタップ起動（quick mode） | 小 | 大 | 離脱率を劇的に下げる |
| **3** | シーン共有（Web Share API） | 中 | 大 | プレイヤーが自発的に拡散する仕組み |
| **4** | 縦画面モード | 大 | 特大 | スマホ体験が根本的に変わる |
| **5** | 埋め込みプレイヤー | 中 | 中 | ブログ・まとめサイトからの流入 |
| **6** | 縦動画クリップ生成 | 大 | 特大 | TikTok/Shorts での爆発的拡散 |
| **7** | 縦スクロールモード | 大 | 中 | ニッチだが刺さる層がいる |

### Phase 分け

```
Phase 1（リリース時・工数小）:
  ✅ 動的 OGP
  ✅ ワンタップ起動
  ✅ Web Share API でシーン共有

Phase 2（リリース後1ヶ月・工数中）:
  📱 縦画面モード
  🔲 埋め込みプレイヤー

Phase 3（リリース後3ヶ月・工数大）:
  🎬 縦動画クリップ生成
  📜 縦スクロールモード
```

## まとめ

「ブラウザで動く」の本当の価値は**配信力**。ゲームの質ではティラノや Ren'Py に追いつけなくても、**「URL 1つでスマホから即プレイ、そのまま SNS にシェア、縦動画で TikTok に拡散」**という体験は他のツールには絶対に作れない。

Phase 1（OGP + ワンタップ + シェア）は工数小で効果が大きく、リリース時に必ず入れるべき。
