## 1. プロジェクト操作

### 1-1. プロジェクト新規作成（`init-project.mjs`）

ジャンル別テンプレートから、アセットインポート済みのプロジェクトをワンコマンドで作成する。

```bash
# 利用可能なジャンル一覧
node scripts/cli/project/init-project.mjs --list

# ファンタジープロジェクト作成
node scripts/cli/project/init-project.mjs --config fantasy

# タイトル指定
node scripts/cli/project/init-project.mjs --config horror --title "深夜の廃校"
```

**対応ジャンル**: fantasy, horror, school, romance, mystery, comedy, longstory

作成されるもの:
- プロジェクト本体
- 公式アセット（背景・キャラ画像）の自動インポート
- キャラクタークラスの作成
- 初期ページ（start → bg → ch → text ブロック）

### 1-2. テキスト台本からの投入（`import-story.mjs`）

プレーンテキストの台本ファイルをブロックに変換してプロジェクトに投入する。

```bash
# プレビュー（API不要・変換結果の確認のみ）
node scripts/cli/project/import-story.mjs --preview story.txt

# プロジェクトに投入
node scripts/cli/project/import-story.mjs <projectId> story.txt
```

**台本フォーマット**:

```
# 第1章 出発
@bg forest
@ch kaede normal center
[楓] 今日から旅に出よう。
[楓] 準備はいい？

--- 選択肢 ---
- はい
- もう少し待って

@jump page-2

# 第2章 旅立ち
@bg road
[楓] さあ、行こう！
```

| 記法 | 変換先 |
|------|--------|
| `# ページ名` | 新しいページ |
| `[キャラ名] セリフ` | text ブロック（speaker 付き） |
| `@bg <assetId>` | bg ブロック |
| `@ch <slug> <表情> <位置>` | ch ブロック |
| `@effect <type>` | effect ブロック |
| `@jump <pageId>` | jump ブロック |
| `--- 選択肢 ---` + `- 選択肢` | choice ブロック |

### 1-3. KS スクリプト変換（`ks-convert.mjs` / `ks-upload.mjs`）

TyranoScript 形式の `.ks` ファイルをエディタ JSON に変換し、プロジェクトとしてアップロードする。

```bash
# Step 1: KS → エディタJSON（オフライン）
node scripts/cli/ks-convert.mjs --config fantasy ./my-scenario/

# Step 2: エディタJSON → プロジェクト作成
node scripts/cli/ks-upload.mjs ./my-scenario/editor-json/
```

**ジャンル別ワンショット変換**:

```bash
node scripts/cli/ks-to-project-comedy.mjs
node scripts/cli/ks-to-project-horror.mjs
node scripts/cli/ks-to-project-mystery.mjs
node scripts/cli/ks-to-project-romance.mjs
node scripts/cli/ks-to-project-drama.mjs
node scripts/cli/ks-to-project-longstory.mjs
```

## 2. ブロックエディタ操作（`editor-cli.mjs`）

プロジェクト内のブロック（シーン命令）をコマンドラインから操作する。

```bash
CLI="node scripts/cli/block/editor-cli.mjs"
```

### 2-1. 閲覧・検証

```bash
# プロジェクト一覧
$CLI list

# プロジェクト内の全ブロックを表示
$CLI blocks <projectId>

# 利用可能なアセット・キャラ・変数を確認
$CLI context <projectId>

# 参照の整合性チェック（存在しないアセットID等を検出）
$CLI validate <projectId>
```

### 2-2. ブロック追加

```bash
# テキスト（セリフ）
$CLI add <projectId> <pageId> text --speaker "楓" --body "こんにちは"

# 背景
$CLI add <projectId> <pageId> bg --assetId <id>

# キャラクター表示
$CLI add <projectId> <pageId> ch --characterId <slug> --expressionId <expr>

# 選択肢
$CLI add <projectId> <pageId> choice --text "どうする？" --options '[{"label":"戦う","toPageId":"page-battle"},{"label":"逃げる","toPageId":"page-escape"}]'

# ジャンプ
$CLI add <projectId> <pageId> jump --toPageId <pageId>

# 変数セット
$CLI add <projectId> <pageId> set_var --varName "flag_met" --value "true"

# エフェクト
$CLI add <projectId> <pageId> effect --effectType "shake"

# オーバーレイ
$CLI add <projectId> <pageId> overlay --assetId <id>

# その他: screen_filter, if, battle, timeline, ksc
```

### 2-3. ブロック編集・削除・並替

```bash
# プロパティ更新
$CLI update <projectId> <blockId> --body "新しいセリフ"
$CLI update <projectId> <blockId> --speaker "楓" --body "変更後のセリフ"

# 削除（start ブロックは削除不可）
$CLI remove <projectId> <blockId>

# 並べ替え
$CLI move <projectId> <blockId> up
$CLI move <projectId> <blockId> down
```

### 2-4. ページ操作

```bash
# ページ追加（start ブロック付きで作成）
$CLI add-page <projectId> "第2章"

# ページ名変更
$CLI rename-page <projectId> <pageId> "新しい章名"

# ページ削除（インデックス指定）
$CLI remove-page <projectId> <pageIndex>
```

### 2-5. エクスポート / インポート

```bash
# プロジェクトをJSON出力（バックアップ・共有）
$CLI export <projectId> > backup.json

# JSONファイルからブロックを投入
$CLI import <projectId> <pageId> blocks.json
```

## 3. キャラクター管理（`character-cli.mjs`）

```bash
CHCLI="node scripts/cli/character/character-cli.mjs"
```

```bash
# 公式キャラアセット一覧
$CHCLI official
$CHCLI official --subcategory 楓

# 公式アセットからキャラクタークラス作成（表情自動取得）
$CHCLI create-from-official <projectId> --slug kaede --name 楓 --subcategory 楓

# プロジェクト内キャラ一覧
$CHCLI list <projectId>

# キャラ詳細表示
$CHCLI show <projectId> <slug>

# 手動でキャラ作成
$CHCLI create <projectId> --slug villain --name "魔王" \
  --expressions '{"angry":"asset-id-1","laugh":"asset-id-2"}'

# キャラ更新
$CHCLI update <projectId> <slug> --name "新しい名前"

# キャラ削除
$CHCLI delete <projectId> <slug>

# 公式アセットへの自動バインド
$CHCLI bind <projectId>
```

## 4. AI 執筆支援（`assist-cli.ts`）

設定ファイルから4段階パイプラインでシナリオを自動生成する。

```bash
# 全段階を実行
npx tsx scripts/cli/ai/assist-cli.ts all --settings <project-dir>

# 途中から再開
npx tsx scripts/cli/ai/assist-cli.ts all --settings <dir> --resume <output-dir>

# 特定の章・話だけ生成
npx tsx scripts/cli/ai/assist-cli.ts all --settings <dir> --chapter 2 --episode 3

# ドライラン（生成せず計画のみ）
npx tsx scripts/cli/ai/assist-cli.ts all --settings <dir> --dry-run

# モック（API呼び出しなし）
npx tsx scripts/cli/ai/assist-cli.ts all --settings <dir> --mock
```

**4段階パイプライン**:

| Stage | モデル | 内容 |
|-------|--------|------|
| 0 | gemini-2.5-pro | アイデア → 構造化プロット |
| 1 | gemini-2.5-pro | プロット → 章アウトライン |
| 2 | gemini-2.5-flash | 章 → 話サマリー |
| 3-4 | gemini-2.5-flash | 話 → フルテキスト → エディタJSON |

### 設定ファイル自動生成（`upstream-cli.ts`）

```bash
# アイデアメモから設定ファイル一式を生成
npx tsx scripts/cli/ai/upstream-cli.ts generate-settings idea.md ./output/
```

## 5. マップ操作（`map-cli.mjs`）

```bash
MCLI="node scripts/cli/map/map-cli.mjs"
```

```bash
# タイルセット一覧
$MCLI tilesets

# タイル一覧（タグ絞り込み可）
$MCLI tiles <tilesetId>
$MCLI tiles <tilesetId> --tag water

# マップ一覧・詳細
$MCLI list
$MCLI show <mapId>

# マップ新規作成
$MCLI create <mapId> --name "森" --label "深い森" \
  --width 20 --height 15 --tileset outdoor

# マップ検証
$MCLI validate <mapId>

# レイヤー生成（stdin からテキスト地形入力）
echo "grass grass water" | $MCLI gen-layer <tilesetId> <mapId> <layerId>

# 衝突データ自動生成
$MCLI gen-collision <mapId>

# イベント追加・削除
$MCLI add-event <mapId> --name "宝箱" --label "chest01" \
  --x 5 --y 3 --trigger action
$MCLI remove-event <mapId> <eventId>
```

## 6. ユーティリティ

### 画像変換ツール（`kaede-tools`）

```bash
# MP4 → PNG フレーム分割
npx kaede-tools mp4topng input.mp4 --fps 12

# 背景除去
npx kaede-tools rmbg input.png --color green

# PNG連番 → アニメーション WebP
npx kaede-tools webp ./frames/ --fps 12 --quality 80

# ワンショット（MP4 → PNG → 背景除去 → WebP）
npx kaede-tools convert input.mp4
```

### タイルセット画像生成（`gen-official-asset.ts`）

```bash
# AI（Imagen 4.0）でタイルセット画像を生成
npx tsx scripts/cli/asset/gen-official-asset.ts outdoor
npx tsx scripts/cli/asset/gen-official-asset.ts interior
```

### RAG インデックス更新（`rag-index.ts`）

```bash
# docs/ のドキュメントをベクトル検索用にインデックス化
npx tsx scripts/cli/rag/rag-index.ts
```

## 典型的なワークフロー

### A. ゼロからプロジェクトを作って台本を流し込む

```bash
# 1. プロジェクト作成
node scripts/cli/project/init-project.mjs --config fantasy --title "冒険の始まり"
# → projectId が出力される

# 2. キャラクター追加
node scripts/cli/character/character-cli.mjs create-from-official <projectId> \
  --slug kaede --name 楓 --subcategory 楓

# 3. 台本を投入
node scripts/cli/project/import-story.mjs <projectId> story.txt

# 4. 確認
node scripts/cli/block/editor-cli.mjs blocks <projectId>
node scripts/cli/block/editor-cli.mjs validate <projectId>
```

### B. 既存プロジェクトのシーンを編集する

```bash
CLI="node scripts/cli/block/editor-cli.mjs"

# 1. ブロック確認
$CLI blocks <projectId>

# 2. セリフ変更
$CLI update <projectId> <blockId> --body "修正後のセリフ"

# 3. 新しいページを追加して分岐を作る
$CLI add-page <projectId> "バッドエンド"
$CLI add <projectId> <newPageId> bg --assetId <id>
$CLI add <projectId> <newPageId> text --speaker "楓" --body "ここで終わりね…"
```

### C. AI にシナリオを書かせる

```bash
# 1. アイデアメモから設定ファイル生成
npx tsx scripts/cli/ai/upstream-cli.ts generate-settings idea.md ./my-story/

# 2. 設定ファイルを確認・修正（手動）
# 3. 全段階生成
npx tsx scripts/cli/ai/assist-cli.ts all --settings ./my-story/

# 4. 生成されたエディタJSONをアップロード
node scripts/cli/ks-upload.mjs ./my-story/editor-json/
```
