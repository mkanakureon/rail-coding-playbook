## 1. あなたが「書くべきではない」もの

**以下は全て AI に任せるべきもの。あなたが 1 行たりとも書く必要がない。**

| カテゴリ | 理由 | 担当 AI |
|---------|------|---------|
| コード実装 | スキル + CLAUDE.md + 設計書があれば AI が書ける | Claude Code / Gemini |
| テスト | テスト計画書を書けば AI が全件実装する | Claude Code |
| ドキュメント | コミット履歴 + コードから自動生成できる | Claude Code |
| リファクタリング | 型チェック + lint が通れば安全 | Claude Code |
| バグ修正 | ログ + 再現条件があれば AI が特定・修正する | Claude Code |
| デプロイ | GitHub Actions で自動化済み | CI/CD |
| コードレビュー | typecheck + lint + E2E が通れば合格 | pre-push hook |
| 開発日誌 | git log から自動生成 | devlog スキル |
| コミット | commit スキルが自動実行 | commit スキル |

## 2. あなたが集中すべき 3 つのこと

### (1) 「次に何を作るか」を決める（戦略判断）

> AI は「どう作るか」は得意だが、「何を作るか」は人間にしか決められない。

今日の Phase 1 → Phase 2 の一気貫通は、あなたが「ノベル特化に舵を切る」と決断したから実現した。12 本の分析文書は AI が書いたが、「ツクールを止めてノベルに集中する」という決断は人間にしかできなかった。

**あなたのタスク:**
- 朝イチで「今日の目標」を 1 行で書く
- 「Phase 3 のどの機能を先にやるか」を優先順位付けする
- 「この機能は要らない」と切り捨てる判断をする

**所要時間: 15 分/日**

### (2) 「体験を判定する」（品質ゲート）

> AI はテストを通せるが、「面白いか」「気持ちいいか」は判定できない。

ペイウォールの文言「続きが気になる？」が正しいかどうか、シェアボタンの位置が右上で正しいかどうか、OGP 画像のグラデーションがブランドに合っているかどうか——これらは人間が 5 秒見て判断するほうが、AI が 100 行のテストを書くより正確。

**あなたのタスク:**
- AI が作ったものをブラウザで触る（スマホ実機推奨）
- 「ここ違う」「これでいい」を 1 行で返す
- スクリーンショットに赤丸を付けて返すだけでも十分

**所要時間: 30 分/日**

### (3) 「外に出す」（マーケティング + コミュニティ）

> AI はコードを書けるが、ユーザーを連れてくることはできない。

5 万行のコードも、誰にも使われなければゼロ。あなたが専念日にやるべき最大の仕事は、**作品を作って公開し、SNS に流すこと。**

**あなたのタスク:**
- デモ作品を 1 本作る（`/create-project` スキルで 5 分）
- X / TikTok に OGP 付きリンクを投稿する
- ユーザーの反応を見て、次の機能の優先順位を調整する

**所要時間: 1-2 時間/日**

## 3. 「5 万行/日」ワークフロー

```
06:00  あなた: 「今日の目標」を 1 行で書く（Slack or CLAUDE.md に追記）
       例: 「Phase A のリファクタリング 4 項目 + デモ作品 1 本」
         ↓
06:05  Claude Code: 目標を読み、実装計画を立てる
       Gemini CLI: 別ブランチで並行タスクを開始
         ↓
06:05〜12:00  AI が自走（コード・テスト・ドキュメント）
       あなた: 別のことをしていい（デモ作品制作、SNS、企画）
         ↓
12:00  AI: 「午前の成果: 15コミット / +8,000行 / typecheck PASS」と報告
       あなた: ブラウザで 10 分触る → 「OK」or「ここ直して」
         ↓
12:15  AI: フィードバック反映 → 午後タスクに着手
         ↓
12:15〜18:00  AI が自走（第 2 ラウンド）
       あなた: デモ作品を仕上げて SNS に投稿
         ↓
18:00  AI: 「本日の成果: 30コミット / +50,000行」と報告
       あなた: 5 分で確認 → push 承認
         ↓
18:05  自動デプロイ → Azure に反映
18:10  devlog スキルが開発日誌を自動生成
```

**あなたの実作業時間: 約 2 時間。残りは AI が自走。**

### まだ足りないもの

#### (A) 「朝の指示書テンプレート」

あなたが毎朝 1 分で書く指示書。これだけで AI が 1 日自走できる。

```markdown
# 今日の指示 — 2026-03-20

## 目標
Phase A リファクタリング（DB接続プール・認証キャッシュ・インデックス・ページネーション）

## 優先順位
1. A-1 DB接続プール（最重要）
2. A-2 認証キャッシュ
3. A-3 インデックス
4. A-4 ページネーション上限

## 制約
- 既存テストが全て通ること
- 本番 DB に影響しないこと

## 判断が必要な場面
- Redis を入れるかどうか → 聞いて
- それ以外は自走していい
```

#### (B) 「成果報告の自動化」

AI が作業完了時に自動で報告を生成する仕組み。現在は手動で「どこまで終わった？」と聞いている。

```bash
# hooks で実装: push 成功時に自動レポート生成
# .claude/settings.json に追加
{
  "hooks": {
    "post-push": "generate-daily-report"
  }
}
```

## 5. 結論

**あなたが「書かない」ことが、最大のアウトプットを生む。**

| あなたの役割 | 所要時間 | AI の役割 | 所要時間 |
|------------|---------|---------|---------|
| 戦略判断（何を作るか） | 15 分 | 実装（コード・テスト・ドキュメント） | 12 時間 |
| 品質判定（触って確認） | 30 分 | リファクタリング・バグ修正 | 自動 |
| マーケティング（作品・SNS） | 1-2 時間 | デプロイ・日誌・レポート | 自動 |

あなたの脳を最も効率的にハックする方法は、「朝 1 分で指示を書き、昼に 10 分触り、夕方に push を承認する」こと。残りの時間は **作品を作り、外に出す** ことに使う。

AI は 5 万行のコードを書ける。だが、「この作品を世界に届けたい」という意志だけは、あなたにしか持てない。
