## 1. プロジェクト作成

### テンプレートから作成（`init-project.ts`）

```bash
# ジャンル一覧
npx tsx scripts/cli/project/init-project.ts --list

# 作成（背景・キャラ自動インポート、初期ブロック付き）
npx tsx scripts/cli/project/init-project.ts --config fantasy
npx tsx scripts/cli/project/init-project.ts --config horror --title "深夜の廃校"
```

**対応ジャンル**: fantasy / horror / school / romance / mystery / comedy / longstory

### テキスト台本から作成（`import-story.ts`）

```bash
# プレビュー（API不要）
npx tsx scripts/cli/project/import-story.ts --preview story.txt

# プロジェクトに投入
npx tsx scripts/cli/project/import-story.ts <projectId> story.txt
```

**台本フォーマット**:

```
# 第1章 出発
@bg forest
@ch kaede normal center
[楓] 今日から旅に出よう。
[楓] 準備はいい？

--- 選択肢 ---
- はい
- もう少し待って

@jump page-2
```

| 記法 | 変換先 |
|------|--------|
| `# ページ名` | 新しいページ |
| `[キャラ名] セリフ` | text ブロック |
| `@bg <assetId>` | bg ブロック |
| `@ch <slug> <表情> <位置>` | ch ブロック |
| `@effect <type>` | effect ブロック |
| `@jump <pageId>` | jump ブロック |
| `--- 選択肢 ---` + `- 項目` | choice ブロック |

### KS スクリプトから作成

```bash
# KS → エディタJSON（オフライン）
node scripts/cli/ks-convert.mjs --config fantasy ./my-scenario/

# エディタJSON → プロジェクト
node scripts/cli/ks-upload.mjs ./my-scenario/editor-json/

# ジャンル別ワンショット
node scripts/cli/ks-to-project-comedy.mjs
node scripts/cli/ks-to-project-horror.mjs
node scripts/cli/ks-to-project-mystery.mjs
node scripts/cli/ks-to-project-romance.mjs
node scripts/cli/ks-to-project-drama.mjs
node scripts/cli/ks-to-project-longstory.mjs
```

## 2. ブロック操作（`editor-cli.ts`）

```bash
CLI="npx tsx scripts/cli/block/editor-cli.ts"
```

### 閲覧・検証

| コマンド | 用途 |
|---------|------|
| `$CLI list` | プロジェクト一覧 |
| `$CLI blocks <projectId>` | 全ブロック表示 |
| `$CLI context <projectId>` | 利用可能アセット・キャラ・変数 |
| `$CLI validate <projectId>` | 参照整合性チェック |

### ブロック追加

```bash
# セリフ
$CLI add <projectId> <pageId> text --speaker "楓" --body "こんにちは"

# 背景
$CLI add <projectId> <pageId> bg --assetId <id>

# キャラクター表示（pos: L / LC / C / RC / R）
$CLI add <projectId> <pageId> ch --characterId <slug> --expressionId <expr> --pos C

# 選択肢
$CLI add <projectId> <pageId> choice \
  --options '["戦う","逃げる"]'

# ジャンプ
$CLI add <projectId> <pageId> jump --toPageId <pageId>

# 変数セット
$CLI add <projectId> <pageId> set_var --varName "flag_met" --value "true"

# エフェクト / オーバーレイ / スクリーンフィルタ
$CLI add <projectId> <pageId> effect --effect "shake"
$CLI add <projectId> <pageId> overlay --assetId <id>
$CLI add <projectId> <pageId> screen_filter --filterType "sepia"
```

### 編集・削除・並替

```bash
$CLI update <projectId> <blockId> --body "新しいセリフ"
$CLI remove <projectId> <blockId>          # start は削除不可
$CLI move <projectId> <blockId> up|down
```

### ページ操作

```bash
$CLI add-page <projectId> "第2章"
$CLI rename-page <projectId> <pageId> "新しい章名"
$CLI remove-page <projectId> <pageIndex>
```

### エクスポート / インポート

```bash
$CLI export <projectId> > backup.json
$CLI import <projectId> <pageId> blocks.json
```

## 3. アセット管理（`asset-cli.ts`）

プロジェクトへの画像・音声アップロード、マイアセット（個人ライブラリ）の操作。

```bash
ACLI="npx tsx scripts/cli/asset/asset-cli.ts"
```

### プロジェクトアセット

```bash
# アセット一覧（カテゴリ絞り込み可）
$ACLI list <projectId>
$ACLI list <projectId> --category bg
$ACLI list <projectId> --category ch-img

# アセット詳細
$ACLI show <projectId> <assetId>

# ローカル画像をプロジェクトにアップロード
$ACLI upload <projectId> ./forest.png --category bg
$ACLI upload <projectId> ./forest.png --category bg --slug my_forest

# ディレクトリ内の画像を一括アップロード（キャラ表情差分など）
$ACLI upload-dir <projectId> ./kaede_faces/ --category ch-img

# slug 変更
$ACLI rename <projectId> <assetId> --slug new_name

# アセット削除
$ACLI delete <projectId> <assetId>

# 公式アセットをプロジェクトにインポート
$ACLI import-official <projectId> <officialAssetId>

# マイアセットからプロジェクトにインポート
$ACLI import-library <projectId> <userAssetId>
```

### マイアセット（個人ライブラリ）

プロジェクトに依存しない個人の画像ライブラリ。複数プロジェクトで使い回せる。

```bash
# マイアセット一覧
$ACLI my-list
$ACLI my-list --category bg

# ローカル画像をマイアセットに追加
$ACLI my-upload ./forest.png --category bg
$ACLI my-upload ./forest.png --category bg --subcategory outdoor

# ディレクトリ一括追加
$ACLI my-upload-dir ./kaede_faces/ --category ch-img --subcategory 楓

# マイアセットから削除
$ACLI my-delete <userAssetId>
```

**対応フォーマット**: png / jpg / jpeg / webp / gif / mp3 / ogg / wav / m4a

## 4. キャラクター作成・管理（`character-cli.ts`）

公式アセットまたはマイアセットの画像を使って、プロジェクトにキャラクターを新規作成する。
公式アセットの場合、表情差分は自動取得される。

```bash
CHCLI="npx tsx scripts/cli/character/character-cli.ts"
```

```bash
# 公式アセットのキャラクター一覧を確認
$CHCLI official
$CHCLI official --subcategory 楓

# 公式アセットの画像でキャラクターを新規作成（表情差分も自動取得）
$CHCLI create-from-official <projectId> --slug kaede --name 楓 --subcategory 楓

# プロジェクト内のキャラクター一覧・詳細
$CHCLI list <projectId>
$CHCLI show <projectId> <slug>

# マイアセットの画像でキャラクターを新規作成（asset ID を直接指定）
$CHCLI create <projectId> --slug villain --name "魔王" \
  --expressions '{"angry":"asset-id-1","laugh":"asset-id-2"}'

# 更新・削除
$CHCLI update <projectId> <slug> --name "新しい名前"
$CHCLI delete <projectId> <slug>

# 公式アセットへの自動バインド
$CHCLI bind <projectId>
```

## 典型的なワークフロー

### A. テンプレートから始めて台本を流し込む

```bash
# 1. プロジェクト作成
npx tsx scripts/cli/project/init-project.ts --config fantasy --title "冒険の始まり"
# → projectId が出力される

# 2. キャラクター作成
npx tsx scripts/cli/character/character-cli.ts create-from-official <projectId> \
  --slug kaede --name 楓 --subcategory 楓

# 3. 台本を投入
npx tsx scripts/cli/project/import-story.ts <projectId> story.txt

# 4. 確認
npx tsx scripts/cli/block/editor-cli.ts blocks <projectId>
npx tsx scripts/cli/block/editor-cli.ts validate <projectId>
```

### B. ローカル画像を使ってキャラクターを作る

```bash
# 1. 表情画像をプロジェクトにアップロード
npx tsx scripts/cli/asset/asset-cli.ts upload-dir <projectId> ./villain_faces/ --category ch-img

# 2. アップロードしたアセットの ID を確認
npx tsx scripts/cli/asset/asset-cli.ts list <projectId> --category ch-img

# 3. キャラクターを作成（asset ID を指定）
npx tsx scripts/cli/character/character-cli.ts create <projectId> \
  --slug villain --name "魔王" \
  --expressions '{"angry":"<assetId1>","laugh":"<assetId2>"}'
```

### C. 既存シーンを手直しする

```bash
CLI="npx tsx scripts/cli/block/editor-cli.ts"

# 1. 現状確認
$CLI blocks <projectId>

# 2. セリフ修正
$CLI update <projectId> <blockId> --body "修正後のセリフ"

# 3. バッドエンド分岐を追加
$CLI add-page <projectId> "バッドエンド"
$CLI add <projectId> <newPageId> bg --assetId <id>
$CLI add <projectId> <newPageId> text --speaker "楓" --body "ここで終わりね…"
```
