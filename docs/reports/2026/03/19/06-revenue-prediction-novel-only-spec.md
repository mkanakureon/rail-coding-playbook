## 2. 収益モデル（ノベルゲーム特化版）

ツクール型の「UIテンプレート販売」「RPGバトルシステム」を削除し、ノベルゲームに特化した収益源に絞る。

| # | 収益源 | 単価 | 転換率/想定 |
|---|--------|------|-----------|
| 1 | 容量課金 | 300円/月 | 作者の 5% |
| 2 | マルチプラットフォーム書き出し | 200円/月 | 作者の 8%（RPGより低い） |
| 3 | 販売手数料 | 流通の 10% | ARPU 5〜10円 |
| 4 | 広告収益 | RPM 200円 | 1人5PV（ノベルはRPGよりPV少ない） |
| 5 | Switch 出版サポート | 50,000円/本 + 売上10% | Month 6〜 |

### ノベルゲーム特化で変わる点

- **広告 PV が減る**: RPG は戦闘・ダンジョンで PV が増えるが、ノベルは「読む」だけなので PV/人が少ない（10PV → 5PV）
- **書き出し転換率が下がる**: RPG は「アプリとして売りたい」モチベが高いが、ノベルは Web 公開で満足する作者も多い（10% → 8%）
- **販売単価が下がる**: ノベルゲームは RPG より安価（1,000〜2,000円 vs 2,000〜5,000円）
- **Switch 出版のハードルが下がる**: ノベルゲームは技術的にシンプルで移植しやすい。品質基準もクリアしやすい

## 4. ツクール込み vs ノベル特化 比較

| 指標 | ツクール込み（元） | ノベル特化（今回） | 差 |
|------|------------------|------------------|-----|
| M12 MAU | 1,000,000 | 300,000 | ▲70% |
| M12 作者数 | 50,000 | 8,000 | ▲84% |
| M12 月間売上 | 1,125万円 | 138万円 | ▲88% |
| M12 営業利益 | 1,013万円 | 122万円 | ▲88% |
| 損益分岐 | M1（初月） | M2（2ヶ月目） | +1ヶ月 |
| Switch 出版開始 | M3 | M6 | +3ヶ月 |

### 差が大きい理由

1. **MAU の天井が低い**: ノベルゲームは RPG より市場が小さい（1/3 程度）
2. **PV/人が少ない**: 読むだけの体験は PV が伸びにくい（広告収益 ▲50%）
3. **Switch 出版が遅れる**: ノベル専門では品揃えが揃うまで時間がかかる

### ノベル特化の優位点

1. **技術リスクが低い**: ツクール型エディタの開発が不要で、リリースが早い
2. **品質管理が容易**: ノベルゲームは技術的にシンプルで Switch 審査に通りやすい
3. **差別化が明確**: 「Switch に出せるノベルゲームエンジン」は競合がゼロ
4. **開発リソース集中**: エンジン改善と作者サポートに全振りできる

## 5. 収益を伸ばすための施策（ノベル特化版）

### 短期（M1〜M3）: 作者獲得

| 施策 | 効果 |
|------|------|
| ティラノビルダーからの移行ガイド | 既存ノベル制作者を取り込む |
| テンプレートシナリオ（ファンタジー/ホラー/学園） | 初心者の制作ハードル低減 |
| 公式ショーケース作品（3〜5本） | 「こんなものが作れる」の実例 |
| ノベルゲームジャム | 作品数を短期間で増やす |

### 中期（M4〜M6）: プレイヤー獲得

| 施策 | 効果 |
|------|------|
| URL 共有で即プレイ（ダウンロード不要） | 実況者・SNS との相性が良い |
| 動的 OGP（作品のサムネイル自動生成） | SNS でのクリック率向上 |
| スマホ最適化（縦画面テンプレート） | 通勤・通学の隙間時間を狙う |

### 長期（M6〜M12）: Switch 展開

| 施策 | 効果 |
|------|------|
| Switch 出版サポート（審査代行） | 作者にとっての最大の価値 |
| 「Web で人気 → Switch で販売」の成功事例 | 他の作者のモチベーション |
| レベニューシェアモデル | プラットフォームと作者の利害一致 |

## 6. リスクと対策

| リスク | 確度 | 対策 |
|--------|------|------|
| MAU が伸びない（M3 で 25,000 未達） | 中 | ゲームジャム・実況者コラボで初動を作る |
| ティラノから移行しない | 中 | 移行ツール + 機能差の明確なアピール |
| Switch 審査に通らない | 低 | 技術的にシンプル。任天堂の審査基準を事前確認 |
| サーバーコスト超過 | 低 | 静的配信中心。CDN 従量課金で制御可能 |
| 競合（Ren'Py 等）の Web 対応 | 低 | Ren'Py は Python ベースで Web 対応が弱い |

## 7. 結論

ツクール型エディタを外すと、M12 の月間利益は **1,013万円 → 122万円** と約1/8に縮小する。しかし：

- **M2 で損益分岐を超える**（運営費が低いため）
- **開発リスクがゼロ**（追加開発なしでリリース可能）
- **「Switch に出せるノベルゲームエンジン」は競合ゼロ**
- **年間利益 約1,500万円**（M12 ベースで計算）は個人〜小規模チームとして十分

ツクール型エディタは将来の拡張として温存し、まずはノベルゲーム特化でリリースして市場の反応を見るのが現実的な戦略。
