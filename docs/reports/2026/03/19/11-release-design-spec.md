## 4. ポイント課金システム設計（モック）

### 4-1. コンセプト

本番決済（Stripe 等）を実装する前に、**モック決済でポイント購入→作品購入の体験を完成させる**。モック時は「購入」ボタンを押すだけで即座にポイントが加算される。

### 4-2. DB スキーマ

```prisma
// ポイント残高
model Wallet {
  id        String   @id @default(cuid())
  userId    String   @unique
  user      User     @relation(fields: [userId], references: [id])
  balance   Int      @default(0) // 現在のポイント残高
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("wallets")
}

// ポイント取引履歴
model PointTransaction {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  type      String   // "charge" | "purchase" | "refund" | "earn"
  amount    Int      // 変動ポイント（チャージ: +, 購入: -）
  balance   Int      // 取引後の残高
  workId    String?  // 購入対象の作品（purchase 時のみ）
  work      Work?    @relation(fields: [workId], references: [id])
  metadata  Json?    // モック/本番の区別、決済ID 等
  createdAt DateTime @default(now())

  @@index([userId, createdAt])
  @@map("point_transactions")
}

// 作品の購入済みフラグ
model WorkPurchase {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  workId    String
  work      Work     @relation(fields: [workId], references: [id])
  points    Int      // 支払ったポイント
  createdAt DateTime @default(now())

  @@unique([userId, workId])
  @@map("work_purchases")
}

// 作品の価格設定（Work モデルに追加）
// Work テーブルに以下のフィールドを追加:
//   price       Int     @default(0)    // 0 = 無料、100 = 100pt
//   freeUntil   String? // ペイウォール位置（ブロックID or ページID）
```

### 4-3. API エンドポイント

| メソッド | パス | 説明 | 認証 |
|---------|------|------|------|
| GET | `/api/wallet` | 自分の残高を取得 | 必須 |
| POST | `/api/wallet/charge` | ポイントをチャージ（**モック: 即加算**） | 必須 |
| GET | `/api/wallet/history` | 取引履歴 | 必須 |
| POST | `/api/works/:id/purchase` | 作品を購入（残高から差し引き） | 必須 |
| GET | `/api/works/:id/access` | 作品へのアクセス権確認（購入済み？無料？） | 必須 |

### 4-4. チャージプラン

| プラン | 価格 | 付与ポイント | ボーナス |
|:---|:---|:---|:---|
| お試し | 300円 | 300 pt | なし |
| 標準 | 500円 | 520 pt | +20 pt |
| **推奨** | **1,000円** | **1,100 pt** | **+100 pt（1本無料相当）** |

### 4-5. モック決済フロー

```
読者が「1,000円チャージ」ボタンを押す
  ↓
POST /api/wallet/charge { plan: "recommended", mock: true }
  ↓
サーバー側:
  1. Wallet.balance += 1100
  2. PointTransaction 作成 (type: "charge", amount: +1100, metadata: { mock: true })
  3. レスポンス: { balance: 1100 }
  ↓
UI: 「1,100 pt チャージしました！」
```

```
読者が「100 pt で続きを読む」ボタンを押す
  ↓
POST /api/works/:id/purchase
  ↓
サーバー側:
  1. 残高チェック: Wallet.balance >= 100
  2. Wallet.balance -= 100
  3. PointTransaction 作成 (type: "purchase", amount: -100, workId)
  4. WorkPurchase 作成 (userId, workId, points: 100)
  5. 作者の Wallet.balance += 90 (90% 分配)
  6. PointTransaction 作成 for 作者 (type: "earn", amount: +90)
  7. レスポンス: { balance: 1000, purchased: true }
  ↓
UI: ペイウォール解除 → 続きのシーンが表示される
```

### 4-6. 本番移行時の変更点

モック → 本番への移行は `POST /api/wallet/charge` のみ変更:

| | モック | 本番 |
|---|---|---|
| 決済 | ボタン押すだけ | Stripe Checkout Session |
| ポイント加算 | 即座 | Stripe Webhook 受信後 |
| metadata | `{ mock: true }` | `{ stripeSessionId: "..." }` |
| その他の API | **同じ** | **同じ** |

## 5. 縦画面モード設計

### 5-1. 解像度

| モード | 論理幅 | 論理高さ | アスペクト比 | 用途 |
|--------|--------|---------|------------|------|
| landscape | 1280 | 720 | 16:9 | Switch / PC / 横持ちスマホ |
| **portrait** | **720** | **端末に追従** | **≈ 9:19.5** | **スマホ縦持ち（Web 専用）** |

### 5-2. ボトムアンカーUI

```
┌──────────────────┐ ← safe area top
│                  │
│   背景 (cover)    │ ← 可変領域（端末に応じて伸縮）
│   キャラ (center)  │    背景は cover で黒帯なし
│                  │
├──────────────────┤ ← text window top（下端から30%固定）
│  テキストウィンドウ  │ ← 固定高さ（画面の30%、最低280px）
│  「セリフテキスト」  │
│  [選択肢A]        │
│  [選択肢B]        │
└──────────────────┘ ← safe area bottom
```

### 5-3. 実装

```typescript
// packages/core/src/constants/layout.ts に追加
export const PORTRAIT_WIDTH = 720;
export const PORTRAIT_BASE_HEIGHT = 1560; // 基準値（≈ 9:19.5）
export const TEXT_WINDOW_RATIO = 0.3;     // 画面高さの30%
export const TEXT_WINDOW_MIN_H = 280;     // 最低高さ

// レイアウト計算
const screenH = app.renderer.height;
const safeTop = env.safeAreaInsetTop;
const safeBottom = env.safeAreaInsetBottom;
const textWindowH = Math.max(screenH * TEXT_WINDOW_RATIO, TEXT_WINDOW_MIN_H);
const textWindowY = screenH - safeBottom - textWindowH;
const bgAreaH = textWindowY - safeTop; // 背景+キャラの可変領域
```

### 5-4. プロジェクト設定

```json
{
  "screenMode": "portrait",  // "landscape"（デフォルト）or "portrait"
}
```

- 作者がプロジェクト設定で選択
- Switch ビルド時は強制 landscape 変換

### 7-2. 挙動

```

  3. 残りアセットはバックグラウンドでロード
```

## 8. シーン共有設計

### 8-1. UI

プレイ中に画面右上のシェアアイコン → 共有メニュー

### 8-2. 実装

```typescript
async function shareScene(sceneId: number) {
  // Canvas の現在フレームをキャプチャ
  const canvas = app.renderer.extract.canvas(app.stage);
  const blob = await new Promise<Blob>(r => canvas.toBlob(r, "image/png"));
  const file = new File([blob], "scene.png", { type: "image/png" });

  const shareData = {
    title: `${workTitle} - シーン${sceneId}`,
    text: "この作品おすすめ！",
    url: `${location.origin}/play/${workId}?scene=${sceneId}`,
    files: [file],
  };

  if (navigator.canShare?.(shareData)) {
    await navigator.share(shareData);
  } else {
    // フォールバック: Twitter / LINE share URL
    window.open(`https://twitter.com/intent/tweet?text=...&url=...`);
  }
}
```
