## 実装すべき項目の評価

ドキュメントの「今後優先すべきタスク」3件 + 追加候補を評価。

### 1. アセット検索・フィルタ強化（推奨：実装すべき）

**現状**: AssetPanel.tsx に種類別フィルタ（bg/ch/bgm/all）のみ。名前検索なし。
**ユーザー影響**: アセットが50件超えると目視で探すのが困難。
**実装コスト**: 小（テキスト入力 + filter で即実装可能）
**提案**: AssetPanel に検索ボックスを追加。名前・スラッグで部分一致フィルタ。

```typescript
// AssetPanel.tsx に追加
const [search, setSearch] = useState('');
const filtered = assets.filter(a =>
  a.slug?.includes(search) || a.filename?.includes(search)
);
```

### 2. UIレイアウトのプリセット拡充（推奨：プリセット追加で十分）

**現状**: 4種（novel-standard, rpg-classic, message-top, message-center）。
**ビジュアルエディタ**: 大規模すぎる。リリースには不要。
**提案**: プリセットを 8-10 種に増やす。`rpg-battle`, `rpg-menu`, `novel-fullscreen`, `visual-novel-wide` 等。JSON 定義を追加するだけなのでコスト極小。

### 3. 並列イベント（推奨：後回し）

**現状**: イベントは逐次実行。
**ユーザー影響**: タイマーや背景演出の同時実行ができない。
**実装コスト**: 大（OpRunner の並列実行、タイミング制御が必要）。
**提案**: リリース後に検討。現状は `set_var` + `if` の組み合わせで擬似並列可能。

### 4. アセットタグ付け（推奨：後回し）

**現状**: カテゴリ（bg/ch-img）とサブカテゴリ（ファンタジー等）はある。
**ユーザー影響**: 公式アセットは既にカテゴリ分類済み。ユーザーアセットが増えた場合に必要。
**提案**: リリース後、ユーザーフィードバックを見てから実装。
