## エクスポート方式（プラットフォーム別）

| Platform | 方式 | 作者の操作 |
|----------|------|-----------|
| macOS | サーバーで .app 生成 → zip ダウンロード | ボタン1つ |
| Windows | サーバーで .exe 生成 → zip ダウンロード | ボタン1つ |
| Android | サーバーで APK 生成 → ダウンロード | ボタン1つ |
| iOS | Xcode テンプレート + game.pak → 作者が Xcode でビルド | game.pak ダウンロード → Xcode |
| Switch | 自社で審査・ビルド・提出 | エディタから申請 |

- iOS だけ作者に Xcode を要求（Apple の仕組み上不可避）
- エンジンは `libkaedevn.a`（静的ライブラリ）としてバイナリ配布、ソースコード非公開

## ブラウザ版作品販売（マーケットプレイス）

### 手数料

| プラットフォーム | 手数料 |
|----------------|--------|

### 原価

| 項目 | コスト |
|------|--------|
| Stripe 決済手数料 | 3.6% |
| 残り（利益） | 6.4% |

### 戦略的意義

- 10% は最安クラス → 「kaedevn で売れば手数料10%」が作者への強い訴求力
- 作品が増える → プラットフォームに遊び手が集まる → 作者がさらに集まる好循環
- ブラウザ版はエクスポート不要（Web プレビューがそのまま製品になる）→ Free ユーザーでも販売可能
- 販売作品が増えるほど、ストレージ課金・エクスポート課金への転換率も上がる
