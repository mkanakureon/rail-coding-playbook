## 「ツクール臭」問題への対応状況

引き継ぎ資料 C 節で挙げられた3層の問題:

| 層 | 問題 | kaedevn の対応状況 |
|----|------|-------------------|
| ① UI レイヤー | ツクール標準 UI から逃げられない | kaedevn はゼロベース設計のため「ツクール臭」は構造的に発生しない。ただし現状はプリセット選択 or JSON 直編集しかなく、**ビジュアル UI エディタを実装して UX を改善する** |
| ② デフォルト値の重力 | ツクール標準を壊す作業 | **ゼロベース設計**のため不要。GameDb で自由に定義 |
| ③ 演出表現の天井 | プラグイン依存 | **ノベル演出が標準搭載**。16 種のスクリーンフィルタ、タイムライン、カメラ演出 |

**補足: 「ツクール臭」と UI エディタの関係**

kaedevn にはツクールの「デフォルト UI テンプレート」が存在しないため、「みんな同じ見た目」問題はそもそも起きない。UI エディタは「ツクール臭の解消」ではなく、「プレイ画面の UI を楽に作れるようにする UX 改善」。ただしツクール作者にとって「GUI で UI を編集できる」は強力な訴求ポイントになるため、実装する価値は高い。

## UI レイアウトエディタ 実装計画

### 概要

エディタ内でプレイ画面の UI 要素（テキストウィンドウ、ネームボックス、メニュー等）の位置・サイズ・色をドラッグ＆ドロップ + スライダーで編集できる画面。編集結果は PlayLayout JSON として保存。

### 実装方式

1. **新タブ「レイアウト」** を TabNavigation に追加
2. **LayoutEditor.tsx** コンポーネント（新規）:
   - 左: 1280x720 のプレビューキャンバス（PixiJS or Canvas 2D）
   - 右: 選択中の要素のプロパティパネル（x, y, width, height, fontSize, color 等）
   - 要素をクリックで選択、ドラッグで移動
3. **編集対象**: PlayLayout JSON の全要素
   - textWindow: { x, y, width, height, fontSize, bgColor, bgAlpha }
   - nameBox: { x, y, fontSize }
   - quickMenu: { visible, position }
   - その他 RPG UI 要素
4. **保存**: project.data.playLayout に JSON として保存（既存の PUT API で対応可能）

### 実装方針

**1つのブランチで実装する。** こちら（レビュー側）では実装しない。PR が来たらレビュー＆テストを行う。
