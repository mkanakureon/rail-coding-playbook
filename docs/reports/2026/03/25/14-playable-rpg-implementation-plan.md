# 遊べる RPG 実装計画

> Generated by Claude Opus 4.6

現在のエディタプロトタイプ（7タブ、49テスト）を「一応遊べる RPG」にするための実装計画。

---

## 前提

- エディタ機能は完成。ここからはランタイム（ゲーム実行側）の実装
- 目標：「マップを歩いて、NPCに話しかけて、敵と戦って、宝箱を開けて、別のマップに移動できる」
- PixiJS でレンダリング、キーボードで操作

---

## 実装順序（依存関係順）

### F1: ゲーム画面のPixiJS レンダリング基盤

**依存**: なし
**内容**: テストプレイ画面を「テキスト出力」から「PixiJS のゲーム画面」に置き換える

- PixiJS Application の初期化（1280×720、Test Play タブ内）
- レイヤー構成: タイル層 → イベント層 → プレイヤー層 → UI層
- タイルマップの描画（SpatialView の描画ロジックを流用）
- メッセージウィンドウの描画（半透明背景 + テキスト表示）

**成果物**: マップが表示される画面（まだ動かない）

---

### F2: プレイヤー移動 + 衝突判定

**依存**: F1
**内容**: 矢印キーで32pxグリッド単位で移動、壁タイルで止まる

- プレイヤースプライト（色付き四角でOK）
- キー入力 → 移動先のタイル判定 → 壁（tileId=4）なら移動しない
- グリッドスナップ移動（アニメーション付き、4フレームで1タイル移動）
- プレイヤーの向き（上下左右）

**成果物**: マップ上を自由に歩ける

---

### F3: イベントトリガー実行

**依存**: F2, 既存 Runtime
**内容**: マップ上のイベント Entity に到達 → トリガー条件に応じて Runtime を実行

- `interact`: プレイヤーが向いている方向の隣接タイルに決定キー → `executeEvent()`
- `player_touch`: プレイヤーが踏んだタイルにイベントがあれば自動実行
- `auto`: マップロード時に条件を満たしていれば自動実行（1回）
- Runtime の出力（TEXT, WAIT_CLICK, CHOICE, ITEM_ADD 等）をゲーム画面に表示

**UI統合**:
- TEXT → メッセージウィンドウに表示、Z/Enter で次へ
- CHOICE → 選択肢を表示、上下キーで選択、Z/Enter で決定
- WAIT_CLICK → Z/Enter で続行

**成果物**: NPCに話しかけてテキストが出る、アイテムをもらえる

---

### F4: マップ遷移

**依存**: F3
**内容**: MAP_TRANSFER 命令でマップを切り替える

- 複数マップの管理（Map<mapId, { tiles, events }>）
- MAP_TRANSFER 時にフェードアウト → タイル再描画 → プレイヤー配置 → フェードイン
- サンプルデータに2つ目のマップ（ダンジョン）を追加

**成果物**: 階段を踏むと別マップに移動

---

### F5: バトルシステム（コア）

**依存**: F1, 既存 Entity Graph（actor, enemy, skill の Entity）
**内容**: ターン制コマンドバトル

#### 5a: バトル画面

- 専用のバトルシーン（マップ画面から切り替え）
- 上半分: 敵スプライト（色付き四角 + 名前 + HPバー）
- 下半分: コマンドウィンドウ + パーティステータス

#### 5b: バトルフロー

```
バトル開始
  ↓
ターンループ:
  1. プレイヤーのコマンド選択（たたかう / スキル / アイテム / にげる）
  2. 敵のAI行動決定
  3. 素早さ順にアクション実行
  4. ダメージ計算（Entity の formula を使用）
  5. HP更新 → 戦闘不能チェック
  6. 全滅 or 勝利で終了
  ↓
勝利: EXP + Gold 獲得、ドロップアイテム判定
敗北: ゲームオーバー表示
```

#### 5c: ダメージ計算

- 既存の `balanceAnalyzer.simulateDamage()` をランタイムでも使用
- Entity の formula プロパティ（`a.atk * 4 - b.def * 2`）をそのまま評価
- 乱数の揺れ（±10%）

#### 5d: 必要なデータ

Entity Graph から取得：
- **パーティ**: actor Entity（HP, MP, ATK, DEF）
- **敵グループ**: enemy Entity（HP, ATK, DEF, EXP, Gold）
- **スキル**: skill Entity（power, mpCost, element, formula）
- **アイテム**: item Entity（回復量）
- **ドロップ**: enemy → item の drops Relation（rate）

すべて既にサンプルデータに入っている。

#### 5e: バトルの状態管理

```typescript
interface BattleState {
  phase: "command" | "execute" | "result";
  turn: number;
  party: BattleActor[];    // プレイヤーキャラ
  enemies: BattleEnemy[];  // 敵
  log: string[];           // 戦闘ログ
  selectedCommand?: { type: "attack" | "skill" | "item" | "flee"; targetIndex?: number; skillId?: string; itemId?: string };
}

interface BattleActor {
  entityId: string;
  name: string;
  hp: number;
  maxHp: number;
  mp: number;
  maxMp: number;
  atk: number;
  def: number;
  skills: string[];  // skill Entity IDs
}

interface BattleEnemy {
  entityId: string;
  name: string;
  hp: number;
  maxHp: number;
  atk: number;
  def: number;
  exp: number;
  gold: number;
  drops: { itemId: string; rate: number }[];
}
```

---

### F6: バトルとマップの接続

**依存**: F3, F5
**内容**: BATTLE_START 命令 → バトル画面 → 勝利/敗北 → マップに戻る

- Runtime の BATTLE_START 出力を検知 → バトルシーンに遷移
- troopId から enemy Entity を取得 → BattleState を構築
- 勝利 → EXP/Gold をパーティに加算、ドロップアイテムをインベントリに追加
- 敗北 → ゲームオーバー画面
- バトル終了後、マップの Runtime 実行を再開

## 工程まとめ

```
F1 ゲーム画面基盤 ──→ F2 プレイヤー移動 ──→ F3 イベントトリガー ──→ F4 マップ遷移
                                                    │
                                                    ↓
F5 バトルシステム（5a画面→5b フロー→5c ダメージ→5d データ→5e 状態管理）
                                                    │
                                                    ↓
                                              F6 バトル接続
```

| Phase | 内容 | 成果 |
|-------|------|------|
| F1 | PixiJS ゲーム画面 | マップが見える |
| F1+F2 | プレイヤー移動 | マップを歩ける |
| F1+F2+F3 | イベント実行 | NPCと会話できる |
| F1〜F4 | マップ遷移 | 町→ダンジョン→町 |
| F1〜F6 | バトル込み | **一応遊べるRPG** |

## サンプルゲーム（完成時のプレイフロー）

1. 町マップで老人に話しかける → クエスト受諾 + ポーション3個
2. 宝箱を開ける → 鉄の剣を入手
3. 門番に話しかける → 「気をつけろ」
4. 階段を踏む → ダンジョンマップに遷移
5. ダンジョンを歩く → スライムとエンカウント → **バトル**
6. バトルに勝利 → EXP + Gold + ドロップ
7. 奥に進む → ドラゴンとバトル
8. 勝利 → Boss Defeated フラグ ON → 町に戻る
9. 門番に話しかける → 「王国は平和になった！」
