## 問題点：優先度 高

### 1. E2E テストが CI で実行されていない

93 件の E2E テストが `npx playwright test` の手動実行のみ。GitHub Actions (`ci.yml`) では typecheck・lint・ユニットテストだけ。

**影響**: UI の回帰を検知できない。ブロック操作・認証フロー・ゲストモードの破壊に気づけない。

**対応案**:
- CI に Playwright ジョブを追加（`tests/shared/` のみ、`tests/local/` は除外）
- PostgreSQL サービスコンテナ + dev サーバー起動が必要
- 実行時間が長いので、main push 時のみ or 手動トリガー

### 2. API エンドポイントのテスト未作成（7件）

| エンドポイント | 状態 |
|-------------|------|
| `export.ts` | テストなし |
| `gamedb.ts` | E2E のみ（ユニットなし） |
| `maps.ts` | E2E のみ（ユニットなし） |
| `assist.ts` (メインルート) | コンポーネントテストのみ |
| `analytics.ts` (メインルート) | assist 用のみ |
| `thumbnails.ts` | 部分的 |

### 3. シェーダー/ネイティブエンジンのテスト不在

直近で追加された機能:
- 20 個の GLSL フィルターシェーダー移植
- GLRenderer ポストプロセスパイプライン

これらのユニットテストがない。ネイティブエンジンのテスト（7ファイル）は CI でビルド・実行されていない。
