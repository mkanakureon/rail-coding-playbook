## 標準版からの変更点

### 実装済み機能による上方修正要因

| 機能 | 実装状態 | 収益への影響 |
|------|---------|------------|
| **フィルター43種 + 重ね掛け** | 実装済み | 背景画像不足の解消 → カクヨム/なろう作者の参入障壁が大幅低下 |
| **縦画面対応** | 実装済み | モバイルファーストの制作・閲覧 → スマホ流入の質が向上 |
| **動画クリップ自動生成** | 実装済み | TikTok/Shorts/Reels への投稿導線 → 動画流入の実現 |
| **Discord サポート埋め込み** | 実装済み | 作者コミュニティの形成 → 継続率向上 |
| **読了アナリティクス** | 実装済み | 作者が離脱ポイントを改善 → 作品品質向上 → 読者リテンション |
| **フレッシュ・ピックアップ** | 実装済み | 新作が必ず読まれる → 作者の「ゼロ再生」恐怖を解消 |
| **リアクション + チップ** | 実装済み | モデレーション0で読者の反応が作者に届く → 継続モチベーション |
| **ゲストモード** | 実装済み | 登録不要で72時間お試し → コンバージョン率向上 |

### カクヨム・なろう作者の流入チャネル（新規）

**背景画像のみでノベルゲームが作れる** — これがカクヨム・なろうの小説作者にとって決定的。

- 小説作者は「キャラ立ち絵」を用意できない → kaedevn はキャラなし・背景+フィルターだけで成立
- 背景3枚 × フィルター43種 × 重ね掛け = **数百パターンの場面**
- 小説のテキストをそのまま流し込める
- 完成したら動画を自動生成 → TikTok に投稿 → 読者流入

**想定流入規模**: カクヨム月間1,000万PV、なろう月間数億PV の作者層の1%がkaedevnを試す → 月数千人の新規作者候補

## 8. 楽観修正の根拠

### なぜ MAU が標準版より 150,000 多いか

1. **カクヨム・なろう作者チャネル**: 「背景+フィルターだけでノベルゲームが作れる」という訴求が小説作者に刺さる。フィルター43種+重ね掛けは他エンジンにない差別化要素
2. **縦動画の完成度**: 動画生成が実装済み + 投稿導線が明確。「作って→動画にして→SNS に投稿」が1クリック
3. **モバイルファースト**: 縦画面対応済み。スマホだけで制作→閲覧→投稿が完結

### なぜ作者課金が標準版より50万多いか

1. **アクティブ作者 3,500人**（標準版 2,200人）: Discord コミュニティ + フレッシュピックアップ + リアクション/チップで継続率が高い
2. **容量課金率 55%**（標準版 50%）: フィルター重ね掛けで背景バリエーションが増える → より多くのシーンを作る → 容量を使う

### なぜ Switch が月8本か

- 作者母数が 8,000人（標準版 5,500人）に増えることで、クオリティの高い作品が増加
- 読了アナリティクスによる作品改善サイクルが機能

## 9. Claude Code CLI の位置づけ

### 初期（M1〜M3）: 直接的な影響は小さい

- 作者数がまだ少ない（80〜500人）
- CLI を使えるのは開発者寄りの一部の作者のみ
- **初期は公式アセット + フィルター43種 + Discord 画像リクエストで十分**
- ゲストモードで登録不要 → 背景を選ぶ → フィルターで雰囲気を変える → 公開。CLI は不要

### 中期（M4〜M6）: 素材セルフサービスのツールとして需要発生

- 作者が 2,000〜3,800人に増え、運営が手動で画像リクエストに応えきれなくなる
- 「自分で背景を AI 生成したい」作者が出てくる
- **Claude Code CLI ガイド**（公式ドキュメント）の需要が生まれる
- CLI で生成した画像をアップロード → フィルターで調整 → 公開の導線

### 後期（M7〜M12）: 作者の自立と作品品質の底上げ

- CLI を使いこなす作者が増え、独自の背景素材を量産
- 公式アセットに頼らない高品質作品が増加 → Switch 候補の質が向上
- CLI 作者は容量を多く使う → **容量課金率の押し上げ要因**

### 収益予測への反映

| フェーズ | CLI の影響 | 反映箇所 |
|---------|-----------|---------|
| M1〜M3 | ほぼなし | — |
| M4〜M6 | 作者継続率 +5%程度 | アクティブ作者数に間接反映済み |
| M7〜M12 | 容量課金率 +3%、Switch 候補品質向上 | 容量課金 55%、Switch 月8本に反映済み |

## 12. 最終結論

実装済み機能を楽観的に評価すると:

- **M12 MAU: 550,000**（標準版+37%）
- **M12 登録作者: 8,000人**（標準版+45%）
- **M12 営業利益: 232万円**（標準版+74%）
- **年間営業利益: 1,534万円**（標準版+92%）
- **M1〜M6 の人件費: 0円**（自分1人で運営）

上振れの主因は:
1. カクヨム/なろう作者の「背景+フィルターだけで作れる」参入
2. 動画投稿→SNS流入の実装完了
3. Discord/アナリティクス/リアクションによる作者継続率向上
4. Claude Code CLI による素材セルフサービス（M4以降の作者自立）

### フェーズ別の成長ドライバー

| フェーズ | 主な成長ドライバー |
|---------|-----------------|
| M1〜M3 | ゲストモード + 公式アセット + フィルター → 「30分で作品公開」体験 |
| M4〜M6 | 動画投稿→SNS流入 + Discord コミュニティ → 作者の相互刺激 |
| M7〜M9 | CLI セルフサービス + 読了アナリティクス → 作品品質の自律改善 |
| M10〜M12 | Switch 候補の蓄積 + 容量課金の安定 → 収益構造の確立 |

**作者課金で守り、動画+小説作者で伸ばし、CLI で自立させ、Switch で回収する。**
