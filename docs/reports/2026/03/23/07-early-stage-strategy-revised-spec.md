# 初期成長戦略（改訂版）：0〜100 MAU

> Generated by Claude Opus 4.6

## 改訂方針

前版（06-early-stage-strategy-spec.md）から以下を変更:

| 変更 | 理由 |
|------|------|
| AI アシスタント → **プラットフォームでは実装しない** | 生成 AI API のコストとメンテナンス負荷。作者は自身の Claude Code / Gemini CLI で対応可能 |
| 共同制作モード → **実装しない** | 権限管理の複雑さに対して100人規模では需要が薄い |
| エクスポート機能 → **実装しない** | ブラウザ体験に集中。作って→動画にして→SNS投稿の導線を優先 |
| 画像リクエストフォーム → **新規追加** | 作者100人までは専用フォームで背景・キャラのリクエストを受付。運営が手動で対応 |

**コアコンセプト**: ブラウザで作って、動画にして、SNS に投稿する。

## 作者数に応じた注力ポイント

| 作者数 | フェーズ | 運営の役割 | 開発の優先度 |
|--------|---------|-----------|------------|
| **1〜10人** | 密着期 | 全員と直接会話し、要望を即実装 | エディタのバグ修正、ゲストモード、フィルター |
| **10〜50人** | 改善期 | 素材リクエストに手動対応、テンプレート整備 | チュートリアル、動画投稿導線、リクエストフォーム |
| **50〜100人** | 循環期 | 作者同士が反応し合う仕組み | タイムライン・通知・ランキング・アナリティクス |

## コアフロー

```
作者が kaedevn にアクセス
  → ゲストモードで即開始（登録不要）
  → テンプレートを選ぶ or 公式アセットから背景を選ぶ
  → フィルターで雰囲気を変える（朝・夜・雨・ホラー...）
  → セリフを入力
  → プレビューで確認
  → 公開ボタン
  → 動画を自動生成
  → 「Xに投稿」ボタンでSNSシェア
  → 反応が来る → 続きを書く意欲が湧く
```

**AI アシスタントはこのフローに入らない。** 素材が足りなければリクエストフォームで運営に頼む or 自分の Claude Code で生成する。
