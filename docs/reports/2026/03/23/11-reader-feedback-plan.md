## 方針

テキストコメントは作らない。モデレーションコスト 0 の「非テキスト」リアクションのみ。

- **シーン・リアクション**: プレイ中の特定ページで ❤️ 😂 😭 ⚡️ をタップ
- **読了チップ**: 読み終わった後に「最高！」「天才」等の定型タグを送信
- **管理コスト 0**: 言葉ではないため検閲不要

## データモデル

```prisma
model PageReaction {
  id        String @id @db.VarChar(31)
  workId    String @map("work_id") @db.VarChar(31)
  pageIndex Int    @map("page_index")
  emoji     String @db.VarChar(10)  // "heart", "laugh", "cry", "shock"
  sessionId String @map("session_id") @db.VarChar(50)  // 重複防止用
  createdAt BigInt @map("created_at")

  work      Work   @relation(fields: [workId], references: [id], onDelete: Cascade)

  @@unique([workId, pageIndex, sessionId, emoji])
  @@index([workId, pageIndex])
  @@map("page_reactions")
}

model ReadingChip {
  id        String @id @db.VarChar(31)
  workId    String @map("work_id") @db.VarChar(31)
  chipType  String @map("chip_type") @db.VarChar(30)  // "awesome", "genius", "crying", "addicted"
  sessionId String @map("session_id") @db.VarChar(50)
  createdAt BigInt @map("created_at")

  work      Work   @relation(fields: [workId], references: [id], onDelete: Cascade)

  @@unique([workId, sessionId])  // 1セッション1チップ
  @@index([workId])
  @@map("reading_chips")
}
```
