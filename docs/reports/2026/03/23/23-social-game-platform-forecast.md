## 概要

kaedevn の2Dゲームエンジン・インフラ・公式アセットを活用し、外部のゲーム企画者/会社に運営型ゲームアプリのプラットフォームを提供する事業。kaedevn はインフラ運営に徹し、ユーザー対応はゲーム会社が行う。

### 収益配分

```
プレイヤー課金 100%
  ├─ App Store / Google Play: 30%（ストア手数料）
  └─ 残り 70% を分配:
       ├─ ゲーム会社: 70%（= 総額の 49%）
       └─ kaedevn: 30%（= 総額の 21%）
```

| 配分先 | 取り分 | 根拠 |
|--------|-------|------|
| ストア | 30% | Apple/Google 固定 |
| ゲーム会社 | 49% | 企画・IP・マーケ・運営を担当。旧Mobage パートナーと同等 |
| **kaedevn** | **21%** | エンジン+インフラ+素材+決済。旧Mobage の20-30%より低めで競争力を持つ |

## 公式アセットが参入障壁を破壊する

### 通常の運営型ゲームアプリ開発コスト vs kaedevn 利用

| 項目 | 自前で用意 | kaedevn 利用 |
|------|-----------|------------|
| ゲームエンジン | Unity学習 + 開発（3-6ヶ月） | **不要（提供）** |
| サーバー構築 | 自前構築（月100-200万） | **不要（提供）** |
| ガチャ/課金システム | 自作（1-2ヶ月） | **不要（提供）** |
| キャラ素材 | 発注（500-1,000万） | **公式アセット＋α** |
| マップ素材 | 発注（200-500万） | **公式アセット＋α** |

**参入コストが1/10以下になる。** これなら個人や2-3人のチームでも運営型が出せる。

### アセット投資（22版）との相乗効果

M6 から始めるアセット発注が、運営型ゲームアプリ事業のパートナー獲得を加速する。

```
M6:   アセット発注開始（月30万〜）
        ↓ ドット絵、マップ、キャラチップが蓄積
M9:   ツクール型リリース → 作者がアセットを使う
        ↓ アセットライブラリが数千点に成長
M12:  運営型ゲームアプリ基盤リリース
        ↓ 同じアセットで運営型も作れる
M15:  最初のパートナー
        ↓ 「公式アセット＋独自キャラ数体」で運営型を出す
M24:  アセット累計 ~7,000点
        ↓ 競合が追いつくのに2年+2,400万必要（参入障壁成立）
```

**アセット投資は3事業全てに効く:**

| 事業 | アセットの使われ方 |
|------|-----------------|
| ノベルゲーム | 背景 + フィルター |
| ツクール型 | タイルセット + キャラチップ + マップ |
| **運営型ゲームアプリ** | **上記全部 + ガチャ用キャラ + ボス素材 + アイテムアイコン** |

アセットライブラリが厚くなるほど、パートナーの「自分で用意するもの」が減る。M24 以降は独自キャラすら公式アセットの組み合わせでカバーできる規模になる。

## 責任分界点

```
kaedevn（プラットフォーム）        ゲーム会社（パートナー）
  エンジン提供                      ゲーム企画
  サーバー運用                      プレイヤー集客（広告・SNS・ストアASO）
  課金/ガチャ基盤                   App Store / Google Play 掲載・審査
  アセット提供                      ユーザーサポート
  アナリティクス                    イベント運営・マーケティング
```

**プレイヤーは App Store / Google Play からダウンロードする。kaedevn のノベルゲーム/ツクール事業とは完全に独立した集客経路。** kaedevn がプレイヤーを集める必要はない。ゲーム会社がプレイヤーを連れてくる。kaedevn はインフラを動かすだけ。プレイヤーが来なければサーバーコストも ¥0（従量課金）。

## リスクと対策

### kaedevn 側のリスク（プラットフォーム運営）

| リスク | 確率 | 影響 | 対策 |
|--------|------|------|------|
| パートナー（ゲーム会社）が見つからない | 中 | 大 | 初期はコミュニティ内からスタート。開発費150万のみで月額固定費ゼロなので待てる |
| パートナーの運営放棄 | 中 | 小 | サービス終了手順を契約に含める。kaedevn 側の損失はほぼゼロ |
| 課金トラブル（返金、不正） | 中 | 中 | IAP レシート検証、ストア側の保護に依存 |
| ガチャ規制強化 | 低 | 大 | 天井・確率表示を最初から実装 |

### ゲーム会社側のリスク（kaedevn は無関係）

| リスク | kaedevn への影響 | 理由 |
|--------|----------------|------|
| プレイヤーが集まらない | **なし** | 集客はゲーム会社の責任。サーバーコストは従量課金で ¥0 |
| ゲームがコケる（83%が3年で失敗） | **最小** | kaedevn の損失は1本あたり月5-28万のインフラ費のみ。ゲームが終了すれば ¥0 に戻る |
| マーケティング費用の回収失敗 | **なし** | マーケ費はゲーム会社負担 |
| ストア審査でリジェクト | **なし** | ストア対応はゲーム会社の責任 |

### kaedevn 固有の強み（リスク軽減要因）

| 強み | 効果 |
|------|------|
| インフラコストが従量課金 | パートナーゼロ or ゲーム失敗でもコスト ¥0。損失は開発費150万のみ |
| 公式アセットが使える | パートナーの初期投資を大幅削減 |
| 既にエンジンが動いている | プラットフォーム開発が3ヶ月で済む |
| ノベルゲーム/ツクール作者がいる | 初期パートナー候補のプール |
| 開発成果がツクール型に流用可能 | パートナーが来なくてもガチャ・ランキング等がツクール型の機能になる |

|------|-------------|-------------|
| サーバー費用 | ゲームが動いた分だけ | **¥0（誰も使っていない）** |
| 開発費（3ヶ月） | 150万 | **150万（同じ）** |
| 月額維持費 | ゲーム数に比例 | **¥0** |

**パートナーが見つからなくても、損失は開発費150万のみ。月額の固定費はゼロ。**

さらに、開発した機能は他事業に流用できるため、150万も実質的には無駄にならない。

### 開発成果の流用先

| 開発した機能 | 流用先 | 効果 |
|------------|--------|------|
| ガチャシステム | ツクール型エディタ | 作者が自作ゲームにガチャを組み込める |
| ランキング/PvP | ツクール型エディタ | 対戦・ランキング機能付きゲームが作れる |
| ログインボーナス | ノベルゲーム本体 | 読者のリテンション向上 |
| イベントスケジューラ | ノベルゲーム本体 | 期間限定イベント・キャンペーン |
| 課金/IAP 基盤 | 全事業共通 | アプリ内課金の基盤として利用 |
| プッシュ通知 | 全事業共通 | 新作通知、イベント告知 |

**つまり「パートナーが来れば運営型ゲームアプリ事業として大きなリターン、来なくてもツクール型の機能強化として回収」という構造。どちらに転んでも損失がない。**
