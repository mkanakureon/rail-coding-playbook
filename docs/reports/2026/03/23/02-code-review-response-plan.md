## 中優先度（今回対応）

### M-1. resolveAssetUrl と storage.getUrl の役割統合（05-CODE_REVIEW）

**指摘**: 2つの URL 解決関数が混在して混乱の原因
**対応**: サムネイル・動画など storage 経由のアセットは `storage.getUrl()` に統一。`resolveAssetUrl` は DB に保存された正規パス用に限定
**ファイル**: 使用箇所を grep して統一

### M-2. タイトル画像の setTimeout → イベント駆動化（10-CODE_REVIEW）

**指摘**: 固定 `setTimeout(6000)` で描画完了を待つのは不安定
**対応**: `postMessage` でエンジンの描画完了を待つ。タイムアウトは保険として残す
**ファイル**: `apps/next/app/(private)/projects/[id]/page.tsx`

### M-3. window.open 失敗時のリスナー解除漏れ（10-CODE_REVIEW）

**指摘**: メッセージリスナーが解除されないケース
**対応**: `handleAutoCapture` / `handleTakeCapture` のリスナーをクリーンアップ
**ファイル**: `apps/next/app/(private)/projects/[id]/page.tsx`

### M-4. 動画生成の多重実行防止強化（08-CODE_REVIEW）

**指摘**: ブラウザ閉鎖時のポーリング喪失
**対応**: `useEffect` のクリーンアップで `clearInterval` を確実に実行
**ファイル**: `apps/next/app/(private)/projects/[id]/page.tsx`

### M-5. フィルター Enum 一元管理（02-CODE_REVIEW）

**指摘**: フィルター種別が editor / web / hono に散在
**対応**: `packages/core/src/types/FilterType.ts` にフィルター種別を定義し、各パッケージから参照
