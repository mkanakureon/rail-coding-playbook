## 3. プロダクト面の評価: 未知数

100万MAUに届くかは技術ではなくプロダクト・マーケティングの問題。

### 解決済みの課題

| 課題 | 状態 |
|------|------|
| 作品を作るツールがあるか | エディタ実装済み |
| 作品を公開できるか | API + プレビュー稼働中 |
| Switch に出せる設計か | 抽象化済み |
| スケールできるインフラか | 計画策定済み |

### 未検証の課題

| 課題 | リスク | 検証方法 |
|------|-------|---------|
| 最初の100人の作者が作品を公開するか | **高** | 早期アクセスで招待、作品作成までサポート |
| 読者が「ここで読む理由」を感じるか | **高** | 独占コンテンツ、UI/UX の差別化 |
| TikTok/Shorts のバイラルが起きるか | **中** | 自動動画生成機能 → 投稿 → 流入の導線 |
| Switch 出版が実際に収益化するか | **中** | 1作品目の出版・売上実績 |
| 作者の継続率（作って終わりにならないか） | **高** | 読者リアクション、アナリティクス、Discord コミュニティ |
| 課金転換率が計画通りか（作者45%、読者5%） | **中** | 無料→有料の導線設計、価格テスト |

### 成長シナリオの依存関係

```
作者が作品を作る
  → 読者が読む
    → リアクションが作者に届く
      → 作者が次の作品を作る
        → コンテンツが増える
          → SEO/SNS で新規読者が来る
            → このループが回れば MAU は伸びる
```

**このループの最初の1回転を起こせるかどうかが全て。**

## 4. 100万MAU運用経験の市場価値

### インフラコスト削減の知見は再現可能

itch.io、NomadList、37signals の事例は公開情報。インフラを安くする方法自体は誰でも真似できる。

### 再現が難しい＝価値が高い部分

| 要素 | 再現可能性 | 理由 |
|------|-----------|------|
| インフラを安くする | 誰でもできる | 記事を読めばわかる |
| 100万MAUまで育てる | **難しい** | プロダクト・マーケティング・継続の問題 |
| 障害を1人で捌き続ける | **経験でしか得られない** | 記事には書かれない判断の連続 |
| AI活用の運用フロー構築 | **事例がほぼない** | 2026年時点で確立した方法論がない |

### 具体的な市場価値

| 項目 | 価値 |
|------|------|
| SRE/インフラ採用市場 | 100万MAU運用経験者は年収800-1500万が相場 |
| 技術顧問・副業 | 月30-80万で「同規模にしたい」の相談需要 |
| 登壇・発信 | 「1人+AIで100万MAU運用した話」は技術カンファレンスで確実に通る |
| AI活用の実績 | 「生成AIで運用コストを1/5にした」は2026年で最も注目される話題 |
| 次の起業の信用 | 「前回100万MAUやりました」で人も資金も集まる |

## 5. 結論

### 技術的には100万MAUに耐えられる

- コンパイラ・エンジン・エディタ・API が実装済み
- CI/CD が自動化済み
- スケーリング計画が策定済み（月¥27万で100万MAU）
- Kubernetes・シャーディング・マルチリージョンは不要（実例で証明済み）

### 100万MAUに届くかはプロダクト次第

- 箱はできている
- 中身（ユーザー）を入れるフェーズ
- インフラで死ぬことはない
- 死ぬとしたら「誰も使わなかった」

### 今やるべきこと

| 優先度 | やること | 理由 |
|--------|---------|------|
| **1** | 最初の作者10人に作品を作ってもらう | 成長ループの最初の1回転 |
| **2** | アラート通知を設定する（30分） | 落ちてることに気づけない状態は解消すべき |
| **3** | 読者が来る導線を作る | 作品があっても読者が来なければループが回らない |
| **4** | 残りのインフラ整備 | MAU が伸び始めてからで間に合う |
