# プレスリリース用デモ動画撮影・YouTube配信テスト

> Generated by Claude Opus 4.6

## 背景

kaedevn のプレスリリースに添付するデモ動画を Playwright E2E テストで自動撮影する。また、YouTube Live 配信の接続テストと、配信中にデモを実行する運用フローを確立する。

## 方針

1. Playwright の `video: 'on'` + `headless: false` で画面操作を録画
2. OBS WebSocket 経由で配信制御・テキストオーバーレイ表示
3. 成功パターンを再利用し、新しいデモは既存スクリプトをコピーして拡張

## デバッグで判明した問題と修正

### エディタプレビューのバグ2件

1. **タイトル画面が残留**: `ksc-demo.ts` の `previewScript` メッセージハンドラで `titleScreen.hide()` が呼ばれていなかった
2. **テキスト未表示**: `buildSnapshotScript` がテキストブロックを処理していなかった。`lastText` 変数で追跡し、スクリプト末尾に出力するよう修正

### Phase 8 キャラ編集の入力セレクタ

- `input[type="number"][step="0.01"][max="5"]` → `SidebarInspector` の `SliderRow` が描画する `input[type="number"]` に一致するはずだが、ブロック選択が不完全で TransformSliders が表示されなかった
- 修正: `label:has-text("位置・スケール")` の存在確認 + 再クリックロジック追加

### Playwright 録画でプレビューが映らない

- `context.waitForEvent('page')` で新タブにプレビューが開くが、Playwright はタブごとに別動画を生成
- 修正: 同一ページで `page.goto(previewUrl)` に遷移する方式に変更

### 背景画像選択の安定性（未解決）

- `bg-showcase.spec.ts` でアセット選択モーダルの「公式」タブクリックが不安定
- 原因: `button:has-text("公式")` がモーダル外要素にマッチ、または初期モーダルがブロック
- 対策: press-demo の成功パターンをそのままコピーして使うべき（新規作成しない）

## 教訓

1. **動いてるコードを再利用する**: 新しいテストは成功済みスクリプトをコピーしてから変更する。ゼロから書き直さない
2. **セレクタはモーダル内に限定**: `page.locator()` ではなく `modal.locator()` でスコープを絞る
3. **`--window-position` は不安定**: Playwright の `launchOptions.args` でモニター指定するより、OS 側で制御した方が確実
4. **配信中は実況モードON**: テキスト VTuber スタイルで作業の様子をリアルタイム解説
