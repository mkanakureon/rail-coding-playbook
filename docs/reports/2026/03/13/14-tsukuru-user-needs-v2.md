# ツクールユーザー 不満点 → kaedevn 対応表

**作成日**: 2026-03-13
**用途**: ツクール系ユーザーの不満に対し、kaedevn の既存機能と新規開発（ツクール型エディタ）で何が解決できるかを明確にする

### 凡例

| マーク | 意味 |
|--------|------|
| ✅ 既存 | **kaedevn に実装済み**。今すぐ訴求できる |
| 🆕 ツクール | **ツクール型エディタで新規対応**。実装中 or 計画中 |
| 🔄 ロードマップ | 将来対応予定（ツクール型エディタとは別） |

---

## 1. 環境・動作

| ツクールの不満 | kaedevn の対応 | 区分 |
|--------------|--------------|:----:|
| Windows 専用、Mac で使えない | ブラウザ動作、OS不問（Windows / Mac / Linux / iPad） | ✅ 既存 |
| 動作が重い、起動が遅い | ブラウザベースで軽快、インストール不要 | ✅ 既存 |
| バージョン管理できない（ファイルが壊れる） | JSON形式でgit管理可能、差分が人間に読める | ✅ 既存 |
| クラウド保存できない | APIサーバー保存、どこからでもアクセス可能 | ✅ 既存 |
| アップデートで既存プロジェクトが壊れる | スキーマバージョン管理あり、後方互換性を保持 | ✅ 既存 |
| プロジェクトファイルが肥大化する | 構造化JSONで軽量、アセットは外部ストレージ管理 | ✅ 既存 |

→ **全6項目が既存機能で対応済み**

---

## 2. Switch / マルチプラットフォーム

| ツクールの不満 | kaedevn の対応 | 区分 |
|--------------|--------------|:----:|
| Switch販売のハードルが異常に高い | SDL2ネイティブでSwitch出力が目標、同じJSONから出力 | 🔄 ロードマップ |
| スマホ対応が難しい・別途有料 | ブラウザ版でスマホ対応済み、追加費用なし | ✅ 既存 |
| PC / スマホ / コンソールで別々の作業が発生 | 同一JSONからWeb / Switch / Android を出力 | 🔄 ロードマップ |
| iOS に出せない | ブラウザ版でiOS Safari 対応済み | ✅ 既存 |
| ゲームのファイルサイズが大きくなりすぎる | 必要なアセットのみバンドル可能 | ✅ 既存 |

→ **既存3 / ロードマップ2**。Switch 出力は SDL2 ランタイム完成待ち

---

## 3. UI・見た目のカスタマイズ

| ツクールの不満 | kaedevn の対応 | 区分 |
|--------------|--------------|:----:|
| UIの見た目をスキン以外に変えられない | ツクール型エディタでレイアウトをビジュアル編集 | 🆕 ツクール |
| メッセージウィンドウの位置・サイズが固定 | PlayLayout JSON で X/Y/W/H を自由に設定 | 🆕 ツクール |
| ボタンの配置を変えられない | クイックメニューの位置・表示ボタンを選択可能 | 🆕 ツクール |
| フォントの変更が面倒 | options フィールドでフォントサイズ・色を指定 | 🆕 ツクール |
| ノベル標準・RPG表示を切り替えたい | プリセット切替（ノベル標準/RPG寄り/メッセージ上/中央） | 🆕 ツクール |
| NVLモード（全画面テキスト）がない | メッセージ中央プリセットで対応 | 🆕 ツクール |

→ **全6項目がツクール型エディタの新規機能**。これがツクール型エディタの存在意義

|--------------|--------------|:----:|

| 条件分岐をネストするとすぐ破綻する | ページ単位の構造で整理しやすい | ✅ 既存 |

| デバッグしにくい | ブラウザの devtools + エンジンのデバッグモード | ✅ 既存 |

→ **全5項目が既存機能で対応済み**

| 背景素材を探すのが大変 | セマンティック検索（ベクトル埋め込み）でアセット発見 | ✅ 既存 |

→ **全5項目が既存機能で対応済み**

## 6. AI との相性

| ツクールの不満 | kaedevn の対応 | 区分 |
|--------------|--------------|:----:|
| AIにスクリプトを書かせても使えない形式が多い | KNFはAI生成最適設計（JS系構文、JSON出力） | ✅ 既存 |
| シナリオをAIで量産しにくい | 4段階パイプライン（章→話→テキスト→エディタJSON） | ✅ 既存 |
| エディタがGUI操作前提でAIが触れない | JSON / API / CLIベースでAIがそのまま操作可能 | ✅ 既存 |
| AIが生成したテキストをゲームに入れる作業が手間 | パイプラインで自動コンパイル・インポート | ✅ 既存 |
| AIと対話しながら作りたい | Claude Code CLI で会話→即プロジェクト反映 | 🆕 ツクール |
| UIレイアウトもAIに任せたい | PlayLayout JSON をAIが直接生成・編集 | 🆕 ツクール |

→ **既存4 / ツクール新規2**。Claude Code CLI 連携が新たな差別化

## 優先度まとめ

ツクールユーザーが最初に感じる不満の重さ順：

| 順位 | 不満 | 対応 | 区分 |
|:---:|------|------|:----:|
| 1 | Switch に出したい | SDL2 ランタイム | 🔄 ロードマップ |
| 2 | UIを自由に変えたい | ツクール型エディタ | 🆕 ツクール |
| 3 | Windows以外で使いたい | ブラウザ対応 | ✅ **既存** |
| 4 | AIと一緒に作りたい | パイプライン + Claude Code CLI | ✅ 既存 + 🆕 ツクール |
| 5 | 販売まで一箇所で完結 | マーケットプレイス | 🔄 ロードマップ |
| 6 | 公式素材を無料で使いたい | 600点無料 | ✅ **既存** |
| 7 | スマホで遊んでもらいたい | ブラウザ版 | ✅ **既存** |

**既存だけで訴求できるもの**: 3・6・7（環境・アセット・スマホ）
**ツクール型エディタで訴求力が跳ねるもの**: 2・4（UI自由度・AI連携）
**リリース後の課題**: 1・5（Switch・販売）
