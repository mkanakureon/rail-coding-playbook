## スマホ特化

### 現状: 80%（エディタ）/ 70%（エンジン）

**実装済み:**
- モバイル判定（useMediaQuery 640px/1024px）でレイアウト自動切替

- キーボード出現時のビューポート自動調整

- ブロックのドラッグ並び替え未対応（↑↓ボタンのみ）

## 動画あり

### 現状: 2層ある

**層 1: ゲームの外（完成）**
- Playwright でゲーム画面を MP4 化（generate-clip.ts）
- YouTube アップロード CLI（OAuth 2.0、API v3）
- OBS WebSocket 制御（配信・録画）
- MP4 → PNG → 透過 → WebP パイプライン（Midjourney 動画からの立ち絵生成）

**層 2: ゲームの中（未実装）**
- Op 型に VIDEO/MOVIE なし
- タイムラインに動画トラックなし
- ゲーム内での動画再生ができない

### A になるか

層 1 だけでは A にならない。YouTube アップロードや OBS 連携はワークフロー効率化であり、ユーザーに見える差別化ではない。

ただし「動画書き出し」は A の増幅装置。

| レベル | 内容 | A になるか |
|--------|------|-----------|
| 現状（書き出し + YouTube） | 開発者ワークフロー | ならない |
| ゲーム内動画再生（VIDEO Op 追加） | OP ムービー・ED 演出 | B+ |
| ワンクリック SNS 動画書き出し | エディタ UI に「TikTok 書き出し」ボタン | A- |
| ユーザーが自作ゲームをショート動画で共有 | 投稿 → 閲覧 → 「作りたい」→ 新規登録 | **A（バイラル装置）** |
