## テスト 1: PWA manifest（packages/web）

## テスト 3: アセットグリッド モバイル最適化

### 確認環境
- Chrome DevTools > Toggle device toolbar（375px 幅）
- エディタ: `http://localhost:5176/projects/editor/{projectId}`

### 手順
1. モバイルビューポート（375px）でエディタを開く
2. 背景ブロックの「アセット選択」ボタンをタップ
3. 3 つのタブ（プロジェクト / 公式 / マイライブラリ）をそれぞれ確認

### 確認ポイント — プロジェクトタブ
- [ ] グリッドが **3 列** で表示される（以前は 2 列）
- [ ] サムネイル下のテキストが **1 行のみ**（ファイル名）、slug 行は非表示
- [ ] テキスト部分のパディングが小さい（px-1 py-0.5）
- [ ] タップで即座にアセットが選択される（モバイル動作）

### 確認ポイント — 公式タブ
- [ ] グリッドが **3 列** で表示される
- [ ] サムネイル下のテキストが **1 行のみ**（displayName）、description 行は非表示
- [ ] 検索バー・カテゴリフィルタが正常に動作する
- [ ] AI 検索トグルが正常に動作する
- [ ] タップで即座に公式アセットが追加される

### 確認ポイント — マイライブラリタブ
- [ ] グリッドが **3 列** で表示される
- [ ] サムネイル下のテキストが **1 行のみ**（ファイル名）、category 行は非表示

### 確認ポイント — デスクトップ（回帰テスト）
- [ ] デスクトップ幅（1280px）では従来通り 2〜4 列表示
- [ ] テキストは 2 行表示（名前 + slug/description/category）
- [ ] フッターの「選択」「キャンセル」ボタンが表示される

## テスト 4: SF ジャンル

### 確認環境
- ターミナル（CLI）

### 手順
```bash
# config 一覧に scifi が表示されることを確認
npx tsx scripts/cli/project/init-project.ts --list

# SF プロジェクトを作成（アセット画像未登録のため BG/CH は空）
npx tsx scripts/cli/project/init-project.ts --config scifi
```

### 確認ポイント
- [ ] `--list` に `scifi` が表示される
- [ ] プロジェクト作成が成功する（タイトルに「SF」が含まれる）
- [ ] `bgOfficialIds` が空のため BG アセットは 0 件（エラーにならない）
- [ ] キャラクターの `officialId` が空のため CH は 0 件（エラーにならない）
- [ ] chapterTitles が SF テーマ（「目覚めの航路」等）

### ジャンルルール確認
```bash
# API 起動後に確認
curl -s -X POST http://localhost:8080/mcp \
  -H "Content-Type: application/json" \
  -d '{"method":"tools/call","id":1,"params":{"name":"search_docs","arguments":{"query":"SF 宇宙","topK":3}}}' | jq .
```
- [ ] SF キーワード（SF, 宇宙, 未来）でジャンルルールがマッチする
