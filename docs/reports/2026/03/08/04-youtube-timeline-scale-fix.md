## 説明文案

```
AI（Claude Opus 4.6）がビジュアルノベルエンジン「kaedevn」のタイムラインスケール問題を修正し、E2E テストで動作確認する様子を録画しました。

■ 修正した問題
タイムラインブロックがあるプロジェクトで、プレイ画面のキャラクターの頭が切れて表示される問題。
エディタのプレビューでは正常に表示されるが、プレイ画面のみで再現。

■ 根本原因
WebOpHandler.ts の applyTimelineResult() で、タイムラインの scale 値を PixiJS スプライトに絶対値として上書きしていた。
- キャラの初期スケール: VIRTUAL_HEIGHT / texture.height ≈ 0.703
- タイムラインで scale: 1.0 を適用 → スケールが 0.703 → 1.0 に上書き → 拡大されて頭が切れる

エディタプレビューは CSS transform（相対乗算）なので正常だが、プレイ画面は PixiJS sprite.scale.set()（絶対値上書き）なのでずれていた。

■ 修正内容
- spriteBaseScales Map を追加し、スプライト作成時の初期スケールを記録
- applyTimelineResult で baseScale * props.scale に変更（乗算方式）
- 既存の spriteBasePositions（位置のオフセット計算）と同じパターン

■ テスト内容
1. API でプロジェクトを新規作成（ファンタジーアセット付き）
2. タイムラインブロックをAPI経由で追加（キャラ演出 + 背景フェードイン、3秒）
3. マイページからプロジェクトを選択
4. エディタでタイムラインブロックを選択
5. シークバー操作（0% → 50% → 100%）でプレビュー確認
6. 実行ボタンからプレイ画面を起動 → シナリオ完走

■ 教訓
- 原因が特定できていない問題に対して、推測でソースコードを変更してはいけない
- ユーザーの「TLを削除したら正常になった」の一言で即原因特定できた
- 既存パターン（spriteBasePositions）を確認すれば最初から正しい修正ができた

■ 技術スタック
- エンジン: kaedevn（PixiJS / WebGL ベースのビジュアルノベルエンジン）
- エディタ: React + Zustand（Vite, port 5176）
- プレビュー: PixiJS OpRunner（Vite, port 5175）
- API: Hono + Prisma + PostgreSQL（port 8080）
```
