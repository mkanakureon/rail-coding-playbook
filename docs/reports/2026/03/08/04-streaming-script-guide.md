# 配信台本ガイド

## 台本の効果

台本（アジェンダ）を用意すると発言量が増える。

- 沈黙が減る（「次何話そう」の思考時間がなくなる）
- 話題の抜け漏れが減る
- 構成が安定する（脱線しても戻れる）

## 台本の粒度

| 粒度 | 効果 | リスク |
|------|------|--------|
| キーワードだけ | 自然な語り、柔軟 | 慣れないと沈黙が発生 |
| 箇条書き（要点） | バランスが良い、おすすめ | — |
| 全文台本 | 発言量最大 | 棒読みになりやすい、ライブ感が薄れる |

## 開発配信での推奨

開発配信では**箇条書きレベル**が最適。

### テンプレート

```
- 今日やること: （メインタスク）
- 背景: （なぜやるか、前回からの流れ）
- デモ: （修正前→修正後、動作確認）
- まとめ: （成果、次回の予定）
```

### 作業をそのまま配信する場合

台本なしでも narrate モード（実況モード）で Claude が作業内容を逐次解説するため、最低限の発言は自動的に発生する。

追加で意識すると良いこと:

- 配信開始時に「今日やること」を一言で宣言する
- 作業の区切りで「ここまでで○○が完了」と振り返る
- 詰まったときは「今こういう問題が起きている」と状況を言語化する
- 終了時に「今日の成果」と「次回の予定」をまとめる
