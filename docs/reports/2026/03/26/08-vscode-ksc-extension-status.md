## 3. LSP 実装時の方針メモ

### 活用できる既存コード

コンパイラ（`packages/compiler/src/`）がそのまま LSP のバックエンドになる：

| コンパイラモジュール | LSP での用途 |
|-------------------|------------|
| `tokenizer/Tokenizer.ts` | トークン分割 → シンタックスエラー検出 |
| `parser/Parser.ts` | AST 構築 → 構造的なエラー検出 |
| `registry/commandRegistry.ts` | 58+ コマンド定義 → 補完候補・ホバー情報 |
| `tokenizer/lineClassifier.ts` | 行分類 → セマンティックトークン |
| エラー型 (`UNKNOWN_COMMAND`, `INVALID_ARGS` 等) | → Diagnostic メッセージ |

### 推奨スタック

- `vscode-languageserver` + `vscode-languageclient`（公式 LSP ライブラリ）
- コンパイラを直接 `import` して使う（別プロセスで動かす）

### 最小実装スコープ（やるとき）

1. LSP サーバー作成（`packages/vscode-ks-ksc/server/`）
2. `textDocument/publishDiagnostics` — コンパイラのエラーをリアルタイム表示
3. `textDocument/completion` — commandRegistry からコマンド名補完
4. 拡張の `package.json` に LSP クライアント設定追加

## 4. VSCode + コンパイラ watch の開発フロー

LSP がなくても、以下のワークフローで即座に開発を始められる。

### 手順

```bash
# ターミナル: コンパイラ watch モード（保存のたびに自動コンパイル）
kaedevn-compile -w -o output.json my-game.ksc

# VSCode で .ksc ファイルを編集・保存
# → コンパイラが変更を検知し、自動で JSON に再コンパイル

# プレビューしたいときにブラウザでリロード
# http://localhost:5175/ksc-demo.html?file=output.json
```

### 仕組み

```
VSCode (.ksc 編集・保存)
  ↓  fs.watchFile (500ms interval)
kaedevn-compile --watch
  ↓  自動再コンパイル → JSON 出力
output.json
  ↓  手動ブラウザリロード
ksc-demo.html でプレビュー
```

- コンパイラの `--watch` は `fs.watchFile()` で 500ms 間隔の変更検知
- コンパイルエラーはターミナルに表示される
- プレビューの自動リロードは不要（手動リロードで十分）
- 追加実装ゼロ。既存機能のみで完結

### 関連する既存ツール

| ツール | 用途 |
|-------|------|
| `kaedevn-compile -w` | ファイル監視 + 自動コンパイル |
| `kaedevn-preview <file.ks>` | Vite dev server 起動 + ブラウザで開く |
| KSC Editor (`localhost:5177`) | Monaco エディタ + プレビュー（ブラウザ完結型の代替） |
