# kaedevn RPG Studio エディタ操作ガイド

> Generated by Claude Opus 4.6

`apps/rpg-studio/` — localhost:5178 で起動

---

## 起動方法

```bash
npm run build -w @kaedevn/entity-graph
cd apps/rpg-studio && npx vite --port 5178
```

初回起動時はサンプルデータ（Hero, Mage, Slime, Dragon, 町マップ, ダンジョン等）が自動ロードされる。以降は LocalStorage に自動保存（5秒間隔）。

---

## 1. Database タブ

**目的**: ゲームデータの CRUD（Actor, Skill, Enemy, Item, Variable, Event, Map）

### 画面構成

| 領域 | 内容 |
|------|------|
| 左サイドバー | Entity Type 一覧（件数付き）。クリックでテーブル切替 |
| 中央テーブル | 選択した Type の Entity を一覧表示 |
| 右パネル | 選択した Entity の詳細編集 + Relation 管理 |

### できること

| 操作 | 方法 |
|------|------|
| **新規作成** | ツールバーの `+ New {Type}` ボタン。Schema のデフォルト値で作成 |
| **名前編集** | Name 列をダブルクリック → テキスト入力 → Enter で確定 |
| **プロパティ編集** | 各列をダブルクリックでインライン編集（数値/文字列/enum/boolean） |
| **詳細編集** | 行クリック → 右パネルに全プロパティ表示（json, rich_text 含む） |
| **削除** | 右パネルの `Delete` ボタン。関連する Relation も自動削除 |
| **フィルタ** | ツールバーのテキストボックスで名前/プロパティの部分一致検索 |
| **ソート** | 列ヘッダーをクリック（▲▼切替） |

### Relation 管理（右パネル下部）

| 操作 | 方法 |
|------|------|
| **関係を見る** | 「→ learns」「← uses」のようにグループ表示。クリックで相手に移動 |
| **関係を追加** | 「+ Add Relation...」ドロップダウンで関係種別を選択 → ターゲット Entity を選択 → Add |
| **関係を削除** | 各 Relation の `×` ボタン |

### 利用可能な Relation

| Entity Type | Relation | Target | 意味 |
|------------|----------|--------|------|
| Actor | learns | Skill | このスキルを覚える |
| Actor | equips | Item | この装備を使える |
| Enemy | uses | Skill | このスキルを使う |
| Enemy | drops | Item | このアイテムをドロップ |
| Event | writes | Variable | この変数を書く |
| Event | reads | Variable | この変数を読む |

## 2. Map タブ

**目的**: タイルマップの描画とイベント配置

### 画面構成

| 領域 | 内容 |
|------|------|
| 左サイドバー | マップ一覧。クリックで切替 |
| 中央キャンバス | PixiJS マップ（32px グリッド） |
| 右パネル | 選択したイベントの DetailView |

### ツール

| ツール | 動作 |
|--------|------|
| **Pen** | クリック/ドラッグでタイルを描画 |
| **Eraser** | クリック/ドラッグで Grass(0) に戻す |
| **Bucket** | クリックで同色領域をフラッドフィル |
| **Event** | クリックでイベント Entity を配置（既存があれば選択） |

### タイルパレット

| ID | タイル | 色 | 通行 |
|----|--------|-----|:----:|
| 0 | Grass | 緑 | ○ |
| 1 | Dirt | 茶 | ○ |
| 2 | Water | 青 | × |
| 3 | Stone | 灰 | ○ |
| 4 | Wall | 暗灰 | × |

### 表示切替

- **Grid**: グリッド線の表示/非表示
- **Collision**: 衝突判定表示（UI のみ、実装未完了）

### イベントマーカー

| アイコン | トリガー | 意味 |
|---------|---------|------|
| **!** | interact | プレイヤーが Z キーで話しかける |
| **T** | player_touch | プレイヤーが踏むと自動発火 |
| **A** | auto | マップロード時に自動実行 |
| **P** | parallel | バックグラウンド実行 |

**タイルデータは Entity Store に自動保存される**。タブを切り替えても消えない。

## 3. Events タブ

**目的**: マップイベントのコマンド（ゲームロジック）を編集

### 画面構成

| 領域 | 内容 |
|------|------|
| 上部 | イベント選択ドロップダウン |
| 左パネル | ◆ コマンドリスト |
| 右パネル | 選択コマンドのプロパティ編集 |
| 下部 | FlowView（フローチャート自動生成） |

### 操作

| 操作 | 方法 |
|------|------|
| **イベント選択** | ドロップダウンから map_event を選択 |
| **コマンド追加** | `+ Add` → カテゴリからコマンド選択 |
| **コマンド編集** | コマンドをクリック → 右パネルでパラメータ編集 |
| **コマンド並替** | ↑↓ ボタン |
| **コマンド削除** | × ボタン |

### コマンド一覧

| カテゴリ | コマンド | パラメータ |
|---------|---------|-----------|
| **Expression** | Show Text | Speaker, Text |
| | Show Choices | Choices（カンマ区切り） |
| | Play BGM | Asset, Volume |
| | Play SE | Asset |
| **State** | Set Variable | Variable（Entity参照）, Value |
| | Add Item | Item（Entity参照）, Count |
| **Flow** | Condition | Variable, Operator, Value（子コマンドを持つ） |
| **Scene** | Transfer Map | Target Map, X, Y |
| | Start Battle | Troop（Enemy Entity参照） |
| **Call** | Call Event | Target Event |

**変更は即座に Entity Store に保存される**（イベント Entity の `properties.commands` に書き戻し）。

## 4. UI Layout タブ

**目的**: ゲーム画面の UI 要素をビジュアルで配置（1280×720）

### 操作

| 操作 | 方法 |
|------|------|
| **要素追加** | `+ Add Element` ボタン |
| **選択** | キャンバス上の要素をクリック |
| **移動** | ドラッグ |
| **リサイズ** | 右下のハンドルをドラッグ |
| **プロパティ編集** | 右パネル（Name, X, Y, Width, Height, Color, Opacity, Label） |

### 注意

**このタブのデータは Entity Store に保存されない**（ローカル state のみ）。タブ切替やリロードで消える。

## 5. Graph タブ

**目的**: Entity 間の関係をグラフとして可視化

### 操作

| 操作 | 方法 |
|------|------|
| **全体表示** | Entity 未選択時、最大50件のノード + Relation を表示 |
| **起点指定** | Entity を選択すると、そこから3ホップ以内の関連を表示 |
| **ノード移動** | ドラッグ |
| **ノード選択** | クリック → 右パネルに DetailView |

### ノードの色

| 色 | Entity Type |
|----|-------------|
| 青 | Actor |
| 赤 | Skill |
| オレンジ | Enemy |
| 緑 | Item |
| 紫 | Variable |
| ティール | Event |

力学シミュレーション（ノード間反発 + エッジ引力 + 中心重力）が常時動作。

## 6. Analysis タブ

**目的**: ゲームデータの問題を自動検出

### Static Analysis（上部）

| 検出内容 | レベル | 例 |
|---------|:------:|-----|
| 読まれるが書かれない変数 | Error | 「Boss Defeated は読まれるが書かれない（常に初期値）」 |
| 書かれるが読まれない変数 | Warning | 「Quest Accepted は書かれるが読まれない」 |
| どこからも参照されない変数 | Info | 「変数 X はどのイベントからも参照されない」 |
| 循環呼出（A→B→C→A） | Error | 「イベント間の循環呼出チェーン」 |

クリックすると該当 Entity の Database タブに移動。

### Balance Simulator（下部）

| 検出内容 | 表示 |
|---------|------|
| ワンショット（1撃撃破） | 黄色の警告 |
| ゼロダメージ | 赤色の警告 |
| ダメージシミュレーション表 | 全 Attacker × Skill × Defender の計算結果 |

ダメージ計算式（`a.atk * 4 - b.def * 2`）を実際にパースして評価。`learns` / `uses` Relation で紐付いた組み合わせのみ計算。

## 7. Play Game タブ

**目的**: エディタで作ったゲームを実際にプレイ

### 起動

`Start Game` ボタン → Entity Graph からマップとイベントを読み込み → PixiJS でゲーム開始。

### 操作

| キー | 動作 |
|------|------|
| ↑↓←→ | 移動（タイル単位、壁/水で停止） |
| Z / Enter | 話しかける / テキスト送り / 決定 |
| X / Escape | バトル中のキャンセル |

### ゲームフロー

1. **マップ移動**: 矢印キーで歩く。壁(4)と水(2)は通行不可。interact イベントも壁扱い
2. **NPC会話**: NPC の隣で向いて Z → メッセージウィンドウ表示 → Z で送り
3. **イベント発火**: player_touch イベントを踏むと自動実行
4. **マップ遷移**: `Transfer Map` コマンドで別マップへ移動
5. **バトル**: `Start Battle` コマンドでターン制戦闘に遷移

### バトルシステム

| コマンド | 動作 |
|---------|------|
| **Attack** | 通常攻撃。ターゲット選択 → ダメージ計算 |
| **Skill** | スキル一覧サブメニュー（MP コスト表示、不足なら灰色）→ ターゲット選択 |
| **Item** | インベントリ一覧 → 使用で HP 100 回復（最も HP が低い味方に自動適用） |
| **Flee** | 50% の確率で逃走成功 |

ダメージ計算: Skill Entity の `formula` プロパティを評価（±10% ランダム揺れ）。

勝利: EXP + Gold 獲得、drops Relation に基づくアイテムドロップ判定。

### サイドバー

| 項目 | 内容 |
|------|------|
| Party | 各キャラの名前と HP |
| Gold | 所持金 |
| Inventory | アイテム一覧 |
| Log | 直近20件のゲームログ |

### 制限事項

- 各イベントは1セッション中1回のみ実行（リスタートでリセット）
- ゲーム中のセーブ/ロードなし
- EXP はログに表示されるがレベルアップには反映されない
- アイテムの回復量は一律 100 HP（Item Entity の `effect` プロパティは未参照）

## 自動保存

- 5秒間隔で LocalStorage に全 Entity + Relation + Schema を保存
- ブラウザリロードで復元
- プロジェクト ID: `rpg-studio-default`
- localStorage をクリアするとサンプルデータに戻る
