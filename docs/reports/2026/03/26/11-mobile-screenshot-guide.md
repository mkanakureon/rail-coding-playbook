## iOS

### 方法 1: Xcode Simulator（Mac 必須）

```bash
# シミュレーター起動
open -a Simulator

# デバイス一覧
xcrun simctl list devices available

# 特定デバイスで起動
xcrun simctl boot "iPhone 16 Pro"

# アプリインストール（.app）
xcrun simctl install booted /path/to/MyApp.app

# スクリーンショット
xcrun simctl io booted screenshot screenshot.png

# 動画録画
xcrun simctl io booted recordVideo video.mp4
# Ctrl+C で停止
```

**デバイスサイズ指定：**
```bash
# iPhone SE (3rd) — 375x667
xcrun simctl boot "iPhone SE (3rd generation)"

# iPhone 16 Pro — 393x852
xcrun simctl boot "iPhone 16 Pro"

# iPhone 16 Pro Max — 430x932
xcrun simctl boot "iPhone 16 Pro Max"

# iPad Pro 13" — 1024x1366
xcrun simctl boot "iPad Pro 13-inch (M4)"
```

### 方法 2: 実機（USB 接続）

```bash
# Xcode → Window → Devices and Simulators → デバイス選択 → Take Screenshot

# または libimobiledevice（Homebrew）
brew install libimobiledevice
idevicescreenshot screenshot.png
```

### 方法 3: 実機（ワイヤレス）

```
設定 → デベロッパ → （有効化後）
電源ボタン + 音量上ボタン 同時押し → カメラロールに保存
AirDrop or iCloud で Mac に転送
```

## Android

### 方法 1: Android Emulator + adb（推奨）

```bash
# エミュレーター起動（Android Studio でAVD作成済み）
emulator -avd Pixel_8_API_35

# または Android Studio → Device Manager → ▶

# スクリーンショット
adb exec-out screencap -p > screenshot.png

# 動画録画（最大180秒）
adb shell screenrecord /sdcard/recording.mp4
# Ctrl+C で停止
adb pull /sdcard/recording.mp4
```

**デバイスサイズ指定（AVD 作成時）：**

| デバイス | 解像度 | 密度 |
|---------|--------|------|
| Pixel 8 | 1080x2400 | 420dpi |
| Pixel 8 Pro | 1344x2992 | 560dpi |
| Pixel Fold | 2208x1840 | 420dpi |
| Galaxy S24 | 1080x2340 | 420dpi |
| Nexus 5X (軽量) | 1080x1920 | 420dpi |

### 方法 2: 実機（USB 接続）

```bash
# USB デバッグを有効化
# 設定 → デベロッパーオプション → USB デバッグ ON

# スクリーンショット
adb exec-out screencap -p > screenshot.png

# 特定のアクティビティを起動してから撮影
adb shell am start -n com.example.app/.MainActivity
sleep 2
adb exec-out screencap -p > screenshot.png
```

### 方法 3: 実機（ワイヤレス）

```
電源ボタン + 音量下ボタン 同時押し
→ ギャラリーに保存
→ Google フォト or USB 転送で PC に
```

## 自動化（CI/バッチ）

### iOS（fastlane）

```bash
# インストール
gem install fastlane

# Fastfile
lane :screenshots do
  capture_ios_screenshots
end

# 実行
fastlane screenshots
```

### Android（fastlane + screengrab）

```bash
# Fastfile
lane :screenshots do
  capture_android_screenshots
end
```

### 両OS 共通（maestro）

```bash
# インストール
curl -Ls "https://get.maestro.mobile.dev" | bash

# フロー定義（screenshot.yaml）
appId: com.example.app
---
- launchApp
- tapOn: "ログイン"
- takeScreenshot: "login_screen"
- tapOn: "ホーム"
- takeScreenshot: "home_screen"

# 実行
maestro test screenshot.yaml
```

## 比較

| 方法 | セットアップ | 速度 | 自動化 | 用途 |
|------|:-----------:|:----:|:-----:|------|
| Xcode Simulator | 中（Xcode 必要） | 速い | ○ | iOS 開発中の確認 |
| Android Emulator + adb | 中（Android Studio 必要） | 速い | ○ | Android 開発中の確認 |
| 実機 USB | 小 | 速い | △ | 実機動作の確認 |
| 実機ボタン操作 | なし | 速い | × | 手軽な1枚 |
| fastlane | 大 | 遅い | ◎ | ストア申請用の全デバイス一括 |
| maestro | 小 | 中 | ◎ | E2E テスト + スクリーンショット |

**1枚撮るだけなら:** `adb exec-out screencap -p > screenshot.png`（Android）か `xcrun simctl io booted screenshot screenshot.png`（iOS）が最速。

**ストア申請用に全デバイス一括なら:** fastlane か maestro。
