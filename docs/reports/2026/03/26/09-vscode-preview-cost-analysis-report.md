# VSCode → Azure プレビューのコスト分析

> Generated by Claude Opus 4.6

---

## 1. 背景

KSC スクリプトを VSCode で編集し、Azure 上のプレビューサーバーで確認するワークフローを検討中。大量ユーザーが同時プレビューした場合のサーバー負荷と費用を試算する。

## 3. プレビュー1回あたりのコスト構造

VSCode で保存 → Azure にプレビューリクエスト、の1回の流れ：

```
VSCode → compile（ローカル） → POST script JSON → Azure API
                                                    ↓
                                              プレビュー HTML
                                                    ↓
                                         ブラウザがアセット取得
                                         (画像・音声を Blob から)
```

### リクエストごとの費用

| 項目 | データ量 | 単価 | 1回あたり |
|------|---------|------|---------|
| **スクリプト POST** | 5〜200KB | Container Apps 処理 | ほぼ ¥0 |
| **プレビュー API レスポンス** | 50〜200KB | Container Apps 処理 | ほぼ ¥0 |
| **アセット取得（画像）** | 1〜10MB | Blob 読み取り ¥0.001/万件 | ほぼ ¥0 |
| **帯域（Egress）** | 1〜10MB | ¥13/GB（5GB/月 無料） | ¥0.013〜0.13 |

**1回のプレビューで最も高いのは帯域（Egress）。** ただし画像は `Cache-Control: immutable`（1年キャッシュ）が設定済みなので、**同じアセットの2回目以降はブラウザキャッシュから読み込まれ、Egress ゼロ。**

## 4. ユーザー規模別の月額試算

### 前提条件

- 1ユーザーあたり1日50回プレビュー（頻繁に編集する開発者）
- プロジェクトあたり平均アセット: 背景10枚 + キャラ20枚 = 約50MB
- 初回アクセス時のみアセットDL、以降はブラウザキャッシュ

### 試算

| ユーザー数 | プレビュー/日 | Egress/月 | API 負荷 | 追加コスト/月 |
|-----------|:----------:|----------|---------|:----------:|
| **10人** | 500 | 1.5GB | min replica で処理可 | **¥0**（無料枠内） |
| **100人** | 5,000 | 15GB | 1 replica で十分 | **¥130** |
| **1,000人** | 50,000 | 150GB | 2 replicas 必要 | **¥2,250** |
| **10,000人** | 500,000 | 1.5TB | 3 replicas + DB 負荷 | **¥20,000+** |

※ Egress は初回DL分のみ。キャッシュ効果で実際はこの 1/3〜1/5 程度

### コスト急増のボトルネック

| 規模 | ボトルネック | 対策 |
|------|-----------|------|
| 〜100人 | なし | 現状のまま |
| 〜1,000人 | **Egress 帯域** | CDN 導入 |
| 〜10,000人 | **API サーバー + DB 接続数** | スケールアップ or キャッシュ層 |

## 5. コスト削減策

### 5.1. 即効性あり（工数小）

| 対策 | 効果 | 工数 |
|------|------|------|
| **Front Door 削除** | ¥1,552/月 削減。Container Apps の標準 SSL で代替 | 1日 |
| **ブラウザキャッシュ徹底** | 同じアセットの再DLゼロ。既に `immutable` ヘッダー設定済み | 済み |
| **Service Worker キャッシュ** | オフラインでもアセット利用可。Egress さらに削減 | 2日 |

### 5.2. 1,000人超で必要

| 対策 | 効果 | 月額 |
|------|------|------|
| **Cloudflare CDN（無料プラン）** | アセットをエッジキャッシュ。Azure Egress 90%削減 | ¥0 |
| **Cloudflare R2（Blob 代替）** | Egress 完全無料。Azure Blob から移行 | ストレージ ¥2/GB のみ |

### 5.3. 10,000人超で必要

| 対策 | 効果 | 月額 |
|------|------|------|
| **Redis キャッシュ** | API レスポンスキャッシュ。DB 負荷 80%削減 | ¥5,000〜 |
| **PostgreSQL スケールアップ** | B1ms → D4s_v3（4 vCPU） | ¥10,000〜 |
| **PgBouncer（接続プール）** | DB 接続数の壁を突破 | ¥1,000〜 |

## 6. アーキテクチャ比較: 3つのパターン

### パターン A: 現状維持（API 直接）

```
VSCode → compile → POST → Azure API → Blob → ブラウザ
```

| 項目 | 評価 |
|------|------|
| 追加実装 | ゼロ |
| 〜100人 | ✅ 問題なし |
| 1,000人〜 | ⚠️ Egress 費用増 |
| 10,000人〜 | ❌ API + DB がボトルネック |

### パターン B: CDN + キャッシュ

```
VSCode → compile → POST → Azure API → ブラウザ
                                         ↓
                              アセットは CDN から取得
```

| 項目 | 評価 |
|------|------|
| 追加実装 | CDN 設定（1日） |
| 〜100人 | ✅ |
| 1,000人〜 | ✅ Egress ほぼゼロ |
| 10,000人〜 | ⚠️ API + DB がボトルネック |

### パターン C: CDN + キャッシュ + アセット初回バンドル

```
VSCode → compile → POST script のみ → Azure API（軽量）
                                         ↓
プロジェクト初回: アセット .pak を CDN から1回DL → ローカルキャッシュ
以降: スクリプト差分のみ通信
```

| 項目 | 評価 |
|------|------|
| 追加実装 | .pak バンドル + Service Worker（1〜2週間） |
| 〜100人 | ✅ |
| 1,000人〜 | ✅ |
| 10,000人〜 | ✅ API は script POST のみで超軽量 |

## 8. 結論

| 問い | 回答 |
|------|------|
| プレビューの API コストは問題か？ | **今は問題ない。** 10人規模なら無料枠内 |
| 何人から費用が気になるか？ | **1,000人超で Egress 費用。** CDN（無料）で解決 |
| 最大のコスト要因は？ | **アセット帯域（Egress）。** スクリプト POST は誤差 |
| Cloudflare R2 に移行すべきか？ | **1,000人超えたら。** Egress 完全無料なので劇的に安くなる |
| 今やるべきことは？ | **Front Door 削除のみ。** 月 ¥1,552 節約、デメリットなし |
