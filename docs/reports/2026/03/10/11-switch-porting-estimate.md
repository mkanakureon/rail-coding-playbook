# Nintendo Switch 移植 — コード量見積もり

> Generated by Claude Opus 4.6

## 背景

kaedevn のネイティブエンジン（`packages/native-engine`）は SDL2 + C++ で構築されており、現在 iOS / Android で動作している。Nintendo Switch への移植に際して、追加で必要になるコード量を見積もる。

### アーキテクチャの特徴

- `SDL2Engine` が描画・入力・オーディオを統括
- `Interpreter` → `Parser` → `Evaluator` のスクリプト実行パイプライン
- プラットフォーム分岐は `#ifdef __ANDROID__` / `#ifdef __APPLE__` で `main.cpp` に集約
- IInput / IAudio / IStorage の抽象化は Web 版（PixiJS）側で定義済み。ネイティブ側はまだ直接実装

## Switch 対応で必要な作業と行数見積もり

### 1. プラットフォーム初期化（200–300 行）

**ファイル**: `platform/switch_init.cpp`

```
nn::os::Initialize()
nn::fs::MountSaveData()
nn::account::Initialize()
nn::nv:: ドライバ初期化
メモリアロケータ設定
```

Nintendo SDK（`nn::` 名前空間）の初期化は冗長で、各サブシステムを個別にセットアップする必要がある。アカウント選択 UI の呼び出しも含む。

### 2. 入力システム（300–500 行）

**ファイル**: `platform/switch_input.cpp`

Switch の入力は最も複雑なプラットフォーム層。

| 機能 | 行数目安 | 備考 |
|------|---------|------|
| Joy-Con 基本入力 | 80–100 | `nn::hid::NpadButton` のマッピング |
| コントローラ着脱検知 | 50–80 | 携帯モード ↔ 分離 ↔ Pro コン |
| 6 軸ジャイロ | 40–60 | ビジュアルノベルでは優先度低 |
| HD 振動 | 50–80 | テキスト送りや選択肢決定のフィードバック |
| タッチスクリーン | 40–60 | 携帯モード時のタッチ操作 |
| `IInput` 抽象への統合 | 40–60 | `dispatch(Action)` パターンへの変換 |

SDL2 の Switch ポート（devkitPro）がゲームパッド基本入力を吸収するが、HD 振動やコントローラ着脱は `nn::hid` 直叩きが必要。

### 3. セーブデータ（200–300 行）

**ファイル**: `platform/switch_storage.cpp`

Nintendo Switch のセーブデータはジャーナル付きファイルシステム。

```
nn::fs::MountSaveData("save")  // マウント
nn::fs::OpenFile()              // 書き込み
nn::fs::FlushFile()             // フラッシュ
nn::fs::Commit("save")          // コミット（原子的）
```

既存の `FileStorage` の Switch 実装として、`IStorage` 抽象に合わせて実装する。セーブスロット管理、破損検知、容量確認も含む。

### 4. オーディオ（100–150 行）

**ファイル**: `platform/switch_audio.cpp`

SDL_mixer の Switch ポートが動作すれば、追加コードは最小限。`nn::audio` を直接使う場合は 300 行以上になるが、SDL 経由であれば初期化と音量制御のラッパー程度。

### 5. 画面モード切替（80–120 行）

**ファイル**: `platform/switch_screen.cpp`

| モード | 解像度 | 備考 |
|--------|--------|------|
| ドッキング | 1920×1080 | TV 出力 |
| 携帯 | 1280×720 | 本体スクリーン |
| テーブル | 1280×720 | スタンド置き |

論理解像度 1280×720 で設計済みなので、携帯モードはそのまま。ドッキング時のアップスケーリング処理が主な追加。

### 6. main.cpp 分岐追加（50–80 行）

既存の `#ifdef __ANDROID__` / `#ifdef __APPLE__` に `#ifdef __SWITCH__` を追加。Switch 固有の初期化シーケンスと、HOME メニュー割り込みの処理。

### 7. ビルド設定（80–120 行）

| ファイル | 行数 | 内容 |
|---------|------|------|
| `CMakeLists.txt` (Switch) | 50–80 | devkitPro ツールチェーン、NX ライブラリリンク |
| `build-switch.sh` | 30–40 | `.nsp` / `.nro` パッケージングスクリプト |

### 8. Switch 固有 UI（150–250 行）

| 機能 | 行数 | 備考 |
|------|------|------|
| HOME メニュー統合 | 40–60 | サスペンド / レジュームのハンドリング |
| エラーダイアログ | 30–50 | `nn::err::ShowError()` のラッパー |
| ユーザー選択画面 | 40–60 | `nn::account::ShowUserSelector()` |
| コントローラアプレット | 40–80 | `nn::hid::ShowControllerSupport()` |

### 9. ロット検査対応（100–200 行）

Nintendo のロット検査（Lotcheck）で要求される品質基準への対応。

| 要件 | 行数 | 内容 |
|------|------|------|
| スリープ復帰 | 30–50 | 状態の保存と復元 |
| コントローラ切断 | 30–50 | 操作不能にならないガード |
| エラーハンドリング | 40–60 | SD カード抜き、メモリ不足等 |
| 言語設定追従 | 20–40 | `nn::settings::LanguageCode` の反映 |

### 10. テスト（200–300 行）

Switch 固有のユニットテスト。入力マッピング、セーブ/ロード、モード切替の検証。

## リスク要因

| リスク | 影響 | 対策 |
|--------|------|------|
| SDL2 Switch ポートの完成度 | SDL が吸収できない部分が増えると行数が 1.5 倍に | devkitPro の SDL2 ポートを事前検証 |
| Nintendo SDK のバージョン | API 変更で実装量が増減 | 開発者ポータルで最新ドキュメントを確認 |
| ロット検査の要件追加 | 予想外の対応が必要になる | 他タイトルの検査レポートを参考に |
| シェーダー互換性 | Switch の GPU (NVIDIA Tegra) で GLSL が動かない場合 | NVN API への移行で +200–500 行 |

### AI 生成が効く理由

1. **SDK のヘッダーファイルが最高のドキュメント**: `nn::hid::NpadButton` の定数名を見れば、AI はボタンマッピングを正確に書ける
2. **サンプルコードがテンプレート**: SDK 付属の `SimpleApp` サンプルを AI に渡せば、初期化コードはほぼコピー＆カスタマイズ
3. **既存エンジンが 2,020 行と小さい**: コンテキストウィンドウに全部入る。AI がエンジン全体を理解した上で Switch 分岐を追加できる
4. **プラットフォーム層が分離済み**: `platform/ios_screen.m` のパターンがあるので、同じ構造で `platform/switch_*.cpp` を生成するだけ

### AI だけでは完結しない部分

| 作業 | 理由 | 対処 |
|------|------|------|
| 実機デバッグ | Switch 実機でしか再現しないバグがある | AI がコード書く → 実機で試す → エラーを AI に返す のサイクル |
| HD 振動の感触調整 | 数値だけでは「気持ちいい」がわからない | 実機で触りながら微調整 |
| ロット検査の隠れ要件 | 任天堂の非公開ガイドラインに依存 | 検査レポートのフィードバックを受けて修正 |

ロット検査対応は要件が明らかになってから対処するため、初回提出までの期間に含めない。検査のフィードバック対応は別途 **1–2 週間** を見込む。

## 前提条件

- Nintendo Switch 開発者登録済み（Nintendo Developer Portal）
- devkitPro + libnx がインストール済み
- SDL2 の Switch ポート（`switch-sdl2`）が利用可能
- セーブスキーマは Web 版と同一（`save_schema_version: 1`）
- 論理解像度 1280×720 は変更なし
- AI 開発環境（Claude Code）が SDK ヘッダーを読める状態
