# IRenderer + GLRenderer + パーティクルシステム 実装計画書

> Generated by Claude Opus 4.6
> 前提文書: [02-mementomori-benchmark-renderer-design.md](./02-mementomori-benchmark-renderer-design.md)

## 目的

1. Native エンジンでメメントモリ級のポストプロセスを動かす
2. パーティクル（粒子演出）を Web / Native 共通で実装する
3. Web の GLSL シェーダー資産（10本 / 1,169行）をそのまま Native に持ち込む

---

## 現状の課題

| 項目 | 現状 |
|------|------|
| Native の描画 | `SDL_Renderer`（2D API）で `SDL_RenderCopy` のみ |
| シェーダー | `Shader` クラスはあるが render パイプラインに未接続 |
| OpenGL 利用 | macOS のみリンク、iOS/Android は no-op |
| パーティクル | Web に雨シェーダーがあるだけ。Native はゼロ |
| フィルター | Web に 39 種。Native はゼロ |

---

## 全体構成

```
Phase 1: IRenderer インターフェース定義 + SoftRenderer 抽出
Phase 2: GLRenderer 実装（OpenGL ES 3.0）
Phase 3: GLSL シェーダー共有化（Web ↔ Native）
Phase 4: パーティクルシステム（環境演出 / バトル・ガチャ演出）
```

## Phase 1: IRenderer + SoftRenderer（3日）

### 目的

既存の `SDL2Engine::render()` を壊さずに、描画の抽象化層を作る。

### 作業内容

#### 1-1. IRenderer インターフェース定義

```cpp
// packages/native-engine/src/engine/IRenderer.hpp

struct SpriteParams {
    float x, y;
    float scaleX = 1.0f, scaleY = 1.0f;
    float rotation = 0.0f;       // radians
    float alpha = 1.0f;
    uint32_t tint = 0xFFFFFF;
    BlendMode blendMode = BlendMode::Normal;
    float anchorX = 0.5f, anchorY = 0.5f;
};

enum class BlendMode { Normal, Add };

using TextureHandle = uint32_t;

class IRenderer {
public:
    virtual ~IRenderer() = default;

    // ライフサイクル
    virtual bool init(SDL_Window* window, int logicalW, int logicalH) = 0;
    virtual void beginFrame() = 0;
    virtual void endFrame() = 0;    // present

    // テクスチャ
    virtual TextureHandle loadTexture(const std::string& path) = 0;
    virtual TextureHandle loadTextureRW(SDL_RWops* rw, const std::string& key) = 0;
    virtual void destroyTexture(TextureHandle tex) = 0;
    virtual void queryTextureSize(TextureHandle tex, int& w, int& h) = 0;

    // スプライト描画
    virtual void drawSprite(TextureHandle tex, const SpriteParams& params) = 0;
    virtual void drawSpriteCover(TextureHandle tex, int viewW, int viewH) = 0;

    // 矩形（UI 用）
    virtual void drawRect(int x, int y, int w, int h,
                          uint8_t r, uint8_t g, uint8_t b, uint8_t a) = 0;

    // テキスト
    virtual void drawText(const std::string& text, int x, int y,
                          SDL_Color color, TTF_Font* font, int wrapWidth = 0) = 0;

    // ポストプロセス
    virtual void beginPostProcess() = 0;     // シーンをフレームバッファに描画開始
    virtual void applyFilter(FilterType type, const FilterParams& params) = 0;
    virtual void endPostProcess() = 0;       // フレームバッファを画面に出力

    // パーティクル
    virtual void emitParticles(const ParticleEmitterDef& def) = 0;
    virtual void updateParticles(float dt) = 0;
    virtual void renderParticles() = 0;
};
```

#### 1-2. SoftRenderer 抽出

今の `SDL2Engine::render()` の描画部分をそのまま `SoftRenderer` に移す。

| 移動元 | 移動先 | 内容 |
|--------|--------|------|
| `SDL2Engine::drawTexture()` | `SoftRenderer::drawSprite()` | テクスチャ描画 |
| `SDL2Engine::drawTextureCover()` | `SoftRenderer::drawSpriteCover()` | Cover 描画 |
| `SDL2Engine::renderText()` | `SoftRenderer::drawText()` | テキスト描画 |
| テキストウィンドウ描画 | `SoftRenderer::drawRect()` | 矩形描画 |
| `SDL_RenderPresent` | `SoftRenderer::endFrame()` | 画面出力 |
| `applyFilter` | no-op | SoftRenderer ではフィルターなし |
| パーティクル | no-op | SoftRenderer ではパーティクルなし |

#### 1-3. SDL2Engine の書き換え

```cpp
// Before
class SDL2Engine : public IEngineAPI {
    SDL_Renderer* renderer;
    // ...
};

// After
class SDL2Engine : public IEngineAPI {
    std::unique_ptr<IRenderer> renderer;  // SoftRenderer or GLRenderer
    // ...
};
```

`render()` メソッドは `renderer->drawSprite()` / `renderer->drawRect()` を呼ぶだけになる。

## Phase 2: GLRenderer 実装（1週間）

### 目的

OpenGL ES 3.0 で描画する `GLRenderer` を実装し、シェーダーが使える状態にする。

### 作業内容

#### 2-1. OpenGL コンテキスト初期化

```cpp
// GLRenderer::init()
SDL_GL_SetAttribute(SDL_GL_CONTEXT_PROFILE_MASK, SDL_GL_CONTEXT_PROFILE_ES);
SDL_GL_SetAttribute(SDL_GL_CONTEXT_MAJOR_VERSION, 3);
SDL_GL_SetAttribute(SDL_GL_CONTEXT_MINOR_VERSION, 0);
glContext = SDL_GL_CreateContext(window);
```

| プラットフォーム | GL API | 備考 |
|----------------|--------|------|
| macOS | OpenGL 4.1 (Core) | ES 3.0 のスーパーセット |
| iOS | OpenGL ES 3.0 | Metal fallback は将来検討 |
| Android | OpenGL ES 3.0 | API Level 18+ (99%カバー) |
| Switch | NVN | シェーダー変換が必要（Phase 2 では対象外） |

#### 2-2. スプライト描画パイプライン

```
頂点バッファ（Quad）
  → 頂点シェーダー（位置・UV 変換）
  → フラグメントシェーダー（テクスチャサンプリング + Tint + Alpha）
  → フレームバッファ or 画面
```

```cpp
// GLRenderer の核心部分
void GLRenderer::drawSprite(TextureHandle tex, const SpriteParams& params) {
    spriteShader.use();
    spriteShader.setUniform("uAlpha", params.alpha);
    spriteShader.setUniform("uTint", params.tint);
    // transform matrix を計算して uniform にセット
    glBindTexture(GL_TEXTURE_2D, tex);
    glDrawElements(GL_TRIANGLES, 6, GL_UNSIGNED_SHORT, 0);
}
```

#### 2-3. フレームバッファ管理（ポストプロセス用）

```cpp
void GLRenderer::beginPostProcess() {
    glBindFramebuffer(GL_FRAMEBUFFER, sceneFBO);
    glClear(GL_COLOR_BUFFER_BIT);
}

void GLRenderer::applyFilter(FilterType type, const FilterParams& params) {
    // sceneFBO のテクスチャを入力として、フィルターシェーダーで描画
    Filter& filter = filters[type];
    filter.shader.use();
    filter.shader.setUniform("uTime", params.time);
    filter.shader.setUniform("uIntensity", params.intensity);
    glBindTexture(GL_TEXTURE_2D, sceneFBOTexture);
    glDrawElements(GL_TRIANGLES, 6, GL_UNSIGNED_SHORT, 0);
}

void GLRenderer::endPostProcess() {
    glBindFramebuffer(GL_FRAMEBUFFER, 0);  // デフォルトに戻す
    // 結果を画面に描画
}
```

#### 2-4. テキスト描画

SDL_ttf でサーフェスを作り、OpenGL テクスチャに変換して描画。

```cpp
void GLRenderer::drawText(...) {
    SDL_Surface* surface = TTF_RenderUTF8_Blended(font, text, color);
    GLuint tex = surfaceToGLTexture(surface);
    drawSprite(tex, ...);
    glDeleteTextures(1, &tex);  // or キャッシュ
    SDL_FreeSurface(surface);
}
```

## Phase 3: GLSL シェーダー共有化（4日）

### 目的

Web の GLSL シェーダー 10 本を Native で動かす。

### 作業内容

#### 3-1. シェーダーソースの集約

```
packages/
  shaders/                          ← 新設
    shared/
      utils.glsl                    ← GLSL_LUMINANCE, GLSL_HASH, GLSL_NOISE2D
    filters/
      bloom.frag                    ← BloomFilter.ts から抽出
      crt.frag
      glitch.frag
      underwater.frag
      old_film.frag
      noise.frag
      gameboy.frag
      pixelate.frag
      chromatic_aberration.frag
      rain.frag
    vertex/
      filter.vert                   ← 共通頂点シェーダー
```

#### 3-2. Web 側の修正

```typescript
// Before: GLSL がインラインで TypeScript に埋め込み
const fragmentSrc = `#version 300 es
precision highp float;
// ... 100行のシェーダーコード
`;

// After: 外部ファイルから import
import bloomFrag from '../../../shaders/filters/bloom.frag?raw';
import filterVert from '../../../shaders/vertex/filter.vert?raw';
```

Vite の `?raw` import で .frag ファイルを文字列として読める。

#### 3-3. Native 側の読み込み

```cpp
// GLRenderer が起動時にシェーダーファイルを読み込む
std::string bloomSrc = readFile("shaders/filters/bloom.frag");
std::string utilsSrc = readFile("shaders/shared/utils.glsl");

// utils.glsl を #include 相当で結合
std::string fullSrc = utilsSrc + "\n" + bloomSrc;
bloomShader.load(filterVertSrc, fullSrc);
```

#### 3-4. GLSL の互換性調整

WebGL2 と OpenGL ES 3.0 の差異は極めて小さい。調整が必要な箇所：

| 差異 | WebGL2 | OpenGL ES 3.0 | 対応 |
|------|--------|---------------|------|
| precision 宣言 | 必須 | 必須 | 共通で OK |
| texture() 関数 | あり | あり | 共通で OK |
| `#version 300 es` | 必須 | 必須 | 共通で OK |
| PixiJS uniform 名 | `uSampler` | 任意 | `#define uSampler uTexture` で吸収 |
| PixiJS 入力座標 | `vTextureCoord` | 任意 | 同上 |

**実質的な差異は PixiJS 固有の uniform 名だけ。** プリプロセッサの `#define` で吸収可能。

```glsl
// shaders/filters/bloom.frag の先頭に追加
#ifndef PIXI
#define uSampler uTexture
#define vTextureCoord vUV
#endif
```

#### 3-5. ScreenFilter 相当の実装（Native 側）

Web の `ScreenFilter.applyMix()` と同じロジックを C++ で実装。

```cpp
void GLRenderer::applyFilterMix(const std::vector<FilterEntry>& filters) {
    // 1. ColorMatrix 系を合成
    float matrix[20] = { /* identity */ };
    for (auto& f : filters) {
        if (isColorMatrixType(f.type)) {
            multiply4x5(matrix, getColorMatrix(f.type, f.intensity));
        }
    }
    applyColorMatrix(matrix);

    // 2. Weather 系（最後の1つ）
    for (auto& f : filters) {
        if (isWeatherType(f.type)) {
            applyFilter(f.type, f.params);
        }
    }

    // 3. PostProcess 系（最後の1つ）
    for (auto& f : filters) {
        if (isPostProcessType(f.type)) {
            applyFilter(f.type, f.params);
        }
    }
}
```

#### 実装方針

既存の `rain.frag` を汎用化して `particles_ambient.frag` にする。

```glsl
// packages/shaders/filters/particles_ambient.frag
uniform float uTime;
uniform int   uType;       // 0:雨 1:光の粒 2:花びら 3:塵 4:キラキラ 5:虹渦 6:星 7:雪
uniform float uDensity;    // 粒子密度
uniform float uSpeed;      // 落下/浮遊速度
uniform vec2  uWind;       // 風向き
uniform float uParticleSize;

// 雨: 縦長の線、高速落下
// 光の粒: 丸、ゆっくり浮遊、sin波で上下
// 花びら: 回転＋横揺れ、中速落下
// 塵: ノイズベースの不規則浮遊
// キラキラ: ランダム配置、α点滅
// 虹渦: 極座標回転、色相シフト
// 星: 大量、高速落下、加算合成
// 雪: 丸、横揺れ、低速落下
```

#### IRenderer での呼び出し

```cpp
// 環境演出は applyFilter の一種として呼ぶ
renderer->applyFilter(FilterType::ParticleAmbient, {
    .subType = ParticleType::Sakura,
    .density = 0.5f,
    .speed = 0.3f,
    .wind = {0.1f, 0.0f},
    .time = elapsedTime
});
```

### 4-B. CPU パーティクル — バトル・ガチャ演出（4日）

#### 対象演出

| 演出 | 粒子数 | 特徴 |
|------|:------:|------|
| 火の粉 | 50-200 | 加算合成、放射、重力なし |
| 氷の結晶 | 20-50 | テクスチャスプライト、拡大→停止→消滅 |
| 斬撃の軌跡 | 30-100 | キャラ位置追従、短寿命、トレイル |
| 爆発の破片 | 100-300 | 放射、減速、重力あり |
| 回復の光 | 30-80 | 螺旋上昇、加算合成 |
| ガチャ集束 | 200-500 | フェーズ: 集束→爆発→拡散 |
| 虹色の粒子 | 100-300 | 色相時間変化、渦 |
| 星の雨（ガチャ） | 200-500 | 画面全体、加算合成 |

#### データ構造

```cpp
// packages/native-engine/src/engine/Particle.hpp

struct Particle {
    float x, y;           // 位置
    float vx, vy;         // 速度
    float life;           // 残り寿命（秒）
    float maxLife;        // 最大寿命
    float size;           // 現在サイズ
    float sizeStart, sizeEnd;  // サイズ変化
    float alpha;          // 現在α
    float alphaStart, alphaEnd;
    float rotation;       // 回転角
    float rotationSpeed;  // 回転速度
    uint32_t color;       // 現在色
    uint32_t colorStart, colorEnd;
};

struct ParticleEmitterDef {
    TextureHandle texture;        // 粒子のテクスチャ
    BlendMode blendMode;          // Normal or Add
    int maxParticles;
    float emitRate;               // 個/秒

    // 初期値の範囲（min-max でランダム化）
    Range<float> life;            // 寿命
    Range<float> speed;           // 初速
    Range<float> angle;           // 射出角度
    Range<float> size;
    Range<float> rotation;
    Range<float> rotationSpeed;
    Range<uint32_t> color;

    // 物理
    float gravityX = 0, gravityY = 0;
    float friction = 0;           // 減速係数

    // エミッター形状
    EmitterShape shape;           // Point, Circle, Rect, Ring
    float shapeWidth, shapeHeight;
};
```

#### ParticleSystem クラス

```cpp
// packages/native-engine/src/engine/ParticleSystem.hpp

class ParticleSystem {
public:
    // エミッター管理
    EmitterId create(const ParticleEmitterDef& def);

    void setPosition(EmitterId id, float x, float y);
    void burst(EmitterId id, int count);   // 一度に大量放出
```

#### IRenderer との関係

```
ParticleSystem（エンジン層・IRenderer の上）
  │
  │ update(dt):  位置・速度・寿命・色・サイズを計算
  │ render():    生存中の粒子ごとに renderer.drawSprite() を呼ぶ
  │
  └─→ IRenderer.drawSprite(tex, {x, y, scale, alpha, tint, blendMode=Add})
          │
     GLRenderer / PixiJS / SoftRenderer
```

ParticleSystem は IRenderer の**消費者**。レンダラーの中には入らない。

#### Web 側の対応

TypeScript で同等の ParticleSystem を実装するが、ロジックは同じ：

```typescript
class ParticleSystem {
    update(dt: number): void { /* 座標と速度を足す */ }
    render(stage: Container): void {
        for (const p of this.particles) {
            sprite.x = p.x;
            sprite.alpha = p.alpha;
            // ...
        }
    }
}
```

言語は違うが、やることが「座標 + 速度 × dt」だけなので互換性の問題は起きない。

## スケジュール

```
Phase 1 (3日)    IRenderer + SoftRenderer 抽出
  │               既存動作を壊さずに描画を分離
  │
Phase 2 (5日)    GLRenderer 実装
  │               OpenGL ES 3.0 で背景・キャラ・テキスト・FBO

  │
Phase 4 (5日)    パーティクルシステム
  │  4-A (2日)     GLSL 環境パーティクル（光の粒・花びら・雪…）
```

### パーティクルのブロックエディタ統合

#### 環境パーティクル → FilterMixBlock に統合

GLSL 環境パーティクルはフィルターの一種。既存の FilterMixBlock でそのまま使える。

```
FilterMixBlock:
  Layer 1: moonlight（色調）       ← 既存
  Layer 2: p_sakura（花びら）      ← 新規（パーティクルカテゴリから選択）
  Layer 3: bloom（ポストプロセス）  ← 既存
```

合成ルール:
- 環境パーティクルは **weather カテゴリ** として扱う（雨と同じ枠）
- 1つの FilterMix につき weather / パーティクル系は **最大1つ**（既存ルールそのまま）

#### バトル/ガチャパーティクル → 新規 ParticleBlock

位置・タイミング制御が必要なため、独立ブロックとして追加。

```typescript
// apps/editor/src/types/index.ts に追加
type ParticleBlock = {
    id: string;
    type: 'particle';
    preset: string;          // 'p_fire_sparks' | 'p_explosion' | ...
    x?: number;              // エミッター位置 X（0-1280、省略で中央）
    y?: number;              // エミッター位置 Y（0-720、省略で中央）
    duration?: number;       // 持続時間 ms（省略で無限）
    intensity?: number;      // 粒子数倍率 0.1-3.0（デフォルト 1.0）
    blendMode?: 'normal' | 'add';
};
```
