## Phase 3: WebOpHandler 統合（3日）

### 3-1. キャラ表示コマンドの拡張

既存の `@show` コマンドで Live2D モデルも表示できるようにする。

```
; 従来（静止画）
@show alice normal

; Live2D（同じコマンド、アセットタイプで自動判定）
@show alice normal
; → アセット DB に alice が live2d タイプで登録されていれば Live2D で表示
```

### 3-2. アセットタイプの判定

```typescript
// AssetRef に animationType を追加
type AssetRef = {
  id: string;
  slug: string;
  type: 'bg' | 'ch' | 'overlay';
  animationType?: 'static' | 'live2d' | 'spine';  // 追加
  url: string;
};
```

WebOpHandler の `handleShow` でアセットタイプを見て分岐：

```typescript
async handleShow(slug: string, expression: string) {
  const asset = this.getCharacterAsset(slug);

  if (asset.animationType === 'live2d') {
    // Live2D モデルをロード・表示
    const model = await Live2DModel.from(asset.url);
    // 表情適用
    if (expression) model.expression(expression);
    this.layers.characterLayer.addChild(model);
  } else {
    // 従来の静止画表示
    // ...既存コード
  }
}
```

### 3-3. 表情マッピング

エディタの ChBlock で設定した表情名と Live2D モデルの表情名を対応付ける。

```typescript
// モデルごとの表情マッピング（エディタ or JSON で設定）
const expressionMap: Record<string, string> = {
  'normal': '',           // デフォルト
  'happy': 'smile',       // Live2D 側の表情名
  'sad': 'sad',
  'angry': 'angry',
  'surprised': 'surprised',
};
```

### 3-4. 音声再生との連携

既存の `@playVoice` コマンド実行時に、表示中の Live2D モデルのリップシンクを連動させる。

```typescript
handlePlayVoice(slug: string) {
  // 既存の音声再生
  this.audio.playVoice(slug);

  // Live2D リップシンク連動
  const activeModel = this.getActiveLive2DModel();
  if (activeModel) {
    activeModel.speak(this.resolveVoiceUrl(slug));
  }
}
```

## Phase 5: メモリ・パフォーマンス調整（1日）

### 5-1. Cubism メモリ設定

```typescript
import { configureCubismSDK } from 'untitled-pixi-live2d-engine';

// デフォルト 16MB → 32MB に増加（複数モデル対応）
configureCubismSDK({ memorySizeMB: 32 });
```

### 5-2. モデルキャッシュ

一度ロードしたモデルをキャッシュして、シーン切替時の再ロードを防ぐ。

```typescript
class Live2DModelCache {
  private cache = new Map<string, Live2DModel>();

  async get(modelPath: string): Promise<Live2DModel> {
    if (this.cache.has(modelPath)) {
      return this.cache.get(modelPath)!;
    }
    const model = await Live2DModel.from(modelPath);
    this.cache.set(modelPath, model);
    return model;
  }

  clear(): void {
    for (const model of this.cache.values()) {
      model.destroy();
    }
    this.cache.clear();
  }
}
```

### 5-3. モバイル対応

| 項目 | 対策 |
|------|------|
| テクスチャサイズ | 2048×2048 以下を強制（FREE 版の制限と一致） |
| 同時表示数 | モバイルは最大 2 体に制限 |
| 物理演算 | 低スペック端末では無効化オプション |

- 複数モデル（2-3体）同時表示でフレーム落ちしない
- モデルの切替がスムーズ（キャッシュ効果）
- モバイルブラウザで動作確認
