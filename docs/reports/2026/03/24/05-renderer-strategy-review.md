# レポート：IRenderer & パーティクルシステムによる技術的優位性と差別化戦略

## 1. エグゼクティブサマリー

本計画は、kaedevn モノレポにおける描画基盤を `IRenderer` インターフェースによって抽象化し、Web（PixiJS）と Native（OpenGL ES 3.0）で **100% 同一の視覚表現** を可能にするものである。これにより、Unity 製ノベルゲームを凌駕する「軽量さ」と、メメントモリ級の「リッチな演出」を両立させる。

## 2. Unity に対する圧倒的優位性

Unity は汎用エンジンとして強力だが、ノベルゲームや 2D アドベンチャーにおいては以下の 3 点で kaedevn が優位に立つ。

### A. シェーダー資産の完全共有（WebGL2 ↔ OpenGL ES 3.0）
*   **Unity の弱点:** WebGL 出力時のシェーダー変換（HLSL → GLSL）に癖があり、プラットフォーム間で見た目の差異やバグが発生しやすい。
*   **kaedevn の強み:** `packages/shaders` に集約された GLSL ES 3.0 コードを、Web と Native の両方が直接利用する。エディタ（Web）で調整した Bloom やパーティクルの質感が、Switch やスマホ（Native）でも「そのまま」再現される。

### B. ゼロ・オーバーヘッド・アーキテクチャ
*   **Unity の弱点:** 実行時エンジン（Runtime）が巨大（数十MB〜）で、メモリ消費量も多い。特に Web 版はロードが重く、ユーザー離脱の原因となる。
*   **kaedevn の強み:** ノベル演出に特化した `IRenderer` 抽象化により、不要な機能を一切排除。低スペック端末や Switch でも、バッテリー消費を抑えつつ 60fps の滑らかな演出を維持できる。

### C. ブロックエディタによる高速な演出イテレーション
*   **差別化:** Unity でリッチなパーティクルやポストプロセスを制御するには C# スクリプトや複雑な Prefab 管理が必要だが、kaedevn では「ブロックエディタ」上でパラメータを叩くだけで、プログラマを介さずにメメントモリ級の演出（花びらの舞い、光の粒、被写界深度）を組み込める。

## 3. 「メメントモリ級」表現を実現する 3 つの柱

本計画（Phase 4, 5）により、以下の「空気感」の演出が可能になる。

1.  **環境パーティクル (GLSL 方式):**
    *   画面全体に漂う「塵・埃」「光の粒」「舞い落ちる花びら」を、GPU のポストプロセスとして実装。数千個の粒子を CPU 負荷ゼロで描画し、静止画に「命」を吹き込む。
2.  **重層的なポストエフェクト:**
    *   Bloom（発光）、Vignette（周辺暗転）、Color Grading（3層同時合成）を組み合わせることで、水彩画のような淡く深い色彩表現を実現する。
3.  **ハイブリッド・パーティクル (CPU 方式):**
    *   バトルやガチャ演出における「火の粉」「氷の結晶」などは CPU で物理挙動を制御。ゲームロジックと密結合した「手応えのある」エフェクトを提供し、RPG 要素を強化する。

本計画は、kaedevn を「単なるノベルエンジン」から「高付加価値な 2D 演出プラットフォーム」へと進化させる極めて重要なステップである。
