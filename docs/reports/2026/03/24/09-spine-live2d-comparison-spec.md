# Spine vs Live2D — kaedevn 統合比較・導入方針

> Generated by Claude Opus 4.6

## 背景

kaedevn エンジンにキャラクターの骨格/メッシュアニメーション（呼吸・瞬き・髪揺れ・待機モーション）を導入するにあたり、Spine と Live2D の 2 つの選択肢を比較し、導入方針を決定する。

---

## 見た目の比較

| 観点 | Spine | Live2D |
|------|-------|--------|
| **アニメーション方式** | 骨格ベース（ボーン + スロット） | メッシュ変形（2Dイラストを直接変形） |
| **動きの質** | メカ・モンスター・アクションが得意 | **呼吸・瞬き・髪揺れが圧倒的に自然** |
| **表現力** | 良い（ゲーム向き） | **非常に高い**（イラストがそのまま動く） |
| **ノベルゲーム適合度** | 十分（Ren'Py 等で実績多数） | **最適**（メメントモリ・FGO・アズレン級） |
| **バトル演出** | **得意**（骨格で剣振り・エフェクト） | やや不向き（全身アクション向きではない） |
| **ドット絵** | **対応**（ツクール型に最適） | 不向き（高解像度イラスト前提） |

### 使用タイトル例

| ゲーム | 使用技術 | 用途 |
|--------|---------|------|
| メメントモリ | **Live2D** | ホーム画面のキャラ演出 |
| FGO | **Live2D** | バトル中キャラ・マイルーム |
| アズールレーン | **Live2D** | 着せ替え・秘書艦 |
| ブルーアーカイブ | **Spine** | バトル中キャラ |
| 崩壊：スターレイル | **Spine** | 2D キャラ演出 |
| Hades | **Spine** | キャラアニメーション全般 |

## 統合難易度

### Spine（簡単）

| 項目 | 内容 |
|------|------|
| PixiJS v8 対応 | **公式対応** `@esotericsoftware/spine-pixi-v8` |
| インストール | `npm i @esotericsoftware/spine-pixi-v8` |
| 実装工数 | **3-5日** |
| コード量 | ~200行（ロード + 表示 + モーション切替） |
| 性能 | v7 比 **50% 高速・50% 省メモリ**。WebGPU 対応 |
| 既存への影響 | LayerManager にスプライトとして addChild するだけ |

```typescript
// Spine: 3行で動く
import { Spine } from '@esotericsoftware/spine-pixi-v8';
const spine = Spine.from({ skeleton: 'character.json', atlas: 'character.atlas' });
layerManager.characterLayer.addChild(spine);
spine.state.setAnimation(0, 'idle', true);
```

### Live2D（中〜難）

| 項目 | 内容 |
|------|------|
| PixiJS v8 対応 | コミュニティ製 (`untitled-pixi-live2d-engine`) |
| インストール | SDK 手動セットアップ + ラッパーライブラリ |
| 実装工数 | **2-3週間** |
| コード量 | ~500行（SDK 初期化 + モデル管理 + リップシンク + 表情） |
| 性能 | WebGL コンテキスト共有の調整が必要 |
| 既存への影響 | WebGL コンテキストの初期化順序に注意が必要 |

```typescript
// Live2D: SDK 初期化 + モデルロード + 設定
import { Live2DModel } from 'untitled-pixi-live2d-engine';
await Live2DModel.registerTicker(app.ticker);
const model = await Live2DModel.from('character.model3.json');
layerManager.characterLayer.addChild(model);
model.internalModel.motionManager.startMotion('idle', 0);
model.expression('smile');
// WebGL コンテキスト共有の問題が出る可能性あり
```

### 難易度の差が出るポイント

| ポイント | Spine | Live2D |
|---------|-------|--------|
| WebGL コンテキスト | PixiJS と統合済み | 独自コンテキストとの共有が必要 |
| アセット形式 | `.json` + `.atlas` + `.png` | `.model3.json` + `.moc3` + `.physics3.json` + テクスチャ |
| モーション制御 | `setAnimation(track, name, loop)` | MotionManager + MotionPriority + MotionGroup の理解が必要 |
| 表情切替 | パラメータスライダーで直接制御 | Expression ファイル（`.exp3.json`）の管理 |
| リップシンク | 手動（口ボーンのアニメーション） | SDK 内蔵（音声ボリューム連動も可） |
| 物理演算（髪揺れ等） | 手動（ボーンアニメ） | SDK 内蔵（`.physics3.json`で自動） |

## パートナーが Live2D モデルを持っている場合

### 有用なケース

運営型ゲームアプリ事業のパートナーが**既に Live2D モデルを持っている**場合、kaedevn が Live2D に対応していることは強力なアドバンテージになる。

| ケース | 状況 | kaedevn の価値 |
|--------|------|---------------|
| **IP ホルダー** | 漫画/ラノベの IP で既に Live2D モデルを制作済み | モデルをそのまま流用してゲーム化できる。制作コスト大幅削減 |
| **VTuber 事業者** | 配信用 Live2D モデルを保有 | ゲーム版 VTuber コンテンツへの転用 |
| **既存ゲーム会社** | サービス終了予定のゲームに Live2D アセットがある | アセットを次のプロジェクトで再利用可能 |
| **Live2D クリエイター** | Live2D モデルを商品として販売している | kaedevn で動作するゲームにそのまま組み込み |

### Live2D 対応の判断基準

```
パートナーが Live2D モデルを持っている？
  │
  ├─ YES → kaedevn が Live2D 対応済みなら、そのまま使える（大きな差別化）
  │         ライセンス費用はパートナー側で解決済みの可能性が高い
  │
  └─ NO  → Spine で十分（コスト ¥0、統合 3-5日）
            新規に Live2D モデルを作るコスト: 1体 10〜50万円
```

### パートナー持ち込み Live2D モデルの技術的な課題

| 課題 | 対策 |
|------|------|
| SDK バージョンの違い（Cubism 2/3/4/5） | `untitled-pixi-live2d-engine` は Cubism 2〜5 対応 |
| モーション名の不統一 | モーションマッピング設定を UI で提供 |
| テクスチャサイズの最適化 | アセットアップロード時に自動リサイズ |
| Live2D ライセンスの確認 | パートナー契約時にライセンス確認チェックリストを用意 |

## 日本市場での受容性

| 観点 | Spine | Live2D |
|------|:-----:|:------:|
| 日本のノベルゲーム採用率 | 少ない | **圧倒的に多い** |
| 日本の VTuber 業界 | ほぼゼロ | **業界標準** |
| 日本のソシャゲ採用率 | 一部（ブルアカ等） | **主流**（FGO, アズレン, メメントモリ, プロセカ） |
| 日本のユーザーの見慣れ度 | 「ゲームっぽい動き」 | **「あ、Live2D だ」と認識できるレベルで浸透** |
| クリエイター人口（日本） | 少ない | **非常に多い**（pixiv, ココナラ, BOOTH で大量出品） |

### なぜ日本では Live2D が強いか

- **イラスト文化との相性が完璧。** 日本のイラストレーターが描いた絵をそのまま動かせる
- **VTuber 産業が巨大。** Live2D モデルの需要と供給が確立されている
- **「イラストが動く」感動が日本のユーザーに刺さる。** Spine の「骨格で動く」は欧米ゲーム的で日本ではやや違和感
- **日本製という信頼感。** ドキュメントが日本語、サポートが日本語

## Live2D FREE 版の機能制限

| 項目 | FREE | PRO |
|------|:----:|:---:|
| **パラメータ数** | **30 まで** | 無制限 |
| **パーツ数** | **30 まで** | 無制限 |
| **アートメッシュ数** | **100 まで** | 無制限 |
| **テクスチャ枚数** | **1枚のみ** | 無制限 |
| **テクスチャサイズ** | **2048×2048 まで** | 無制限 |
| **ブレンドシェイプ** | 3個以上で制限あり | 無制限 |
| **メッシュ自動生成プリセット** | 追加不可 | カスタム保存可能 |
| **利用期限** | **無期限** | サブスク（年 $100） |
| **商用利用** | 年商 1,000万円以下は可 | 制限なし |

SDK 側に FREE/PRO の区別はない。FREE 版エディタで作ったモデルも SDK で問題なく動く。

### メメントモリ級は FREE 版で作れるか？

**作れない。** メメントモリのキャラは推定パラメータ 50+、メッシュ 200+ で FREE 版の上限を超える。

ただし**ノベルゲームの立ち絵（バストアップ）なら FREE 版で十分。**

| 要素 | 必要パラメータ | FREE 上限内？ |
|------|:------------:|:-----------:|
| 呼吸（胸・肩・首） | 3 | ○ |
| 瞬き（左右） | 2 | ○ |
| 目の開閉（左右） | 2 | ○ |
| 瞳の動き（左右 × XY） | 4 | ○ |
| 口の開閉 | 2 | ○ |
| 髪揺れ・前髪（左右 × XY） | 4 | ○ |
| 髪揺れ・横髪（左右 × XY） | 4 | ○ |
| 頭の傾き（XYZ） | 3 | ○ |
| 体の傾き（XY） | 2 | ○ |
| **合計** | **26 / 30** | **○（残り 4）** |

**呼吸・瞬き・髪揺れ・頭の傾きが入る。** メメントモリとの差は「衣装の揺れ」「装飾品」「全身の動き」がないこと。ノベルゲームの表示領域（バストアップ）では問題にならない。

### FREE 版 vs 静止画 vs メメントモリ

```
静止画の立ち絵:       表情差分を切り替えるだけ。動かない。
FREE 版 Live2D:      呼吸・瞬き・髪揺れ・頭の傾き。「イラストが生きている」感。
PRO 版 Live2D:       全身・衣装・装飾品・物理演算。メメントモリ級。
```

**「静止画」→「FREE 版 Live2D」の差は、「FREE 版」→「PRO 版」の差よりはるかに大きい。** ユーザーが最も感動するのは「動かない → 動く」の瞬間。

## 導入方針（改訂版）

### ~~当初の方針: Spine 先行~~ → **Live2D 先行に変更**

| 変更理由 | 詳細 |
|---------|------|
| 日本市場での圧倒的な普及度 | VTuber 産業・ソシャゲ業界で標準 |
| クリエイター供給の厚さ | BOOTH/ココナラで Live2D モデルが大量に流通 |
| パートナー獲得への効果 | 「Live2D 対応」は日本のゲーム会社への訴求力が高い |
| 日本製エンジン × 日本製ミドルウェア | ブランドストーリーとして強い |
| FREE 版でもノベルゲームに十分 | 追加コスト ¥0 で「動く立ち絵」が実現 |
| Ren'Py との差別化 | Ren'Py の Live2D 対応は不完全。kaedevn が本格対応すれば日本市場で唯一の選択肢 |

### Phase A: Live2D 統合（最優先、2-3週間）

| 項目 | 内容 |
|------|------|
| 対象 | ノベルゲーム + 運営型ゲームアプリ |
| 理由 | 日本市場での差別化に直結 |
| 工数 | 2-3週間 |
| ライブラリ | `untitled-pixi-live2d-engine`（PixiJS v8 + Cubism 2-5 対応） |

実装内容:
1. `untitled-pixi-live2d-engine` の統合
2. WebGL コンテキスト共有の解決
3. モーション/表情切替 API（`@show` コマンド拡張）
4. リップシンク連携（音声再生と連動）
5. `ChBlock` にアニメーションタイプ（static / live2d）の選択肢を追加
6. アセット管理に `.model3.json` + `.moc3` + テクスチャのセット管理を追加
7. エディタでモーション/表情プレビュー

### Phase B: Spine 統合（後続、3-5日）

| 項目 | 内容 |
|------|------|
| 対象 | ツクール型 + バトル演出 |
| 理由 | ドット絵キャラのアクションに向いている |
| 工数 | 3-5日 |
| コスト | エディタ $349（1回）、ランタイム無料 |

実装内容:
1. `npm i @esotericsoftware/spine-pixi-v8`
2. `WebOpHandler.ts` に Spine ロード/表示/モーション切替を追加
3. `ChBlock` にアニメーションタイプ `spine` を追加
4. アセット管理に `.json` + `.atlas` + `.png` のセット管理を追加

### ポジショニング

```
kaedevn のキャラアニメーション対応:

  Live2D → ノベルゲームの「動く立ち絵」（日本市場の差別化ポイント）
  Spine  → ツクール型のバトルキャラ（アクション向き）
  静止画 → 従来通り対応（最も軽量）

対応しているキャラアニメーション:
  ✓ 静止画（PNG/WebP）
  ✓ Live2D（Cubism 2-5、FREE/PRO 両対応）
  ✓ Spine（骨格アニメーション）

→ 「どんなキャラアセットでも動く」プラットフォーム
```
