# Live2D × PixiJS v8 パラメータ上書きのタイミング問題と解決策

> Generated by Claude Opus 4.6

## 問題

Live2D モデルのパラメータ（目の開閉、表情、顔の向き等）を外部から制御したい場合、**PixiJS の ticker でパラメータを書き換えても反映されない。**

## 原因

`untitled-pixi-live2d-engine`（および `pixi-live2d-display`）では、Live2D モデルの `update()` が **PixiJS の render パイプライン内**で呼ばれる。

```
PixiJS の 1 フレームの処理順序:

  1. ticker callbacks（ユーザーコード）  ← ここで上書きしても...
  2. render pipeline 開始
  3.   Live2DModel.renderLive2D()
  4.     internalModel.update()
  5.       motionManager.update()    ← モーションがパラメータを上書き
  6.       physics.update()          ← 物理演算がパラメータを上書き
  7.       eyeBlink.update()         ← 自動まばたきがパラメータを上書き
  8.     coreModel.draw()            ← この値で描画される
  9. render pipeline 終了

  → ticker (1) で設定した値は、(5)(6)(7) で上書きされて消える
```

## 解決策

`internalModel` の `afterMotionUpdate` イベントを使う。このイベントは **モーション・物理演算の適用後、描画前**に発火する。

```typescript
// パラメータ上書き用のマップ
const overrides: Record<string, number> = {};

// 生のパラメータ配列に直接アクセス
const rawModel = model.internalModel.coreModel._model;
const paramIds = rawModel.parameters.ids;
const paramValues = rawModel.parameters.values;

// パラメータ名 → インデックスのマップを作る
const paramMap: Record<string, number> = {};
for (let i = 0; i < paramIds.length; i++) {
  paramMap[paramIds[i]] = i;
}

// afterMotionUpdate: モーション適用後、描画前に呼ばれる
model.internalModel.on('afterMotionUpdate', () => {
  for (const [name, value] of Object.entries(overrides)) {
    const idx = paramMap[name];
    if (idx !== undefined) {
      paramValues[idx] = value;
    }
  }
});

// 使用例: 目を閉じる
overrides['ParamEyeLOpen'] = 0;
overrides['ParamEyeROpen'] = 0;

// 使用例: モーションに戻す（上書きを解除）
delete overrides['ParamEyeLOpen'];
delete overrides['ParamEyeROpen'];
```

## 正しい処理順序

```
  1. ticker callbacks
  2. render pipeline 開始
  3.   Live2DModel.renderLive2D()
  4.     internalModel.update()
  5.       motionManager.update()      ← モーションがパラメータを設定
  6.       physics.update()            ← 物理演算
  7.       eyeBlink.update()           ← 自動まばたき
  8.       emit('afterMotionUpdate')   ← ★ここで上書きする★
  9.     coreModel.draw()              ← 上書きされた値で描画される
  10. render pipeline 終了
```

## 注意点

### パラメータ ID はオブジェクト

Cubism 5 SDK では `coreModel.getParameterId(i)` は **文字列ではなく CubismId オブジェクト**を返す。文字列でパラメータ名を取得するには `_model.parameters.ids[i]` を直接参照する。

```typescript
// ❌ CubismId オブジェクトが返る（toString すると [object Object]）
const id = coreModel.getParameterId(i);

// ✅ 文字列が返る
const id = coreModel._model.parameters.ids[i]; // "ParamEyeLOpen"
```

### stopAllMotions() は使えない

`untitled-pixi-live2d-engine` の `motionManager.stopAllMotions()` は Cubism 5 SDK との内部 API 不整合で **クラッシュする**（`_motions.begin is not a function`）。描画ループが壊れて黒画面になる。

```typescript
// ❌ クラッシュする
model.internalModel.motionManager.stopAllMotions();

// ✅ 代わりに afterMotionUpdate で上書き
```

### model.motion() の優先度

`model.motion(group, index, priority)` の `priority` はデフォルト `NORMAL (2)`。同じ優先度のモーションが再生中だと **拒否される**（`result: false`）。`FORCE (3)` を指定しても、state の管理が不完全で連続呼び出しが効かないことがある。

```typescript
// 確実にモーションを切り替えるには state をリセットしてから
model.internalModel.motionManager.state?.complete();
await model.motion('TapBody', 0, 3); // FORCE
```

## 主要パラメータ一覧（Hiyori モデル）

| パラメータ ID | 用途 | 範囲 |
|-------------|------|------|
| `ParamAngleX` | 顔の左右 | -30 〜 30 |
| `ParamAngleY` | 顔の上下 | -30 〜 30 |
| `ParamAngleZ` | 顔の傾き | -30 〜 30 |
| `ParamEyeLOpen` | 左目の開閉 | 0（閉）〜 1（開） |
| `ParamEyeROpen` | 右目の開閉 | 0 〜 1 |
| `ParamEyeLSmile` | 左目の笑い | 0 〜 1 |
| `ParamEyeRSmile` | 右目の笑い | 0 〜 1 |
| `ParamEyeBallX` | 瞳の左右 | -1 〜 1 |
| `ParamEyeBallY` | 瞳の上下 | -1 〜 1 |
| `ParamMouthOpenY` | 口の開閉 | 0 〜 1 |
| `ParamMouthForm` | 口の形（笑顔） | -1 〜 1 |
| `ParamBodyAngleX` | 体の左右 | -10 〜 10 |
| `ParamBreath` | 呼吸 | 0 〜 1 |

## この問題が発生する条件

- Live2D モデルを PixiJS で表示している
- モーション再生中にパラメータを外部から制御したい
- PixiJS の ticker や requestAnimationFrame でパラメータを設定している

**Live2D + PixiJS を使う全てのプロジェクトで該当する。** ライブラリの README やドキュメントには記載されていない。
