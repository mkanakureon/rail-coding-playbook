# ブロックエディタ browser-verify テスト計画

> Generated by Claude Opus 4.6

## 目的

browser-verify スキル（Playwright MCP）を使って、ブロックエディタの UI を DOM / アクセシビリティツリーで構造的に検証する。画像判定ではなく、要素の存在・テキスト・状態を正確に確認する。

## 対象

- URL: `http://localhost:5176/projects/editor/{projectId}`
- 認証: API login → localStorage 注入（`test1@example.com` / `DevPass123!`）

## 実行順序

```
Phase 1 (基本構造)        ← まずここで画面が壊れてないか確認
  ↓
Phase 2 (追加メニュー)    ← メニューが開くか
  ↓
Phase 3 (ブロック追加)    ← 各ブロックが追加できるか
  ↓
Phase 5 (テキスト編集)    ← 最重要ブロックの編集
  ↓
Phase 4 (ブロック操作)    ← 並び替え・削除・複製
  ↓
Phase 6 (bg/ch 編集)      ← アセット選択系
  ↓
Phase 7 (ページ操作)      ← ページ管理
  ↓
Phase 8-10 (サイドバー・キーボード・レスポンシブ)
```

## 検証時の注意

- **START ブロックは不変**: 削除・移動不可。テストで誤って操作しない
- **認証必須**: API login → localStorage 注入してからエディタを開く
- **ブロック ID**: `data-block-id` 属性でブロックを特定する
- **data-testid**: SidebarInspector と TimelinePreview にのみ存在。他のコンポーネントは `data-block-id` やテキストで特定する
- **非同期ロード**: プロジェクトデータの読み込みを `browser_wait_for` で待つ
- **オートセーブ**: 60秒間隔。テスト中の変更は API に保存される可能性がある
