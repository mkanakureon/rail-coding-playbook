# エディタ UI 要素調査ログ

> Generated by Claude Opus 4.6

## 背景

デモ動画録画（Playwright `recordVideo`）のために、エディタ画面のボタン・ブロック要素のセレクタを調査した。
ゲストログイン → エディタ遷移後の UI 要素を Playwright で自動取得。

## 調査方法

- `scripts/inspect-editor.ts` — 全ボタン・`data-testid`・`data-block-*` 要素を列挙
- `scripts/inspect-editor2.ts` — 「+ ブロック追加」クリック後のメニュー要素を列挙
- 両スクリプトとも headless Chromium で実行

## セレクタガイド（Playwright テスト用）

```typescript
// ブロック追加メニューを開く
await page.locator('button:has-text("ブロック追加")').click();

// ブロックタイプを選択
await page.locator('button:has-text("テキスト")').first().click();

// 既存ブロックをクリック
await page.locator('[data-block-id^="bg-"]').click();

// プレビュー実行
await page.locator('button:has-text("実行")').click();

// サイドバータブ切替（短いテキストでフィルター）
await page.locator('button').filter({ hasText: 'KS' }).first().click();

// 保存
await page.locator('button:has-text("保存")').click();
```

## デモ動画録画結果

上記調査に基づき `scripts/record-demo.ts` を更新して録画成功。

**録画フロー:**
1. ランディングページ → ゲストログイン
2. エディタで既存ブロック（背景・キャラ・テキスト）クリック
3. 5種類のブロック追加（選択肢・変数・IF文・FX・ジャンプ）
4. サイドバータブ切替（KS・キャラ・アセット・作品設定・ブロック）
5. ▶実行 → プレビュー画面でクリック進行

**保存先:** `recordings/*.webm`

## 教訓

- エディタは `data-testid` を一切使っていない → テキストベースのセレクタが必須
- `button:has-text()` が最も安定したセレクタ
- ブロック追加メニューはクリックで展開されるインラインUI（`role="menu"` 等は未使用）
- ゲストログインボタンは2箇所にある → `.first()` が必要
