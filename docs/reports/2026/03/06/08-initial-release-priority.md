## 優先度: Must（リリースブロッカー）

初期リリース前に**必ず**対応が必要なもの。

### M1. Editor CORS 設定

**現状**: Editor SWA → API 間の CORS が未設定。アセット管理パネルは開くがアセット一覧は空。
**影響**: 本番環境でアセット選択・保存が動かない。
**作業量**: 小（Azure SWA の設定 or API 側の CORS origin 追加）

### M2. Preview SPA fallback routing

**現状**: Preview SWA で SPA fallback routing 未設定。ディープパス（`/preview/{id}/{page}`）が 404。
**影響**: 「実行」ボタンでプレビューが表示されない。
**作業量**: 小（`staticwebapp.config.json` に fallback ルール追加）

### M3. 保存前の最低限バリデーション

**現状**: assetId/characterId が空のまま保存できてしまう。プレビューで空コマンドが生成される。
**影響**: 作者が意図しない壊れたスクリプトが生成される。
**作業量**: 小〜中（buildPreviewScript / buildSnapshotScript で空チェック + UI に警告表示）
**方針**: 保存をブロックするのではなく、未設定ブロックに視覚的な警告（黄色枠等）を出す

## 優先度: Should（リリース品質向上）

なくてもリリースできるが、あると初期ユーザーの体験が大幅に向上するもの。

### S1. オートセーブ

**現状**: 手動保存のみ。ブラウザを閉じると編集内容が消える。
**影響**: 作者が保存を忘れて作業内容を失う。
**作業量**: 中
**方針**: 30秒〜1分間隔でデバウンス付きオートセーブ。保存中インジケーター表示。変更検知は `updatedAt` で判定

### S2. ページリネーム

**現状**: TODO コメントあり、未実装。ページ名は「第N話」固定。
**影響**: 作者がページの内容を識別しにくい。
**作業量**: 小（HamburgerMenu / PageNavigator にインライン編集追加）

### S3. 未保存変更の離脱警告

**現状**: なし。ブラウザバック/タブ閉じで確認なく消える。
**影響**: S1（オートセーブ）があれば緩和されるが、それでも直近の変更は失われる。
**作業量**: 小（`beforeunload` イベント + 変更フラグ管理）

### S4. config/frontend.ts の本番 URL 設定

**現状**: TODO コメント「本番環境では適切な URL に変更」。
**影響**: プロジェクト一覧・マイページへのリンクが本番で壊れる可能性。
**作業量**: 小（環境変数 or ビルド時定数で切り替え）

**判断**: 初期ターゲットは日本語ユーザー。需要が出てから対応。

### N3. ページ並べ替え

**現状**: 未実装。
**判断**: ページ数が少ないうちは手動対応可能。

### N4. ブロック検索

**現状**: 未実装。
**判断**: ブロック数が少ないうちは目視で対応可能。

### N5. README / ドキュメント整備

**現状**: なし。
**判断**: 初期は限定公開。利用者が増えてから整備。
