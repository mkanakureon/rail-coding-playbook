# ゲーム画面 状態管理統一設計書 — Web (PixiJS) / Native (SDL2) 共通化

> Generated by Claude Opus 4.6
> 作成日: 2026-03-28

## 背景

### 動機

Qiita 記事「[MV(R)P パターン](https://qiita.com/toRisouP/items/5365936fc14c7e7eabf9)」を参考に、ゲーム画面（プレイ画面）の UI 更新パターンを見直す。対象はブラウザのプレイ画面（`packages/web`）と SDL2 ネイティブエンジン（`packages/native-engine`）の 2 つ。エディタやランディングページは対象外。

### 現状の問題

1. **Native 側のブロッキングループ**: `waitForClick()` / `showChoices()` がスレッドをブロックし、自前で render ループを回している。idle アニメーション・パーティクル・フィルター時間更新ができない
2. **状態管理が完全に別実装**: Web は `WebOpHandler` 内の Map/変数（2,100 行）、Native は `SDL2Engine` 内の struct/float（800 行）。同じゲーム状態を別々に管理している
3. **Handler インターフェースの乖離**: Web は `IOpHandler`（Promise ベース）、Native は `IEngineAPI`（void, ブロッキング）。同じ Op IR を実行するのに別インターフェース

### 参考: MV(R)P パターンの適用方針

記事の核心は「**状態変更→表示更新を分離する**」こと。ゲーム画面は状態が少ない（BG、キャラ数体、テキスト、選択肢）ため、本格的な ReactiveProperty フレームワークは不要。考え方だけ取り入れ、実装はステートマシン + シンプルな Observer で十分。

## 現状のアーキテクチャ

### 共通部分（packages/core）

```
Op IR (types/Op.ts)  — 30+ 命令型、プラットフォーム非依存
OpRunner.ts          — async/await の実行ループ（Web 用）
IOpHandler.ts        — Promise<void> ベースの 40+ メソッド
ViewState.ts         — { bgId, characters[], bgmId, bgmVol }
```

### Web 実装（packages/web）

```
OpRunner (core)
  ↓ await handler.bgSet() / handler.waitClick() ...
WebOpHandler (2,100+ 行)
  ├── 状態: currentBgId, currentCharacters (Map), textBuffer, skipMode ...
  ├── アニメーション: Ticker + fadeTo() → Promise
  ├── 描画: PixiJS DisplayObject 直接操作
  └── 入力: InputManager → Action イベント
```

**特徴:**
- `waitClick()` は `await messageWindow.show()` — ノンブロッキング。Ticker が回り続ける
- アニメーションは `Promise.all([fadeTo(old), fadeTo(new)])` で並列実行
- テキストは Ticker で 1 文字ずつ表示、skip 時は即時表示

### Native 実装（packages/native-engine）

```
Interpreter.step() — 同期実行、@l で return
  ↓ engine.showBackground() / engine.waitForClick() ...
SDL2Engine (800+ 行)
  ├── 状態: currentBg, prevBg, transitionProgress, activeCharacters[], currentText
  ├── アニメーション: render() 内で transitionProgress += 0.015f
  ├── 描画: IRenderer::drawSprite() / drawText()
  └── 入力: SDL_PollEvent
```

**特徴:**
- `waitForClick()` は `while(!clicked){ render(); SDL_Delay(16); }` — **スレッドブロック**
- `showChoices()` も同様にブロッキング
- メインループが 2 重構造（main loop + wait 内 loop）

### 差分の詳細

| 項目 | Web | Native | 差の原因 |
|------|-----|--------|---------|
| waitClick | `await` (ノンブロッキング) | `while` loop (ブロッキング) | JS は非同期必須、C++ は同期可能 |
| showChoices | `await` overlay | `while` loop + キー検出 | 同上 |
| BG 遷移 | fadeTo() Promise | `transitionProgress` float | アニメーション管理方式の違い |
| キャラ管理 | `Map<string, Container>` | `vector<ActiveChar>` | 言語の違い |
| テキスト送り | Ticker で charAccum 加算 | render() で毎回全文描画 | PixiJS Text vs TTF_Render |
| Handler I/F | `IOpHandler` (Promise) | `IEngineAPI` (void) | 別インターフェース |

## 設計: GameScreenState による統一

### 設計原則

```
MV(R)P 的分離:
  Model     = GameScreenState（ゲーム画面の状態データ）
  View      = Renderer（PixiJS / SDL2+GL、プラットフォーム固有）
  Presenter = update() ロジック（状態遷移・アニメーション進行、共通化可能）
```

### GameScreenState の定義

`packages/core` に配置。両エンジンがこの型を共有する。

```typescript
// packages/core/src/types/GameScreenState.ts

/** 背景の遷移状態 */
interface BgTransition {
  fromId: string;
  toId: string;
  effect: "fade" | "slide_left" | "slide_right";
  progress: number;  // 0.0 → 1.0
  durationMs: number;
  elapsedMs: number;
}

/** キャラクター表示状態 */
interface CharacterState {
  name: string;
  pose: string;
  pos: "left" | "center" | "right";
  alpha: number;       // 0.0 → 1.0（フェード中の透明度）
  focused: boolean;    // 話者ならtrue
}

/** テキスト表示状態 */
interface TextState {
  content: string;     // 全文
  speaker: string;
  displayedChars: number;  // 表示済み文字数（文字送り用）
  speed: number;       // 文字/秒
}

/** 選択肢状態 */
interface ChoiceState {
  options: { label: string; jump: number }[];
  selected: number;    // -1 = 未選択
}

/** ゲーム画面の全状態 */
interface GameScreenState {
  // シーン状態
  sceneState: "executing" | "wait_click" | "wait_choice" | "transitioning";

  // 背景
  bgId: string | null;
  bgTransition: BgTransition | null;

  // キャラクター
  characters: Map<string, CharacterState>;

  // テキスト
  text: TextState | null;

  // 選択肢
  choices: ChoiceState | null;

  // オーディオ（参照のみ、再生はプラットフォーム固有）
  bgmId: string | null;
  bgmVol: number;

  // ユーザー操作モード
  skipMode: boolean;
  autoMode: boolean;

  // クリック待ちアイコン表示
  waitingForClick: boolean;
}
```

### update() の共通化

アニメーション進行ロジックを core に置く。Renderer は状態を読んで描画するだけ。

```typescript
// packages/core/src/engine/GameScreenUpdater.ts

function update(state: GameScreenState, deltaMs: number): void {
  // BG 遷移の進行
  if (state.bgTransition) {
    state.bgTransition.elapsedMs += deltaMs;
    state.bgTransition.progress = Math.min(
      state.bgTransition.elapsedMs / state.bgTransition.durationMs,
      1.0
    );
    if (state.bgTransition.progress >= 1.0) {
      state.bgId = state.bgTransition.toId;
      state.bgTransition = null;
    }
  }

  // テキスト文字送り
  if (state.text && state.text.displayedChars < state.text.content.length) {
    const charsToAdd = state.text.speed * (deltaMs / 1000);
    state.text.displayedChars = Math.min(
      state.text.displayedChars + charsToAdd,
      state.text.content.length
    );
  }

  // キャラクターフェード進行
  for (const ch of state.characters.values()) {
    // alpha の補間（追加/退場時）
    // ...
  }
}
```

C++ 側も同等のロジック:

```cpp
// packages/native-engine/src/engine/GameScreenUpdater.hpp

void update(GameScreenState& state, float deltaMs) {
    // BG 遷移
    if (state.bgTransition.has_value()) {
        auto& t = state.bgTransition.value();
        t.elapsedMs += deltaMs;
        t.progress = std::min(t.elapsedMs / t.durationMs, 1.0f);
        if (t.progress >= 1.0f) {
            state.bgId = t.toId;
            state.bgTransition = std::nullopt;
        }
    }
    // テキスト文字送り
    if (state.text.has_value()) {
        auto& txt = state.text.value();
        float charsToAdd = txt.speed * (deltaMs / 1000.0f);
        txt.displayedChars = std::min(
            txt.displayedChars + charsToAdd,
            (float)txt.content.size()
        );
    }
}
```

## 実装計画: 3 ステップ

### Step 1: Native のブロッキング廃止（最優先）

**目的:** メインループを 1 つにして、待ち中もアニメーション・描画が継続するようにする。

**Before:**

```cpp
// SDL2Engine.cpp — waitForClick()
void SDL2Engine::waitForClick() {
    if (skipping) return;
    bool clicked = false;
    while (!clicked) {
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_MOUSEBUTTONDOWN) clicked = true;
        }
        render();       // ← ここで自前 render ループ
        SDL_Delay(16);
    }
}

// main.cpp — Interpreter が呼ぶ
while (!interpreter.isFinished()) {
    interpreter.step();  // ← step() 内で waitForClick() がブロック
}
```

**After:**

```cpp
// SDL2Engine — 状態を変えるだけ、ブロックしない
void SDL2Engine::requestWaitClick() {
    sceneState = SceneState::WaitClick;
}

void SDL2Engine::requestChoices(const std::vector<std::string>& choices) {
    sceneState = SceneState::WaitChoice;
    pendingChoices = choices;
}

// main.cpp — メインループは 1 つだけ
while (!quit) {
    float deltaMs = /* 前フレームからの経過 */;

    // 1. イベント処理
    while (SDL_PollEvent(&event)) {
        switch (sceneState) {
            case SceneState::WaitClick:
                if (isClick(event)) {
                    sceneState = SceneState::Executing;
                    interpreter.step();  // 次の命令へ
                }
                break;
            case SceneState::WaitChoice:
                if (isChoiceSelect(event, &selected)) {
                    sceneState = SceneState::Executing;
                    interpreter.jumpTo(selected);
                    interpreter.step();
                }
                break;
            default:
                break;
        }
    }

    // 2. 状態更新（アニメーション進行）
    update(gameScreenState, deltaMs);

    // 3. 描画（状態を読むだけ）
    render(gameScreenState);

    SDL_Delay(16);
}
```

**変更対象ファイル:**
- `packages/native-engine/src/main.cpp` — メインループ書き換え
- `packages/native-engine/src/engine/SDL2Engine.cpp` — waitForClick/showChoices をノンブロッキング化
- `packages/native-engine/src/engine/SDL2Engine.hpp` — SceneState enum 追加
- `packages/native-engine/src/interpreter/Interpreter.cpp` — step() のブロッキング呼び出し除去

**リスク:** Interpreter.step() の呼び出しタイミングが変わる。step() が複数命令を一度に実行する現在の挙動を、1 命令ずつに変更する必要がある。

### Step 2: GameScreenState を core に定義し両エンジンで共有

**目的:** 状態管理のコードを共通化し、WebOpHandler と SDL2Engine から状態管理の責務を分離する。

**新規ファイル:**
- `packages/core/src/types/GameScreenState.ts` — 状態型定義
- `packages/core/src/engine/GameScreenUpdater.ts` — update() ロジック
- `packages/native-engine/src/engine/GameScreenState.hpp` — C++ 版の同等定義
- `packages/native-engine/src/engine/GameScreenUpdater.hpp` — C++ 版 update()

**変更ファイル:**
- `packages/web/src/renderer/WebOpHandler.ts` — 内部状態を GameScreenState に移行。描画ロジックのみ残す
- `packages/native-engine/src/engine/SDL2Engine.cpp` — 同上
- `packages/core/src/types/ViewState.ts` — GameScreenState から導出する形に変更

**WebOpHandler の分割イメージ:**

```
Before (2,100+ 行):
  WebOpHandler = 状態管理 + アニメーション進行 + 描画 + 入力

After:
  GameScreenState (core)     = 状態データ（200行）
  GameScreenUpdater (core)   = アニメーション進行（150行）
  WebOpHandler (web)         = 描画 + 入力のみ（700行程度）
  PixiJSRenderer (web)       = GameScreenState → PixiJS 描画（500行程度）
```

**SDL2Engine の分割イメージ:**

```
Before (800+ 行):
  SDL2Engine = 状態管理 + render() 内アニメーション + 描画 + 入力

After:
  GameScreenState (hpp)      = 状態データ（100行）
  GameScreenUpdater (hpp)    = アニメーション進行（100行）
  SDL2Engine (cpp)           = 描画 + 入力のみ（400行程度）
```

### Step 3: 低頻度状態に Observer を追加

**目的:** BG 切替・キャラ登退場など、変更頻度が低い状態の変更時に描画側のリソース管理（テクスチャロード、スプライト生成/破棄）を自動化する。

**対象と方式:**

| 状態 | 変更頻度 | Observer の役割 |
|------|---------|----------------|
| `bgId` | シーン切替時 | 新テクスチャのプリロード開始 |
| `characters` (追加/削除) | シーン切替時 | スプライト生成/破棄 |
| `choices` (null↔有値) | 選択肢表示時 | 選択肢 UI の表示/非表示 |
| `waitingForClick` | テキスト表示完了時 | クリックアイコン表示/非表示 |
| `skipMode` / `autoMode` | ユーザー操作時 | テキスト速度変更 |

**Observer 不要（update() で直接更新）:**

| 状態 | 理由 |
|------|------|
| `bgTransition.progress` | 毎フレーム変化、Reactive のオーバーヘッドが無駄 |
| `text.displayedChars` | 同上 |
| `character.alpha` | フェード中は毎フレーム変化 |

**実装:**

```typescript
// packages/core/src/engine/GameScreenObserver.ts
type Callback<T> = (newVal: T, oldVal: T) => void;

class GameScreenObserver {
  private bgListeners: Callback<string | null>[] = [];
  private charListeners: Callback<Map<string, CharacterState>>[] = [];

  onBgChange(fn: Callback<string | null>) { this.bgListeners.push(fn); }
  onCharactersChange(fn: Callback<Map<string, CharacterState>>) { this.charListeners.push(fn); }

  // Handler が状態変更時に呼ぶ
  notifyBgChange(newId: string | null, oldId: string | null) {
    if (newId === oldId) return;
    this.bgListeners.forEach(fn => fn(newId, oldId));
  }
}
```

```cpp
// C++ 版
template<typename T>
class Observable {
    T value_;
    std::vector<std::function<void(const T&, const T&)>> listeners_;
public:
    void set(const T& v) {
        if (v == value_) return;
        T old = value_;
        value_ = v;
        for (auto& fn : listeners_) fn(value_, old);
    }
    const T& get() const { return value_; }
    void subscribe(std::function<void(const T&, const T&)> fn) {
        listeners_.push_back(std::move(fn));
    }
};
```

## 差分の変化

### 共通化できる部分（差がなくなる）

| 層 | Before | After |
|---|---|---|
| ゲーム状態定義 | Web: WebOpHandler 内 Map/変数, Native: SDL2Engine 内 struct | `GameScreenState` 1 つ |
| アニメーション進行 | Web: Ticker callback, Native: render() 内 float 加算 | `update(state, dt)` 共通関数 |
| セーブ/ロード | Web: ViewState, Native: GameState (別物) | GameScreenState からスナップショット導出 |
| 実行モデル | Web: async/await, Native: ブロッキング | 両方ステートマシン（executing/wait_click/wait_choice） |

### 残る差（プラットフォーム固有、消せない）

| 層 | Web | Native | 理由 |
|---|---|---|---|
| 描画 API | `sprite.alpha = x` | `glUniform1f(uAlpha, x)` | グラフィックス API の違い |
| テクスチャ読み込み | URL fetch + PixiJS Assets | ファイルパス + stb_image | I/O モデルの違い |
| テキスト描画 | PixiJS Text (Canvas 2D) | TTF_RenderUTF8 + GL テクスチャ | フォントレンダリングの違い |
| 入力取得 | PointerEvent / KeyboardEvent | SDL_PollEvent | イベントシステムの違い |
| 音声再生 | Web Audio API | SDL_mixer | オーディオ API の違い |
| シェーダー | PixiJS Filter (GLSL ES 1.0) | GLSL ES 3.0 | シェーダー言語バージョン |
