## 説明文案

```
AI（Claude Opus 4.6）がビジュアルノベルエンジン「kaedevn」のブロックエディタ E2E テストを自律的に実行する様子を録画しました。

■ テスト内容
1. API でプロジェクトを新規作成（タイムスタンプ付き）
2. ブラウザでログイン → マイページ → プロジェクト詳細
3. エディタで「+ ブロック追加」→ 背景ブロックを追加
4. 背景ブロックの「変更」→ 公式アセット → ファンタジーカテゴリ → 画像選択 → 確定
5. 「+ ブロック追加」→ テキストブロックを追加 → 台詞を入力
6. 全6ブロックを順番にクリック確認（赤パルスエフェクト付き）
7. 実行ボタンからプレイ画面を起動 → canvas クリックでシナリオ完走

■ 特徴
- OBS テキストオーバーレイ連携: 各フェーズ・クリックごとに「→ 〇〇をクリック」と表示
- 公式アセット選択: モーダル内で公式タブ → ファンタジーカテゴリ → 画像選択 → 確定
- テキスト入力: テキストブロックを展開し textarea に台詞を自動入力（50ms delay）
- プレイ完走: Scenario completed をコンソールログで検出

■ 技術スタック
- エンジン: kaedevn（PixiJS / WebGL ベースのビジュアルノベルエンジン）
- エディタ: React + Zustand（Vite, port 5176）
- プレビュー: PixiJS OpRunner（Vite, port 5175）
- API: Hono + Prisma + PostgreSQL（port 8080）
- フロントエンド: Next.js（port 3000）
- テスト: Playwright（headless: false で実行画面を表示）
- OBS 連携: obs-websocket-js でテキストオーバーレイを動的更新
- AI: Claude Opus 4.6（Claude Code CLI）

■ ポイント
- UI 操作でブロック追加（API ではなくブラウザ上の「+ ブロック追加」ボタン）
- AssetSelectModal の公式タブからファンタジー背景を選択
- テキストブロック展開 → textarea 入力まで全自動
- 所要時間: 約2.5分

■ リポジトリ
kaedevn-monorepo（プライベート）
```

## OBS テキスト連携

`node scripts/obs-stream.mjs text "メッセージ"` で OBS の StatusText ソースを動的更新。
- 各フェーズ開始時にフェーズ名を表示
- `pulseAndClick` 内で「→ 〇〇をクリック」を表示
- テスト完了時に「テスト完了！」、終了時にクリア

## 修正した問題

- **AssetSelectModal のサムネイルクリック問題**: `renderThumbnail` の `onClick` で `stopPropagation` + プレビュー表示 → カードの選択が発火しない。修正: サムネイル画像クリックで選択、プレビューはホバー時の🔍ボタンに変更
- **「ファンタジー」ボタン重複**: サイドバーのキャラ管理と公式カテゴリチップの両方にマッチ → `getByRole('button', { name: 'ファンタジー', exact: true })` に変更
- **2回目の「+ ブロック追加」が開かない**: アセット選択後のページ状態でクリックが効かない → リトライロジック追加
