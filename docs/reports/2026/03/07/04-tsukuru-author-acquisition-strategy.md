# ツクール作者獲得戦略: kaedevn で「現代のツクール」を作る

## 概要

kaedevn は現在ノベルゲーム PF としてベータ版運用中。次の成長段階として **RPGツクール作者の取り込み** を目指す。

ビジョン: **現代の技術でツクールを作るとどうなるか？ それを作者に提供する**

## 元ネタ文書の分析

### 文書1: ChatGPT 壁打ち (9754行)

技術設計の深掘り。主要な成果物:

1. **プレイ画面のレイヤー構造** が確定（PlayRoot → SceneLayer + UiLayer）
2. **制約付きスロット配置システム** の設計（完全自由配置は危険、制約付きが正解）

4. **収益モデル**: 作成・公開は無料、販売時に 20-25% 手数料
5. **必須エディタ機能**: DB管理、イベントコマンドGUI、シーンフロー可視化、リアルタイムプレビュー

## 戦略の全体像

### kaedevn がツクールに勝てる理由

| 観点 | RPGツクール | kaedevn |
|------|-----------|---------|
| エディタ | Windows 専用デスクトップアプリ | **ブラウザ（OS不問）** |
| Switch 出版 | 個人ではほぼ不可能 | **PF が Switch 出版を代行** |
| Web 公開 | 別途対応必要 | **標準対応** |
| スマホ対応 | ほぼ不可 | **Android/iOS 対応** |
| マルチ PF | 各 PF 個別対応 | **1プロジェクトから全 PF 同時リリース** |
| PC 要件 | 普通（プラグインで重くなる） | **メモリ 8GB の普通の PC** |
| AI 補助 | なし | **AI 執筆支援パイプライン** |
| レイアウト管理 | Prefab（diff 困難） | **JSON（git diff 可能、AI 生成向き）** |
| 初期コスト | 買い切り（MZ: 約9,000円） | **無料** |

### kaedevn が勝てない部分（正直に認める）

| 観点 | RPGツクール | kaedevn |
|------|-----------|---------|
| RPG システム | 戦闘・マップ・DB 完備 | **Phase 1 では対象外** |
| プラグイン資産 | 膨大なコミュニティ資産 | **まだない** |
| 実績 | 数千タイトル、商用成功事例多数 | **ベータ版** |
| コミュニティ | 大規模 | **これから** |

→ **正面衝突は避け、「ノベル/ADV でSwitch に出したい」層を狙う**

## 技術的な根拠: なぜ実現可能か

### 壁打ちで確認済みの設計

1. **ツクール UI は2種類で再現できる**
   - 固定配置 UI（メッセージ、HP表示等）= 絶対座標で配置
   - リスト UI（アイテム一覧、セーブ等）= ListView コンポーネント
   - SDL2 で Window, List, Button, Panel の4クラスを作れば十分

2. **Unity UGUI より実装コストが低い**
   - Anchor/Pivot/Stretch の複雑な組み合わせが不要（1280x720 固定）
   - JSON レイアウト = git diff 可能、AI が生成しやすい
   - ブラウザエディタが自由に進化できる（SDL 側に影響しない）

3. **プレイ画面 → エディタ → Switch の一気通貫**
   - ブラウザエディタで配置変更 → JSON 保存
   - SDL2 ランタイムは確定座標を読むだけ（編集機能なし）
   - `compileLayoutForSdl()` で変換、Switch 向けは固定レイアウト

4. **Switch 解像度は 1280x720 固定で十分**
   - 携帯モード: そのまま表示
   - TV モード: GPU スケーリングで 1080p に拡大
   - 背景は 1920x1080 で作成 → カメラパン・ズーム演出可能

## まとめ

kaedevn がツクール作者を獲得できる核心的な理由:

1. **Switch 出版を PF が代行する** — 個人では実現困難なことを、PF として実現する
2. **ブラウザだけで完結する** — インストール不要、OS 不問、低スペック PC で動作
3. **1プロジェクトから全 PF 同時リリース** — Web/Android/iOS/Switch を1つのプロジェクトから
4. **現代の技術スタック** — JSON レイアウト、AI 補助、git 管理、ブラウザエディタ
5. **無料で始められる** — 売れたときだけ手数料

「ツクールを置き換える」のではなく、「ツクールでは届かない場所（Switch・Web・スマホ）に作品を届ける手段」として位置づける。
