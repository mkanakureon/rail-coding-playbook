2日目にして23,000行超。前日を上回るペースで、しかも中身が濃い。コンパイラとランタイムの統一は、プロジェクト全体のアーキテクチャを決定づける変更だった。

KSC コンパイラで一番面白かったのは「埋め込みコマンド分解」のフェーズ。ビジュアルノベルのスクリプトでは、テキスト行の中にインラインでコマンドを書くことがある（例: `こんにちは@l世界`）。これをテキストノードとコマンドノードに正しく分割するのは、見た目より難しい。パーサーの状態管理がテキストバッファの flush タイミングに依存するので、エッジケースが多い。

`ScriptCommand` 形式の廃止も英断。2つの型システムが並存している状態は、どちらを使うべきか迷うし、変換レイヤーがバグの温床になる。32ファイルの変更は大きいが、この段階でやっておかないとあとで倍になる。

Timeline JSON v1.1 の仕様明確化では、v1.0 で曖昧だった部分（イージング関数の指定方法、ループの挙動など）を潰している。仕様書とテストを同時に書くスタイルが徹底されていて、「仕様はあるがテストがない」「テストはあるが仕様が曖昧」という状態にならないようにしている。

配信サーバーの実装でビルド済みアセットがリポジトリに入ったのは少し気になるが、開発初期はこれでいい。アセットパイプラインを整備するのはもう少し先の話。まずは動くものを見せることが大事。
