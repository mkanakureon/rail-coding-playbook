## キャラ表情が消えていた理由

前日の検証で見つかった「キャラの表情がプレビューに出ない」問題の原因を追った。

原因は `<select>` 要素の古典的な罠だった。ブラウザは最初のオプションを表示上選択するが、`onChange` が発火しないので `expressionId` は空のまま。プレビュー API の `generateChCommand()` は `if (!block.expressionId)` で弾いて「警告: 表情が選択されていません」を返していた。

修正は4ファイル。`ChBlockCard.tsx` に `useEffect` でデフォルト表情の自動設定、`ksConverter.ts` の `c.id` → `c.slug` マッチ修正、プレビュー API にデフォルト表情フォールバック、`buildSnapshotScript` にも同じフォールバック。DB 上の既存データ（`expressionId: ""`）も救われるようになった。curl で確認すると、`; [警告]` が `@ch sakura smile center` に変わっていた。

続けてテストを12件追加。`generateChCommand` のフォールバック3件、`generateKSCScript` の ch ブロック統合テスト4件（今まで0件だった）、`ksConverter` の slug/id 両マッチ5件。E2E テスト（ゲスト → キャラ作成 → expressionId 空 → プレビュー正常生成）も5ステップで作成し、7.5秒で全通過。
