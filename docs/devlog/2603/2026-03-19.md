## Phase 1 実装——OGP・ワンタップ起動・シーン共有

設計書が固まると、すぐに Phase 1 の実装に入った。play ページを Server Component と Client Component に分離し、`generateMetadata()` で作品ごとの動的 OGP を生成。curl で叩くと og:title から twitter:card まで12タグが綺麗に出力される。

`?quick=1` パラメータでタイトル画面をスキップする「ワンタップ起動」は、`ksc-demo.ts` の既存 `autostart=1` 条件に1行追加するだけで実現。プレイ中の右上にはシェアボタンを配置し、iframe 間の `postMessage` でキャンバスをキャプチャ → Web Share API で SNS 共有。非対応ブラウザ向けに X / LINE / URLコピーのフォールバックダイアログも用意した。

Pillow で生成したデフォルト OGP 画像（1200x630、グラデーション + ロゴ）も `/public/ogp-default.png` に配置。5ファイル、570行の追加で Phase 1 完了。

## Phase 2 前半——ポイント課金の全レイヤーを貫通

午後は Phase 2 に突入。Prisma スキーマに Wallet / PointTransaction / WorkPurchase の3テーブルを追加し、Work モデルに `price` / `freeUntil` フィールドを追加。`prisma.$transaction` で購入者の残高減算と作者への90%分配を7ステップでアトミック実行する購入APIを実装した。

フロントは `/mypage/wallet`（チャージ画面 + 取引履歴）と `/mypage/revenue`（作者収益ダッシュボード）の2ページを新規作成。300円/500円/1,000円の3プランでモック決済が動くようになった。

途中で Prisma の `output` パスが monorepo 環境で `@prisma/client` の型と衝突する問題が発覚。カスタム output を削除して標準パスに戻すことで解決——踏まないとわからない罠だった。

## Phase 2 後半——ペイウォール・埋め込み・販売設定

ペイウォールは `ksc-demo.ts` の ticker で `_page` 変数を毎フレーム監視し、`freeUntil` ページを超えたら `runner.stop()` → `postMessage({ type: 'paywallReached' })` で親ウィンドウに通知する方式。購入完了後は `paywallUnlocked` メッセージで `runner.resume()` を呼び、ページリロードなしで続きが読める。

`/embed/[id]` で外部サイト埋め込み用の軽量プレイヤーも追加。公開ダイアログには無料/有料ラジオ、価格入力、無料区間設定の GUI を追加した。

## テスト——API 8件 + E2E 購入フロー 10ステップ

締めくくりはテスト。Wallet API と Works Purchase API の認証チェック計8件、OGP メタタグ検証3件、ウォレットフロー6ステップ。

ペイウォールの ticker 監視も印象に残っている。シナリオスクリプトに特殊コマンドを追加せず、メタデータ（freeUntil）だけでペイウォールが機能するのは、作者の負担を考えると正しい判断だったと思う。`paywallUnlocked` → `runner.resume()` でリロードなしに続きが読める体験は、没入感の維持に直結する。
