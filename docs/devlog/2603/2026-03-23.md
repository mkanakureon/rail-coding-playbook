## タイトル画像が壊れていた理由

朝一番に取り掛かったのは、タイトル画像のリンク切れ修正。原因を追ったところ、`resolveAssetUrl(CONTAINER, blobPath)` と2引数で呼んでいたが、関数は1引数しか受け取らない——という、TypeScript なのに型で防げなかったバグだった。全サムネイルの URL が `/uploads/assets` になっていた。`storage.getUrl()` に統一して解消。ついでに「実行画面から選択」ボタンの撮影ボタンが表示されない問題も修正。`captureWinRef` が `useRef` ではなく普通のオブジェクトだったため、毎レンダーで `null` にリセットされていた。3ファイル、23行の追加。Playwright で Jaeger にトレースを送って動作確認まで行った。

## Claude Code の動きを OpenTelemetry で可視化する

Zenn の記事に触発されて、Claude Code のセッションログ（JSONL）を OpenTelemetry のトレースに変換するツールを作った。`tools/otel-viewer/` に 3ファイル、470行。Jaeger（Docker）で可視化すると、55ターン・467ツール呼び出しの3層ウォーターフォール（session → turn → tool）が一望できる。OTel SDK v2 の API 変更と Jaeger の時刻単位（マイクロ秒）に2時間ほどハマったが、最終的には `BatchSpanProcessor` + 明示的な parent context 指定で解決。

## 背景3枚で30場面——フィルター22種と重ね掛け

この日最大の変更。17ファイル、855行の追加。

カクヨム・なろうの小説作者がノベルゲームを作るとき、背景画像の少なさが最大のハードル。フィルターで1枚の「教室」から朝・夕・夜・雨・回想・ホラーと変化させれば、3枚の背景で30場面が作れる。この発想をもとに、既存18種に加えて22種のフィルターを新設した。

時間帯（朝・夕方・黄昏・月明かり）、天候（曇り）、季節感（紅葉・冬）、雰囲気（ほんわか・緊張・哀愁・懐かしい・ロマンチック・ホラー・ミステリアス・穏やか）、色調（色褪せ・暖色・寒色・鮮やか・くすみ・明るく・暗く）。すべて `ColorMatrixFilter` のみで実装し、GLSL を1行も書かずに済んだ。

さらに `filter_mix` ブロックを新設。2〜4層の重ね掛けで「夜+雨+哀愁」のような組み合わせが可能。色調系は ColorMatrix の行列積で1パスに合成するため、何枚重ねてもパフォーマンスコストはゼロ。天候系・ポスト処理系は各1つまでの制約を設け、UI に黄色い警告を表示するようにした。

コンパイラ（`@filter_mix` コマンド）、エンジン（`FILTER_MIX` Op）、エディタ UI、プレビュー、API スキーマの全レイヤーを一括で通した。コンパイラテスト7件、E2Eテスト（Playwright）も追加。

## レビュー18件を読んで対応する

外部レビュー18件 + PENDING_ACTION_ITEMS + スケーラビリティ分析 + 成長ロードマップを精読し、コードで対応できる指摘を洗い出した。

セキュリティ系を最優先で処理。動画生成の `execSync` → `exec` 非同期化、ファイル削除のパストラバーサル対策（`validatePathParam` 共通関数）、`/storage/*` のディレクトリトラバーサル防止。続けて安定性改善——自動キャプチャの `setTimeout(6000)` → `engineReady` イベント駆動化（8秒フォールバック付き）、メッセージリスナーの解除漏れ修正、動画ポーリングの `useRef` 管理と `useEffect` クリーンアップ。

設計改善では、`FilterType` を `packages/core` に一元管理。`Project.data` の `as any` を10箇所除去して `ProjectData` 型を定義。サムネイル一覧に `role` / `aria-label` / `aria-current` を追加。フィルター22種のエディタスクリーンショットを仕様書に添付。

## スケーラビリティのタイムライン

MAU 100 → 1,000 → 5,000 → 10,000 → 100,000 の5フェーズに分け、各フェーズの発動条件を Health Dashboard の数値で定義した。「DB latency が100msを超えたら Phase 2」「UGC が日10件を超えたら Phase 3」のように、判断に迷わない閾値を設定。

Phase 0（今月）として Front Door CDN のセットアップワークフローを作成。`setup-frontdoor.yml` を GitHub Actions で手動実行すれば、Front Door 作成 → `ASSET_BASE_URL` 自動設定まで一気通貫で完了する。
