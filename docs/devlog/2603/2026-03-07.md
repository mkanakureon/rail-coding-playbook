## パフォーマンス最適化三部作

午後いちばんに取り掛かったのは、プレイ画面の重さ問題。計画書を書いてから実装に入る流れで、3つの最適化を連続で仕上げた。

まず **フィルタ品質の自動調整システム**。`PerformanceMonitor` が FPS を実測し、品質レベルを high / medium / low で自動切り替えする。Bloom の 25 サンプルを 9 に減らしたり、Rain の 3 レイヤーを間引いたり。low でも雰囲気は残す方針で、Bloom は vignette 代替に、Rain は darkening で "それっぽさ" だけ残すようにした。設定画面に Effect Quality の選択 UI も追加して、Auto / High / Medium / Low を手動で選べるようにもしている。

次に **テクスチャ LRU キャッシュ**。最大 30 エントリ / 128MB 上限で、`loadPremultiplied()` にキャッシュチェックを挟む。VN は同じ背景やキャラが繰り返し出るジャンルなので、キャッシュヒット時に fetch が丸ごとスキップされる効果は大きい。`TextureCache.ts` 103 行で収まった。

最後に **アニメスプライトのストリーミングロード**。フレームを先頭から順次読み込んで、揃った分から再生を始める。全フレームのダウンロード完了を待たずにアニメーションが動き始める。

3 つとも計画書の Phase 1 → 2 → 3 をそのまま実装した形。合計で 400 行ほど。
