50 コミットは、おそらくこのプロジェクト最多の 1 日。複数のフィーチャーブランチが同時に育っていて、この日にまとめて main に合流した感がある。マージコミットの差分が `package-lock.json` で膨れ上がるのはモノレポの宿命で、実質的なコード変更量とは乖離する。

KS エディタのコメント保持は、見た目は小さな機能だが実装は意外と厄介だった。GUI ブロックから KS スクリプトを再生成するとき、元のスクリプトにあったコメント行をどこに挿入するかを判断しなければならない。ブロックの対応関係が 1:1 であればシンプルだが、ブロックの追加・削除・並び替えがあると、コメントの「帰属先」が曖昧になる。最終的にはブロック境界にアンカーを設けて、直前のアンカーにコメントを紐づける方式にした。

RAG の MCP 統合は、Claude Code のスキルとして RAG 検索を使えるようにする試み。`/mcp` エンドポイントを追加する際、Next.js の pages ディレクトリの認証ミドルウェアが `/mcp` パスにも反応してしまう問題があった。パスの prefix マッチが広すぎたのが原因で、認証対象のパスを明示的にリストアップすることで解決した。

ローカル/Azure テスト乖離は、CI がない期間に蓄積した技術的負債。`next dev` は ESLint を走らせないが `next build` は走らせる、Vite dev server は型チェックしないが `tsc -b` はする、という非対称性がその根本にある。GitHub Actions の導入でこの乖離は縮小しつつあるが、ゼロにするにはまだ時間がかかる。Editor の ESLint 既存エラーを「インシデント」として文書化したのは良い判断だった。見て見ぬふりをせず、かといって今すぐ全部直すこともせず、記録として残す。
