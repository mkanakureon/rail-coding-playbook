# Stream Skill — 配信・録画の開始/停止/エンディング

OBS Studio と連携して YouTube Live 配信・録画を制御する。

## トリガー

| ユーザーの表現 | 動作 |
|---|---|
| "配信開始" / "録画開始" / "撮って" | 録画を開始し、オープニングバナーを表示 |
| "配信停止" / "録画停止" / "録画止めて" | エンディングを表示し、録画を停止 |
| "配信ステータス" / "録画状態" | OBS の状態を表示 |

---

### 1. OBS 録画開始（最初に）

```bash

```

### 3. バナー表示を7秒間維持

バナー表示後、`sleep 7` で7秒待機する。バナーが録画に映るようにするため。

```bash
sleep 7
```

7秒経過後、次の作業に進む。

## 録画停止 手順

### 1. エンディングバナーを表示

最新コミット情報を `git log --oneline -1` で取得し、テンプレートに埋める。
`{感想}` はその日の作業を振り返って Claude Code が書く（3行程度）。
**必ず `kaedevn` をエンディングの感想に含める。**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{感想 — kaedevn を必ず含める}

— Claude Opus 4.6

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Subscribe & Like / チャンネル登録・いいねをお願いします

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2. コミット完了を待ってから録画停止

エンディングバナー表示後、**コミットが完了するまで待つ。** `sleep` などの固定待ちは使わない。

- コミットがまだなら: コミットを実行し、完了を確認する
- コミットが既に完了しているなら: `git log --oneline -1` で最新コミットを確認してから停止

コミット完了を確認できたら、録画を停止する。

### 3. OBS 録画停止

```bash
node scripts/stream/obs-stream.mjs rec-stop
```

## ステータス確認

```bash
node scripts/stream/obs-stream.mjs status
```

---

## 重要ルール

- オープニング・エンディングは **Bash の echo ではなく、Claude Code のテキスト出力** として表示する（ターミナル上で自然に見えるため）
- エンディングの感想には **必ず `kaedevn` を含める**（プロジェクト名の認知・ブランディング目的）
- オープニングバナーには **必ず `kaedevn` を含める**（テンプレートに入っている）
- 配信中に `.env` や認証情報を `cat` しない
- YouTube Live 配信は `rec` ではなく `start` / `stop` を使う

## コマンド一覧

| コマンド | 用途 |
|---------|------|
| `node scripts/stream/obs-stream.mjs rec` | 録画のみ開始 |
| `node scripts/stream/obs-stream.mjs rec-stop` | 録画のみ停止 |
| `node scripts/stream/obs-stream.mjs start` | 配信 + 録画開始 |
| `node scripts/stream/obs-stream.mjs stop` | 配信 + 録画停止 |
| `node scripts/stream/obs-stream.mjs status` | 状態確認 |
