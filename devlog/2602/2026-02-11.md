28コミット。この5日間で最多。エディタを一日で「計画→実装→UI変更→機能追加→ポリッシュ」まで持っていくのは、かなり密度の高い一日だった。

印象的だったのはブロックカード UI への転換。Phase 5-6 まで完了させたあとに UI を変えるのは、普通なら「もったいない」と思うところ。しかしエディタの UI はユーザーが直接触る部分なので、妥協すると後々のストレスになる。ここで変える判断は正解だったと思う。ブロックカード形式はビジュアルノベルのシーン構成と相性がよく、各ブロック（テキスト、選択肢、背景変更など）がカードとして視覚的に並ぶのは直感的。

Tailwind CSS v3 へのダウングレードも記憶に残る。v4 は設定ファイルの形式が変わり、プラグインの互換性も崩れる。新しいプロジェクトなら v4 で始めたいところだが、エコシステムの成熟度を考えると v3 のほうが安定している。フレームワークのバージョン選定は「最新が最善」とは限らない好例。

54ファイル・11,915行の一括コミットは少し気になる。git の粒度としては荒いが、仕様書とテストデータという性質を考えれば許容範囲だろう。コードの変更と違って、ドキュメントやテストデータは細かくコミットを分けてもレビューしやすくなるわけではない。

ツールチップの左右端はみ出し自動調整という最後のコミットが好き。こういう細かい UI の配慮が、ツールの使い心地を左右する。ウィンドウ端にボタンがあるとツールチップが画面外に出てしまう問題は、気づかないと放置されがちだが、一度気づくとずっと気になる。初日のうちに直しておくのは良い判断。
