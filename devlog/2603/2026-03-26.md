## バックアップの自動化

夕方、データベースのバックアップまわりに取り掛かった。

`node-cron` で毎朝 9 時（JST）にバックアップ状態をチェックしてメール通知するスケジューラを `apps/hono` に追加。Azure Communication Services の再構築も必要になり、リソースグループ `rg-next-aca-min` に新たにメールサービスを作り直した。`backup-dump.sh` の `pg_dump` パスがバージョン固定（16.12）でエラーになっていたのを自動検出に修正。

ファイアウォール → バージョン不一致 → リソース消失と、想定外のハードルが連続したが、最終的にメールが届くところまで確認できた。

この日は振れ幅が大きかった。朝は RPG エディタという新しい世界を 1 万行で立ち上げ、昼は CLI のコマンド体系を整備し、夕方にはインフラのバックアップ自動化。レイヤーが全然違う作業が 1 日に詰まっている。

Entity Graph の設計は、RPG のデータ構造をグラフ理論で抽象化するアプローチだ。キャラクターがスキルを持ち、スキルがステータスを参照し、敵がドロップテーブルでアイテムとつながる——この関係性をノードとエッジで表現すれば、静的解析で整合性チェックが自動でできる。49 テストが最初から入っているのは、前日のテスト監査の教訓が効いている。

CLI の 4 連続コミットは、1 つずつ積み上げていく開発のリズムが出ている。EditorAPI → コマンド自動収集 → バッチ実行 → JSON 出力と、毎回前のコミットの上に乗る形で機能が広がっていった。

バックアップの件は、Azure Communication Services のリソースが消えていたのが一番の驚きだった。2 月に動いていたものが 3 月には消えている。クラウドリソースの管理は定期的に確認しないと痛い目に遭う、という教訓。
