## ネイティブエンジンに描画の「抽象」を入れる

この日の最大の仕事は、C++ ネイティブエンジンに `IRenderer` インターフェースを導入したこと。SoftRenderer（SDL_Renderer ベース）を実装し、続けて GLRenderer（OpenGL ES 3.0）を845行で書き上げた。VAO/VBO/EBO、シェーダーコンパイル、オーソグラフィック射影行列——大学の講義で習ったものが、Switch 向けエンジンのコードに降りてきた感覚。`IRenderer.hpp` に 30 メソッドの純粋仮想関数を定義し、Web（PixiJS）と Native（SDL2/OpenGL）のコードパスを完全に分離した。

メメントモリの描画戦略を分析したベンチマークレポートもついでに書いた。Live2D + パーティクル + ポストエフェクトを 60fps で回すには、描画コール数を 50 以下に抑えるのが鍵。

## パーティクルが空から降ってきた

環境パーティクルフィルター5種（雪・花びら・光の粒・キラキラ・塵）を WebGL シェーダーで実装。各100行前後。続けて CPU パーティクルシステム（435行）も追加し、バトルやガチャ演出用の `ParticleBlockCard` をエディタに新設した。`filterCategories.ts` にパーティクル階層を追加し、エディタ UI からプリセットを選べるようにした。

さらに色調補正ブロック（明るさ/コントラスト/彩度/色温度スライダー）も追加。パーティクル・色調補正ともに、エンジン正式統合として Op 型定義 → コンパイラ → WebOpHandler → KS 生成まで全レイヤーを通した。統合テスト3件も追加。

## Live2D、3時間の格闘

Live2D Cubism 5 の統合は試作版から始まって、デモ拡張（7モデル切替・リップシンク・位置操作）、キャラブロック新設、エンジン正式統合、プレビュー統合まで一気に進んだ。合計7コミット。

が、途中で `afterMotionUpdate` のタイミング問題に3時間ハマった。API が `result: true` を返しても画面に反映されない。ライブラリのソースを最初に読んでいれば30分で解決していた。この教訓は CLAUDE.md に「外部ライブラリのソースコードを先に読め」ルールとして残した。

CubismCore の CDN 読み込み失敗、API サーバーのコード変更後に古いプロセスが残る問題、KS スクリプト生成が2箇所（エディタ + API）にあって片方だけ更新した問題——6つの問題と対策を振り返り文書にまとめた。

## Playwright MCP で目視確認を自動化

検証の過程で、公式アセットが0件になっている問題を発見して COPY 同期で復旧し、キャラクターの表情画像設定後にプレビューで表示されない問題も発見。この問題は翌日の修正につながる。
