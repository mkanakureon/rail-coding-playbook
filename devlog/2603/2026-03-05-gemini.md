## テスト駆動によるインフラとUIの同期

UIの変更に追従できなくなった E2E テストが大規模に修正され、全 91 件のテストが完全にパスする状態へ復旧した。この過程で、ローカル用の 7 フェーズにわたるテストランナーが新設され、UI と E2E テストの乖離を事前に検知する `pre-commit` フックのチェッカーも導入された。テストが壊れる原因（13件の失敗ルート）の分析と再発防止策もドキュメント化されている。

## KSCエディタ（Phase 1 MVP）の投入

新たなスクリプト形式である KSC (Kaede Script Compiler 用フォーマット) に対応したエディタが、MVP（Minimum Viable Product）として追加された。API レスポンスのハンドリングやデフォルトスクリプトの調整が行われ、独自の E2E テストも 10 件追加。スクリプトタブの名称が「KS」に変更されるなど、エディタ内の棲み分けも明確になった。

## セキュリティ強化とデプロイの安定化

API に対する curl を用いたセキュリティ・拡張テストが追加され、認証基盤（Phase 4）の安全性がより強固に検証されるようになった。また、デプロイワークフローにおいて Next.js の ESLint を一時的に無効化し、デプロイが不要な Lint エラーでブロックされる問題が解消されている。
