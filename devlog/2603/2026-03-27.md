## 図形だけでキャラクターを作るエディタ

PixiJS の Graphics API だけで人体を描くキャラクターエディタが生まれた。`packages/web/chara-editor.html` で動く。

頭、胴体、腕、脚をベジェ曲線で構成し、スライダーで頭のサイズ、顎の幅、肩幅、ウエスト、ヒップ、腕の長さ、脚の太さなどを調整できる。髪はキャップ（頭頂部を覆う帽子型）＋サイドヘア＋前髪の3層構造で、前髪のギザギザは一体型の塗りで描いている。

前髪とキャップの間に肌色が見える問題を何度も修正した。結局、前髪をキャップの頂点まで覆い被せる形にして解消。Playwright でスクリーンショットを撮って、顔の拡大画像で肌色がないことを1ピクセル単位で確認した。

## 表情 18 種 × ポーズ 16 種 = 288 パターン

表情差分を追加。目の開閉、瞳の縮小、眉の角度、口の開き具合をパラメータ化して、笑顔・怒り・泣き・驚き・ウインクなど 18 種類のプリセットを用意した。瞬きのためにまぶたのオーバーレイも実装。閉じ目は曲線1本で表現。

ポーズも同じアプローチで、腕と脚に肘・膝の関節角度を導入。上腕と前腕を分離して描画することで、手を振る、万歳、腕組み、ファイティングポーズなど 16 種類のポーズが切り替わる。最初は変化が小さすぎて「動いてない？」となったが、角度の範囲を広げて解決。

30 FPS のアイドルアニメーションも入れた。呼吸（体の伸縮）、頭と腕の揺れ、ランダムな瞬き。`setInterval` で回している素朴な実装だが、キャラクターが生きているように見える。

## リップシンク — テキストから口パク

リップシンク機能を3段階で実装した。

最初はパターンベース。Talk / Whisper / Shout の 3 モードで、あらかじめ並べた母音パターン配列を順に再生する仕組み。テキストがない場面で使う。

次にテキスト駆動。ひらがな・カタカナの全文字を母音（A/I/U/E/O/N）にマッピングするテーブルを作り、テキストの 1 文字ごとに口形状を変える。五十音表の規則性のおかげで、マッピングは素直に書けた。モード（Talk/Whisper/Shout）は口の開き具合の強度に影響する。同じ「あ」でも Whisper なら小さく、Shout なら大きく開く。

テキスト内容からモードを自動判定する `detectLipMode` も作ったが、正規表現で文脈は読めないと判断して削除。モードは作者がエディタで設定するか、Claude Code がシナリオ生成時に `lip=` で埋め込む方式に落ち着いた。

## Live2D 風のリッチアニメーション

夕方、`chara-editor.ts` に 461 行の追加。これまでの素朴なアイドルアニメーションを、Live2D を模したスムーズな動きに書き換えた。

ポーズと表情の切り替えにイージング付きの遷移を導入。肩の回転、重心のシフト、髪の揺れ物理が加わり、棒立ちだった図形キャラクターが「呼吸している」ような動きになった。表情遷移も瞬間的な差し替えではなく、パラメータを時間軸で補間する方式に変更。Live2D SDK を使わずに、パラメータ補間だけでそれっぽい動きを再現している。

## インラインディレクティブとテキストウィンドウ

`[e:smile]` で表情切り替え、`[p:wave]` でポーズ変更、`[cam:shake]` でカメラ演出、`[click]` でページ送り待ち——テキスト中に埋め込めるインラインディレクティブを実装した。4 ファイル、+996 行。

Ren'Py の仕様調査も同時に行い、`05-renpy-spec-reference.md`（279行）にまとめた。Ren'Py の ATL（Animation and Transformation Language）やスクリーン言語との比較から、[PROJECT_NAME] のディレクティブ設計の方向性を固めている。

テキストウィンドウも追加。キャラクター名とテキスト本文を表示する半透明のウィンドウが画面下部に表示され、ビジュアルノベルらしい見た目になった。

リップシンクの五十音→母音マッピングは、日本語の構造がきれいだからこそ成立する機能だ。英語だと綴りと発音が一致しないから同じアプローチは取れない。`detectLipMode` を作って消した判断は正しかったと思う。正規表現で「静かにしろ」を shout と判定するのは原理的に無理がある。文脈を読むのは人間か LLM の仕事。

Live2D 風アニメーションの実装は、SDK なしでどこまでやれるかの実験でもある。結論として、パラメータ補間とイージングだけでかなりそれっぽくなる。Live2D の真価はモデリングツールとアセット制作のエコシステムにあるのであって、再生側のロジックはそこまで魔法ではない、ということが手を動かして分かった。
